# P0.514 Final External Action Ledger Backend URL Priority Design

## Context

The local iPhone showcase reports now agree on the first three operator actions:

1. write the ignored iOS deploy config with `DEVELOPMENT_TEAM`
2. start `backend-device-demo`
3. set `PMF_BACKEND_BASE_URL` to an iPhone-reachable LAN URL

Fresh `final-external-action-ledger` evidence still lists provider key handoff actions before the backend URL action. This weakens the external action ledger as a real operator handoff because it sends the user toward provider/key work before the local iPhone backend link is actionable.

## Decision

Extend the final external action ledger operator-action priority pass so that when both `backend-device-demo` and the iPhone-reachable backend URL action are present, the backend URL action is placed immediately after `backend-device-demo` and before provider/key/print/Xcode work.

Desired top ordering:

1. `DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`
2. `start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight`
3. `set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL; rerun make mobile-deploy-preflight`

If no backend-device-demo action is present, preserve the existing ordering.

If backend-device-demo is already in the correct second position, the prioritizer must still move the backend URL action behind it. This prevents future upstream reports from preserving a provider-first tail only because the backend demo action was already correctly placed.

## Scope

In scope:

- Add a focused external ledger regression test.
- Add a focused helper regression for the already-second backend-device-demo case.
- Update the existing backend-device-demo prioritizer to include backend URL as part of the device chain.
- Refresh `final-external-action-ledger` evidence after merge.

Out of scope:

- Launch closure packet ordering.
- Real device checks.
- Xcode signing.
- Meshy/OpenAI/Treatstock calls or any cost-incurring action.

## Validation

- The new focused test must fail before implementation.
- `tests/test_final_external_action_ledger.py` must pass.
- `make backend-lint` and `make backend-test` must pass before merge.
- `make final-external-action-ledger` must show deploy writer, backend-device-demo, and backend URL as the first three operator actions.
