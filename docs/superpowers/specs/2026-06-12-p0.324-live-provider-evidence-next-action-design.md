# P0.324 Live Provider Evidence Next Action Design

## Context

`live-provider-evidence.json` is the configured-provider evidence gate for the final iOS demo. It checks provider handoff, configured Meshy 3D evaluation, configured OpenAI NPC evaluation, configured final acceptance, and configured launch evidence.

The report currently exposes `first_blocker` and normalized `operator_actions`, but it does not expose a top-level `next_action`. Adjacent reports already do, so the final launch surface can point at one machine-readable action without re-parsing operator text.

## Decision

Add `next_action` to `live_provider_evidence_report`:

- ready reports set `next_action` to `null`
- blocked, missing, or partial reports copy `first_blocker`
- the copied action adds `source: "first_blocker"`
- the copied action adds `validation_command: "make live-provider-evidence"`
- when the blocker is a known evidence slot, the action includes `requires_live_provider_consent`

This is report-only. It does not call Meshy, OpenAI, Treatstock, Xcode, Keychain, or any payment/order service. It does not write secrets or mutate global state.

## iOS Handoff

`LiveProviderEvidenceReport` decodes optional `next_action` with the same model as `first_blocker`. `LiveProviderEvidenceSlot` accepts optional `source` and optional `validation_command`.

`FinalLaunchMobileSummary` shows a `Next action:` row before the existing blocker row when live provider evidence is not ready. The row is sanitized like the existing live evidence rows.

## Acceptance

- Backend tests prove missing live provider evidence reports expose `next_action` for `provider_handoff`.
- Backend tests prove ready live provider evidence reports set `next_action` to `null`.
- Backend tests prove configured live-provider blockers preserve `requires_live_provider_consent`.
- Swift contract tests prove final launch payloads decode `live_provider_evidence.next_action`.
- Swift summary tests prove the iPhone final launch view shows `Next action:` plus `make live-provider-evidence`.
- `make live-provider-evidence` still exits `2` in the current local missing/blocked state and only writes ignored JSON.
