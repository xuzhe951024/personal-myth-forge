# P0.274 Configured Provider Action Design

## Context

The final configured preflight report aggregates resource, provider, launch,
and iOS deploy readiness before the configured iPhone demo path. Its top-level
`operator_actions` are meant to be the safe handoff checklist a human follows.

Most resource actions now include validation commands, but the configured
preflight report can still expose the provider handoff as a raw CLI command:
`cd services/backend && uv run python -m myth_forge_api.cli provider-handoff ...`.
That is less stable than the project-root `make provider-handoff` target and is
inconsistent with the rest of the final handoff surface.

## Goal

`final-configured-preflight` should normalize provider handoff CLI actions to a
stable `make provider-handoff` top-level operator action.

## Non-Goals

- Do not change provider readiness status calculations.
- Do not change nested child reports such as `provider_handoff.next_commands`.
- Do not run live Meshy, OpenAI, Treatstock, Sculpteo, Xcode, signing,
  Keychain, or print order actions.
- Do not write or apply secrets, `.env` files, or iOS deploy config.
- Do not broaden action normalization beyond existing shared helper rules.

## Design

Reuse `normalize_operator_action()` from `myth_forge_api.operator_actions` in
the configured preflight top-level action aggregation. The configured preflight
builder should normalize each assembled action before adding resource validation
commands and deduping.

The expected pipeline is:

```text
child report actions
  -> normalize_operator_action()
  -> add_final_resource_validation_command()
  -> _dedupe()
```

This keeps raw child evidence intact while ensuring the top-level operator list
points to stable project-root `make` targets.

## Acceptance

- RED/GREEN test proves `final-configured-preflight.operator_actions` includes
  `make provider-handoff`.
- The same test proves no top-level configured preflight operator action still
  contains `myth_forge_api.cli provider-handoff` or
  `--output .local/provider-handoff.json`.
- Existing configured preflight tests still pass.
- Refreshed `.local/final-configured-preflight.json` remains blocked by missing
  external resources, but top-level operator actions no longer expose the raw
  provider handoff CLI.
- Safety flags remain read-only with no provider calls, env writes, iOS config
  writes, global mutation, Xcode/signing, or Keychain writes.
