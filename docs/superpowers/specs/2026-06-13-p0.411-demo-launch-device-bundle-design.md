# P0.411 Final Demo Launch Device Bundle Design

## Context

`final-demo-launch-local.json` is the main local launch report for the final demo flow. It already embeds both `final_showcase_readiness` and `ios_device_evidence_bundle`, and its top-level blocker/next action already use the concrete iPhone deploy action from final showcase readiness.

The report still lacks a top-level `device_action_bundle`, so consumers must know which nested report to inspect. Other final handoff reports now expose a top-level device bundle, including final showcase readiness, iOS deploy runbook, iOS device evidence bundle, iOS device launch certificate, and final launch closure packet.

## Goal

Add a sanitized, read-only `device_action_bundle` to `final-demo-launch-local.json` and configured demo launch reports.

## Scope

- Add top-level `device_action_bundle` to `build_final_demo_launch_report`.
- Prefer `final_showcase_readiness.device_action_bundle` because it carries the deploy child action aligned with final demo launch's top-level next action.
- Fall back to `ios_device_evidence_bundle.device_action_bundle` if final showcase readiness lacks a bundle.
- Preserve existing phase summary, first blocker, next action, operator actions, embedded source reports, and status behavior.
- Add focused blocked and child-action tests.
- Update the roadmap with P0.411.

## Non-Goals

- Do not run backend, Xcode, signing, Keychain, provider, iPhone, or paid-print commands.
- Do not change final demo launch blocker ordering.
- Do not change mobile UI models in this iteration.
- Do not write backend env, iOS deploy config, provider secrets, or print quote evidence.
- Do not embed secrets, local absolute paths, raw media, or payment links.

## Design

The final demo launch report will include:

- `id`: `final_demo_launch_device_actions`
- `label`: `Final Demo Launch Device Actions`
- `source_report`: `final_showcase_readiness` or `ios_device_evidence_bundle`
- `status`: copied from the source bundle
- `actions`: compact sanitized copies of source device actions
- `first_action`: compact sanitized copy of source bundle first action
- `summary`: counts for actions, ready, missing, blocked, manual, provider calls, global actions, and Xcode/signing
- `safety`: read-only no-mutation flags

The helper will copy only already-sanitized report data. It will not execute commands or inspect local machine state. If no source bundle exists, it will produce a stable blocked bundle with no actions and a `missing_source_bundle` summary.
