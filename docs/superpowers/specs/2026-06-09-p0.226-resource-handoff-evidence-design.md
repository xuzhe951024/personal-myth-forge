# P0.226 Resource Handoff Evidence Design

## Context

`resource-handoff` already exists as a backend CLI command and report builder.
The final showcase readiness report uses that builder to decide whether
provider/iOS resources are ready. However, the repo does not expose a matching
Make target and the local rehearsal refresh does not persist
`services/backend/.local/resource-handoff.json`. That leaves one of the final
provider-key handoff inputs without a standard evidence file.

## Decision

Add a first-class `resource-handoff` Make target that writes
`services/backend/.local/resource-handoff.json`, then include the same report in
`final-local-report-refresh`. This keeps the operator flow consistent with
`provider-handoff`, `final-resource-apply-preview`, and the other `.local`
evidence files.

## Scope

- Add `make resource-handoff`.
- Add a `resource_handoff` step to `final-local-report-refresh`.
- Add resource template acceptance and Make dry-run tests for the new output
  path.
- Add ProjectChecks coverage for the Make target and output path.
- Document how to inspect the persisted report.

## Non-Goals

- Do not apply resources or write `services/backend/.env`.
- Do not write iOS deploy config.
- Do not call Meshy, OpenAI, Treatstock, or Sculpteo.
- Do not run Xcode, signing, keychain, or global developer settings.
- Do not expose provider secret values or absolute local paths.

## Acceptance

- `make resource-handoff` writes
  `services/backend/.local/resource-handoff.json`.
- `make final-local-report-refresh` writes the same report as one of its
  managed steps.
- Focused backend tests and Swift ProjectChecks pass.
- The report remains sanitized and read-only.
