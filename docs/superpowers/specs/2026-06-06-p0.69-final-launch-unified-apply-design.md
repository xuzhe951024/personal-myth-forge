# P0.69 Final Launch Unified Apply Design

## Context

P0.66 added `make final-apply-resources` so final demo resources can be provided through one ignored file:

`services/backend/.local/final-resources.env`

That target applies backend provider configuration and iOS deployment configuration without writing global machine state. However, `final-demo-launch` still presents the older low-level writer commands as first-class launch phases:

- `make backend-write-provider-env`
- `make mobile-write-deploy-config`

Those commands remain useful for separate handoff debugging, but the final demo operator path should point at one primary apply command.

## Goal

Make the launch report and operator checklist treat `make final-apply-resources` as the primary resource application step for both local and configured final demo modes.

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, or other live providers.
- Do not remove low-level writer scripts or Makefile targets.
- Do not change iOS signing, Apple account state, firewall, or any global machine state.
- Do not broaden product behavior beyond final launch readiness reporting.

## Behavior

`final-demo-launch` should expose a launch phase:

- id: `apply_final_resources`
- command: `make final-apply-resources`
- status in configured mode: combined core backend provider status plus iOS deploy config status
- status in local mode: blocked when iOS deploy config is missing, partial when iOS is ready but final Meshy/OpenAI resources are not, and ready only when the unified apply command has all required resources

The launch report command list should no longer include lower-level writer commands as primary next commands. The lower-level commands remain discoverable in resource handoff and docs for debugging.

## Acceptance

- Configured launch with missing resources blocks on `apply_final_resources`.
- Configured launch with Meshy/OpenAI/Treatstock and iOS deploy resources marks `apply_final_resources` ready.
- Local launch still succeeds with partial status when iOS device resources are missing.
- Local launch marks `apply_final_resources` partial when iOS deploy config is ready but final Meshy/OpenAI resources are absent.
- Local and configured `commands` include `make final-apply-resources`.
- Local and configured `commands` do not include `make backend-write-provider-env` or `make mobile-write-deploy-config`.
- Existing secret/path sanitization remains intact.
