# P0.526 Provider Key Capability Validation Design

## Goal

Make the final showcase readiness `provider_key_handoff` capability expose the same validation-aware provider key action that already appears in operator actions.

## Context

The current final showcase report is blocked by external iOS and provider inputs, which Codex must not invent or spend against. The report already preserves safe, copy-pasteable operator actions such as `provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight`, but the `provider_key_handoff` capability row can still expose the weaker child command `provide MESHY_API_KEY in final-resources.env`.

That mismatch makes the final handoff less executable because mobile and report consumers may read the capability row instead of the normalized operator action list.

## Design

When `_provider_key_handoff_capability()` detects blocked final resources, it should continue using the child report's next action, but it must append the child validation command to the surfaced capability command. If the child action is `provide MESHY_API_KEY in final-resources.env` and the child validation command is `make final-resources-preflight`, the capability command should be `provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight`.

This does not change provider execution, secret storage, live-call consent, or external resources. It only improves the read-only readiness contract.

## Data Flow

`final_resources` report -> `_report_next_action_command()` -> `_provider_key_handoff_capability()` -> `_capability()` -> `capabilities_by_id.provider_key_handoff.command` and nested `next_action.command`.

## Error Handling

If there is no child validation command, keep the current command unchanged. If there is only a generic fallback command, keep `make final-resource-apply-preview`.

## Tests

Add a focused TDD regression in `services/backend/tests/test_final_showcase_readiness.py` proving that the blocked provider key capability command and nested next action include `; rerun make final-resources-preflight`, while `validation_command` remains `make final-resources-preflight`.
