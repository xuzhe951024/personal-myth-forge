# P0.75 Mobile NPC Agent Mode Design

## Goal

Make the iPhone demo explicitly show whether NPC behavior is running in local
deterministic mode or OpenAI-backed AI Agent mode, and show what is needed to
reach AI Agent readiness.

## Context

The backend already supports `NPC_PROVIDER=openai` through `OpenAINPCDirector`
and `OpenAINPCTickRuntime`. The mobile app decodes NPC traces, ticks, provider
readiness, and runtime names, but the operator has to infer the active NPC mode
from scattered labels. For the final demo, the iPhone should make the NPC Agent
state obvious: local demo mode, OpenAI configured, waiting for a session, or
missing `OPENAI_API_KEY`.

## Design

Add a pure Swift mobile core builder:

- `NPCAgentModeStatus`: `waiting`, `localDemo`, `aiReady`, and `needsSetup`.
- `NPCAgentModeSummary`: title, detail, provider label, runtime label,
  trace/tick counts, missing environment names, and privacy notes.
- `NPCAgentModeSummaryBuilder.build(session:latestTick:tickHistoryCount:providerReadiness:providerReadinessError:)`.

The builder combines:

- current `MythSession.npcAgentRuntime` and `npcAgentTraces`;
- latest `NPCAgentTick.agentRuntime` and tick traces when available;
- `ProviderReadinessResponse.providers[kind == "npc"]`;
- backend readiness errors.

It treats runtime names containing `openai` or a real-ready NPC provider as
AI-ready. It treats `selectedProvider == local` or local runtime names as local
demo mode. Missing env names are shown as names only, never values.

## App Wiring

Add `NPCAgentModeView` above `NPCTickView` in `ForgeRootView`. It is a compact
status card with:

- `NPC Agent Mode` title;
- status badge (`WAIT`, `LOCAL`, `AI`, `SETUP`);
- provider/runtime labels;
- traces and saved tick count;
- missing env names when setup is incomplete;
- privacy copy that keys and raw personal data stay backend-only.

This view does not start NPC ticks and does not call OpenAI. Existing `Run
Autonomy` remains the user action for generating more NPC behavior.

## Out Of Scope

- New OpenAI API calls.
- Voice or Realtime NPCs.
- NPC memory/belief systems.
- Provider key creation or global environment mutation.
- Device signing, simulator builds, or Apple SDK license changes.

## Testing And Evidence

- Swift core contract tests cover waiting, local demo, OpenAI ready, missing
  OpenAI key, latest tick runtime override, and unsafe text redaction.
- Project checks require the core builder, SwiftUI view, root wiring, and Xcode
  project membership.
- iOS showcase acceptance gains a source gate for `mobile_npc_agent_mode`.
- Static 390x844 evidence shows local demo, AI-ready, and setup-needed states.
