# P0.491 External Ledger Device Action Normalization Design

## Context

`final-external-action-ledger.json` is one of the top-level operator handoff
reports for the final iPhone showcase. Current evidence still exposes:

- `run Xcode build gate manually on the Mac: make mobile-xcode-build; rerun make mobile-xcode-build-evidence`
- bare `make mobile-deploy-preflight`

Those entries are safe, but weaker than the current canonical handoff used by
final showcase, final demo launch, final local refresh, and final launch closure
packet.

## Design

Keep the structured external ledger action rows intact. They still need to record
that Xcode build, signing, and physical device work are global/manual actions.

Only normalize top-level `operator_actions`:

- Prefer `accept the Xcode license outside Codex, then rerun make mobile-xcode-build-evidence`
  over the older `run Xcode build gate manually on the Mac...` text when both
  are present.
- Drop bare `make mobile-deploy-preflight` when a more specific action already
  says `rerun make mobile-deploy-preflight`.
- Preserve `make ios-device-launch-rehearsal`, provider handoff, print quote
  handoff, and resource handoff actions.

## Acceptance

- A RED/GREEN backend test proves the external ledger no longer surfaces the
  older Xcode build gate prose in top-level `operator_actions`.
- A RED/GREEN backend test proves bare `make mobile-deploy-preflight` is removed
  when validation-aware mobile actions are present.
- Existing external ledger and shared operator-action tests pass.
- `make final-external-action-ledger` refreshes ignored `.local` evidence while
  preserving the same blocked first action: project-local iOS deploy config.

## Safety

This phase does not run Xcode, change signing, write Keychain state, call live
providers, or place print orders. It only changes report generation and tests.
