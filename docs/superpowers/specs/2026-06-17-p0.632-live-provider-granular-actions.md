# P0.632 Live Provider Granular Actions Spec

## Goal

Make `live-provider-evidence` itself guide operators through the configured Meshy
3D and OpenAI NPC evidence prerequisites before configured final acceptance.

## Problem

The live provider evidence report currently chooses the first blocker by status
severity. A blocked downstream `final_acceptance_configured` report can therefore
appear before missing upstream `3d-evaluation-configured` and
`npc-evaluation-configured` reports. Most higher-level reports have already corrected
this by surfacing the granular 3D/NPC actions, but this source report can still point
operators at broad configured acceptance too early.

## Acceptance

- When provider handoff is ready but configured 3D/NPC evidence is missing,
  `live-provider-evidence.first_blocker` is configured 3D evaluation.
- `live-provider-evidence.operator_actions` contains both configured 3D and NPC live
  consent commands before any broad configured acceptance action.
- Existing provider handoff, failed evaluation, redaction, and ready cases remain
  unchanged.
- No live providers are called and no secrets or global/Xcode state are written.
