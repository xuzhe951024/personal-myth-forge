# P0.62 Treatstock Print Quote Adapter Design

## Purpose

The product loop already creates a local `print_candidate` and mobile can request
a safe local draft quote. The remaining print gap is that `PRINT_PROVIDER` still
cannot route to a real third-party quote provider. P0.62 adds the first live
print quote adapter path for Treatstock while keeping provider calls opt-in and
approval-gated.

Treatstock's official API documentation describes this minimum quote flow:

1. Upload or reference a 3D print file through `POST /api/v2/printable-packs`
   with `private-key`.
2. Include `location[country]` or another location signal if price is needed.
3. Retrieve costs with `GET /api/v2/printable-pack-costs/`.
4. Use returned quote URLs for checkout handoff; creating an order is a separate
   `place-order/create` call that requires shipping/contact data.

P0.62 implements steps 1-3 only. Automatic order placement is intentionally out
of scope.

## Scope

In scope:

- Serve local demo print candidates as `/v1/print-candidates/{session_id}/print.3mf`.
- Update local session URL rewriting so mobile sees a downloadable print
  candidate URL when using the local backend.
- Add `TreatstockPrintProvider` behind `PRINT_PROVIDER=treatstock`.
- Add `TREATSTOCK_API_BASE_URL` for test and sandbox injection.
- Update provider readiness so Treatstock with an API key is real-provider-ready
  for quote handoff.
- Sanitize Treatstock errors and report bodies before returning through the API.
- Keep all live Treatstock calls opt-in through backend configuration.

Out of scope:

- Creating Treatstock orders or collecting shipping/payment/contact data.
- Uploading private local files directly from disk.
- Making Treatstock calls during automated tests.
- Guaranteeing the local placeholder 3MF is production-printable.
- Implementing Sculpteo.

## Provider Behavior

`TreatstockPrintProvider.create_print_candidate(generated_asset)` should derive a
reviewable `PrintCandidate`:

- If `generated_asset.variants` contains `role="print_source"` with format
  `stl`, `ply`, or `3mf`, use that variant URI and format.
- Otherwise, derive a 3MF candidate from the generated asset URI, matching the
  existing local behavior.
- Always set `requires_user_approval=True`.

`TreatstockPrintProvider.create_print_quote(request)` should:

- Require `TREATSTOCK_API_KEY`.
- Require a supported candidate format: `stl`, `ply`, or `3mf`.
- Require an `http://` or `https://` candidate URL, preferring
  `print_candidate.uri` over `source_asset_uri`.
- POST to `/api/v2/printable-packs` with `files-urls[]`, `location[country]`,
  and `private-key`.
- If quantity is greater than 1 and upload response parts include uids, PUT
  `qty[<uid>]` updates to `/api/v2/printable-packs/{id}` before pricing.
- GET `/api/v2/printable-pack-costs/` with `printablePackId`, `private-key`,
  and `location[country]`.
- If Treatstock returns `{"reason": "not_calculated_yet", "success": false}`,
  retry a bounded number of times without sleeping in tests.
- Choose the lowest returned `price`, convert it to cents, set currency to USD,
  and return a `draft_quote` with a checkout URL from the chosen cost result when
  present.
- Never return provider keys, private-key query strings, local file paths,
  payment identifiers, or raw provider error payloads.

## API and Readiness

`build_print_provider(Settings(print_provider="treatstock"))` should return
`TreatstockPrintProvider`.

`/v1/print-quotes` should catch Treatstock provider errors and return sanitized
HTTP 502 details, matching Meshy/OpenAI provider handling.

`/v1/provider-readiness` should report:

- `PRINT_PROVIDER=treatstock` without `TREATSTOCK_API_KEY` as
  `missing_configuration`.
- Treatstock with a key as `ready`, `is_demo_ready=True`, and
  `is_real_provider_ready=True`.
- Capabilities for URL upload, minimum quote retrieval, checkout handoff, and
  manual approval.
