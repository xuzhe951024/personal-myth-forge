# P0.588 Resource Requirements Ready Actions Design

## Goal

Remove the self-refresh operator action from `final_resource_requirements_report` when every required resource is already ready.

## Context

The saved local handoff currently has:

```text
final-resource-requirements.status = ready
final-resource-requirements.operator_actions = ["make final-resource-requirements"]
```

That command is useful when the report is blocked or missing resource values, because it tells the operator how to revalidate the filled handoff file. When the report is already `ready`, it becomes noise and leaks into `final-demo-launch-local.operator_actions` beside the real remaining live-provider consent action.

## Design

Keep `make final-resource-requirements` as a validation command and keep it in blocked/missing operator handoff paths.

Change only `myth_forge_api.final_resource_requirements._operator_actions()`:

- collect resource blockers from missing/blocked requirement rows
- collect nested preflight operator actions
- append `make final-resource-requirements` only when at least one actionable item exists
- return an empty list when the requirements report is already ready

## Scope

In scope:

- source report operator action behavior
- focused resource requirements regression tests
- final demo launch regression check for downstream consumption
- roadmap entry for P0.588

Out of scope:

- writing resource files
- applying resources to backend `.env` or iOS deploy config
- live Meshy/OpenAI/Treatstock calls
- iOS signing or Xcode execution

## Verification

This phase is valid when:

- RED proves the ready report still emits `make final-resource-requirements`
- GREEN proves ready reports emit no operator actions
- blocked/missing resource reports still emit validation-aware handoff commands
- refreshed final local reports no longer include the ready self-refresh action
