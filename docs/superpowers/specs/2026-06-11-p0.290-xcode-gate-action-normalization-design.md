# P0.290 Xcode Gate Action Normalization Design

## Goal

Normalize vague Xcode build gate operator actions to the concrete evidence command already used by the final iPhone handoff.

## Context

The final showcase remains blocked by real iOS deploy evidence and external resources. The current saved reports already expose the concrete Xcode handoff action:

```text
accept the Xcode license outside Codex, then rerun make mobile-xcode-build-evidence
```

Some final acceptance and nested rehearsal actions still surface the vaguer wording:

```text
resolve Xcode build gate outside the app
final_rehearsal_local: final_acceptance_local: resolve Xcode build gate outside the app
```

That wording is accurate, but it does not tell the operator which project command refreshes the local evidence file.

## Design

Keep the behavior centralized in `myth_forge_api.operator_actions`.

Add a small generic action replacement that maps:

- `resolve Xcode build gate outside the app`

to:

- `accept the Xcode license outside Codex, then rerun make mobile-xcode-build-evidence`

The same mapping should work when the action is nested behind a source prefix, preserving the prefix:

```text
final_rehearsal_local: final_acceptance_local: accept the Xcode license outside Codex, then rerun make mobile-xcode-build-evidence
```

Existing detail suffix handling must remain intact, so any ` | detail` text stays attached after normalization.

## Scope

In scope:

- shared operator action normalization
- focused tests for direct, prefixed, and detail-suffixed Xcode gate actions
- focused tests for final acceptance/readiness and final showcase aggregation
- roadmap entry for P0.290
- `.local` refresh verification

Out of scope:

- accepting the Xcode license
- running Xcode build/signing/device commands
- editing Keychain or global Xcode state
- real Meshy/OpenAI/Treatstock calls
- writing API keys or deploy credentials

## Verification

This iteration is valid when:

- RED tests show vague Xcode gate actions still leak through
- GREEN tests show direct and prefixed actions normalize to the concrete evidence command
- refreshed `.local` reports no longer expose the unprefixed vague Xcode gate action
- backend lint and tests pass
