# P0.373 Final Demo Local Apply Preview Safety Design

## Goal

Keep the local final demo launch operator actions aligned with the final resource safety model: preview local resource writes before suggesting the apply writer.

## Current Evidence

- `final-showcase-readiness` already prioritizes `make final-resource-apply-preview` over `make final-apply-resources`.
- `final-external-action-ledger` already keeps the apply writer out of top-level operator actions when preview is still actionable.
- `final-demo-launch-local.json` still exposes `unblock apply_final_resources: make final-apply-resources` in its top-level operator actions, which can make the local iPhone demo handoff look like it should skip the preview gate.

## Behavior

`final_demo_launch` should:

- Keep `make final-resource-apply-preview` in the full command list.
- Keep `make final-apply-resources` in the full command list for documented manual execution after preview.
- Exclude any top-level local operator action whose root is `make final-apply-resources` when `make final-resource-apply-preview` is present.
- Preserve the first blocker, provider handoff actions, print handoff actions, and iOS deploy next action ordering.

## Non-Goals

- Do not run `make final-apply-resources`.
- Do not write backend `.env` or iOS deploy config files.
- Do not change configured provider or Treatstock behavior.
- Do not mark iOS deploy, provider handoff, or print fulfillment ready.

## Acceptance

- A RED/GREEN test proves local final demo launch top-level operator actions include `make final-resource-apply-preview` and exclude all `make final-apply-resources` variants.
- Existing final demo launch tests remain green.
- Adjacent final showcase/local refresh tests remain green.
- Refreshed `.local/final-demo-launch-local.json` no longer shows `unblock apply_final_resources: make final-apply-resources`.
