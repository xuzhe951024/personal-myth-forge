P0.484 Provider Chain Normalization Design

Goal

Make final operator handoff surfaces prefer the complete provider-key handoff chain:

`make final-resource-apply-preview; rerun make provider-handoff; rerun make live-provider-evidence`

over weaker variants that jump directly from resource apply preview to live provider evidence.

Context

The final showcase readiness report already surfaces the complete provider path, which is the safest next-step sequence for the final configured demo. The external action ledger and launch closure packet can still expose weaker actions such as:

- `make final-resource-apply-preview; rerun make live-provider-evidence`
- `make provider-handoff; rerun make live-provider-evidence`
- bare `make final-resource-apply-preview` when it only duplicates the richer chain

This weakens the final handoff because Meshy/OpenAI/Treatstock setup should pass through provider handoff before any live evidence refresh or configured acceptance command.

Non-Goals

- Do not call Meshy, OpenAI, Treatstock, Xcode, or any paid/global provider action.
- Do not change provider readiness semantics, resource templates, or actual key destinations.
- Do not remove independent resource input actions for missing keys or iOS deploy values.
- Do not mark provider, functional, or iOS readiness as complete without real external evidence.

Design

Add provider-chain preference to:

- `final_external_action_ledger._operator_actions()`
- `final_launch_closure_packet._operator_actions()`

The rule should:

- keep the complete chain when present,
- drop weaker provider-only actions that the complete chain supersedes,
- preserve independent missing input actions,
- preserve the standalone apply-preview action when no complete provider chain exists,
- preserve existing deploy-writer and print quote action normalization.

Acceptance Criteria

- Final external action ledger replaces weak provider evidence action hints with the complete apply-preview -> provider-handoff -> live-provider-evidence chain when provider handoff is the configured child blocker.
- Final launch closure packet surfaces the same complete provider chain through its top-level `operator_actions`.
- Weak direct live-provider actions are absent when the complete chain is present.
- Existing iOS deploy writer, print quote, and resource input action tests continue to pass.
- Safe local evidence refresh remains blocked on real external conditions and does not perform provider calls.

Safety

This phase changes only report action normalization. Verification may write ignored `.local` JSON reports through safe Make targets. It must not write secrets, set signing values, call live providers, place print orders, or mutate global machine state.
