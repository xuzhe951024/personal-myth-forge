# P0.266 Resource Validation Actions Design

## Goal

Make final resource requirement operator actions include the validation command the operator should rerun after providing a missing resource.

## Context

The final goal depends on a safe API/key and iOS deploy handoff: Codex must not write secrets into git or trigger external provider spend, but the operator needs precise next steps.

`final_resource_requirements` already records `validation_command` for each missing resource, and `final_local_report_refresh` already knows how to append validation commands when the selected `next_action` includes one. The current gaps are that `final_resource_requirements.operator_actions` still emits short strings such as `provide MESHY_API_KEY in final-resources.env`, and `final_local_report_refresh.next_action` does not preserve the validation command from the selected blocker.

## Design

Update only resource requirement operator-action wording:

- Missing required values become `provide <KEY> in final-resources.env; rerun <validation_command>`.
- Blocked values keep their existing action wording and also append `; rerun <validation_command>` when present.
- Preserve `validation_command` when `final_local_report_refresh` promotes its first blocker into `next_action`.
- The existing first blocker payload shape remains unchanged.
- Preflight-generated actions, init guidance, de-duplication, safety flags, and source summaries remain unchanged.
- The change must not write any resource file, call live providers, launch Xcode, alter Keychain, or mutate global state.

## Test Plan

Extend `test_requirements_report_lists_missing_required_resources_without_leaks()` to assert the top resource actions include validation commands:

- `provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight`
- `provide OPENAI_API_KEY in final-resources.env; rerun make final-resources-preflight`

Extend `test_final_local_report_refresh_operator_actions_use_concrete_next_actions()` so the refresh-level first action matches the same validation-aware string.

The tests should fail before implementation because the current output omits `; rerun make final-resources-preflight`.

## Self-Review

- No placeholders.
- Scope is one resource-handoff formatting improvement.
- The change moves the project closer to final key handoff without weakening privacy or provider safety.
