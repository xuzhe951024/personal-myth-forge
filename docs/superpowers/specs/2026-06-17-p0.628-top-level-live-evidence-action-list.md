# P0.628 Top-Level Live Evidence Action List Spec

## Goal

Make the top-level final handoff reports preserve the current configured live evidence
action list instead of collapsing it to only the first Meshy action.

## Problem

`final-configured-evidence-plan` and `configured-live-evidence-bundle` now correctly
surface the current live provider prerequisites:

- configured Meshy 3D evaluation
- configured OpenAI NPC evaluation

The higher-level external action ledger and closure packet can still expose only the
first action because their 3D/NPC action rows lack the configured-evidence validation
suffix and are folded into the broader configured evidence gate.

## Acceptance

- External action ledger action rows for configured 3D and NPC evaluation include
  `rerun make final-configured-evidence-plan`.
- When the saved configured live evidence bundle includes 3D and NPC granular actions,
  external ledger operator actions preserve both in order.
- Closure packet operator action normalization preserves both validated 3D and NPC
  actions and still prunes configured final acceptance before prerequisite evidence.
- No report calls live providers or writes secrets.
