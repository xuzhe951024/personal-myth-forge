# P0.273 Resource Source Actions Design

## Context

Recent phases normalized final-resource actions in downstream launch and
readiness aggregators. The remaining inconsistency is at the source reports:
`final-resources-preflight`, `final-resource-requirements`, and
`final-resource-apply-preview` can still expose top-level operator actions that
edit `final-resources.env` without immediately rerunning
`make final-resources-preflight`.

## Goal

Every source-level top-level operator action that asks the operator to provide a
known key in `final-resources.env` should include
`; rerun make final-resources-preflight`.

## Non-Goals

- Do not change first-blocker or next-action command fields.
- Do not rewrite embedded child reports.
- Do not apply resources, write env/config files, call providers, invoke Xcode
  or signing, touch Keychain, or place print orders.
- Do not change readiness status calculations.

## Design

Reuse the shared `add_final_resource_validation_command()` helper in the three
source report builders before final action dedupe. This keeps one canonical rule
for known `final-resources.env` edit actions across source reports and
aggregators.

The helper only affects known `provide <KEY> in final-resources.env` actions. It
does not modify broader guidance such as `set PRINT_PROVIDER to local or
treatstock`, `set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL`, or
first-blocker command fields.

## Acceptance

- RED/GREEN tests prove each source report has no known bare final-resource
  top-level action and no final-resource action rerunning a non-resource command.
- Existing source report tests still pass.
- Refreshed `.local` source reports show no known bare top-level
  `final-resources.env` actions.
- Safety flags remain read-only with no provider calls, env writes, iOS config
  writes, global mutation, Xcode/signing, or Keychain writes.
