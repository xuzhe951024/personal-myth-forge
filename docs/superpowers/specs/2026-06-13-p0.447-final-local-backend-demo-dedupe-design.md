# P0.447 Final Local Backend Demo Dedupe Design

## Goal

Remove the redundant bare `make backend-device-demo` action from the final local report refresh handoff when a validated backend-device-demo action is already present.

## Context

`final-local-report-refresh` now gives a clear validated device action:

```text
start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight
```

The same report can still include a later bare:

```text
make backend-device-demo
```

That duplicates the same operator step without the validation command and makes the final iPhone handoff noisier.

## Behavior

When final local operator action dedupe sees an action that starts backend-device-demo and reruns `make mobile-deploy-preflight`, it should drop bare `make backend-device-demo`.

Other backend demo variants, detail-bearing validated actions, and unrelated make targets must keep existing behavior.

## Safety

This only changes report text. It must not start the backend, run iOS preflight, call providers, write keys, touch Xcode signing, write Keychain entries, or create payment/order artifacts.

## Tests

Add a focused unit test for `_operator_actions()` that includes:

- a validated backend-device-demo action
- a bare `make backend-device-demo`

The expected output keeps only the validated backend-device-demo action.
