# P0.374 Final Local Self-Rerun Actions Design

## Goal

Make `final-local-report-refresh` operator actions copy-pasteable by removing redundant self-rerun suffixes.

## Current Evidence

`services/backend/.local/final-local-report-refresh.json` can emit actions like:

- `make final-rehearsal-local; rerun make final-rehearsal-local`
- `run make final-handoff-index; rerun make final-handoff-index`

Those strings are technically harmless, but they make the final Mac-side handoff less clear than the rest of the launch evidence chain.

## Behavior

When a report action has the same command and validation command, `final-local-report-refresh` should emit one normalized command:

- `make final-rehearsal-local`
- `make final-handoff-index`

The report should still retain validation-aware actions when command and validation differ, such as:

- `make provider-handoff; rerun make live-provider-evidence`
- `DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`

## Non-Goals

- Do not run any Xcode, signing, device, provider, or print provider command.
- Do not change readiness status or first blocker selection.
- Do not change final showcase capability classification.
- Do not write keys or local deploy config.

## Acceptance

- A RED/GREEN test proves self-rerun actions normalize to one command.
- Existing final local report refresh tests remain green.
- Adjacent final demo/showcase tests remain green.
- Refreshed `.local/final-local-report-refresh.json` no longer contains `; rerun` where both sides are the same command.
