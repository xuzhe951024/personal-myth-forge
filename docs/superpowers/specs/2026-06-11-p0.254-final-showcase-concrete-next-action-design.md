# P0.254 Final Showcase Concrete Next Action Design

## Context

P0.253 made `mobile-deploy-preflight-evidence.json` expose a concrete
`next_action`, such as `provide DEVELOPMENT_TEAM in Deployment.local.xcconfig`,
plus `validation_command: make mobile-deploy-preflight`. The top-level
`final-showcase-readiness.json` still reports the iOS blocker next action as
`make mobile-deploy-preflight`, which is a validation command, not the concrete
operator task needed to unlock the current iPhone path.

## Goal

When `ios_deployable` is the first final showcase blocker, prefer the saved
mobile preflight evidence `next_action.command` for the top-level final showcase
`next_action.command`, while preserving the preflight command as
`next_action.validation_command`.

## Design

Keep the final showcase report read-only. It should not run Xcode, write deploy
config, call providers, or mutate global state.

The report should:

- continue to keep `first_blocker.command` as `make ios-device-launch-rehearsal`
  because the blocker is still the overall iOS deployable capability.
- set `next_action.command` to the concrete child action when saved mobile
  preflight evidence has a non-ready `next_action.command`.
- set `next_action.validation_command` to the child validation command
  (`make mobile-deploy-preflight`), falling back to the device action validation
  command when needed.
- keep `next_action.detail` aligned with the enriched blocker detail so the
  operator sees all blocked preflight check details.

## Testing

Add focused tests proving:

- when saved preflight evidence contains `next_action.command`, final showcase
  top-level `next_action.command` uses it.
- final showcase `next_action.validation_command` preserves
  `make mobile-deploy-preflight`.
- existing device action detail and source acceptance gates still pass.

## Safety

This only reshapes local readiness JSON from existing ignored reports. It does
not provide secrets, write repo-local deploy config, perform signing, call
Meshy/OpenAI/Treatstock, or place print orders.

## Out of Scope

This iteration does not supply `DEVELOPMENT_TEAM`, set a real
`PMF_BACKEND_BASE_URL`, run a physical iPhone deployment, or mark the full goal
complete.
