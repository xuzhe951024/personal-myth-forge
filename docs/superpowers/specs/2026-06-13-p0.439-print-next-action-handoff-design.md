# P0.439 Print Next Action Handoff Design

## Context

The current final showcase has local print proof ready, but configured print
fulfillment is still partial because Treatstock quote evidence requires explicit
cost consent. The backend emits a precise next action:

```json
{
  "command": "after explicit Treatstock cost consent, save a sanitized services/backend/.local/print-quote-configured.json from POST /v1/print-quotes",
  "requires_cost_consent": true,
  "validation_command": "make print-fulfillment-readiness"
}
```

The iPhone Final Launch fixture and model still treat the next action as a plain
`make print-fulfillment-readiness` command and drop `requires_cost_consent` plus
`validation_command`. That weakens the final print handoff: the operator needs
to see both the cost-consent requirement and the rerun command after the quote
evidence is saved.

## Decision

Make `PrintFulfillmentReadinessNextAction` decode `requires_cost_consent` and
`validation_command`. Render both in the Final Launch print fulfillment section:

- append `cost consent required` when `requiresCostConsent` is true
- append `rerun <validation_command>` when a validation command is present

The backend behavior remains unchanged; this is a mobile contract and display
alignment with the existing report.

## Scope

Included:

- Update the Swift fixture to use the real configured Treatstock handoff command.
- Decode `requires_cost_consent` and `validation_command` for print next actions.
- Display cost consent and rerun validation in the print fulfillment readiness
  rows.
- Update the roadmap.

Not included:

- No Treatstock API calls, quote creation, checkout URLs, payment links, or
  external cost actions.
- No provider key writes, Xcode signing, Keychain writes, or backend behavior
  changes.

## Testing
