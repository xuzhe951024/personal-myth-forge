# P0.360 Final Showcase Live Provider Operator Actions Design

## Context

P0.359 made the final showcase 3D and NPC capability rows inherit the live
provider evidence report's concrete next action:

- `command`: `make provider-handoff`
- `validation_command`: `make live-provider-evidence`

The top-level `operator_actions` list still includes the raw report actions as
separate bare commands. That makes the final demo handoff less clear than the
capability rows because the operator sees `make live-provider-evidence` and
`make provider-handoff` independently instead of the intended sequence.

## Goal

When a non-ready final showcase capability has both `command` and
`validation_command`, the top-level `operator_actions` list should expose one
validation-aware action:

```text
make provider-handoff; rerun make live-provider-evidence
```

The list should prefer that combined action over bare duplicate actions for the
same command roots.

## Approach

Add a small capability-action formatter in `final_showcase_readiness.py`. The
formatter uses the same shape already used for `next_action`:

- command only: `command`
- command plus validation: `command; rerun validation_command`

Use this formatter for capability-derived operator actions. Then update
operator action dedupe to prefer validation-aware actions over their bare command
roots. The rule is generic enough for other capability rows but the test will
focus on the live provider handoff path.

## Non-Goals

- Do not run Meshy, OpenAI, Treatstock, Xcode, signing, Keychain, or global
  configuration.
- Do not change provider readiness semantics.
- Do not change capability row status, first blocker selection, or safety fields.

## Testing

Extend the final showcase readiness regression coverage so a live provider
partial scenario asserts:

- `operator_actions` contains
  `make provider-handoff; rerun make live-provider-evidence`
- bare `make provider-handoff` is not retained when the combined action exists
- bare `make live-provider-evidence` is not retained when it only duplicates the
  combined validation command
