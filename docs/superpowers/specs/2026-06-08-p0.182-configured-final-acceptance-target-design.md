# P0.182 Configured Final Acceptance Target Design

## Context

The project already has a safe local rehearsal target:

```text
make final-rehearsal-local
```

That target intentionally uses local providers and accepts blocked reports so it
can run without API keys, iOS signing credentials, provider calls, or Xcode
global state. It writes the local evidence set used by the final demo handoff.

The configured final acceptance run is different. It is the explicit operator
step for final validation after the user provides `MESHY_API_KEY`,
`OPENAI_API_KEY`, iOS deployment values, and a reachable backend URL. Existing
reports describe the raw CLI command, but there is no top-level Make wrapper for
the operator to run.

## Goal

Expose a single manual `make final-acceptance-configured` target that runs the
configured final acceptance command and writes
`services/backend/.local/final-acceptance-configured.json`, without adding any
live-provider path to automatic local rehearsal.

## Design

Add a shell wrapper:

```text
services/backend/scripts/write_final_acceptance_configured.sh
```

The wrapper will:

1. Create `services/backend/.local`.
2. Run `uv run python -m myth_forge_api.cli final-acceptance` from
   `services/backend`.
3. Use:
   - `--profile quick`
   - `--provider-mode configured`
   - `--require-real-core`
   - `--allow-live-provider-calls`
   - `--repo-root ../..`
   - `--output .local/final-acceptance-configured.json`
4. Accept exit code `0` or `2`, matching the local wrapper pattern where
   `2` means a usable blocked report was written.
5. Fail on any other exit code because no usable configured acceptance report is
   guaranteed.

Add a Make target:

```text
make final-acceptance-configured
```

The target only invokes the configured wrapper. It must not be a dependency of
`final-rehearsal-local`, `final-demo-launch`, or any other automatic local
rehearsal path.

## Safety

This target is intentionally manual because it includes
`--allow-live-provider-calls`. It can make provider calls when configured API
keys are present. Tests will use a fake `uv` binary and `make -n` dry runs, not
the real configured target.

## Non-Goals

- Do not change provider adapters.
- Do not call Meshy, OpenAI, Treatstock, or any real provider during local
  verification.
- Do not add the configured target to `final-rehearsal-local`.
- Do not change iOS signing, Xcode, `.env`, or deployment config writers.
- Do not replace local acceptance as the default rehearsal gate.

## Acceptance

- `services/backend/scripts/write_final_acceptance_configured.sh` exists.
- The wrapper passes the configured final acceptance flags listed above.
- The wrapper exits `0` for CLI exit code `0` or `2`.
- The wrapper exits nonzero for CLI exit code `1`.
- `make -n final-acceptance-configured` shows the configured wrapper.
- `make -n final-rehearsal-local` does not show the configured wrapper.
- Focused script tests pass.
- Backend lint/tests and Swift mobile project checks still pass.

## Self-Review

- Placeholder scan: no TBD/TODO/open placeholders.
- Safety check: real configured execution remains manual and consent-gated.
- Scope check: one Make/script entrypoint slice; no provider, iOS, or mobile UI
  behavior changes.
- Ambiguity check: blocked configured reports are accepted only when the CLI
  returns exit code `2`, consistent with existing local report wrappers.
