# P0.417 Configured Live Evidence Device Bundle Design

## Context

The configured/live handoff path now carries device action bundles through `final_configured_preflight_report` and `final_configured_evidence_plan_report`. `configured_live_evidence_bundle_report` is the next aggregate report in that chain: it composes the configured evidence plan with saved live-provider evidence files, but it still does not expose a top-level `device_action_bundle`.

That leaves the final configured live evidence packet weaker than the upstream plan. Operators can see resource and live-provider blockers, but the physical-iPhone action sequence is not directly available from the bundle report.

## Goal

Expose a sanitized top-level `device_action_bundle` in `configured_live_evidence_bundle_report`, sourced from `final_configured_evidence_plan.device_action_bundle`.

## Scope

- Add `device_action_bundle` to `build_configured_live_evidence_bundle_report`.
- Source it from the already-built configured evidence plan.
- Preserve existing `current_blocker`, `first_blocker`, `next_action`, live evidence status, command sequence, and consent behavior.
- Add focused TDD coverage for the blocked configured live evidence bundle path.
- Refresh `services/backend/.local/configured-live-evidence-bundle.json` and verify the new field appears.
- Add the P0.417 roadmap entry.

## Non-Goals

- Do not run live provider evaluations or configured final acceptance.
- Do not call Meshy, OpenAI, Treatstock, Sculpteo, or any live provider.
- Do not run Xcode, signing, Keychain, global Mac configuration, or real iPhone deployment.
- Do not write backend `.env`, iOS deploy config, or provider resource files.
- Do not change configured live evidence command ordering or consent gating.

## Report Contract

`configured_live_evidence_bundle_report.device_action_bundle` will contain:

- `id`: `configured_live_evidence_bundle_device_actions`
- `label`: `Configured Live Evidence Bundle Device Actions`
- `source_report`: `final_configured_evidence_plan`
- `status`: normalized from the source bundle status
- `actions`: sanitized copies of source evidence-plan device actions
- `first_action`: sanitized source first action when present, otherwise first non-ready copied action
- `summary`: counts for actions, ready, missing, blocked, manual, partial, live, provider calls, global actions, and Xcode/signing actions
- `safety`: all false for command execution, provider calls, global mutation, Keychain writes, backend env writes, iOS deploy config writes, and Xcode/signing execution from the report builder

If the nested configured evidence plan bundle is missing, the live evidence bundle returns a blocked empty bundle with `source_report: "missing"` and `summary.missing == 1`.
