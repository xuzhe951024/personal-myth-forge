# P0.66 Unified Final Resource Apply Design

## Goal

Create one project-local resource handoff path for the final demo operator. The user should be able to copy a tracked example file, fill backend-only provider keys plus iOS deploy values once, and run one Make target to generate both ignored local config files:

- `services/backend/.env`
- `apps/mobile/ios/Config/Deployment.local.xcconfig`

## Approach

Add a tracked template at `services/backend/final-resources.env.example` and a script at `services/backend/scripts/apply_final_resources.sh`.

The script reads a simple `KEY=value` file, validates required values before writing anything, rejects unknown keys, rejects loopback iPhone backend URLs, rejects multiline/unsafe values by construction, and then delegates to the existing backend and iOS writer scripts. It writes only ignored project-local files and prints redacted status only.

## Required Keys

- `MESHY_API_KEY`
- `OPENAI_API_KEY`
- `DEVELOPMENT_TEAM`
- `PRODUCT_BUNDLE_IDENTIFIER`
- `PMF_BACKEND_BASE_URL`

## Optional Keys

- `PRINT_PROVIDER`, default `local`
- `TREATSTOCK_API_KEY`
- `TREATSTOCK_API_BASE_URL`
- `SCULPTEO_API_KEY`
- `OPENAI_API_BASE_URL`
- `CAPTURE_STORAGE_DIR`
- `MYTH_SESSION_STORAGE_DIR`

If `PRINT_PROVIDER=treatstock`, `TREATSTOCK_API_KEY` is required. Unsupported `PRINT_PROVIDER` values fail before writing local files.

## Safety

The apply path must not call providers, start servers, run Xcode, sign apps, touch keychain, accept Apple licenses, write global config, or print secret values.

## Verification

Add tests that prove missing resources, unknown keys, and loopback backend URLs fail without writing outputs; successful apply writes both ignored local files with redacted output; Treatstock provider values flow into backend settings; and resource template acceptance includes the new unified apply surface.
