# P0.320 Final Resources Preflight Next Action Design

## Context

`final-resources-preflight.json` is the read-only report that tells the operator whether ignored `final-resources.env` has the Meshy/OpenAI keys, print provider slots, iOS deployment values, and backend URL needed for the final configured demo.

The report already has `operator_actions`, but it lacks top-level `first_blocker` and `next_action`. That makes the provider/key handoff path less direct than newer readiness reports.

## Scope

Add top-level `first_blocker` and `next_action` to final resources preflight reports.

The report keeps existing `status`, `summary`, `items`, `unknown_keys`, `malformed_lines`, `operator_actions`, and `safety` fields unchanged.

## Blocker Selection

When the resources file is missing:

- `first_blocker.id` is `final_resources_file`.
- Command is `make final-resource-init`.
- Validation command is `make final-resources-preflight`.

When the file exists but is blocked:

1. Unknown keys are first.
2. Malformed lines are next.
3. Required missing items are next in existing item order.
4. Blocked item values are next in existing item order.

The `next_action` mirrors `first_blocker` and adds `source: "first_blocker"`.

When the report is ready, `first_blocker` and `next_action` are `null`.

## Safety Behavior

This remains read-only. It does not write `.env`, `.xcconfig`, Keychain, Xcode state, global files, or provider services. It does not include secret values, raw local paths, payment links, or private media.

## Acceptance Criteria

- Missing resources-file reports expose a first blocker and next action for `make final-resource-init`.
- Missing `MESHY_API_KEY` exposes a first blocker with `provide MESHY_API_KEY in final-resources.env` and `make final-resources-preflight`.
- Blocked backend URL exposes a first blocker for `PMF_BACKEND_BASE_URL`.
- Ready reports expose `first_blocker: null` and `next_action: null`.
- CLI-written `final-resources-preflight` JSON contains the new fields.
- Focused tests, lint, Swift gates, refreshed local report, and full backend tests pass.

## Self Review

- The design is metadata-only and does not alter resource validation semantics.
- Blocker order follows existing report/operator-action intent.
- No API keys or local absolute paths are added to reports.
