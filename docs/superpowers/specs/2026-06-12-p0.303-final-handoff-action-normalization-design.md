# P0.303 Final Handoff Action Normalization Design

## Goal

Make `final-handoff-index.json` top-level `operator_actions` copy-pastable by
removing source-report prefixes and normalizing simple `run make ...` wrappers.

## Current Context

The final handoff index is the Mac-side checklist that ties local rehearsal,
configured preflight, iPhone deploy handoff, and live acceptance together. It is
currently safe and sanitized, but the top-level operator actions can still
contain provenance text:

```text
final_demo_launch_local: provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
run make final-configured-preflight
run make final-demo-launch-configured
```

Those prefixes are useful in nested source reports, but the top-level handoff
list should be directly executable or directly actionable:

```text
provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
make final-configured-preflight
make final-demo-launch-configured
```

## Design

Add a final-handoff-specific action normalization step inside
`myth_forge_api.final_handoff_index`. It should reuse shared
`normalize_operator_action()` first, then strip only known final handoff source
prefixes such as `final_demo_launch_local:` from the top-level operator action
roots.

The change stays narrow:

- It applies only to `final_handoff_index` top-level `operator_actions`.
- It does not rewrite nested `source_reports`.
- It does not change lane detail or blocker provenance fields.
- It does not run commands, write deploy config, write backend env, call
  providers, or touch Xcode/Keychain.
- It preserves the live acceptance consent wording because that is an operator
  policy action, not a plain make target.

## Verification

- RED/GREEN coverage that final handoff index strips

- RED/GREEN coverage that simple generated `run make ...` handoff actions become

- Run focused backend tests plus lint and whitespace checks before merge.
