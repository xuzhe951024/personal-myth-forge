# P0.310 Final Apply Action Canonical Target Design

## Goal

Normalize final resource apply operator actions to the canonical make target `make final-apply-resources` across final handoff reports.

## Context

The final iOS showcase remains blocked by external resource inputs and device evidence, not by local report generation. Several final readiness reports still surface the same resource-apply handoff as `run make final-apply-resources to apply the filled resource bundle`. That wording is actionable, but it is inconsistent with the surrounding canonical make targets and keeps appearing as a remaining `run make` wrapper in readiness audits.

The project already centralizes this behavior in `myth_forge_api.operator_actions.FINAL_RESOURCE_APPLY_ACTION`. Updating that one shared constant and its tests should normalize downstream reports without touching provider calls, Xcode, Keychain, or real order flows.

## Approach

Change `FINAL_RESOURCE_APPLY_ACTION` from the descriptive wrapper to `make final-apply-resources`.

Keep the existing normalization roots:

- `make final-apply-resources`
- `run make final-apply-resources`

Both roots should normalize to the same canonical command. Provider-selection actions such as `set THREE_D_PROVIDER=meshy` and unblock actions such as `unblock final_apply_resources after final_resource_apply_preview` should also normalize to `make final-apply-resources`.

## Data Flow

Final readiness generators collect child actions, pass them through `normalize_operator_action`, and dedupe them. After this change, any child or legacy action routed through the shared final-resource-apply branch emits `make final-apply-resources`. Reports that carry structured command fields continue to use their existing `command` values.

## Testing

Use TDD on `tests/test_operator_actions.py`:

- `make final-apply-resources`, `run make final-apply-resources`, and `make final-apply-resources; rerun make final-resources-preflight` normalize to `make final-apply-resources`.
- resource apply unblock and provider selection shortcuts normalize to `make final-apply-resources`.

Then update affected report tests that intentionally assert the shared canonical action. Refresh representative local evidence reports and verify that the final configured evidence plan no longer emits the descriptive `run make final-apply-resources...` action.

## Non-Goals

- Do not run `make final-apply-resources`.
- Do not write secrets, apply resource files, touch global Xcode state, or call live providers.
- Do not change resource validation commands or the final apply preview report contract.
