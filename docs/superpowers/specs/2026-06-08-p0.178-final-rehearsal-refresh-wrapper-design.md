# P0.178 Final Rehearsal Refresh Wrapper Design

## Context

P0.177 appended `final-local-report-refresh` to `final-rehearsal-local`, so the
canonical local rehearsal command refreshes the final ignored evidence bundle.

The real command exposed a regression:

```bash
make final-rehearsal-local
# ...
# make: *** [final-local-report-refresh] Error 2
```

`final-local-report-refresh` intentionally returns exit code `2` when it writes a
usable report but the final launch remains blocked by external gates such as
resource keys, device evidence, Xcode signing, or live provider proof. That exit
code is correct for the direct refresh command, but a Make dependency treats it
as a hard failure. The existing local rehearsal chain already handles this
pattern for `final-acceptance-local` and `ios-deploy-runbook-local` through small
shell wrappers that accept exit codes `0` and `2`.

## Goal

Keep `make final-rehearsal-local` usable as the no-key local rehearsal command:
it should return `0` when all local report commands complete and the only
remaining blockers are represented by accepted exit code `2` reports.

## Design

Add a new wrapper script:

```text
services/backend/scripts/write_final_local_report_refresh.sh
```

The script will:

1. create `services/backend/.local`;
2. run:
   ```bash
   cd services/backend &&
   uv run python -m myth_forge_api.cli final-local-report-refresh \
     --repo-root ../.. \
     --output .local/final-local-report-refresh.json
   ```
3. accept exit codes `0` and `2`;
4. print:
   - `accepted final local report refresh exit code $status`
   - `wrote services/backend/.local/final-local-report-refresh.json`
5. return the original non-`0`/non-`2` status as an unusable failure.

Add a Make target:

```make
.PHONY: final-local-report-refresh-local

final-local-report-refresh-local:
	@services/backend/scripts/write_final_local_report_refresh.sh
```

Then change `final-rehearsal-local` to depend on
`final-local-report-refresh-local` instead of the direct
`final-local-report-refresh` target.

The direct `final-local-report-refresh` Make target remains unchanged. Operators
and tests that need the exact blocked/failed status can still call it directly.
The rehearsal command uses the wrapper because its job is to refresh all local
evidence and leave the blocked state inside the saved reports.

## Alternatives Considered

1. Change the direct `final-local-report-refresh` target to accept exit `2`.
   Rejected because the direct command's exit code is useful evidence for
   automation that wants to distinguish ready from blocked.

2. Prefix the Make dependency with `-` to ignore errors.
   Rejected because that would also hide exit code `1`, which means an unusable
   report command failure.

3. Convert `final-rehearsal-local` into one large shell script.
   Rejected because existing Make dependencies are readable and already use
   smaller wrappers for the commands that need accepted blocked semantics.

## Acceptance

- `make final-local-report-refresh` still returns exit code `2` in the no-key
  local fixture when the report status is blocked.
- `make final-rehearsal-local` returns exit code `0` in the same no-key local
  environment and writes `services/backend/.local/final-local-report-refresh.json`.
- `make -n final-rehearsal-local` shows
  `write_final_local_report_refresh.sh` after the older local rehearsal gates.
- The new wrapper accepts exit codes `0` and `2`, and fails exit code `1`.
- Resource template and iOS showcase source acceptance require the wrapper
  target/script so the rehearsal chain cannot regress to the direct failing
  target.
- No live provider calls, Xcode/signing, keychain writes, backend env writes, iOS
  deploy config writes, or global mutation are introduced.

## Test Strategy

1. Add failing wrapper tests in
   `services/backend/tests/test_final_rehearsal_scripts.py`.
2. Add failing dry-run/source acceptance assertions for
   `final-local-report-refresh-local` and
   `write_final_local_report_refresh.sh`.
3. Run focused tests and confirm failure before implementation.
4. Add the wrapper script and Make/source fixture updates.
5. Re-run focused tests and the real `make final-rehearsal-local` command.
6. Re-run backend lint and the full backend test suite before merging.

## Root Cause

The failure is at the Make target boundary. The refresh CLI correctly reports a
blocked but usable evidence bundle with exit code `2`; Make correctly treats a
non-zero dependency as a failed target. The missing component is a wrapper that
translates accepted blocked report exits into successful local rehearsal exits,
matching the established `final-acceptance-local` and `ios-deploy-runbook-local`
patterns.
