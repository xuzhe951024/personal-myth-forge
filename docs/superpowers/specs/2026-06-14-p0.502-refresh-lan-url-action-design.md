# P0.502 Final Local Refresh LAN URL Action Design

## Context

`final-local-report-refresh.json` is the safe local rehearsal aggregator. It refreshes local-only evidence and becomes one of the main reports an operator reads before moving to physical iPhone validation.

After P0.501, the final showcase and final demo launch reports preserve the concrete iPhone backend URL handoff:

`set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL; rerun make mobile-deploy-preflight`

The local refresh report still promotes provider and print handoff actions from the saved final demo launch report, but it does not promote the backend URL handoff. That leaves one of the highest-value safe rehearsal reports without the full physical-device next-action set.

## Decision

Extend `final_local_report_refresh` so final demo launch local report actions promote device handoff actions as well as provider and print actions.

The device handoff scope for this phase is:

- iPhone-reachable backend URL action
- backend-device-demo action

Reuse the shared `prefer_iphone_reachable_backend_url_handoff_actions()` helper during operator-action dedupe so any generic `provide PMF_BACKEND_BASE_URL in Deployment.local.xcconfig` action is replaced by the current LAN URL action.

## Alternatives Considered

1. Only refresh `.local` evidence. This restores freshness, but the next source change can recreate the omission.
2. Add the LAN URL action directly to final local report refresh output. This is brittle because it bypasses the saved child report that already knows the current device actions.
3. Promote final demo launch device handoff actions through the same selection path as provider and print. This keeps the source of truth in the child report and matches existing aggregation style.

## Safety

This is report-only. It does not run Xcode, write signing config, touch Keychain, call Meshy/OpenAI/Treatstock, write ignored env/config files, or spend provider credits.

## Verification

TDD coverage must prove:

- `_operator_actions()` promotes the iPhone-reachable backend URL action from `final_demo_launch_local.report_operator_actions`.
- A generic Deployment.local.xcconfig backend URL action is not emitted when the LAN URL action is present.
- Existing final local refresh and shared operator action focused tests still pass.
- Refreshed `services/backend/.local/final-local-report-refresh.json` includes the LAN URL action and excludes the generic backend URL action.
