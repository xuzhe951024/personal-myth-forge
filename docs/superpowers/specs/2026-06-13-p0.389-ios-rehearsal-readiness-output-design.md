# P0.389 iOS Rehearsal Readiness Output Design

## Context

The final iPhone deployment lane depends on two related reports:

- `ios-device-launch-rehearsal.json`: the rehearsal sequence itself.
- `ios_device_launch_rehearsal_readiness_report`: a wrapper that checks whether
  the saved rehearsal report exists, is fresh against the current product source
  revision, and exposes a first blocker plus next action.

The readiness report is currently available only through in-process builders and
embedded evidence summaries. Operators can save the rehearsal report with
`make ios-device-launch-rehearsal`, but there is no direct CLI or Make output for
`services/backend/.local/ios-device-launch-rehearsal-readiness.json`.

## Decision

Add an explicit handoff path for iOS device launch rehearsal readiness:

- Add a backend CLI command named `ios-device-launch-rehearsal-readiness`.
- Add a root Make target with the same name that writes
  `services/backend/.local/ios-device-launch-rehearsal-readiness.json`.
- Update `services/backend/scripts/write_ios_device_launch_rehearsal.sh` so the
  existing `make ios-device-launch-rehearsal` wrapper writes the readiness report
  immediately after the rehearsal report.

This keeps the current rehearsal report unchanged and adds a directly inspectable
readiness artifact for iOS deploy blockers, freshness, first blocker, and next
action.

## Non-Goals

- Do not run Xcode, sign an app, touch Keychain, or mutate global machine state.
- Do not call Meshy, OpenAI, Treatstock, or any paid/live provider.
- Do not change readiness classification logic.
- Do not make the final showcase ready; missing iOS deploy values and provider
  keys remain real blockers.

## Testing

Add RED/GREEN coverage for:

- CLI writes `ios_device_launch_rehearsal_readiness_report` and returns its
  result exit code.
- Makefile exposes `ios-device-launch-rehearsal-readiness` with the expected
  `.local` output path.
- The iOS rehearsal wrapper calls the readiness CLI after writing the rehearsal
  report and prints the saved readiness path.
- The wrapper still syncs final demo launch after the rehearsal readiness step.
