# P0.579 Final Refresh Consent Action Design

## Context

`final-showcase-readiness.json` correctly routes the remaining live Meshy/OpenAI
work through the guarded command:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make final-acceptance-configured; rerun make live-provider-evidence
```

However, `final-local-report-refresh.json` can still promote the child
`final_configured_evidence_plan` blocker directly:

```text
make backend-evaluate-3d-configured; rerun make final-configured-evidence-plan
```

That child command may call a live provider and has cost risk. Surfacing it as a
top-level refresh action is inconsistent with the explicit live-provider consent
policy and can mislead the operator.

## Goal

When final-local refresh promotes a `final_configured_evidence_plan` blocker
that requires live provider consent, the promoted command must use the guarded
live-provider handoff chain instead of a bare configured evaluation command.

## Approach

- Keep `final_configured_evidence_plan` itself unchanged; it can still explain
  which specific configured evidence slot is missing.
- In `final_local_report_refresh.py`, normalize consent-required live-provider
  hints from `final_configured_evidence_plan_report` to the same guarded live
  provider consent action used by final showcase readiness.
- Preserve detail, classification, and validation command so the report remains
  diagnosable.

## Non-Goals

- Do not run Meshy, OpenAI, Treatstock, Xcode signing, or any live-cost command.
- Do not mark live provider evidence ready.
- Do not change configured evidence planning order.
- Do not write secrets or modify ignored resource files.

## Verification

- Add a RED test for a child `final_configured_evidence_plan_report` with a
  consent-required 3D evaluation blocker.
- Confirm the final-local step, top-level first blocker, next action, and
  operator actions use the guarded live-provider handoff.
- Run focused final-local refresh tests and lint.
- Refresh safe local reports and confirm `final-local-report-refresh.json`
  no longer surfaces bare `make backend-evaluate-3d-configured` as the first
  action.
