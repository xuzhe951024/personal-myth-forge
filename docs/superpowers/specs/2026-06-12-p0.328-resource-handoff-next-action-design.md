# P0.328 Resource Handoff Next Action Design

## Context

`final-showcase-readiness` still blocks on external iOS deploy evidence and
provider/key handoff. Most parent reports now expose machine-readable
`first_blocker` and `next_action`, but `resource_handoff_report` still only
emits a human-oriented `operator_actions` array.

That makes the provider/key handoff lane less precise for the final iPhone
showcase, because consumers must infer the first actionable step from a list.

## Decision

Add top-level `first_blocker` and `next_action` to `resource_handoff_report`.

- `first_blocker` is the first backend or iOS item with `missing` or `blocked`
  status, ordered by the existing report item order.
- `next_action` mirrors that blocker and adds `source: "first_blocker"`.
- Backend secret items use `provide <ID> in final-resources.env` and validate
  with `make final-resources-preflight`.
- iOS deploy items use the existing mobile deploy handoff action and validate
  with `make mobile-deploy-preflight`.
- Ready or partial reports without missing/blocked items return `null` for both
  fields.

## Mobile Contract

The Swift `ResourceHandoffReport` model decodes both optional fields. The Final
Launch summary shows the `next_action` row before falling back to the first
legacy operator action. This keeps the iPhone operator panel aligned with the
backend report contract.

## Non-Goals

- Do not read or write real keys.
- Do not run live Meshy/OpenAI/Treatstock calls.
- Do not mutate Xcode signing, Keychain, or global machine state.
- Do not change `final-showcase-readiness` prioritization.

## Acceptance

- Focused backend tests prove blocked resource handoff exposes a sanitized
  `first_blocker` and `next_action`, and a fully configured handoff returns
  `null` for both.
- Focused Swift contract tests prove the model decodes and summary displays the
  resource handoff next action.
- `make resource-handoff` still exits blocked without local secrets, but writes
  `.local/resource-handoff.json` with top-level action fields.
