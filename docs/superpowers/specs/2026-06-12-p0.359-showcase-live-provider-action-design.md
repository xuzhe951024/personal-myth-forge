# P0.359 Final Showcase Live Provider Action Design

## Context

The final showcase readiness ledger maps the full iOS demo objective to local,
live-provider, and device evidence. The 3D and NPC rows currently become
`partial` when local proofs are ready but live provider evidence is not ready,
and both rows point at the generic `make live-provider-evidence` command.

The live provider evidence report already exposes a more precise
`next_action`. In the current local state that action is `make provider-handoff`,
with `make live-provider-evidence` as the validation command. The final showcase
ledger should reuse that child action instead of making the operator infer it.

## Goal

When local 3D or NPC Agent proof is ready but live provider evidence remains
partial or blocked, the final showcase capability row should surface the live
provider evidence report's concrete `next_action`.

## Approach

Add a small metadata helper in `final_showcase_readiness.py` that reads the
child report's `next_action.command`, `next_action.validation_command`, and
`next_action.requires_live_provider_consent` fields. Use it from:

- `game_asset_3d_generation`
- `ai_agent_npc`

If the child report does not include a next action, keep the current fallback:
`make live-provider-evidence`.

## Data Contract

Partial 3D/NPC rows should include:

- `command`: concrete child next action command, such as `make provider-handoff`
- `validation_command`: validation command from the child report, such as
  `make live-provider-evidence`, when present
- `requires_live_provider_consent`: boolean from the child report when present

Ready rows remain unchanged except for using the same fallback-safe metadata
path when useful. Local-not-ready rows remain blocked on their local evaluation
commands.

## Safety

This iteration is read-only with respect to external systems. It does not run
Meshy, OpenAI, Treatstock, Xcode signing, Keychain writes, or global
configuration. It only changes the saved readiness report shape produced from
existing local evidence.

## Testing

Add a focused regression test to `tests/test_final_showcase_readiness.py` that
builds a report with local 3D and NPC evaluation ready, monkeypatches the live
provider evidence child report to expose a concrete `next_action`, and asserts
both 3D and NPC rows inherit command, validation command, and live-provider
consent metadata.
