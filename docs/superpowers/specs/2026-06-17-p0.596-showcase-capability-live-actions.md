# P0.596 Showcase Capability Live Actions

## Context

Final showcase readiness now reads the configured evidence plan and can use its
guarded live-provider next action. The current capability rows still reuse the
configured plan's first blocker for every live-provider partial row.

That is correct for the top-level first blocker, but it is less useful when the
operator inspects a specific capability. The NPC row should point at the configured
NPC evaluation step while the 3D row points at the configured 3D evaluation step.

## Decision

When live provider evidence is not ready, final showcase capability rows should
prefer a configured evidence plan operator action matching the capability:

- `game_asset_3d_generation` -> `make backend-evaluate-3d-configured`
- `ai_agent_npc` -> `make backend-evaluate-npc-configured`
- `provider_key_handoff` -> the configured evidence plan's first guarded next action

All commands must keep the existing live-provider consent gate.

## Acceptance

- The 3D capability row points at the guarded configured 3D evaluation command.
- The NPC capability row points at the guarded configured NPC evaluation command.
- The provider/key handoff row continues to point at the configured plan's first
  guarded next action.
- No report generation runs Meshy, OpenAI, Treatstock, Xcode signing, or paid actions.
