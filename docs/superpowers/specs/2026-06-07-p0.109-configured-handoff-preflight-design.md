# P0.109 Configured Handoff Preflight Design

## Context

P0.108 added `make final-rehearsal-local`, which writes the local/no-key readiness reports needed for the iPhone final showcase. The remaining final-demo resource gap is the configured handoff path: after the operator fills Meshy/OpenAI/print/iOS values, there should be one safe report that shows whether the resource bundle and current ignored local config are ready for the final configured acceptance flow.

Today the information exists, but it is split across `final-resources-preflight`, `provider-handoff`, `resource-handoff`, `final-demo-launch --mode configured`, and `ios-deploy-runbook --mode configured`. P0.109 combines those read-only views into one saved, sanitized packet.

## Goal

Add a project-root configured handoff preflight command that writes `services/backend/.local/final-configured-preflight.json` without calling live providers, applying secrets, starting servers, running Xcode, or mutating global state.

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, Sculpteo, or any live provider.
- Do not run configured final acceptance automatically.
- Do not run `final-apply-resources`; applying user-provided keys remains an explicit separate command.
- Do not write `services/backend/.env` or `apps/mobile/ios/Config/Deployment.local.xcconfig`.
- Do not accept Apple licenses, touch keychain, run signing, or run Xcode.

## Report Contract

New report kind: `final_configured_preflight_report`.

Top-level fields:

- `kind`: fixed report kind.
- `status`: `ready` only when the final resources preflight is ready, core real providers are configured, the configured final launch report has no missing/blocked configured resources, and the configured iOS deploy runbook is not blocked.
- `summary`: counts `ready`, `missing`, `blocked`, `manual`, `optional`, and `live` statuses across the embedded reports.
- `final_resources_preflight`: existing sanitized report for `services/backend/.local/final-resources.env`.
- `provider_handoff`: provider readiness report with `core_real_ready` and missing backend-only env keys.
- `resource_handoff`: existing backend/iOS resource status from ignored local config.
- `configured_final_launch`: `final-demo-launch --mode configured` report.
- `configured_ios_deploy_runbook`: `ios-deploy-runbook --mode configured` report.
- `operator_actions`: deduped next actions from the embedded reports plus explicit commands for apply/resources/live acceptance.
- `commands`: exact safe next commands, including `make final-resources-preflight`, `make final-apply-resources`, `make final-configured-preflight`, `make backend-device-demo`, `make mobile-deploy-preflight`, and the consent-gated configured final acceptance command.
- `safety`: false for provider calls, writes, command execution from API/report builder, global mutation, Xcode/signing, keychain writes, provider secrets in report, raw media in report, local paths in report, and payment links in report.

The report may exit `2` while still writing a usable JSON packet. Exit `0` means ready for the next configured handoff step. Exit `1` is reserved for unusable runtime errors.

## CLI And Make

New CLI:

```bash
cd services/backend
uv run python -m myth_forge_api.cli final-configured-preflight \
  --repo-root ../.. \
  --output .local/final-configured-preflight.json
```

New Make target:

```bash
make final-configured-preflight
```

The Make target runs the CLI above and writes only the ignored JSON report.

## iOS/Operator Surface

The iPhone app already reads `/v1/final-demo-launch?mode=configured`; P0.109 does not add a new mobile endpoint. The configured preflight packet is a Mac-side operator handoff artifact that makes final key handoff reviewable before live acceptance.

Source gates should ensure the new CLI, Make target, report builder, and tests exist. Documentation should explain that this is the final safe preflight after keys are filled, not a live-provider validation run.
