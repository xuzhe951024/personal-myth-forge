# P0.87 Final Resource Checklist Design

Date: 2026-06-06
Branch: `codex/p0.87-final-resource-checklist`

## Context

The backend final resource preflight already returns a safe, redacted `items`
array. Each item tells the operator whether a final handoff value is ready,
missing, optional, or blocked:

- provider keys such as `MESHY_API_KEY` and `OPENAI_API_KEY`
- print settings such as `PRINT_PROVIDER` and `TREATSTOCK_API_KEY`
- iOS deployment values such as `DEVELOPMENT_TEAM`,
  `PRODUCT_BUNDLE_IDENTIFIER`, and `PMF_BACKEND_BASE_URL`
- launch/storage values such as `PMF_FINAL_LAUNCH_MODE`,
  `CAPTURE_STORAGE_DIR`, and `MYTH_SESSION_STORAGE_DIR`

The iPhone app currently decodes only the resource preflight summary and operator
actions. That is enough to say "copy/provide values", but it does not let the
operator quickly see which final API/key/deploy slots are missing without
switching back to the Mac terminal.

## Approaches Considered

### A. Decode resource items and add concise mobile checklist rows

Add `FinalResourcesPreflightItem` to the Swift model, derive a
`resourceChecklistRows` array in `FinalLaunchMobileSummary`, and render it in
`FinalLaunchStatusView`. Show missing/blocked required values first, then a
compact all-ready row when the file is ready.

This uses existing backend data, keeps secrets redacted, and makes the final
handoff more inspectable on the phone without adding commands or writes.

### B. Expand only `resourceActions`

Fold item names into the existing `resourceActions` strings. This is smaller but
mixes "what is missing" with "what to do next", making the launch panel harder to
scan.

### C. Add a new backend endpoint

Expose a dedicated resource checklist endpoint. This is unnecessary because the
data is already present in `/v1/final-demo-launch`.

## Decision

Use approach A.

The Swift model will decode `items` with only safe fields:

- `id`
- `status`
- `required`
- `configured`
- `redacted`
- optional `classification`
- optional `normalizedValue`

No secret values are added to the model, and the UI never displays raw values.

`FinalLaunchMobileSummary` will gain `resourceChecklistRows: [String]`. The
builder will map final resources preflight items to short, sanitized lines:

- missing required values: `MESHY_API_KEY: missing required`
- blocked values: `PMF_BACKEND_BASE_URL: blocked loopback_url`
- optional/configured hints: only shown when no required missing/blocked rows
  need attention
- ready file: `Required final resources ready.`

`FinalLaunchStatusView` will show a `Resource Checklist` group before the
existing `Resources` actions group.

## Data Flow

```text
final-resources.env
  -> final_resources_preflight_report.items
  -> /v1/final-demo-launch
  -> FinalDemoLaunchReport.finalResourcesPreflight.items
  -> FinalLaunchMobileSummary.resourceChecklistRows
  -> FinalLaunchStatusView "Resource Checklist"
```

## Safety

- Provider secrets remain backend-only.
- The app decodes only item metadata, never raw resource values.
- Secret-bearing IDs may be displayed, but values are never displayed.
- All rows pass through the existing sanitizer before rendering.
- The app remains read-only; it does not apply resources or run shell commands.

## Test Plan

- Swift core contract tests:
  - decode `final_resources_preflight.items`
  - show missing provider and iOS deploy slots in `resourceChecklistRows`
  - show blocked loopback backend URL classification
  - show ready checklist copy when required resources are ready
  - redact unsafe text from item classifications or ids
- Swift project source checks:
  - enforce item model, summary rows, view rendering, root source gate evidence,
    and contract tests
- Backend iOS showcase acceptance:
  - require source evidence for item decoding and mobile checklist rendering
- Visual regression:
  - static 390x844 screenshot with `Resource Checklist`, missing
    `MESHY_API_KEY`, missing `OPENAI_API_KEY`, and blocked
    `PMF_BACKEND_BASE_URL`

## Out of Scope

- No backend report shape changes.
- No writes to `services/backend/.local/final-resources.env`.
- No provider calls.
- No iOS signing, Xcode, Apple SDK license, or global system changes.
