# P0.629 Local Refresh Live Action Completeness Spec

## Goal

Make `final-local-report-refresh` preserve both configured live evidence actions when
the configured evidence plan says Meshy 3D and OpenAI NPC evidence still need explicit
cost consent.

## Problem

The configured evidence plan and higher-level reports can already surface two
granular live provider actions:

- `backend-evaluate-3d-configured`
- `backend-evaluate-npc-configured`

`final-local-report-refresh` currently promotes only the first blocker command from
the configured evidence step. That makes the local refresh report look executable but
causes the operator handoff to omit the NPC live evidence step.

## Acceptance

- `final-local-report-refresh.operator_actions` keeps both configured 3D and NPC live
  provider consent actions when the saved configured evidence plan exposes both.
- The broader guarded configured acceptance action remains hidden while granular
  prerequisite evidence actions exist.
- No live providers are called, no secrets are written, and no global/Xcode state is
  mutated by the report generation path.
