# P0.591 Handoff Index Live Consent Action Design

## Goal

Make `final_handoff_index_report.operator_actions` use the same guarded live-provider consent command as the other final handoff reports.

## Context

The final showcase, final demo launch, local refresh, closure, configured evidence, and external ledger reports now route live Meshy/OpenAI evidence through:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make final-acceptance-configured; rerun make live-provider-evidence
```

`final-handoff-index` still emitted the older review-only text:

```text
approve live provider cost review before make final-acceptance-configured; --allow-live-provider-calls consent required
```

That text is safe, but it is not directly executable and can leak into iOS rehearsal readiness as a weaker handoff.

## Design

Keep the configured acceptance command itself as `make final-acceptance-configured`.

For the final handoff index only:

- emit a guarded live-provider operator action for the `live_acceptance` lane
- normalize the older cost-review action into the guarded command when importing saved source report actions
- do not classify the guarded final-acceptance command as a generic provider handoff for priority ordering, so safe configured preflight actions stay ahead of live calls when local prerequisites are missing

## Scope

In scope:

- `myth_forge_api.final_handoff_index`
- focused RED/GREEN coverage in `test_final_handoff_index.py`
- configured acceptance command handoff regression
- roadmap entry for P0.591

Out of scope:

- changing other reports that intentionally use the shared cost-review text
- live provider calls
- Xcode, signing, or physical-device execution
- changing action groups in downstream reports

## Verification

This phase is valid when:

- RED proves `final-handoff-index` still emits the old cost-review text
- GREEN proves the guarded command is emitted and the old text is absent
- configured preflight remains ahead of live acceptance in missing-prerequisite ordering
- refreshed `.local/final-handoff-index.json` uses the guarded live-provider command
