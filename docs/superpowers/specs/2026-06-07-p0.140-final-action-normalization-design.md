# P0.140 Final Action Normalization Design

## Context

P0.139 added `make final-resource-init` as the safe way to create the ignored
`services/backend/.local/final-resources.env` file. The current source builders
now emit that command when final resources are missing.

`make final-showcase-readiness` can still promote operator actions from older
saved `.local` reports. A stale `ios-device-launch-rehearsal` report may contain
the previous guidance:

```text
copy services/backend/final-resources.env.example to services/backend/.local/final-resources.env
```

That stale text is not a secret leak, but it weakens the final handoff because
the operator sees an outdated manual copy instruction instead of the current
guarded initialization command.

## Goal

Normalize legacy final-resource copy guidance when final showcase readiness
promotes nested operator actions. The final readiness report should direct the
operator to:

```text
run make final-resource-init
```

even if a saved nested report still contains the old copy text.

## Non-Goals

- Do not rewrite ignored `.local` JSON evidence files.
- Do not hide other stale-report signals or freshness blockers.
- Do not call providers, run Xcode, apply resources, start servers, or mutate
  global machine state.
- Do not change final resource validation, apply preview, or writer semantics.

## Approaches Considered

1. Refresh `.local` reports automatically before reading final showcase
   readiness. This would make the stale action disappear, but it would turn a
   read-only ledger into an action runner and could mask which saved evidence was
   stale.
2. Filter out every action from stale saved reports. This would avoid outdated
   instructions, but it would also remove useful nested blockers such as iOS
   deploy config and final handoff index actions.
3. Normalize the specific legacy final-resource copy instruction while keeping
   other promoted actions intact.

Approach 3 is the least disruptive and matches the existing P0.139 operator
contract.

## Design

Add a small action normalization step in
`services/backend/src/myth_forge_api/final_showcase_readiness.py`.

The normalizer maps any action containing both:

- `services/backend/final-resources.env.example`
- `services/backend/.local/final-resources.env`

to:

```text
run make final-resource-init
```

All other action strings pass through unchanged. Existing `_dedupe` behavior then
collapses the normalized command with the current preflight/apply-preview actions
if both appear.

## Testing

Add a RED test to `services/backend/tests/test_final_showcase_readiness.py` that
writes an `ios-device-launch-rehearsal` fixture with the legacy copy action and
asserts:

- `run make final-resource-init` appears in `operator_actions`.
- `services/backend/final-resources.env.example` does not appear in the final
  readiness report text.
- Existing priority actions are still promoted.

Then implement the normalizer and rerun the focused final showcase tests.

## Safety

The change is report-only. It reads and transforms action strings in memory,
does not write config files, does not modify ignored saved reports, and does not
perform external or global actions.
