# P0.424 Refresh Step Source Reports Design

## Goal

Preserve child report `source_reports` inside `final_local_report_refresh.steps` so the top local refresh ledger can trace final iPhone closure evidence down through the closure packet, iOS device evidence bundle, and mobile evidence action bundles.

## Current State

`final_launch_closure_packet_report` now preserves nested source reports for the iOS device evidence bundle. `final_local_report_refresh` runs the closure packet as one refresh step and copies selected step fields into `steps` and `steps_by_id`. It already copies a step report's `device_action_bundle`, but it drops a step report's `source_reports`, so the top refresh ledger loses the nested evidence trace.

## Approach

Extend `final_local_report_refresh._run_step()` to copy `report["source_reports"]` into `step_result["source_reports"]` when the child report exposes a dictionary. This keeps the propagation generic, local, and consistent with the existing `device_action_bundle` handling. Also add a `device_action_bundle` to the refresh-generated safe Xcode evidence snapshot so nested closure evidence has an action bundle even when the refresh intentionally avoids running Xcode.

## Safety

This iteration only carries already-sanitized report summaries into a higher-level ignored/local report. It does not run Xcode signing, write provider env files, call live providers, or place print orders. The full refresh report still passes through the existing sanitizer before returning or writing JSON.

## Testing

Add a focused contract test that runs `run_final_local_report_refresh()` and asserts that `steps_by_id["final_launch_closure_packet"].source_reports.ios_device_evidence_bundle.source_reports` includes the mobile deploy and Xcode device action bundles. The Xcode bundle should point to the safe manual action for running `make mobile-xcode-build-evidence` outside refresh. The test also verifies no repo path or synthetic secret appears in the refresh report.
