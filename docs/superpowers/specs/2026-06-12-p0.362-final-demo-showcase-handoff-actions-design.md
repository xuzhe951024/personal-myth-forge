# P0.362 Final Demo Showcase Handoff Actions Design

## Context

`final-demo-launch-local` embeds `final_showcase_readiness`, and the embedded report already knows the concrete handoff actions for live provider evidence and Treatstock print fulfillment. The top-level `final_demo_launch_report.operator_actions` still only includes the launch `next_action` plus the general operator checklist, so a client or operator that reads only the top-level launch report can miss the game-asset/NPC live provider and print handoff work that remains central to the final iOS showcase goal.

## Decision

Promote a bounded set of final showcase operator actions into the top-level final demo launch `operator_actions`.

Only actions that directly mention the live provider handoff or print fulfillment handoff are promoted:

- `provider-handoff`
- `live-provider`
- `live_provider`
- `Treatstock`
- `print-fulfillment`
- `print_fulfillment`

The launch report keeps its existing first action priority. Final showcase actions are inserted immediately after the first concrete launch action and before the general checklist, then normalized through the existing final action dedupe logic. The top-level action cap becomes a named limit of 12 so the two core showcase handoff actions are visible without dropping existing resource handoff actions.

## Scope

In scope:

- Extend final demo launch report assembly to pass `final_showcase_readiness` into top-level action selection.
- Add a small selector for final showcase handoff actions.
- Add a regression test proving live provider and print handoff actions are promoted while unrelated embedded showcase actions are not.
- Refresh `iteration-roadmap.md` with the P0.362 entry.

Out of scope:

- Running Meshy, OpenAI, Treatstock, Xcode, signing, or physical-device checks.
- Changing readiness status, first blocker selection, provider configuration, or mobile UI.
- Expanding all embedded final showcase actions into the launch report.

## Testing

Use TDD:

1. Add a failing focused backend test in `services/backend/tests/test_final_demo_launch.py`.
2. Confirm the test fails because the top-level final demo launch report is missing the final showcase handoff actions.
3. Implement the selector and report wiring.
4. Re-run the focused test and related final demo launch tests.
5. Run backend lint plus `git diff --check`.

## Safety

This phase only reads and writes repo files and ignored local report evidence during verification. It does not write secrets, call live providers, place print orders, mutate global environment, touch Keychain, or perform Xcode signing.
