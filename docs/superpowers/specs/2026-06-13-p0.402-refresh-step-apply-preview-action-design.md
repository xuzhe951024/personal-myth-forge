# P0.402 Refresh Step Apply Preview Action Design

## Context

The final iPhone showcase remains blocked by external iOS deployment evidence
and provider credentials. Recent phases made top-level operator actions more
copy-pasteable, but the refreshed local report still exposes one old step-level
command:

```text
steps_by_id.final_resource_apply_preview.next_action.command =
rerun make final-resource-apply-preview before applying resources
```

The top-level `operator_actions` already normalize this to
`make final-resource-apply-preview`, but a reader drilling into the failed step
still sees a sentence-shaped command. That is weaker handoff evidence for the
provider/key path because the step-level action cannot be copied directly into a
terminal.

## Goal

Make the final resource apply preview report expose the canonical
`make final-resource-apply-preview` command for its own next action and for
downstream step-level refresh reports.

## Scope

- Change only `myth_forge_api.final_resource_apply_preview`,
  `myth_forge_api.final_local_report_refresh`, and their focused tests.
- Keep the existing top-level `operator_actions` behavior: preview before apply,
  no `make final-apply-resources` while preview is blocked.
- Do not write `.env`, `Deployment.local.xcconfig`, provider keys, signing
  settings, or print order state.
- Do not run live providers, Xcode signing, or external cost actions.

## Design

Replace the old sentence-shaped apply preview action with the canonical make
target at the source report and in the refresh layer's special blocked-preview
step override. The report remains blocked until the same missing resource values
are supplied, but the command becomes a terminal-ready action:

```text
make final-resource-apply-preview
```

The existing final-local-report refresh projection already appends validation
commands when a `validation_command` is present. This phase should keep that
behavior unchanged. It only prevents stale text from leaking into
`steps_by_id.final_resource_apply_preview.next_action.command`.

## Tests

Update focused backend tests:

- `test_apply_preview_operator_actions_use_canonical_preview_command` verifies
  the apply preview source report contains `make final-resource-apply-preview`
  and no old `rerun ... before applying resources` string.
- `test_final_local_report_refresh_operator_actions_gate_apply_behind_preview`
  verifies the refreshed step-level `next_action.command` is also canonical,
  while top-level actions still contain the preview root and still exclude
  `make final-apply-resources`.

Focused verification:

```bash
cd services/backend && uv run pytest \
  tests/test_final_resource_apply_preview.py::test_apply_preview_operator_actions_use_canonical_preview_command \
  tests/test_final_local_report_refresh.py::test_final_local_report_refresh_operator_actions_gate_apply_behind_preview -q
```

Report verification:

```bash
make final-local-report-refresh
jq '.steps_by_id.final_resource_apply_preview.next_action.command' \
  services/backend/.local/final-local-report-refresh.json
```

Expected: the command is `make final-resource-apply-preview`; the refresh still
exits `2` until external keys and physical iPhone evidence are provided.
