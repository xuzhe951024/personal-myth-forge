# P0.585 Final Refresh Showcase Settings Design

## Context

`final-local-report-refresh` rewrites the saved final reports used by the Mac-side and iPhone handoff panels. Its `final_showcase_readiness` step was hard-coded to `LOCAL_SETTINGS`.

That is safe, but it can overwrite a correctly configured `final-showcase-readiness.json` with a local-provider view after the operator has already filled backend provider keys. The resulting report can incorrectly show provider/key handoff blockers even though `services/backend/.env` is configured and `resource-handoff` is nonblocking.

## Decision

Keep `final-local-report-refresh` safe and read-only, but make the `final_showcase_readiness` step load the current backend settings:

- Use `load_settings()` for `build_final_showcase_readiness_report`.
- Keep local deterministic 3D and NPC evaluation steps on `LOCAL_SETTINGS`.
- Keep configured evidence generation behind existing live provider consent gates.

## Acceptance

- The final showcase step in `final-local-report-refresh` receives loaded backend settings.
- Safe refresh still performs no live provider calls, Xcode signing, keychain writes, or global mutations.
- `final-local-report-refresh` no longer overwrites configured final showcase readiness with local-only provider settings.
- Focused final-local report refresh tests remain green.
