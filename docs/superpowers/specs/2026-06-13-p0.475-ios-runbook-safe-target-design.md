# P0.475 iOS Runbook Safe Target Design

## Context

`make ios-deploy-runbook` still calls the backend CLI directly. In the current local environment it writes `services/backend/.local/ios-deploy-runbook-local.json`, reports the expected blocked state for missing `DEVELOPMENT_TEAM` and iPhone-reachable backend configuration, then exits `2`, which makes Make fail.

The project already has `services/backend/scripts/write_ios_deploy_runbook_local.sh`. The wrapper accepts report exit codes `0` and `2`, writes the same ignored JSON evidence, and returns `0` for usable blocked evidence. `ios-deploy-runbook-local` already uses it.

## Decision

Make the canonical `ios-deploy-runbook` target use the same safe wrapper as `ios-deploy-runbook-local`. Keep both target names so existing reports, iPhone panels, and operator docs remain compatible.

## Scope

- Change `Makefile` so `ios-deploy-runbook` and `ios-deploy-runbook-local` both invoke `services/backend/scripts/write_ios_deploy_runbook_local.sh`.
- Update Swift ProjectChecks to validate the wrapper-backed target instead of the old raw CLI text.
- Update backend source-gate tests and complete fixtures so acceptance requires the canonical target to use the safe wrapper.
- Update roadmap with P0.475.

## Non-Goals

- Do not write `DEVELOPMENT_TEAM`, bundle ID, or `PMF_BACKEND_BASE_URL`.
- Do not run Xcode signing, Keychain writes, or device operations.
- Do not change the report semantics: missing deploy configuration must remain blocked.

## Verification

- RED: a dry-run test fails while `make -n ios-deploy-runbook` prints the raw CLI command.
- GREEN: `make ios-deploy-runbook` returns `0` and writes blocked `ios-deploy-runbook-local.json`.
- GREEN: Swift ProjectChecks, backend source-gate tests, lint, and whitespace checks pass.
