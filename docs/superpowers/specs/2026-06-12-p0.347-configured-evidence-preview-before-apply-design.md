# P0.347 Configured Evidence Preview Before Apply Design

## Context

`final-configured-evidence-plan.json` is an operator report for the configured
provider evidence path. In the missing-resource path, its top-level
`operator_actions` currently include both:

```text
make final-resource-apply-preview
make final-apply-resources
```

The direct writer remains a valid command after the preview is reviewed, but the
top-level configured evidence handoff should first route the operator through
the read-only preview.

## Goal

Make `final-configured-evidence-plan` prefer the read-only final resource apply
preview over direct final resource writer actions in top-level
`operator_actions`.

## Non-Goals

- Do not remove `make final-apply-resources` from `commands`.
- Do not change the configured evidence step list or writer command semantics.
- Do not change live-provider consent behavior.
- Do not execute live provider calls, Xcode signing, Keychain writes, resource
  writers, backend `.env` writes, iOS deploy config writes, or print orders.

## Design

Keep the ordered configured evidence steps unchanged. After the existing
operator-action normalization and deduplication, add a small preference filter:
if the top-level action roots include `make final-resource-apply-preview`, drop
exact roots equal to `make final-apply-resources`.

This keeps the command list and detailed step table honest while preventing the
operator-facing action list from skipping the dry-run preview gate.
`configured-live-evidence-bundle` consumes the plan, so it should inherit this
same top-level preview-first behavior while preserving its command list.

## Acceptance Criteria

- Missing-resource configured evidence plan includes
  `make final-resource-apply-preview` in top-level `operator_actions`.
- The same top-level action list does not include direct
  `make final-apply-resources` while preview is present.
- `commands` still includes `make final-apply-resources`.
- `configured-live-evidence-bundle` inherits the same top-level action behavior.
- Existing consent-required behavior remains unchanged when resources are ready.
- Focused configured evidence plan tests, operator action tests, lint, diff
  checks, and backend tests pass.
