# P0.364 Final Showcase Print Handoff Filter Design

## Context

`final_demo_launch.operator_actions` now filters final showcase print handoff actions correctly, but `final_showcase_readiness.operator_actions` itself still includes a validation-only action:

```text
make visual-regression-local; rerun make print-fulfillment-readiness
```

That action is a visual regression refresh with print readiness as a validation command. It is not the Treatstock quote handoff needed to finish the final print fulfillment path. Clients that read `final_showcase_readiness` directly can still surface the wrong print action.

## Decision

Apply the same semantic boundary at the final showcase readiness layer:

- Keep actions that represent the real Treatstock or print quote handoff.
- Keep live provider handoff actions.
- Drop validation-only print readiness actions when they do not mention Treatstock, `print-quote`, `print_quote`, or `/v1/print-quotes`.

This preserves the user-facing final showcase checklist while making the print handoff wording consistent across final showcase and final demo launch.

## Scope

In scope:

- Add a regression test to `test_final_showcase_readiness.py` proving validation-only print readiness actions are not promoted.
- Add a small selector/filter in `final_showcase_readiness.py` before the final operator action limit is applied.
- Add a P0.364 roadmap entry.

Out of scope:

- Changing print fulfillment readiness reports.
- Changing Treatstock API integration or running live quote calls.
- Changing iOS signing, Xcode, device, Meshy, OpenAI, or Treatstock external setup.

## Testing

Use TDD:

1. Change the existing nested-action test expectation and confirm it fails.
2. Implement the filter.
3. Run the focused test, full final showcase readiness tests, final demo launch tests, lint, and diff checks.

## Safety

This phase only edits tracked repo files and may refresh ignored `.local` evidence after merge. It does not write secrets, call live providers, submit print orders, mutate global state, touch Keychain, or perform signing.
