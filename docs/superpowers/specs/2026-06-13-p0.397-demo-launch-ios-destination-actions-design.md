# P0.397 Demo Launch iOS Destination Actions Design

## Context

The final iOS showcase still has external blockers: real provider keys, Treatstock quote evidence, Apple development team settings, and a physical iPhone-reachable backend URL. The reports should therefore make the next operator actions precise and copy-pasteable.

`final-launch-closure-packet` and `final-external-action-ledger` now surface iOS deploy resource actions as `Deployment.local.xcconfig` / `make mobile-deploy-preflight` handoffs. `final-demo-launch-local` still includes top-level actions like:

```text
provide DEVELOPMENT_TEAM in final-resources.env; rerun make final-resources-preflight
```

That is weaker for the iPhone deployment path because the user-facing deploy handoff is the ignored iOS config file and mobile preflight, not the backend provider resource preflight.

## Goal

Make `final-demo-launch` operator actions project iOS deploy resource requirements to their real iOS config destination:

- `provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`
- `provide PRODUCT_BUNDLE_IDENTIFIER in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`
- `provide PMF_BACKEND_BASE_URL in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`

Backend provider secrets should continue to point at `final-resources.env`.

## Scope

- Change only `final_demo_launch` operator checklist projection.
- Reuse `final_resource_requirements.requirements[].destination` to distinguish backend and iOS resource actions.
- Keep final resources preflight behavior unchanged.
- Keep local/configured mode status calculations unchanged.
- Keep live provider calls, Xcode, signing, Keychain, resource writers, and paid provider actions out of this phase.

## Design

Add a small normalization step inside `final_demo_launch._operator_checklist()`:

1. Collect the current preflight, requirements, resource report, and phase actions as before.
2. Before deduplication, rewrite resource actions whose requirement destination is `apps/mobile/ios/Config/Deployment.local.xcconfig`.
3. Preserve backend actions such as `provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight`.
4. For iOS destination actions, rerun `make mobile-deploy-preflight` because that gate validates the deploy config and backend URL for iPhone use.
5. Keep existing dedupe and writer-preference behavior so the auto writer can still supersede lower-level manual team actions when it is the immediate next action.

This keeps the final demo report aligned with the current closure packet and device launch handoff without changing lower-level resource reports.

## Tests

Update `services/backend/tests/test_final_demo_launch.py`:

- `test_local_final_demo_launch_resource_actions_use_final_resources_preflight` should expect iOS deploy actions to target `Deployment.local.xcconfig` and should reject iOS deploy actions pointing to `final-resources.env`.
- `test_configured_final_demo_launch_drops_superseded_resource_team_action` should expect the configured demo report to use the same iOS destination-aware handoff.
- `_operator_actions` unit coverage should stop requiring the old resource-team action to suppress the Deployment.local action.
