# P0.452 Mobile External Ledger Backend Handoff Design

## Context

The backend final external action ledger now emits a complete backend-device-demo handoff in top-level `operator_actions`. The iPhone Final Launch summary still renders only `operatorActions.first` for the `External Actions` group.

In the current saved report, the backend handoff appears after resource, provider, print, and Xcode actions. Showing only the first action means the iPhone UI can hide the physical-device prerequisite:

```text
start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight
```

## Goal

Make the iPhone Final Launch `External Actions` rows show the backend-device-demo handoff whenever it is present in `FinalExternalActionLedgerReport.operatorActions`, while preserving the first operator action and existing redaction.

## Approaches Considered

1. Show the first operator action plus the first backend-device-demo action.
   - Pro: minimal UI change, keeps current first-action behavior, directly surfaces the missing device prerequisite.
   - Con: still hides other non-device actions unless they are already covered by groups.

2. Show the first three operator actions.
   - Pro: simple.
   - Con: backend-device-demo can still be below the first three actions.

3. Render every operator action.
   - Pro: complete.
   - Con: too noisy for a compact iPhone status panel and risks burying the actual next step.

Recommended approach: option 1. It is narrow and aligned with the current handoff gap.

## Design

Add a small helper in `FinalLaunchMobileSummary.swift` that receives the external ledger status and `operatorActions`.

When the ledger status is not `ready`, it should:

- include the first non-empty operator action, preserving existing behavior;
- include the first non-empty operator action containing `backend-device-demo`, if it differs from the first row;
- dedupe exact duplicate strings;
- rely on the existing final `rows.map(sanitize)` pass for privacy and unsafe text redaction.

Do not change backend JSON, iOS models, or visual layout components.

## Testing

Extend the MobileCore contract test fixture so `finalDemoLaunchReport()` can inject external ledger operator actions. Add a RED test where the first action is a resource handoff and the second action is the backend-device-demo handoff. The current summary should fail to show the backend handoff. After implementation, it should include both strings.

Verification:

```bash
swift run --package-path apps/mobile/ios PersonalMythForgeMobileCoreContractTests
swift run --package-path apps/mobile/ios PersonalMythForgeMobileProjectChecks
git diff --check
```

This phase does not build/sign with Xcode, write Keychain state, start the backend, call live providers, or place print orders.
