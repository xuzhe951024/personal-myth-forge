# P0.363 Final Demo Print Handoff Filter Design

## Context

P0.362 promoted final showcase handoff actions into top-level final demo launch `operator_actions`. Local evidence then showed the print selector was too broad: an action such as `make visual-regression-local; rerun make print-fulfillment-readiness` was promoted only because its validation command mentioned `print-fulfillment-readiness`.

That action is not a print quote or Treatstock handoff. It should stay out of top-level final demo launch actions.

## Decision

Keep live provider matching as-is, but narrow print handoff matching to actions that mention actual print quote handoff semantics:

- `Treatstock`
- `print-quote`
- `/v1/print-quotes`
- `print_quote`

Do not treat a generic `print-fulfillment-readiness` validation command as a handoff action by itself.

## Scope

In scope:

- Add a regression test where a final showcase report contains a real Treatstock action and a validation-only `print-fulfillment-readiness` action.
- Update `final_demo_launch.py` selector constants/helper so only the real print handoff is promoted.
- Add a P0.363 roadmap entry.

Out of scope:

- Changing print readiness behavior.
- Running Treatstock, Meshy, OpenAI, Xcode, signing, or physical-device checks.
- Changing final showcase readiness itself.

## Testing

Use focused TDD:

1. Confirm the new test fails with the current broad marker.
2. Narrow the selector.
3. Confirm the new test passes.
4. Run related final demo launch/showcase tests, backend lint, and diff checks.

## Safety

This phase only edits repo files and refreshes ignored local evidence. It does not write secrets, call live providers, place print orders, mutate global state, touch Keychain, or perform signing.
