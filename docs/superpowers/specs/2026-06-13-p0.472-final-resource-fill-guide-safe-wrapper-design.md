# P0.472 Final Resource Fill Guide Safe Wrapper Design

## Context

`make final-resource-fill-guide` is an operator handoff report for filling the ignored final resource file. In the current no-key state it writes usable local evidence:

- `services/backend/.local/final-resource-fill-guide.json`
- `services/backend/.local/final-resource-fill-guide.md`

The CLI exits `2` because required values such as `MESHY_API_KEY` are missing. The Make target currently calls the CLI directly, so Make fails even though the blocked report is expected and useful. Other final handoff report targets already use safe wrappers that accept `0` and `2` while preserving unexpected failures.

## Goal

Make `make final-resource-fill-guide` behave like the other safe final handoff report targets: write ignored JSON and Markdown evidence, accept a blocked report as usable, and return `0` for exit codes `0` or `2`.

## Scope

- Add `services/backend/scripts/write_final_resource_fill_guide.sh`.
- Change the `final-resource-fill-guide` Make target to call the wrapper.
- Extend wrapper tests and source acceptance so the final resource handoff path requires the wrapper.
- Keep `build_final_resource_fill_guide_report()` unchanged.

## Non-Goals

- Do not write `services/backend/.env`.
- Do not write `apps/mobile/ios/Config/Deployment.local.xcconfig`.
- Do not run live Meshy, OpenAI, Treatstock, Sculpteo, Xcode, signing, Keychain, or payment actions.
- Do not mark final resources ready without the required external values.

## Design

The wrapper follows the established safe report pattern:

```sh
mkdir -p services/backend/.local
cd services/backend
uv run python -m myth_forge_api.cli final-resource-fill-guide \
  --repo-root ../.. \
  --output .local/final-resource-fill-guide.json \
  --markdown-output .local/final-resource-fill-guide.md
```

If the CLI exits `0` or `2`, the wrapper prints:

- `accepted final resource fill guide exit code $status`
- `wrote services/backend/.local/final-resource-fill-guide.json`
- `wrote services/backend/.local/final-resource-fill-guide.md`

and exits `0`. Any other exit code is forwarded after a stderr message.

Source acceptance should verify:

- Makefile uses `write_final_resource_fill_guide.sh`.
- the wrapper calls `myth_forge_api.cli final-resource-fill-guide`.
- the wrapper writes both JSON and Markdown outputs.
- the wrapper accepts blocked exit code `2`.

## Verification

- RED: tests fail before the wrapper exists and before the Makefile/source gate use it.
- GREEN: focused tests pass after adding the wrapper and source requirements.
- `make final-resource-fill-guide` exits `0` in the current blocked state and writes both local evidence files.
- `make backend-lint`, `git diff --check`, and backend tests pass.
