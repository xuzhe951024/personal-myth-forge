# P0.215 Final Demo Launch Local Alias Design

## Goal

Add an explicit `make final-demo-launch-local` command for the local final demo launch report, then protect it with source gates and a checked-in 390x844 visual evidence artifact.

## Context

The local report already exists at `services/backend/.local/final-demo-launch-local.json`, and the current `make final-demo-launch` target writes that report with `--mode local`. Other rehearsal commands use explicit `*-local` names, so an operator can reasonably try `make final-demo-launch-local` and hit a missing target even though the underlying local workflow exists.

During the latest local rehearsal check, that exact command miss occurred. It is a small operator-friction bug rather than a provider, iOS signing, or runtime capability gap.

## Approaches Considered

1. **Recommended: add `final-demo-launch-local` as an alias and keep `final-demo-launch`.** This preserves backward compatibility while giving the operator a clear local command that matches `final-acceptance-local`, `ios-deploy-runbook-local`, and `final-local-report-refresh-local`.
2. **Rename `final-demo-launch` to `final-demo-launch-local`.** This is clearer but would churn existing docs, tests, scripts, and historical acceptance source gates.
3. **Only update documentation.** This documents the current target but does not fix the operator footgun.

## Scope

This iteration changes only local command ergonomics and evidence:

- Add `final-demo-launch-local` to the Makefile.
- Keep `final-demo-launch` as a compatibility alias or dependency.
- Update final rehearsal/source gates so the canonical local chain uses the explicit alias.
- Add a visual regression artifact proving the operator-facing local command exists.
- Update README and mobile README wording where they describe the local final demo report command.

No provider calls, Xcode commands, signing changes, resource writes, `.env` writes, deployment config writes, or global state changes are part of this change.

## Visual Artifact

Add `p0.215_final_demo_launch_local_alias` with a static 390x844 iPhone proof. Required visible tokens:

- `Final Demo Launch`
- `Local command ready`
- `make final-demo-launch-local`
- `final-demo-launch-local.json`
- `Local alias`
- `Compatibility`
- `No providers, Xcode, signing, or global mutation.`

The artifact must pass the same unsafe-text checks as the existing visual index.

## Self-Review

- Scope is a single operator-command ergonomics fix plus required visual evidence.
- The design keeps old command compatibility.
- The change moves the final local demo handoff closer to a deployable rehearsal workflow without pretending external keys or Xcode license blockers are solved.
- No global state or live provider action is introduced.
