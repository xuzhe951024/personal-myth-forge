# P0.177 Final Rehearsal Refresh Design

## Context

`final-local-report-refresh` is now the safest complete local evidence refresh
entry point. It writes the ignored final resource reports, local evaluation
reports, configured evidence bundle, final showcase readiness report, and final
launch closure packet.

`final-rehearsal-local` is still the operator-facing project-root rehearsal
command, but it only depends on the older local gates:
`backend-evaluate-local`, `visual-regression-local`, `final-acceptance-local`,
`final-demo-launch`, and `ios-deploy-runbook-local`. That means an operator can
run the canonical rehearsal and still leave the newest closure packet and
configured evidence refresh stale.

## Goal

Make `make final-rehearsal-local` refresh the same safe final local evidence
bundle that the final handoff depends on, without adding live provider calls,
device actions, Xcode/signing, keychain writes, or global configuration writes.

## Design

Add `final-local-report-refresh` as the last dependency of
`final-rehearsal-local`:

```make
final-rehearsal-local: backend-evaluate-local visual-regression-local final-acceptance-local final-demo-launch ios-deploy-runbook-local final-local-report-refresh
```

Keeping the existing dependencies preserves the current local smoke behavior and
operator mental model. Appending `final-local-report-refresh` makes the same
command also regenerate the current ignored final evidence set, including
`services/backend/.local/final-launch-closure-packet.json`.

The refresh target already owns report ordering and safety controls. This change
does not move report-building logic into Make, does not duplicate the refresh
sequence, and does not call any live provider, Xcode, signing, keychain, backend
env writer, or iOS deploy config writer.

Source acceptance will be tightened in three places:

- `services/backend/tests/test_resource_template_acceptance.py` fixture and
  acceptance checks require `final-local-report-refresh` in the
  `final-rehearsal-local` target.
- `services/backend/tests/test_ios_showcase_acceptance.py` Makefile source
  fixture requires the full target dependency string.
- `services/backend/tests/test_visual_regression_readiness.py` and
  `ios_showcase_acceptance.py` source requirements continue to prove
  `visual-regression-local` stays in the rehearsal chain while now also requiring
  the refresh dependency.

## Alternatives Considered

1. Replace all individual rehearsal dependencies with `final-local-report-refresh`.
   Rejected because the existing Make target is a stable operator command and
   still documents the local gates directly.

2. Tell operators to run `make final-local-report-refresh` after
   `make final-rehearsal-local`.
   Rejected because this keeps the stale-evidence footgun in the canonical
   rehearsal path.

3. Make `ios-device-launch-rehearsal` call `final-local-report-refresh`
   separately.
   Rejected because `ios-device-launch-rehearsal` already calls
   `final-rehearsal-local`; fixing the root rehearsal command is simpler and
   covers both direct and nested use.

## Acceptance

- `Makefile` defines `final-rehearsal-local` with
  `final-local-report-refresh` as the last dependency.
- Resource template acceptance fails if the fixture or source loses that
  dependency.
- iOS showcase source acceptance fails if the Makefile source requirement loses
  that dependency.
- Visual regression readiness acceptance continues to require
  `visual-regression-local` and now also protects the refresh dependency.
- `make -n final-rehearsal-local` shows `final-local-report-refresh` after the
  older local rehearsal gates.
- Focused backend acceptance tests pass.
- `make backend-lint` and `make backend-test` pass.

## Test Strategy

Use TDD:

1. Add failing acceptance assertions for `final-local-report-refresh` in the
   rehearsal target.
2. Run the focused acceptance tests and confirm they fail because the Makefile
   target and fixtures still omit the dependency.
3. Add the smallest Makefile and source fixture updates.
4. Re-run focused tests, dry-run `make -n final-rehearsal-local`, backend lint,
   and the full backend test suite before merging.

## Self-Review

- Placeholder scan: no TBD/TODO/open placeholders.
- Scope check: one Make target plus source acceptance fixtures; no runtime
  report builder, mobile UI, provider adapter, Xcode, signing, or resource
  writer behavior changes.
- Safety check: `final-local-report-refresh` remains the only new dependency and
  already writes ignored local reports without live calls or global mutation.
- Ambiguity check: dependency order, target string, tests, and verification
  commands are explicit.
