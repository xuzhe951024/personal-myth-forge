# P0.449 iOS Rehearsal Backend Handoff Design

## Goal

Make `ios-device-launch-rehearsal` preserve the backend-device-demo handoff that is already present in `final-demo-launch-local`.

## Context

The final iPhone showcase needs the backend to be reachable from the device before `make mobile-deploy-preflight` can pass. P0.448 added this local final demo action:

```text
start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight
```

After refreshing rehearsal evidence, `final-demo-launch-local.json` includes that action, but `ios-device-launch-rehearsal.json` does not surface it in top-level operator actions. The rehearsal report still starts with the deploy-config writer and provider/print handoffs, leaving the backend launch step hidden in a downstream source.

## Behavior

When a partial local source such as `final_demo_launch_local` includes a backend-device-demo handoff, `_local_rehearsal_operator_actions()` should keep it alongside existing provider and print handoffs.

The action should retain its source prefix inside the `final_rehearsal_local` sequence row, then normalize to the canonical top-level command through the existing `_top_level_operator_action()` path.

## Safety

This only changes report action selection. It must not start the backend, run iOS preflight, run Xcode, write signing config, call Meshy/OpenAI/Treatstock, write provider secrets, or create print orders.

## Alternatives Considered

- Add a hard-coded backend action in `ios-device-launch-rehearsal`: rejected because the action already exists in the final demo source and should be propagated.
- Prioritize backend actions above all other actions: rejected because the deploy writer remains the first concrete blocker when `DEVELOPMENT_TEAM` is missing.
- Extend the partial source handoff filter to keep backend-device-demo actions: chosen because it is local, preserves source provenance, and reuses existing normalization.

## Tests

Extend the existing `test_ios_device_launch_rehearsal_local_actions_include_partial_final_demo_handoff` case with the backend-device-demo action from `final_demo_launch_local`, and assert it is preserved after provider/print handoffs.
