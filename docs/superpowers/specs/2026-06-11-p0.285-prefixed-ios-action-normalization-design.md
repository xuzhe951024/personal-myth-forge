# P0.285 Prefixed iOS Action Normalization Design

## Context

The latest final showcase report now includes a concrete backend-device-demo
action, but nested iOS device reports can still contribute prefixed legacy
actions such as:

- `ios_device_launch_certificate: provide iOS deploy config and rerun mobile deploy preflight`
- `ios_device_launch_certificate: start backend-device-demo`

Those actions still point in the right direction, but they are less useful than
the rest of the handoff because they omit the project-local config file, the make
target, or the validation command.

## Goal

Normalize prefixed iOS device actions while preserving their source prefix:

- `...: provide iOS deploy config and rerun mobile deploy preflight` becomes
  `...: provide iOS deploy config in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`
- `...: start backend-device-demo` becomes
  `...: start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight`

## Scope

- String-only local report action normalization.
- No backend process is started.
- No iOS deploy, signing, Xcode, Keychain, provider call, or paid action is
  executed.
- No change to readiness status, blocker ordering, or safety metadata.

## Approach

Extend `myth_forge_api.operator_actions.normalize_operator_action()` so it can
normalize exact and prefixed legacy iOS device actions. Existing report
aggregators already call this helper before dedupe, so the change should flow
through final showcase, rehearsal, closure, and local refresh reports.

Rejected alternatives:

- Patch only `final_showcase_readiness`; this leaves other aggregate reports with
  old wording.
- Change `ios_device_launch_certificate` source output directly; it helps one
  source but does not handle older saved reports that still contain legacy
  actions.

## Testing

- Add focused shared normalizer tests for exact and prefixed legacy iOS deploy
  config and backend-device-demo actions.
- Add a final-showcase test that injects legacy certificate actions and asserts
  the final report exposes normalized prefixed actions.
- Refresh safe local reports and verify `final-showcase-readiness.json` no longer
  contains the old prefixed certificate actions.
- Run backend lint, backend tests, and `git diff --check` before merging.
