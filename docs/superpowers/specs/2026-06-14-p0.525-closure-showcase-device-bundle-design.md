# P0.525 Closure Showcase Device Bundle Design

## Context

The refreshed final handoff reports agree on the top-level next command:

```bash
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

However, `final-launch-closure-packet` still exposes `device_action_bundle.first_action` from the lower-level `ios_device_evidence_bundle`, which can point at `make backend-device-demo`. This makes the closure packet carry two different device priorities in the same report.

## Goal

Make the closure packet's top-level `device_action_bundle` use the final showcase readiness device bundle when it is available, because final showcase readiness already merges saved mobile preflight evidence, iOS runbook state, final acceptance blockers, and provider-safe ordering.

## Scope

- Use `final_showcase_readiness.device_action_bundle` for `final-launch-closure-packet.device_action_bundle` when present.
- Preserve the existing `ios_device_evidence_bundle` fallback for older or partial source reports.
- Preserve nested `source_reports.ios_device_evidence_bundle.device_action_bundle` for provenance.
- Add regression coverage that closure packet first action follows the final showcase promoted deploy writer.

## Non-Goals

- Do not change mobile deploy preflight, Xcode signing, live provider, or print provider behavior.
- Do not execute real provider calls, device builds, Keychain writes, or paid print actions.
- Do not remove lower-level iOS evidence from closure packet source reports.

## Acceptance Criteria

- Closure packet top-level `device_action_bundle.source_report` is `final_showcase_readiness` when showcase readiness provides a bundle.
- Closure packet top-level `device_action_bundle.first_action.command` matches the promoted `DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight` action in the current blocked local state.
- Nested `source_reports.ios_device_evidence_bundle.device_action_bundle` remains present.
- Focused closure packet tests pass.
- Backend lint and tests pass before merge.
