# P0.80 Final Launch Mode Policy Summary Design

## Purpose

The iPhone final launch panel can already request either `local` or
`configured` `/v1/final-demo-launch` reports. The operator still has to infer
which report is currently displayed and whether live provider calls are allowed
from dense subtitle text and command rows. P0.80 makes that state explicit in
the mobile launch panel without adding new provider behavior.

## Scope

Add a read-only mobile summary section for:

- the currently loaded final launch mode (`local` or `configured`)
- whether the backend report says live provider calls run by default
- the configured acceptance consent flag when consent is required

The app remains a status surface. It must not execute final launch commands,
read backend secrets, mutate resource files, or call live providers.

## Architecture

The backend already returns `mode` and `live_call_policy` in
`FinalDemoLaunchReport`. Swift models already decode those fields. P0.80 only
extends `FinalLaunchMobileSummary` with `modePolicyRows: [String]` and maps the
existing report fields into sanitized display strings.

`FinalLaunchStatusView` renders a compact `Mode` group above resources so the
operator can see the selected report path before reading readiness details.
Existing redaction remains the only text-safety boundary.

## Data Flow

`ForgeRootView` keeps the existing `FinalLaunchMode` picker and fetch path.
`FinalLaunchMobileSummaryBuilder.build(report:error:)` receives the decoded
report and produces:

- `Mode: Local` or `Mode: Configured`
- `Live calls by default: no`
- `Consent flag: --allow-live-provider-calls` when configured acceptance
  requires consent

If a report is missing or blocked by an API error, `modePolicyRows` is empty.

## Error Handling And Safety

Unsafe text is passed through the existing summary sanitizer before display.
The new rows do not add network calls or command execution. If the backend ever
returns unexpected mode or policy text, the app displays sanitized text rather
than trusting it as a command.

## Tests And Acceptance

Add Swift contract tests for local and configured reports:

- local report shows `Mode: Local`
- configured report shows `Mode: Configured`
- policy rows state that live calls are not default
- consent flag is visible and sanitized

Add source gates so final acceptance requires `modePolicyRows`, the `Mode`
section in `FinalLaunchStatusView`, and the contract tests. Update the roadmap
and verification evidence with a static iPhone-sized screenshot.
