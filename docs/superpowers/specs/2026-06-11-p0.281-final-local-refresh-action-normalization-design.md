# P0.281 Final Local Refresh Action Normalization Design

## Context

P0.280 made `resource_handoff` and the shared operator action normalizer replace
bare provider selector actions such as `set THREE_D_PROVIDER=meshy` with the
safe final resource apply handoff. The saved
`final-local-report-refresh.json` can still surface stale provider selector
actions because `final_local_report_refresh._operator_actions()` dedupes raw
commands without calling the shared normalizer.

That report feeds the final readiness and operator handoff flow. It should not
reintroduce action text that asks the operator to manually set provider selector
values.

## Goal

Make final local report refresh operator actions reuse the shared normalization
rules before deduping and trimming the action list.

## Non-Goals

- Do not add provider selector values to `final-resources.env`.
- Do not change provider readiness, final resource preflight, or live provider
  consent semantics.
- Do not call Meshy, OpenAI, Treatstock, Xcode, Keychain, or any paid/global
  external flow.
- Do not write real secrets or signing values.

## Design

Import `normalize_operator_action()` from `myth_forge_api.operator_actions` in
`final_local_report_refresh.py`.

Change `_dedupe_operator_actions()` so it normalizes every candidate action
before computing validation-aware roots and removing bare duplicates. This keeps
the existing duplicate behavior for validated actions and adds support for
stale saved actions like:

- `set THREE_D_PROVIDER=meshy`
- `set NPC_PROVIDER=openai`

Both should become:

`run make final-apply-resources to apply the filled resource bundle`

The existing `_dedupe()` call should still remove duplicate normalized apply
actions.

## Acceptance

- RED/GREEN test proves `_operator_actions()` converts stale provider selector
  commands to the final apply action.
- RED/GREEN test proves duplicate stale provider selector commands collapse to a
  single final apply action.
- Existing final resource and iOS deploy validation-aware action behavior stays
  unchanged.
- Refreshed `.local/final-local-report-refresh.json` no longer contains
  `set THREE_D_PROVIDER=meshy` or `set NPC_PROVIDER=openai` in
  `operator_actions`.
