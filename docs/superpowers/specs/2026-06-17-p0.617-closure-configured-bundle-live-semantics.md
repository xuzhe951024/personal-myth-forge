# P0.617 Closure Configured Bundle Live Semantics

## Problem

`final-launch-closure-packet` can promote a configured evidence bundle operator action to a live provider consent command while keeping the old `local_stub` detail and classification from `current_blocker`. The resulting handoff tells the operator to approve a live provider call, but describes the blocker as a provider resource handoff issue.

## Scope

- Keep the configured evidence bundle route through explicit live provider consent.
- When a live provider operator action is selected, source classification, detail, and validation metadata from the matching live `next_action` when available.
- Fall back to live-consent semantics if the selected live action has no matching structured source.
- Do not run live provider calls, Xcode signing, Keychain, global writes, or paid actions.

## Acceptance

- RED test reproduces the mixed `local_stub` detail/classification plus live command mismatch.
- GREEN implementation makes closure packet configured bundle actions report coherent live consent metadata.
- Focused closure tests, adjacent closure packet tests, lint, and backend regression gates remain green.
