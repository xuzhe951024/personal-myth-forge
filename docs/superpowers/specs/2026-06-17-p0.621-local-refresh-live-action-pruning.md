# P0.621 Local Refresh Live Action Pruning Spec

## Context

The final local report refresh is now blocked on configured provider evidence. The first actionable command is the narrow Meshy 3D consent gate:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-3d-configured; rerun make final-configured-evidence-plan
```

However, the saved `final-local-report-refresh` handoff can still include lower-priority fallback actions after this narrow consent gate:

- device-demo fallback commands that are already ready
- broad `final-acceptance-configured` live-provider acceptance commands

Those extra actions make the final handoff less precise at the moment when the operator needs one guarded, cost-aware action.

## Requirements

- Treat any `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 ...` action as a live-provider consent action for fallback pruning.
- When a granular live-provider consent action exists, remove the broad final acceptance configured action from the operator action list.
- Preserve the broad final acceptance configured action when it is the only live-provider consent action available.
- Keep existing provider, print, iOS deploy, and validation action preferences intact.
- Do not call live providers, mutate global Xcode state, or write secrets.

## Acceptance

- Focused tests cover granular live-provider consent pruning.
- Existing final local report refresh tests still pass.
- Safe local report refresh writes an operator action list whose first action is the granular 3D consent gate and does not include stale device or broad configured fallback actions after it.
