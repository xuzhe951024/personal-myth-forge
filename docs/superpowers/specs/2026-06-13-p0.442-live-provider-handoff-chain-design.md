# P0.442 Live Provider Handoff Chain Design

## Context

`live-provider-evidence.json` is one of the remaining gates for both
`game_asset_3d_generation` and `ai_agent_npc`. In the current local state, it is
blocked by `provider_handoff`, and the provider handoff report already projects
its child blocker:

```text
make final-resource-apply-preview
source_blocker_validation_command: make provider-handoff
validation_command: make live-provider-evidence
```

However, `live_provider_evidence.operator_actions` currently starts with
`make live-provider-evidence` and then a less actionable rerun hint. That asks
the operator to rerun the final verifier before completing the upstream handoff.

## Design

When the live provider evidence first blocker includes a projected
`source_blocker_validation_command`, emit a chained operator action:

```text
make final-resource-apply-preview; rerun make provider-handoff; rerun make live-provider-evidence
```

For simpler missing-report or failed configured evidence cases without a
projected source blocker, use the blocker command followed by the final
validation command:

```text
make provider-handoff; rerun make live-provider-evidence
make backend-evaluate-3d-configured; rerun make live-provider-evidence
```

The report remains read-only. It does not run commands, write resources, call
Meshy/OpenAI, mutate Xcode, or touch Keychain.

## Scope

- Modify only `myth_forge_api.live_provider_evidence`.
- Add backend tests for the projected provider handoff child blocker.
- Keep redaction and safety behavior unchanged.
- Do not apply provider resources or call live providers.

## Acceptance

- A focused backend test fails before implementation because operator actions do
  not include the source-blocker validation chain.
- The focused backend test passes after implementation.
- `make live-provider-evidence` still writes a blocked report locally without
  executing commands or leaking secrets.
- Backend lint, focused tests, and whitespace checks pass.
