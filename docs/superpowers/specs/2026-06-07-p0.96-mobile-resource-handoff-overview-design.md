# P0.96 Mobile Resource Handoff Overview Design

## Context

The backend final launch report already embeds `resource_report`, a sanitized
consolidated handoff for final provider keys, print provider selection, capture
storage, iOS deploy settings, and local build gates. The iPhone client decodes
`final_resources_preflight` and shows a flat checklist, but it does not decode
or render the grouped `resource_report`.

This leaves the final phone-side demo missing the clearest answer to: "Which
backend provider keys and iOS deploy values still need to be supplied before the
configured run?"

## Design

Add Swift models for the existing backend `resource_report` payload:

- `ResourceHandoffReport`
- `ResourceHandoffSection`
- `ResourceHandoffItem`
- `ResourceHandoffSafety`

`FinalDemoLaunchReport` will gain optional `resourceReport` decoding from
`resource_report`.

`FinalLaunchMobileSummary` will add three row groups:

- `resourceHandoffRows`: overall resource status plus backend and iOS
  destination summaries
- `resourceHandoffBackendRows`: highest-priority backend provider/key rows
- `resourceHandoffIOSRows`: highest-priority iOS deploy rows

`FinalLaunchStatusView` will render these row groups after `Resource Checklist`
and before `Resources`. The iPhone app remains read-only; it will not execute
resource apply commands or write local config.

## Row Rules

- Missing report: show `Resource handoff has not loaded.`
- Overall row: show `Resource handoff <status>: ready <n>, missing <n>, blocked <n>, manual <n>.`
- Section rows: show backend and iOS destinations without absolute paths.
- Backend rows: prefer missing or blocked items; otherwise show required
  real-provider slots such as `THREE_D_PROVIDER`, `MESHY_API_KEY`,
  `NPC_PROVIDER`, `OPENAI_API_KEY`, `PRINT_PROVIDER`, and `TREATSTOCK_API_KEY`.
- iOS rows: prefer missing or blocked deploy items; otherwise show
  `DEVELOPMENT_TEAM`, `PRODUCT_BUNDLE_IDENTIFIER`, and `PMF_BACKEND_BASE_URL`.
- All rows are sanitized with the existing mobile final launch sanitizer.

## Tests

Swift contract tests will be written first:

- decode `resource_report` from final launch payload
- show grouped resource handoff rows for missing backend/iOS resources
- show ready/configured resource handoff rows
- redact unsafe resource handoff text

Project/source guards will require the new models, summary rows, view sections,
and contract tests. Backend iOS showcase and mobile final launch readiness
acceptance will also require the same mobile source strings.

## Visual Regression

Add a 390x844 static visual of the Final Launch panel showing:

- `Resource Handoff`
- `Backend Resources`
- `iOS Resources`
- existing `iOS Deploy Runbook` below the resource handoff rows

## Non-Goals

- No backend endpoint changes
- No new provider adapters
- No command execution from iOS
- No writes to `.env`, `Deployment.local.xcconfig`, Xcode, signing, keychain, or
  global machine state
- No live Meshy, OpenAI, Treatstock, or Sculpteo calls
