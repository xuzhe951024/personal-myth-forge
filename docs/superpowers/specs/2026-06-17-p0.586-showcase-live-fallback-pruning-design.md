# P0.586 Showcase Live Fallback Pruning Design

## Context

`final-showcase-readiness.json` now routes 3D, NPC, and provider/key handoff blockers through the guarded live provider consent command:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make final-acceptance-configured; rerun make live-provider-evidence
```

The top-level final showcase `operator_actions` can still keep configured live-provider fallback actions beside that guarded command, including:

- `unblock final_configured_preflight after provider_handoff`
- `make backend-evaluate-3d-configured`
- `make backend-evaluate-npc-configured`

Those commands are not the final preferred handoff once the guarded configured acceptance path is present.

## Decision

Extend the final showcase live-consent preference:

- Keep the guarded live provider consent action as the single live-provider entry point.
- Drop provider-handoff, bare live-provider-evidence, configured 3D/NPC evaluation, and configured-preflight unblock fallback actions from the same top-level action list.
- Preserve unrelated print, device, and local resource actions.

## Acceptance

- `_dedupe_operator_actions()` removes configured live-provider fallbacks when the guarded live consent action exists.
- Existing final showcase and final demo launch focused tests remain green.
- Safe report generation still performs no live provider calls.
