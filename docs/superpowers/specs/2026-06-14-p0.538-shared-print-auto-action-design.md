# P0.538 Shared Print Auto Action Design

## Context

P0.536 added `PRINT_SOURCE_ASSET_URI=auto PRINT_CANDIDATE_URI=auto make print-quote-request-configured`, and P0.537 made the final launch closure packet prefer that handoff over the bare print readiness command. The shared `operator_actions` module still defines the canonical configured print quote request action with the old `https://...` placeholder. Multiple final reports use that shared helper to normalize and prefer print quote handoff actions.

## Goal

Make the shared operator action normalization and print quote preference layer treat the `auto` print quote request as the canonical configured print request handoff.

## Non-Goals

- Do not remove compatibility with older saved reports that still contain the `https://...` placeholder request.
- Do not call Treatstock or any live print provider.
- Do not write quote request JSON, provider keys, or iOS signing configuration.
- Do not change final readiness status without real iPhone or provider evidence.

## Design

Update `myth_forge_api.operator_actions` so:

- `CONFIGURED_PRINT_QUOTE_REQUEST_ACTION` uses `PRINT_SOURCE_ASSET_URI=auto PRINT_CANDIDATE_URI=auto make print-quote-request-configured`.
- The old placeholder request remains recognized as a legacy request action for dedupe and preference.
- When both a guarded provider quote and any configured request action are present, the request action wins.
- When both old placeholder and new auto request actions are present, the auto action wins.

This centralizes the P0.536/P0.537 behavior for all reports that call `prefer_guarded_print_quote_handoff_actions`.

## Testing

Focused tests will cover:

- Auto print request handoff wins before guarded provider quote calls.
- Auto print request handoff wins over legacy placeholder request actions.
- Non-print operator actions still pass through unchanged.

This phase is report text and normalization only. It does not execute provider calls, device actions, or global writes.
