# P0.79 Deploy Preflight Launch Mode Design

## Context

P0.78 added `PMF_FINAL_LAUNCH_MODE` to final resources, iOS config, and the app runtime. The iPhone safely defaults unknown values to local, and the final resource apply path writes only `local` or `configured`.

One remaining gap is manual device deployment: `apps/mobile/ios/scripts/deploy_preflight.sh` checks Team ID, bundle ID, backend URL, loopback safety, and backend health, but it does not check `PMF_FINAL_LAUNCH_MODE`. A hand-edited `Deployment.local.xcconfig` can therefore contain an unsupported mode and still pass deploy preflight until the app silently falls back to local.

## Decision

Add a deploy-time source/config gate for `PMF_FINAL_LAUNCH_MODE`.

The script will:

- read `PMF_FINAL_LAUNCH_MODE` with the existing `get_value` helper, so `Deployment.xcconfig` supplies the default `local` and `Deployment.local.xcconfig` can override it;
- allow only `local` and `configured`;
- print `PMF_FINAL_LAUNCH_MODE must be local or configured.` and exit 2 for unsupported values;
- keep missing local override valid because the base config already provides `PMF_FINAL_LAUNCH_MODE = local`;
- print `Final launch mode: <mode>` on success.

## Files

- `apps/mobile/ios/scripts/deploy_preflight.sh`: add mode lookup, allowlist, and success output.
- `services/backend/tests/test_ios_deploy_preflight_script.py`: add invalid override and valid configured override tests.
- `apps/mobile/ios/Sources/PersonalMythForgeMobileProjectChecks/main.swift`: require deploy preflight source to mention `PMF_FINAL_LAUNCH_MODE` and the allowlist copy.
- `README.md`, `apps/mobile/README.md`, `docs/iteration-roadmap.md`: document the P0.79 safety gate.
- `docs/superpowers/verification/p0.79-deploy-preflight-launch-mode.html` and PNG: static visual regression evidence.

## Safety

This slice does not run Xcode, mutate signing state, call providers, read provider keys, or touch global configuration. The only runtime command remains the existing backend `/health` check used by deploy preflight.

## Acceptance

- Invalid `PMF_FINAL_LAUNCH_MODE = live` in `Deployment.local.xcconfig` exits 2 before backend health is required.
- Valid `PMF_FINAL_LAUNCH_MODE = configured` passes when backend health is ok.
- Project checks require the preflight source gate.
- Backend lint/test and Swift source gates pass.
