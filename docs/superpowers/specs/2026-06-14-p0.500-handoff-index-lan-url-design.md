# P0.500 Handoff Index LAN URL Design

## Context

`final-handoff-index.json` is the top-level handoff index for the final demo. It summarizes the local rehearsal lane, configured preflight lane, live acceptance lane, and operator actions.

The current final handoff index can include:

- `DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`
- provider handoff
- print quote handoff
- configured preflight
- configured launch
- backend device demo

It does not preserve the `set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL; rerun make mobile-deploy-preflight` action from `final-demo-launch-local.json`. That weakens the highest-level handoff because the real iPhone blocker is not only the Apple Team ID; the backend URL must also be reachable from the phone.

## Decision

Update `final_handoff_index` so local lane handoff actions preserve concrete device handoff actions from the saved final demo launch report:

- iPhone-reachable backend URL action
- backend-device-demo action

Keep provider and print handoff actions. Prioritize the final top-level action list so the six visible actions include:

1. first local blocker action
2. iPhone-reachable backend URL action
3. backend-device-demo action
4. provider handoff
5. print quote handoff
6. configured preflight

This intentionally drops lower-priority configured launch noise from the visible six-action list when device handoff actions need the space.

Saved source reports should keep enough prioritized operator actions to carry these handoffs forward. The final handoff index may still cap its visible top-level actions at six.

## Safety

This is report-only. It does not run Xcode, write signing config, touch Keychain, call Meshy/OpenAI/Treatstock, write ignored env/config files, or spend provider credits.

## Verification

TDD coverage must prove:

- `_local_lane_handoff_actions()` carries LAN backend URL and backend-device-demo actions from final demo launch.
- Saved final demo launch operator actions preserve LAN URL even when it appears after provider, print, and bundle-id handoffs.
- `_operator_actions()` preserves the LAN backend URL action inside the six-action limit.
- Refreshed `final-handoff-index.json` includes the LAN backend URL action.
- Existing final handoff index and shared operator action tests keep passing.
