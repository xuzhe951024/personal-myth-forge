# P0.564 Backend Test Dotenv Isolation Design

## Context

The project now has real provider handoff values in ignored local files. `services/backend/.env` can select `THREE_D_PROVIDER=meshy` and `NPC_PROVIDER=openai` with real keys. That is useful for configured demos, but it makes ordinary backend tests unsafe: `load_settings()` always calls `load_dotenv()`, so tests that delete or override environment variables can still reload real provider keys from `.env`.

This already caused `make backend-test` in the main checkout to start using OpenAI/Meshy paths instead of deterministic local providers. The same class of leak applies to ignored iOS handoff files such as `apps/mobile/ios/Config/Deployment.local.xcconfig`: script tests that copy fixture repos can accidentally inherit the operator's real local deploy state.

## Decision

Keep production behavior unchanged: when running the backend normally, `load_settings()` still loads `.env`.

Add an explicit skip switch for automation:

- `PMF_SKIP_DOTENV=1` makes `load_settings()` skip `load_dotenv()`.
- Passing an explicit `env` mapping to `load_settings(env=...)` also skips `load_dotenv()` because the caller is providing the complete source of truth.
- `make backend-test` sets `PMF_SKIP_DOTENV=1`.
- Backend pytest also sets `PMF_SKIP_DOTENV=1` in `tests/conftest.py` so direct `uv run pytest` is safe from real ignored `.env` values.
- Test fixture repos ignore `Deployment.local.xcconfig` so iOS deploy tests start from the intended unconfigured state.

## Scope

In scope:

- Add focused tests proving explicit env mappings and pytest skip real `.env` values.
- Update `config.py` and `Makefile`.
- Add a pytest `conftest.py` guard for the backend suite.
- Keep copied test fixture repos free of ignored local iOS deploy config.
- Add a roadmap note.

Out of scope:

- Removing `.env` support for local backend/demo runs.
- Changing provider adapters or live provider consent flows.
- Running Meshy, OpenAI, Treatstock, Xcode, signing, or physical device actions.

## Acceptance

- A RED/GREEN test proves `load_settings(env={...})` ignores a real `.env` file in the working directory.
- A RED/GREEN test proves `PMF_SKIP_DOTENV=1` prevents `.env` from repopulating deleted provider keys.
- `make backend-test` no longer reads `.env` because the Make target exports `PMF_SKIP_DOTENV=1`.
- Main-checkout tests that copy iOS config fixtures no longer inherit `Deployment.local.xcconfig`.
- Focused tests, backend lint, backend tests, and whitespace diff checks pass in the isolated worktree.
