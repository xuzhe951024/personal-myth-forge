# P0.528 Device Action Validation Design

## Goal

Make final showcase device action rows expose validation-aware commands when the row command is already an executable `make ...` command.

## Context

The final showcase report now has validation-aware provider and print capability commands. The iOS device action bundle still exposes some rows with a raw command plus a separate `validation_command`, for example:

- `make backend-device-demo` with `make mobile-deploy-preflight-evidence`
- `make mobile-deploy-preflight` with `make mobile-deploy-preflight-evidence`

Mobile and final handoff consumers can surface the device action row directly. Those rows should be copy-pasteable without requiring consumers to manually combine `command` and `validation_command`.

## Design

For final showcase device actions, append `; rerun <validation_command>` to the row command only when:

- `command` starts with `make `
- `validation_command` is present
- the command does not already contain that validation suffix

Do not append validation to prose commands such as `open Xcode and resolve signing/build gate`; those are instructions, not shell commands.

## Safety

This is report-only. It does not run Xcode, mutate signing settings, start servers, call providers, write secrets, or perform external paid actions.

## Tests

Update focused final showcase readiness tests so RED proves executable device actions currently miss their validation rerun suffix. GREEN requires backend/preflight `make` actions and their first-action surfaces to be validation-aware while the Xcode prose action remains prose plus separate validation metadata.
