# P0.545 Deploy Auto Command Visibility Design

## Goal

Make the first iOS deploy handoff command show that the auto writer resolves `PMF_BACKEND_BASE_URL=auto`.

## Current Gap

Final readiness reports currently surface:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

The Make target does set `PMF_BACKEND_BASE_URL=auto` internally, but the report hides that fact. This makes the first blocker less self-explanatory when both Apple Team ID and iPhone-reachable backend URL are missing.

## Intended Behavior

- Canonical auto writer actions should read:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto
```

- Validation-aware actions should append:

```text
; rerun make mobile-deploy-preflight
```

- Existing legacy actions without the explicit `PMF_BACKEND_BASE_URL=auto` should normalize to the new canonical form.
- Reports still perform no Xcode signing, Keychain writes, global mutation, provider calls, or paid actions.
- The placeholder `YOUR_TEAM_ID` remains visibly placeholder-like so the user must supply the real Apple Team ID before running the handoff.

## Acceptance

- `mobile_deploy_preflight_evidence` next action and device action bundle include explicit `PMF_BACKEND_BASE_URL=auto`.
- Operator action normalization converts legacy auto writer commands to the explicit canonical form.
- Final aggregators that consume normalized operator actions preserve the explicit command.
- Safe readiness refresh still writes blocked JSON when external device requirements are missing.
