# P0.336 iOS Detail Writer Normalization Design

## Context

P0.335 made the current mobile deploy preflight evidence promote this safe
project-local writer action:

```bash
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto
```

Some upstream reports still carry older detail-suffixed actions from saved
rehearsal evidence:

```text
provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable
```

That text is technically valid but noisier. It leaves the final operator
handoff with both the writer command and a stale manual-edit command for the
same Team-ID/LAN-backend blocker.

## Decision

Normalize legacy mobile-preflight detail actions that contain both:

- a validated `provide DEVELOPMENT_TEAM in Deployment.local.xcconfig` command;
- a detail mentioning `PMF_BACKEND_BASE_URL` or `iPhone-reachable`.

The normalized action becomes:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable
```

The detail remains visible so reports still explain why the action is needed.
Pure Team-ID-only actions and bundle-id actions are not changed.

## Safety

This is a report text normalization only. It does not write
`Deployment.local.xcconfig`, run Xcode, accept licenses, touch signing or
Keychain, call providers, or create orders.

## Acceptance

- `normalize_operator_action()` rewrites stale Team-ID plus backend-url detail
  actions to the writer command.
- Final showcase and final local refresh operator actions stop surfacing the
  old manual Team-ID command for the combined blocker.
- Safe report refresh still returns blocked with no provider, Xcode, or config
  writes.
