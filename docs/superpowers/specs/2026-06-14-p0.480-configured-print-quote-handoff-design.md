P0.480 Configured Print Quote Handoff Design

Goal

Turn the configured Treatstock quote evidence step from a prose-only handoff into a guarded repo-local command path. The operator should be able to provide ignored local request/config files, explicitly consent to a live print-provider quote call, and receive a sanitized `.local` quote evidence file that `print_fulfillment_readiness` can validate.

Context

The final iOS showcase remains blocked by external resources and physical-device evidence. Print fulfillment is currently partial because `services/backend/.local/print-quote-configured.json` is absent. Existing reports tell the operator to manually save a sanitized response from `POST /v1/print-quotes`, but there is no project command that:

- refuses to run by default,
- documents the required explicit Treatstock cost consent,
- writes the evidence file in the exact validated shape, and
- redacts checkout/payment links before persistence.

Non-Goals

- Do not call Treatstock during automated tests.
- Do not place orders, open checkout, capture payment, or write shipping/customer data.
- Do not write API keys, `.env`, Xcode signing, Keychain, or global environment state.
- Do not change the public `/v1/print-quotes` contract.
- Do not mark final print fulfillment ready without real configured local evidence.

Proposed Behavior

Add a guarded configured quote evidence path:

- `make print-quote-configured` first runs a repo script that requires `PMF_ALLOW_PRINT_PROVIDER_CALLS=1`.
- The target reads an ignored local request file, defaulting to `services/backend/.local/print-quote-request-configured.json`.
- The CLI command calls the configured print provider only after the explicit allow flag is present.
- The CLI writes `services/backend/.local/print-quote-configured.json` with a sanitized `print_quote` JSON payload:
  - keep validation fields: `kind`, `provider`, `status`, `currency`, `estimated_price_cents`, shipping/production estimates, `requires_user_approval`, `approval_reason`, and quote notes when safe,
  - force `checkout_url` to `null`,
  - include no local absolute paths, secrets, bearer tokens, payment links, or raw private context.
- `print_fulfillment_readiness` should point the configured Treatstock quote action at the guarded target and retain `requires_cost_consent: true`.

Acceptance Criteria

- Without `PMF_ALLOW_PRINT_PROVIDER_CALLS=1`, `make print-quote-configured` exits nonzero before running the CLI/provider call.
- The CLI refuses configured print quote creation without its explicit allow argument.
- With a fake provider in tests, the CLI writes a sanitized `print_quote` payload that validates as configured Treatstock evidence and omits checkout/payment links.
- `print_fulfillment_readiness_report` missing configured quote actions mention `PMF_ALLOW_PRINT_PROVIDER_CALLS=1 make print-quote-configured` and still rerun `make print-fulfillment-readiness`.
- Source acceptance/ProjectChecks know about the guarded target and consent variable.
- Focused backend tests and Swift source checks pass.

Safety

This phase must not call Treatstock, Meshy, OpenAI, Xcode, signing, Keychain, or payment services during development verification. Any test that exercises provider behavior must monkeypatch a fake provider or fake shell command.
