# P0.279 Resource Handoff Secret Validation Design

## Context

The final resource and iOS deploy action surfaces now retain validation commands
for most missing inputs. `resource_handoff.py` still emits backend secret
actions as bare text:

- `provide MESHY_API_KEY`
- `provide OPENAI_API_KEY`
- `provide TREATSTOCK_API_KEY`

Those actions are visible in final showcase operator actions and do not tell the
operator which ignored handoff file to edit or which safe validation command to
rerun. This is weaker than the rest of the final handoff flow.

Provider selection actions such as `set THREE_D_PROVIDER=meshy` and
`set NPC_PROVIDER=openai` are not accepted keys in the current
`final-resources.env` preflight, so this iteration will not route them through
that file.

## Goal

`resource_handoff` should make backend secret actions explicit by targeting
`final-resources.env` and appending `; rerun make final-resources-preflight`.

## Non-Goals

- Do not add `THREE_D_PROVIDER` or `NPC_PROVIDER` to final resources.
- Do not change final resource preflight accepted keys.
- Do not change provider readiness semantics, live provider calls, or configured
  acceptance behavior.
- Do not write `.env`, `final-resources.env`, iOS deploy config files, run
  Xcode/signing, touch Keychain, call live providers, or place print orders.

## Design

Import `add_final_resource_validation_command()` in `resource_handoff.py` and
compose it with the existing mobile deploy validation pass over operator
actions.

Change backend secret action roots to include the ignored handoff file:

- `provide MESHY_API_KEY in final-resources.env`
- `provide OPENAI_API_KEY in final-resources.env`
- `provide TREATSTOCK_API_KEY in final-resources.env`

The shared final resource validation helper already recognizes those roots and
will append `make final-resources-preflight`.

## Acceptance

- RED/GREEN test proves missing Meshy/OpenAI actions in `resource_handoff` point
  to `final-resources.env` and include
  `; rerun make final-resources-preflight`.
- RED/GREEN test proves Treatstock missing-key actions use the same validated
  handoff text.
- Existing iOS deploy actions still include
  `; rerun make mobile-deploy-preflight`.
- Refreshed `.local/resource-handoff.json` keeps blocked status while showing
  validated backend secret actions.
- Safety flags continue to report no provider secret leaks, no local path leaks,
  and no payment links.
