# P0.144 iOS Showcase Real Source Gate Design

## Context

`run_ios_showcase_acceptance(repo_root=real_repo)` currently fails one required
feature: `final_resource_requirements_manifest`. The fixture test passes because
the fixture writes a literal `final-resources.env initialized` string into
`init_final_resources.sh`, while the real script emits the same meaning via a
variable-expanded line.

That leaves final showcase readiness with `ios_showcase_acceptance: failed`,
which weakens the final iOS demo gate even though the runtime script behavior is
otherwise correct.

## Decision

Keep the source gate strict and make the real script include the stable literal
acceptance signal. Do not loosen `ios_showcase_acceptance`; instead, add a real
repository regression test so future fixture drift cannot hide current-source
failures.

The script should preserve the existing stdout substring checked by
`test_final_resource_init_script.py`:

`services/backend/.local/final-resources.env initialized`

It should also contain the literal source-gate token:

`final-resources.env initialized`

## Alternatives Considered

1. Relax the source gate to accept variable-expanded output.
   This makes the gate less useful and would miss regressions in the operator
   handoff text.

2. Add a comment containing the literal string.
   This would satisfy the source gate but would not improve the operator-facing
   script output.

3. Update the script output line to include both the destination and the stable
   literal signal.
   This is the smallest behavior-preserving change and keeps the source gate
   meaningful.

## Testing

Add `test_ios_showcase_acceptance_passes_current_repo_source_gates` using the
actual repository root. It should assert:

- exit code `0`
- summary `{"passed": 47, "failed": 0}`
- no required feature has status `failed`

Run that test before changing the script and confirm it fails on
`final_resource_requirements_manifest`.

Then update `init_final_resources.sh` and verify:

- the new real-repo source gate test passes
- existing fixture tests still pass
- existing final resource init script tests still pass

## Safety

This change does not run `make final-resource-init`, does not create or overwrite
`.local/final-resources.env`, does not touch provider keys, does not run Xcode,
and does not mutate global state.
