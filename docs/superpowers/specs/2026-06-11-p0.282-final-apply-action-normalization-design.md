# P0.282 Final Apply Action Normalization Design

## Context

The local final report refresh now normalizes stale provider selector actions,
but its operator checklist can still contain a raw action like:

`make final-apply-resources; rerun make final-resources-preflight`

That text comes from a blocked `final_resource_apply_preview` step whose
contract-level command remains `make final-apply-resources`. The contract field
is used by mobile source tests and should stay stable. The problem is only the
aggregated operator action text: it suggests an unsafe order while resources
are still missing.

## Goal

Normalize bare final resource apply actions in operator action lists to the
existing explicit handoff text:

`run make final-apply-resources to apply the filled resource bundle`

## Non-Goals

- Do not change `final_resource_apply_preview.first_blocker.command`.
- Do not change mobile decoding contracts or Swift fixtures.
- Do not run `make final-apply-resources` or write `.env` /
  `Deployment.local.xcconfig`.
- Do not add, remove, or validate real secrets or Xcode signing values.

## Design

Extend `myth_forge_api.operator_actions.normalize_operator_action()` with exact
normalization for:

- `make final-apply-resources`
- `make final-apply-resources; rerun make final-resources-preflight`
- `run make final-apply-resources`

Each should return `FINAL_RESOURCE_APPLY_ACTION`.

Because `final_local_report_refresh._dedupe_operator_actions()` already routes
actions through the shared normalizer, this keeps the source report contract
stable while cleaning the aggregated operator checklist.

Other report aggregators that already use `normalize_operator_action()` should
inherit the same explicit final apply handoff text while preserving their source
prefixes.

## Acceptance

- RED/GREEN test proves the shared normalizer rewrites bare final apply actions.
- RED/GREEN test proves final local refresh operator actions do not expose
  `make final-apply-resources; rerun make final-resources-preflight`.
- Refreshed `.local/final-local-report-refresh.json` contains the explicit
  final apply handoff and not the raw semicolon action.
- Existing iOS device launch rehearsal action normalization keeps source
  prefixes while using the explicit final apply handoff.
- Existing provider selector normalization and validation-aware action dedupe
  tests remain green.
