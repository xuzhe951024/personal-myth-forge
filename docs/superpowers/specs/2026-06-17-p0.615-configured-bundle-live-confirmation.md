# P0.615 Configured Bundle Live Confirmation

## Problem

`final-configured-evidence-plan` and `live-provider-evidence` now expose explicit live-provider confirmation metadata, but `configured-live-evidence-bundle` drops those fields while projecting command rows, evidence files, and current blockers. That can make the configured evidence bundle less safe or less informative than its source reports.

## Scope

- Preserve `requires_cost_consent`, `live_provider_call`, and `requires_user_confirmation` from configured plan command rows into the bundle command sequence.
- Preserve the same fields from configured plan and live evidence blockers into bundle `current_blocker`, `first_blocker`, and `next_action`.
- Preserve the same fields from live evidence rows into bundle evidence file rows.
- Do not run live provider commands or any external paid/provider action.

## Acceptance

- Focused RED tests prove the bundle drops live confirmation fields from command rows, blockers, and evidence files.
- GREEN implementation copies the fields without changing command ordering or status classification.
- Focused and adjacent backend tests remain green.
