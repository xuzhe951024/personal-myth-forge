# P0.395 External Ledger Live Child Action Design

## Context

The final local refresh, final handoff index, and iOS device launch certificate
now expose the concrete live provider child action:

```text
make final-resource-apply-preview; rerun make live-provider-evidence
```

`final-external-action-ledger` still hard-codes the broader provider action:

```text
make provider-handoff; rerun make live-provider-evidence
```

When live provider evidence already knows a more concrete child action, the
external action ledger should not regress to the broad action.

## Goal

Make `final-external-action-ledger` derive the live provider evidence operator
action from `live-provider-evidence.next_action`, while preserving the broad
provider handoff fallback when no child action is available.

## Behavior

- If `live-provider-evidence.next_action` has a command and validation command,
  use `command; rerun validation_command` for the live provider evidence action.
- If no usable next action exists, keep the existing fallback:
  `make provider-handoff; rerun make live-provider-evidence`.
- Do not alter the 3D/NPC/configured acceptance live cost actions.
- Preserve print fulfillment handoff action behavior.
- Do not call live providers, write secrets, apply resources, run Xcode signing,
  touch Keychain, or place print orders.

## Testing

Add a regression where saved `provider-handoff.json` contains a child
`next_action` with command `make final-resource-apply-preview`. Building the
external action ledger should include:

```text
make final-resource-apply-preview; rerun make live-provider-evidence
```

and should not include the broad provider handoff action.
