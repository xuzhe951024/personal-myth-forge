# P0.584 Showcase Provider Key Live Consent Design

## Context

`final-showcase-readiness.json` already routes the 3D and NPC live-evidence blockers through the live provider evidence report's child `next_action`. In the current local state, those rows point to the guarded consent command:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make final-acceptance-configured; rerun make live-provider-evidence
```

The `provider_key_handoff` capability can still expose a weaker top-level command such as `make live-provider-evidence`. That creates an inconsistent final showcase handoff and can suggest bypassing the configured acceptance consent gate.

## Decision

Make `provider_key_handoff` reuse the same live provider evidence child-action metadata as the 3D and NPC capabilities when live provider evidence is not ready:

- Promote the child command plus validation command.
- Preserve the `requires_live_provider_consent` metadata from the child action.
- Mark partial live provider evidence as requiring live provider consent to complete.
- Leave local-resource-blocked and configured-bundle-blocked branches unchanged.

## Acceptance

- `provider_key_handoff.command` follows live provider evidence `next_action` when live evidence is partial or blocked.
- `provider_key_handoff.validation_command` remains `make live-provider-evidence`.
- `provider_key_handoff.next_action` includes the promoted command and consent metadata.
- Existing final showcase readiness tests remain green.
- No report generation performs live provider calls.
