# P0.298 Final Showcase Xcode Action Dedupe Design

## Goal

Keep the final showcase operator checklist focused by removing a
source-prefixed Xcode gate duplicate when the same concrete Xcode action already
appears as a bare top-level action.

## Current Context

`final-showcase-readiness.json` can currently include both actions:

```text
final_rehearsal_local: final_acceptance_local: accept the Xcode license outside Codex, then rerun make mobile-xcode-build-evidence
accept the Xcode license outside Codex, then rerun make mobile-xcode-build-evidence
```

Both ask the operator to perform the same Mac-side Xcode gate resolution. The
bare action is enough for the top-level final showcase checklist; the source
prefix remains useful in lower-level reports.

## Design

Extend the existing final-showcase-only source-prefix dedupe allowlist.

The dedupe helper should:

- keep the existing exact dedupe and known source-prefix stripping behavior;
- keep provider/resource source-prefixed actions, because those still preserve
  useful source context for missing keys and configured evidence;
- drop `final_rehearsal_local: final_acceptance_local:` Xcode gate actions only
  when the same bare Xcode action exists in the final showcase checklist;
- leave lower-level reports unchanged.

## Out of Scope

- No Xcode, signing, Keychain, iPhone deployment, provider calls, or paid print
  actions.
- No readiness status changes.
- No broad deletion of source-prefixed resource/provider actions.

## Verification

- RED/GREEN unit test proving the prefixed Xcode gate action is removed while
  the bare Xcode gate action remains.
- Focused final showcase readiness regression tests.
- `make final-local-report-refresh` still exits 2, and
  `final-showcase-readiness.json` contains the bare Xcode action once and no
  prefixed Xcode duplicate.
- Full lint, Swift checks, and backend tests before merge.
