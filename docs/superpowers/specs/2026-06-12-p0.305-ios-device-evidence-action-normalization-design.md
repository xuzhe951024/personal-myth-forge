# P0.305 iOS Device Evidence Action Normalization Design

## Goal

Make `ios-device-evidence-bundle.json` top-level `operator_actions`
copy-pastable by stripping saved source-report prefixes from the final operator
checklist.

## Current Context

`final-showcase-readiness.json` still identifies `ios_deployable` as a blocker.
The launch chain now routes through `make ios-device-launch-rehearsal`, and the
device evidence bundle is the compact handoff that tells the operator which
Mac/iPhone evidence slots are still missing.

After P0.304, `ios-device-launch-rehearsal.json` top-level actions are source-free.
However, `ios-device-evidence-bundle.json` can still wrap a saved rehearsal
detail into another top-level action:

```text
run make ios-device-launch-rehearsal | final_rehearsal_local: mobile_deploy_preflight_evidence: provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable
```

That provenance is useful inside `evidence_slots` and source reports, but the
top-level checklist should be directly actionable:

```text
run make ios-device-launch-rehearsal | provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable
```

## Design

Add evidence-bundle-specific normalization at the boundary where
`ios_device_evidence_bundle._operator_actions()` returns top-level actions.
The step should:

- reuse existing `normalize_operator_action()`;
- preserve detail suffixes after `|`;
- recursively strip known evidence-bundle source prefixes from the detail root;
- keep the action root for the evidence slot intact, such as
  `run make ios-device-launch-rehearsal`;
- leave `evidence_slots`, `source_reports`, readiness status, and safety fields
  unchanged so provenance and blocker classification remain available.

## Out of Scope

- No iOS project configuration writes.
- No backend env writes.
- No Xcode signing, Keychain, real iPhone, live provider, 3D generation, print
  quote, paid fulfillment, or external API actions.
- No changes to the lower-level launch rehearsal readiness logic.

## Verification

- RED/GREEN coverage for source-prefixed rehearsal details becoming source-free
  in top-level bundle actions.
- Regression coverage that slot details still preserve source provenance.
- Refresh safe local reports and `make ios-device-evidence-bundle`; verify the
  saved JSON remains blocked on external/device setup but top-level actions no
  longer include known source prefixes after `|`.
- Run focused evidence-bundle tests, operator-action tests, lint, whitespace
  checks, and backend tests before merge.
