# P0.250 Final Local Refresh Next Action Design

## Context

`final-local-report-refresh` regenerates the safe local evidence bundle used before the final iPhone showcase. Its report already exposes `first_blocker`, but the top-level `next_action` field is `null`, while adjacent final handoff reports expose a concrete `next_action` derived from the first blocker.

Current readiness is still blocked by external resources and device evidence: `MESHY_API_KEY`, `OPENAI_API_KEY`, Treatstock configuration, `DEVELOPMENT_TEAM`, an iPhone-reachable backend URL, Xcode/device validation, and live provider evidence. This iteration does not attempt those external actions.

## Goal

Expose a sanitized top-level `next_action` in `final-local-report-refresh.json` so operator-facing handoff tooling can consume the same immediate next step shape as the final launch closure packet.

## Design

`run_final_local_report_refresh` will compute `first_blocker` once and use it for both `first_blocker` and `next_action`. The new `next_action` is a compact dictionary copied from the sanitized blocker fields:

- `id`
- `label`
- `status`
- `classification`
- `command`
- `detail`
- `source`
- `output`
- `step_id`

When no blocker exists, `next_action` remains `None`.

## Safety

The refresh command remains read-only with respect to secrets and external systems. It still writes only ignored `.local` evidence files, does not call live providers, does not run Xcode signing/build gates, and does not write backend env or iOS deploy config. The existing sanitizer continues to redact local paths and secrets from both `first_blocker` and the derived `next_action`.

## Testing

Add a focused regression test that:

- runs `run_final_local_report_refresh` on a temp repo fixture
- verifies the first blocker is `final_resource_requirements`
- verifies `next_action` mirrors the blocker fields plus `source: first_blocker`
- verifies the derived action does not leak fixture secrets

Then run the focused refresh test file, backend lint, backend tests, diff checks, and a real `make final-local-report-refresh` smoke that is expected to exit `2` while writing a blocked report with non-null `next_action`.

## Out of Scope

This iteration does not provide provider keys, run Meshy/OpenAI/Treatstock calls, change Xcode signing, launch on a physical iPhone, or mark the final goal complete.
