# P0.536 Print Quote Request Auto URI Design

## Context

Final showcase readiness is still blocked by device deployment evidence and external provider resources. The print fulfillment capability also remains blocked when `services/backend/.local/print-quote-request-configured.json` is missing. The current readiness report can suggest concrete `PRINT_SOURCE_ASSET_URI` and `PRINT_CANDIDATE_URI` values when both local showcase evidence and an iPhone-reachable backend URL are present, but the request writer still requires operators to copy those values into the shell.

## Goal

Allow the configured print quote request writer to accept `PRINT_SOURCE_ASSET_URI=auto PRINT_CANDIDATE_URI=auto make print-quote-request-configured` and derive the served asset URLs from repo-local ignored evidence. This keeps the final handoff executable after `make local-showcase-smoke` and `make mobile-write-deploy-config-auto` have produced the needed local state.

## Non-Goals

- Do not call Treatstock or any real print provider.
- Do not place orders, upload files, or create payment links.
- Do not write provider keys or signing data into the repository.
- Do not change the final readiness status without real device or provider evidence.

## Design

The `print-quote-request-configured` CLI will accept a new `--repo-root` option. When either configured asset URI is `auto`, both URIs must be `auto`; the CLI will read:

- `services/backend/.local/final-demo-launch-local.json` for the local showcase smoke `session.session_id`.
- `apps/mobile/ios/Config/Deployment.local.xcconfig` for `PMF_BACKEND_BASE_URL`.

If both values are available and the backend URL is HTTP(S), the CLI derives:

- `{PMF_BACKEND_BASE_URL}/v1/generated-assets/{session_id}/game.glb`
- `{PMF_BACKEND_BASE_URL}/v1/print-candidates/{session_id}/print.3mf`

If the required evidence is missing, malformed, or not HTTP(S), the command exits with code 2 and does not write the request file.

The shell wrapper will keep explicit URI support unchanged and pass `--repo-root ../..` to the CLI so `auto` works from the existing `services/backend` working directory. The readiness report command will prefer the new `auto` handoff because it is more stable than copying long local URLs.

## Testing

Focused tests will cover:

- CLI resolves `auto` into served local generated-asset and print-candidate URLs.
- CLI rejects missing local evidence for `auto` and leaves no output file.
- The existing explicit URI path still writes a valid `PrintQuoteRequest`.
- Print fulfillment readiness suggests the `auto` handoff command.

No live provider test is required for this phase because this phase only writes a local ignored quote request handoff.
