# P0.576 Final Handoff Live Lane Separation Design

## Context

The refreshed final reports now expose a clean live provider command:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make final-acceptance-configured; rerun make live-provider-evidence; rerun make ios-device-launch-rehearsal
```

However, `final-handoff-index.json` imports that command from
`final-demo-launch-local.json` into the `local_rehearsal` lane. That makes the
local rehearsal lane look blocked by live provider consent, even though the
handoff index already has a dedicated `live_acceptance` lane for provider-cost
actions.

## Goal

Keep live-acceptance-only handoff actions out of the `local_rehearsal` lane.
Local rehearsal should surface local/device/print work plus provider setup
handoff such as `make provider-handoff; rerun make live-provider-evidence`.
Live Meshy/OpenAI acceptance should remain represented by the `live_acceptance`
lane and its explicit consent action.

## Approach

Change the final handoff local lane selectors so they do not select
`final-acceptance-configured`, `PMF_ALLOW_LIVE_PROVIDER_CALLS`, or bare
`make live-provider-evidence` actions from `final_demo_launch_local`. This
applies to `local_rehearsal.operator_actions`, the scalar
`local_rehearsal.operator_action`, and `local_rehearsal.detail`, because all
three can feed the top-level `first_blocker` and `next_action`. The final
handoff top-level operator actions still include live acceptance consent through
the existing `live_acceptance` lane, while provider setup handoffs remain
available to the local lane.

This is intentionally narrower than changing report statuses. It avoids
reclassifying evidence as ready, avoids running provider calls, and keeps the
remaining live provider blocker visible in the correct lane.

## Non-Goals

- Do not call Meshy/OpenAI.
- Do not mark live provider evidence as ready.
- Do not change iOS/Xcode signing behavior.
- Do not change print quote provider policy.
- Do not delete existing local evidence files.

  `operator_actions`, scalar `operator_action`, local `detail`, or the
  top-level blocker/next action.
- Run focused final handoff, iOS rehearsal, and final showcase tests.
- Run lint, `git diff --check`, and full backend tests.
- Refresh safe `.local` reports and inspect that `final-handoff-index` keeps
  live provider consent under top-level/live acceptance actions rather than as a
  prefixed `final_demo_launch_local:` local lane action.
