# P0.548 Showcase Saved Action Command Design

## Goal

Keep final showcase readiness saved device actions directly executable when the selected operator action includes an explanatory detail suffix.

## Current Gap

`final-showcase-readiness.json` already exposes a clean `device_action_bundle.first_action.command`, but the same action's `saved_next_action.command` can contain a detail suffix:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable
```

Downstream reports promote `saved_next_action.command`, so `final-demo-launch-local.json` and `final-local-report-refresh.json` can end up with command fields that are no longer copy-pasteable shell commands.

## Intended Behavior

- `saved_next_action.command` stores only the executable command part before ` | `.
- `saved_next_action.detail` preserves the explanatory suffix when present, falling back to the action's evidence detail.
- When an operator action includes a detail suffix, `saved_next_action` remains present even if its executable command matches `first_action.command`.
- Validation commands remain unchanged.
- Top-level operator actions may continue to carry detail suffixes for compact human summaries.
- Report builders remain read-only and do not write iOS config, invoke Xcode, call providers, or spend external resources.

## Acceptance

- `test_final_showcase_readiness_saved_next_action_splits_operator_detail_suffix` fails before implementation and passes after implementation.
- The test covers the real evidence shape where the first action command already matches the operator command.
- Refreshed `final-showcase-readiness.json`, `final-demo-launch-local.json`, and `final-local-report-refresh.json` have pure shell commands in promoted device action command fields.
- Focused final showcase/demo/local refresh tests and full backend tests pass.
