# P0.415 Configured Preflight Device Bundle Design

## Context

The final iOS showcase path now exposes `device_action_bundle` in the final showcase readiness report, local/configured final demo launch reports, final handoff index, final external action ledger, iOS deploy runbook, and iOS device launch certificate. The configured handoff preflight report still embeds `configured_ios_deploy_runbook.device_action_bundle` but does not expose a top-level bundle of its own.

That makes `final-configured-preflight.json` less useful as the single Mac-side configured handoff packet: the report can say provider resources are blocked first, but the operator still has to drill into the nested runbook to see the concrete physical-iPhone action sequence.

## Goal

Expose a sanitized top-level `device_action_bundle` in `final_configured_preflight_report`, sourced from the embedded configured iOS deploy runbook, so the configured handoff preflight packet directly carries the iPhone device action sequence.

## Scope

- Add `device_action_bundle` to `build_final_configured_preflight_report`.
- Source the bundle from `configured_ios_deploy_runbook.device_action_bundle`.
- Keep the source actions, first action, validation commands, and safety flags sanitized.
- Add focused TDD coverage for the blocked configured preflight path.
- Refresh `services/backend/.local/final-configured-preflight.json` and verify the new field appears.
- Add the P0.415 roadmap entry.

## Non-Goals

- Do not change existing `first_blocker` or `next_action` ordering.
- Do not call Meshy, OpenAI, Treatstock, Sculpteo, or any live provider.
- Do not run Xcode, signing, Keychain, global Mac configuration, or real iPhone deployment.
- Do not write backend `.env`, iOS deploy config, or provider resource files.
- Do not expose secrets, raw local paths, media payloads, or payment URLs.

## Report Contract

`final_configured_preflight_report.device_action_bundle` will contain:

- `id`: `final_configured_preflight_device_actions`
- `label`: `Final Configured Preflight Device Actions`
- `source_report`: `configured_ios_deploy_runbook`
- `status`: normalized from the source bundle status
- `actions`: sanitized copies of source runbook device actions
- `first_action`: sanitized source first action when present, otherwise first non-ready copied action
- `summary`: counts for actions, ready, missing, blocked, manual, partial, live, provider calls, global actions, and Xcode/signing actions
- `safety`: all false for command execution, provider calls, global mutation, Keychain writes, backend env writes, iOS deploy config writes, and Xcode/signing execution from the report builder

If the nested runbook bundle is missing, the configured preflight report returns a blocked empty bundle with `source_report: "missing"` and `summary.missing == 1`.
