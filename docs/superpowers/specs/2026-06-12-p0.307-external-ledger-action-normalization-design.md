# P0.307 External Ledger Action Normalization Design

## Goal

Make `final-external-action-ledger.json` top-level `operator_actions` directly
copy-pastable by removing ledger-internal `unblock <action_id>:` wrappers.

## Current Context

The final external action ledger groups all non-repo or user-provided work that
still blocks the iOS showcase: missing secrets, safe local writes, live provider
cost consent, global Mac/Xcode actions, and device runtime actions.

The structured rows already carry the useful provenance:

```json
{
  "id": "preview_final_resource_apply",
  "group_id": "safe_local_writes",
  "status": "blocked",
  "command": "make final-resource-apply-preview"
}
```

But the top-level handoff currently wraps blocked commands:

```text
unblock preview_final_resource_apply: make final-resource-apply-preview
unblock apply_final_resources: make final-apply-resources
unblock run_mobile_deploy_preflight: make mobile-deploy-preflight
```

That is less useful for a human operator than the command itself.

## Design

Normalize only the top-level `operator_actions` list:

- for blocked action rows, append `normalize_operator_action(action["command"])`;
- keep missing-resource actions unchanged because they already include destination
  and validation commands;
- keep live-cost and manual/global confirmation wording unchanged because those
  intentionally require explicit human consent before running commands;
- keep `action_groups`, `actions_by_id`, `operator_sequence`, source summaries,
  status, and safety fields unchanged.

This keeps the ledger auditable while making the top-level handoff list
copy-pastable.

## Out of Scope

- No live provider calls.
- No final resource writes.
- No Xcode, signing, Keychain, physical iPhone, payment, or fulfillment action.
- No change to ledger grouping or readiness semantics.

## Verification

- RED/GREEN tests that blocked safe-local-write and device-runtime actions appear
  as concrete commands in top-level `operator_actions`.
- Regression test that structured action rows still retain their ids and
  commands.
- Refresh `make final-external-action-ledger`; expect exit 2 while external
  inputs remain blocked, but no top-level action starts with `unblock `.
- Run focused tests, lint, whitespace check, and backend tests before merging.
