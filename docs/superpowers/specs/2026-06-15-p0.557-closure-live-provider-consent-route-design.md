# P0.557 Closure Live Provider Consent Route Design

## Context

The final launch closure packet is the operator-facing handoff for the end of the local demo path. The current report correctly marks live Meshy/OpenAI/configured acceptance as requiring consent, but its top-level `operator_actions` can still include direct `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 ...` commands.

Those commands are not executed by the report, but they are too direct for the final handoff surface. The safer top-level action is the configured evidence gate:

```text
make final-configured-preflight; rerun make configured-live-evidence-bundle
```

That route keeps the user on the resource/preflight/consent path before any live provider action.

## Design

P0.557 adds a closure-packet action normalization step after existing provider, device, and print preferences. The step replaces any top-level closure operator action that contains `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 ` with the configured evidence gate command. It also folds the bare `make configured-live-evidence-bundle` action into the same validated chain when the closure packet is producing a final operator list. It keeps the live provider consent section details unchanged so the report still explains which configured checks need eventual approval.

The normalization preserves action order and dedupes the configured evidence gate if several live-provider actions are present. It does not run provider calls, write secrets, touch Xcode signing, or change `.local` resource files.

## Acceptance

- A focused unit test proves `_operator_actions()` replaces direct `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 ...` actions with `make final-configured-preflight; rerun make configured-live-evidence-bundle`.
- The test proves no top-level closure operator action contains `PMF_ALLOW_LIVE_PROVIDER_CALLS`.
- A refreshed `services/backend/.local/final-launch-closure-packet.json` contains the configured evidence gate and no top-level `PMF_ALLOW_LIVE_PROVIDER_CALLS` action.
- The refreshed closure packet contains no bare top-level `make configured-live-evidence-bundle` action when the validated configured evidence gate is present.
- Existing closure packet behavior for provider handoff, print handoff, and iOS deploy handoff remains covered by the existing tests.

## Safety

This phase changes report text only. It does not execute live provider calls, does not write backend provider env files, does not write iOS deploy config, and does not mutate global Mac/Xcode state.
