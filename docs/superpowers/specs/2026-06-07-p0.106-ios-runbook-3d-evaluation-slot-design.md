# P0.106 iOS Runbook 3D Evaluation Slot Design

## Goal

Make the iOS deploy runbook explicitly require the local 3D evaluation readiness report before a final physical-device demo handoff.

## Problem

The project now has 3D evaluation readiness in Final Launch Status, Final Showcase, Demo Script, Autopilot, and Final Operator Handoff. The iOS deploy runbook still lists final resources, backend/iOS deploy slots, local final acceptance, and NPC Agent evaluation, but it does not list the 3D evaluation report as an input slot.

That means the deploy runbook can guide the operator toward an iPhone build without naming the 3D provider quality evidence that proves Meshy/local 3D generation output is ready for the demo path.

## Scope

In scope:

- Add a required `three_d_evaluation` input slot to `ios_deploy_runbook`.
- Read the existing `three_d_evaluation_readiness` report from `services/backend/.local/3d-evaluation-local.json`.
- Include missing/blocked 3D evaluation operator actions in the runbook actions.
- Keep all runbook behavior read-only: no commands run, no provider calls, no global mutation.
- Update backend runbook tests, Swift final launch payload fixture, Swift contract tests, ProjectChecks, and backend iOS showcase acceptance gates.
- Add a small mobile visual regression fixture showing the new iOS runbook row.

Out of scope:

- Running `evaluate-3d`.
- Calling Meshy or any live provider.
- Changing the 3D evaluation report schema.
- Changing Xcode signing, Apple SDK license, or device deployment configuration.

## Behavior

When 3D evaluation readiness is missing:

- `ios_deploy_runbook.input_slots` contains `three_d_evaluation` with status `missing`.
- The runbook status is `blocked`.
- `operator_actions` includes `run local 3D evaluation with evaluate-3d and write services/backend/.local/3d-evaluation-local.json`.
- iPhone Final Launch runbook rows surface the missing `three_d_evaluation` row before less relevant deploy inputs when it is one of the first missing items.

When 3D evaluation readiness is ready:

- The slot status is `ready`.
- Ready operator action text `3D evaluation is ready` is not repeated as an action.
- Local runbook status can remain `partial` because the Xcode build gate is still manual.

When 3D evaluation readiness is blocked/failed:

- The slot status is `blocked`.
- Operator actions include the sanitized 3D evaluation repair action.
- Reports never expose provider secrets, raw media, local paths, payment links, or raw personal context markers.

## Acceptance

- `services/backend/tests/test_ios_deploy_runbook.py` covers missing, ready, and blocked 3D evaluation slots.
- Swift contract tests decode the slot from final launch payload and show it in `deployRunbookRows`.
- Source gates require the `three_d_evaluation` slot and tests.
- Full backend lint/test and Swift contract/project/app compile gates pass.
- Final acceptance quick remains allowed to exit 2 only for known local iOS/Xcode blockers, with zero failed checks.
