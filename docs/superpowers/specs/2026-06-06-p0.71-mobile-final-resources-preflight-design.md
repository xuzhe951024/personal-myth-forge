# P0.71 Mobile Final Resources Preflight Design

## Context

P0.70 added a backend `final_resources_preflight` report to
`final-demo-launch`. The iPhone app already loads the final launch report for
the Device Preflight panel, but it only shows one broad `Final Launch` row. The
operator cannot see from the phone whether the unified final resources file is
missing, blocked, or ready.

## Goal

Surface backend `final_resources_preflight` status inside the mobile Device
Preflight summary so the iPhone operator can see final resource readiness before
running the final apply step.

## Non-Goals

- Do not put provider keys in iOS source, app state, logs, screenshots, or UI.
- Do not let iOS read or write `final-resources.env`.
- Do not call Meshy, OpenAI, Treatstock, Apple, Xcode, or signing services.
- Do not change the backend preflight behavior.
- Do not add a new screen; keep this inside the existing Device Preflight panel.

## Behavior

Add Swift models for the sanitized backend preflight shape:

- `FinalResourcesPreflightReport`
- `FinalResourcesFileStatus`
- `FinalResourcesPreflightSummary`

Extend `FinalDemoLaunchReport` with optional
`finalResourcesPreflight: FinalResourcesPreflightReport?`.

Update `DevicePreflightSummaryBuilder`:

- Add an item id `final_resources`.
- Label it `Final Resources`.
- If no final launch report has loaded, status is waiting.
- If `final_resources_preflight.status == ready`, status is ready and detail
  says the final resources file is ready to apply.
- If status is `missing`, status is waiting and detail says the final resources
  file is missing.
- If status is `blocked`, status is blocked and detail says the final resources
  file is blocked.
- If preflight is absent from an older backend report, status is waiting and
  detail says the final resources preflight has not loaded.
- All details pass through existing sanitization.

The existing `Final Launch` row remains as the broader launch-lane summary.

## Acceptance

- Swift decodes `final_resources_preflight` from `FinalDemoLaunchReport`.
- Device Preflight summary includes `final_resources`.
- Missing preflight file maps to waiting, not blocked, so local no-key demo
  remains usable while resources are pending.
- Blocked preflight maps to blocked.
- Ready preflight maps to ready.
- Unsafe backend details are redacted.
- Source acceptance checks require the new model and summary item.
