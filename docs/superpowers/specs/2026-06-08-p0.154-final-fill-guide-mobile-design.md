# P0.154 Final Resource Fill Guide Mobile Design

## Problem

The project already has a backend `final_resource_fill_guide` report that turns
the final one-file resource bundle into operator-friendly required, optional,
and configured input rows. The `/v1/final-demo-launch` payload and iOS Final
Launch panel currently expose resource preflight, resource requirements, and
apply preview rows, but they do not expose this fill guide directly.

That leaves a gap for the final key handoff: when the user is ready to provide
Meshy, OpenAI, Treatstock, Apple Team ID, bundle id, and iPhone backend URL
values, the iPhone status panel can show that resources are missing but not the
more explicit fill-source and command-order summary.

## Goal

Expose the sanitized final resource fill guide through the final demo launch
payload and render a compact `Resource Fill Guide` section in the iOS Final
Launch panel.

## Non-Goals

- Do not read, create, or store real provider keys.
- Do not write `.env`, `Deployment.local.xcconfig`, Xcode, signing, keychain, or
  global machine state.
- Do not call Meshy, OpenAI, Treatstock, Sculpteo, Apple, or any live provider.
- Do not change the final resource apply semantics.
- Do not add new required resources.

## Design

Backend:

- Import `build_final_resource_fill_guide_report` in `final_demo_launch.py`.
- Build the fill guide report next to resource requirements and apply preview.
- Add it to the sanitized final launch payload under
  `final_resource_fill_guide`.
- Keep the existing `final_resource_fill_guide` sanitizer as the only source of
  fill-guide text so paths and secrets remain redacted before mobile decoding.

iOS core:

- Add Codable models for `FinalResourceFillGuideReport`, summary, guide item,
  and safety to `PMFModels.swift`.
- Add an optional `finalResourceFillGuide` field to `FinalDemoLaunchReport`.
- Add `resourceFillGuideRows` to `FinalLaunchMobileSummary`.
- Build rows from the fill guide:
  - summary row: status, required, optional, configured, secret counts
  - up to three required or blocked inputs with id, status, destination, and
    validation command
  - first command row from the fill-guide command order
  - safety row proving no writes or live calls happened

iOS app:

- Render `resourceFillGuideRows` in `FinalLaunchStatusView` under
  `Resource Fill Guide`, near the existing resource sections.

## Safety

The report remains read-only. It must not expose actual resource values, local
absolute paths, bearer tokens, provider keys, raw media, or private LAN URLs.
The iOS panel must show fill instructions and command names only.

## Testing

Use TDD:

- Backend red test: final demo launch payload must include
  `final_resource_fill_guide` with required input counts and no unsafe values.
- Swift red test: contract fixture decoding must expose `finalResourceFillGuide`
  and the mobile summary must produce `Resource Fill Guide` rows.
- Project checks red test: static source checks must require the new model,
  summary rows, and view title.
- Then implement the minimal backend and Swift changes.
- Verify focused backend tests, Swift contract tests, Swift project checks, app
  compile check, backend lint/test, and final local report refresh.
