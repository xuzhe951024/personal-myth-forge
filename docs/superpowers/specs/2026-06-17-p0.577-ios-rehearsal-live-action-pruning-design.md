# P0.577 iOS Rehearsal Live Action Pruning Design

## Context

P0.576 separated live acceptance from the final handoff index local rehearsal
lane. After refreshing the safe reports, `final-handoff-index.json` now points
to the local device rehearsal action, but
`ios-device-launch-rehearsal-readiness.json` can still preserve saved live
acceptance actions such as:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make final-acceptance-configured; rerun make live-provider-evidence
```

Those actions do not belong in the iOS device rehearsal readiness lane. They
can make the device readiness `next_action` look like a live provider command,
even when the actual device blocker is local final acceptance or iPhone
rehearsal evidence.

## Goal

Prune live-acceptance-only actions from iOS device launch rehearsal readiness
reports while preserving concrete local/device actions and provider setup
handoffs.

## Approach

Add source-prefix-aware live acceptance detection to
`myth_forge_api.ios_device_launch_rehearsal_readiness`.

Filter live-acceptance-only text from:

- saved top-level `operator_actions`
- saved sequence row `operator_actions`
- preferred device operator action fallback selection

Actions containing `PMF_ALLOW_LIVE_PROVIDER_CALLS=1`,
`final-acceptance-configured`, or bare `make live-provider-evidence` are
considered live-acceptance-only. Provider setup handoffs such as
`make provider-handoff; rerun make live-provider-evidence` and
`make final-resource-fill-guide; ... provider-handoff; ... live-provider-evidence`
remain valid because they are setup/review commands, not live acceptance.

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, or any live provider.
- Do not run Xcode signing, Keychain, or global Mac mutations.
- Do not mark iOS device deploy evidence ready.
- Do not remove the dedicated live acceptance path from final showcase or
  configured capability reports.

## Verification

  `operator_actions`, sequence row actions, `next_action`, and device action

- Keep provider setup handoff coverage green.
- Run focused iOS rehearsal/final showcase/final handoff tests.
- Run lint, `git diff --check`, and backend tests as needed.
- Refresh safe local reports and inspect that
  `ios-device-launch-rehearsal-readiness.json` no longer exposes live
