# P0.306 Configured Evidence Action Normalization Design

## Goal

Make configured provider evidence reports expose concrete top-level handoff
commands instead of internal `unblock ...` dependency labels.

## Current Context

The final showcase still cannot be marked ready because real/configured provider
evidence is blocked by missing user-supplied resources and live provider proof.
The relevant local source report,
`services/backend/.local/configured-live-evidence-bundle.json`, currently emits
operator actions such as:

```text
unblock provider_handoff before configured evidence bundle
unblock final_configured_preflight after final_apply_resources
unblock three_d_evaluation_configured after final_configured_preflight
```

Those strings are useful as internal dependency labels, but they are poor
handoff text for the operator who needs concrete next commands. The configured
evidence plan already stores the canonical command for each step, and Makefile
targets exist for each handoff.

## Design

Normalize configured evidence top-level actions at the report boundary:

- extend shared `operator_actions.normalize_operator_action()` with configured
  evidence unblock mappings;
- route `final_configured_evidence_plan.operator_actions` through that shared
  normalizer;
- route `configured_live_evidence_bundle.operator_actions` through that shared
  normalizer while leaving `commands` untouched;
- preserve source reports, command sequence rows, safety flags, status, consent
  policy, and blocker metadata.

Canonical mappings:

```text
unblock final_configured_preflight after final_apply_resources -> make final-configured-preflight
unblock provider_handoff after final_configured_preflight -> make provider-handoff
unblock three_d_evaluation_configured after final_configured_preflight -> make backend-evaluate-3d-configured
unblock npc_evaluation_configured after final_configured_preflight -> make backend-evaluate-npc-configured
unblock final_acceptance_configured after final_configured_preflight -> make final-acceptance-configured
unblock final_demo_launch_configured after final_configured_preflight -> make final-demo-launch-configured
unblock live_provider_evidence after final_configured_preflight -> make live-provider-evidence
```

## Out of Scope

- No live Meshy/OpenAI/Treatstock calls.
- No resource file writes.
- No Xcode, signing, Keychain, physical iPhone, payment, or fulfillment action.
- No change to configured evidence readiness semantics or cost-consent gates.
