# P0.74 Mobile Artifact Actions Design

## Goal

Make the iPhone demo explicitly show what the generated 3D artifact can do next:
open in the SceneKit preview, share a cached file, retry a failed download, or
wait for GLB/GLTF conversion.

## Context

P0.15 added the underlying generated-asset handoff: the app can download and
cache backend-returned 3D files, prefer `ios_scene_asset` variants, and pass
local USDZ/USD/DAE/SCN files to SceneKit. The current UI shows a status line, but
it does not turn that status into operator-visible actions. For the final mobile
demo, this leaves a gap: a user can see a generated asset URI but cannot quickly
tell whether it is directly previewable, shareable, blocked by conversion, or
recoverable with a retry.

## Design

Add a mobile core action builder:

- `ArtifactHandoffActionKind`: `openScene`, `shareCachedAsset`, `convertRequired`,
  `retryDownload`, and `waiting`.
- `ArtifactHandoffActionStatus`: `waiting`, `ready`, and `attention`.
- `ArtifactHandoffAction`: a small safe display row with title, detail, enabled
  state, and primary flag.
- `ArtifactHandoffActionSummary`: the title/detail plus derived booleans
  `canOpenScene` and `canShareCachedAsset`.
- `ArtifactHandoffActionBuilder.build(session:preparedAsset:)`.

The builder consumes `MythSession?` and `PreparedArtifactAsset?` instead of
running network or file operations. It is deterministic and testable under
SwiftPM. It never exposes provider keys or full local filesystem paths; details
use safe filenames or redacted text.

## State Mapping

- No session: waiting for forging.
- Session but no prepared asset: preparing generated asset.
- `.localSceneReady` or `.cachedSceneReady`: SceneKit handoff ready; share is
  enabled when a cached URL exists.
- `.cachedRequiresConversion`: cached GLB/GLTF is shareable, but SceneKit direct
  open is blocked until conversion.
- `.downloadFailed`: retry is the primary action.
- `.unsupportedURI`, `.unsupportedFormat`, `.awaitingAsset`, `.awaitingFormat`,
  and `.cancelled`: show one disabled waiting or attention row.

## App Wiring

Add `ArtifactHandoffActionsView` below the SceneKit preview status in
`Artifact3DPreviewView`. The view receives the summary, optional cached URL, and
a retry closure. It renders:

- compact status rows for non-clickable states;
- `ShareLink` for shareable cached files;
- a retry button that reruns asset preparation for the current session.

The app does not open raw remote URLs and does not introduce provider keys or
global configuration. Runtime cache writes remain inside the app sandbox through
the existing `ArtifactAssetPreparer.live()`.

## Out Of Scope

- Implementing GLB-to-USDZ conversion.
- Launching Quick Look or a separate AR preview controller.
- Calling Meshy/OpenAI/Treatstock.
- Device signing, simulator builds, or Apple SDK license changes.

## Testing And Evidence

- Swift core contract tests cover each action mapping plus redaction.
- Project checks require the core builder, SwiftUI action view, `ShareLink`, and
  retry wiring.
- App compile check ensures SwiftUI/SceneKit source still builds under SwiftPM.
- iOS showcase acceptance gains a source gate for artifact action visibility.
- Static 390x844 evidence shows SceneKit ready, share cached asset, conversion
  required, and retry states with no horizontal overflow.
