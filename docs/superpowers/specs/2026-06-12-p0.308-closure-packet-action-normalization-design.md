# P0.308 Closure Packet Action Normalization Design

## Goal

Make `final-launch-closure-packet.json` top-level `operator_actions`
validation-aware for missing final resource inputs.

## Current Context

The closure packet is the final roll-up for the iOS demo handoff. It combines
resource inputs, safe local writes, iOS device evidence, configured provider
evidence, and final acceptance.

The external action ledger already emits concrete actions:

```text
provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight
provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
```

But the closure packet compresses those rows and drops `destination`, so its
top-level actions become weaker:

```text
provide MESHY_API_KEY
provide DEVELOPMENT_TEAM
```

That makes the final closure packet less useful as an operator handoff.

## Design

Preserve validation metadata through the closure packet:

- copy `destination` from compacted external ledger actions into closure actions;
- include `destination` in closure action rows only when present;
- render `provide_*` top-level actions as
  `provide <slot> in <destination filename>; rerun <command>`;
- preserve existing action ids, statuses, first blocker, section summaries,
  source reports, safety flags, and sanitization behavior;
- leave live-cost and global/manual confirmation wording unchanged.

## Out of Scope

- No real provider calls.
- No final resource writes.
- No Xcode, signing, Keychain, physical iPhone, payment, or print fulfillment
  action.
- No changes to final readiness semantics.

## Verification

- RED/GREEN test that closure packet top-level resource actions include target
  file and validation command.
- RED/GREEN test that compacted closure action rows retain destination metadata
  for UI/audit consumers.
- Refresh `make final-launch-closure-packet`; expect exit 2 while external
  inputs remain blocked, but no top-level weak `provide <slot>` actions remain.
- Run focused tests, lint, whitespace check, and backend tests before merging.
