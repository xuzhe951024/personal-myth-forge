# P0.479 Final Live Action Operator Consent Design

## Context

Configured Meshy/OpenAI Make targets are now protected by `services/backend/scripts/require_live_provider_consent.sh`. `live-provider-evidence` also renders consent-prefixed operator actions when its first blocker is a live provider step. The final external action ledger and final launch closure packet still expose Meshy/OpenAI live actions with bare Make targets and no operator-action consent prefix.

## Goal

Make final operator handoff reports show `PMF_ALLOW_LIVE_PROVIDER_CALLS=1` for configured Meshy/OpenAI live actions while preserving canonical Make target commands for structured clients.

## Scope

- Add consent-prefixed `operator_action` values for:
  - `run_configured_3d_evaluation`
  - `run_configured_npc_evaluation`
  - `run_configured_final_acceptance`
- Preserve canonical `command` fields such as `make backend-evaluate-3d-configured`.
- Let `final_launch_closure_packet` inherit those operator actions from `final_external_action_ledger`.
- Do not add the Meshy/OpenAI consent env prefix to Treatstock quote handoff. Treatstock cost consent remains a separate print-provider handoff.
- Do not run Meshy, OpenAI, Treatstock, Xcode, signing, Keychain, or global environment actions.

## Behavior

In `final-external-action-ledger.json`, configured 3D, configured NPC, and configured final acceptance actions keep their `command` values but gain `operator_action` values:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-3d-configured
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-npc-configured
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make final-acceptance-configured
```

In `final-launch-closure-packet.json`, the matching compacted actions carry the same `operator_action` values.

## Testing

- Add failing Python tests for final external action ledger and final launch closure packet operator actions.
- Keep tests proving canonical `command` fields are unchanged.
- Add source acceptance and Swift ProjectChecks coverage for the consent prefix in final external action ledger source.
- Run focused Python tests, Swift contract tests, Swift ProjectChecks, backend lint, and `git diff --check`.

## Safety

This phase only changes generated report fields and tests. It must not set `PMF_ALLOW_LIVE_PROVIDER_CALLS`, run configured Make targets, read provider secrets, or write tracked secrets.
