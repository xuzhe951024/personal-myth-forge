# P0.46 Resource Template Acceptance Design

## Product Intent

The final demo needs a clean handoff point where the operator can provide the real
resources at the end: provider keys, local backend runtime settings, and iPhone
deployment settings. P0.40 and P0.45 already report which resources are missing,
but they do not prove that the checked-in templates still expose every required
slot.

P0.46 adds a machine-verifiable resource template acceptance gate. It answers:

- Are all backend provider/configuration slots present in `.env.example`?
- Are all iPhone deployment slots present in
  `apps/mobile/ios/Config/Deployment.local.xcconfig.example`?
- Are the real local destination files ignored by git?
- Are the templates and report safe to commit, with no secrets, raw media, local
  absolute paths, or payment links?

This is a source-only acceptance gate. It must not call Meshy, OpenAI, print
providers, object storage, Xcode, or network services.

## Scope

Backend template slots:

- `THREE_D_PROVIDER`
- `MESHY_API_BASE_URL`
- `MESHY_POLL_INTERVAL_SECONDS`
- `MESHY_MAX_WAIT_SECONDS`
- `MESHY_API_KEY`
- `NPC_PROVIDER`
- `OPENAI_NPC_MODEL`
- `OPENAI_API_BASE_URL`
- `OPENAI_API_KEY`
- `PRINT_PROVIDER`
- `TREATSTOCK_API_KEY`
- `SCULPTEO_API_KEY`
- `CAPTURE_STORAGE_DIR`
- `MYTH_SESSION_STORAGE_DIR`

iPhone deployment template slots:

- `DEVELOPMENT_TEAM`
- `PRODUCT_BUNDLE_IDENTIFIER`
- `PMF_BACKEND_BASE_URL`

Ignored local destinations:

- `.env`
- `services/backend/.env`
- `apps/mobile/ios/Config/Deployment.local.xcconfig`
- `services/backend/.local/`

## Acceptance Report

The report shape should be stable and JSON friendly:

- `kind`: `resource_template_acceptance_report`
- `status`: `succeeded` or `failed`
- `summary`: counts for passed/failed checks
- `backend_template`: path, destination, required keys, missing keys
- `ios_template`: path, destination, required keys, missing keys
- `gitignore`: ignored local destination entries and missing entries
- `safety`: booleans proving no secrets, raw media, local paths, payment links,
  provider calls, or global mutation were introduced

Exit codes:

- `0`: all template checks passed
- `1`: source/template defect

## Final Acceptance Integration

`final-acceptance` should run this gate before local-machine blockers such as
iOS signing and Xcode license checks. In the current local environment the quick
profile should move from:

- `passed=6, blocked=2, failed=0`

to:

- `passed=7, blocked=2, failed=0`

The two expected blockers remain local/operator owned:

- `mobile_deploy_preflight`
- `mobile_xcode_build`

## Non-Goals

- Do not validate that real key values are present.
- Do not call live providers.
- Do not add provider setup wizards.
- Do not change iOS runtime behavior.
- Do not push or publish the repository.
