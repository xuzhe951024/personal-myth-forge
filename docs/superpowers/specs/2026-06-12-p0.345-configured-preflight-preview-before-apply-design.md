# P0.345 Configured Preflight Preview Before Apply Design

## Context

`final-configured-preflight.json` is the configured-mode gate before live
provider evidence, configured launch, and final acceptance. It aggregates final
resources, provider handoff, resource handoff, configured launch, and iOS
runbook reports.

When final resources are ready but real providers are not ready, configured
preflight still exposes direct local writer action:

```text
make final-apply-resources
```

That writer remains useful after preview review, but the configured preflight
handoff should first route the operator through:

```text
make final-resource-apply-preview
```

## Goal

Make configured preflight prefer the read-only final resource apply preview over
direct resource apply writer actions in top-level `operator_actions`.

## Non-Goals

- Do not remove `make final-apply-resources` from `commands`.
- Do not change resource writer scripts or make targets.
- Do not execute live provider calls, Xcode signing, Keychain writes, resource
  writers, backend `.env` writes, iOS deploy config writes, or print orders.
- Do not change configured acceptance cost-consent behavior.

## Design

Add a configured preflight constant:

```text
make final-resource-apply-preview
```

When final resources are ready but provider handoff is not ready, append this
preview target instead of `make final-apply-resources`.

After existing normalization/deduplication, add a small filter: if the top-level
actions contain the preview target, drop exact `make final-apply-resources`
roots. This also suppresses nested direct apply actions that may arrive from
configured launch or iOS runbook reports.

## Acceptance Criteria

- The provider-missing configured preflight path includes
  `make final-resource-apply-preview`.
- The same top-level action list does not include direct
  `make final-apply-resources` while preview is present.
- `commands` still includes `make final-apply-resources`.
- Existing configured acceptance cost review actions remain visible.
- Focused configured preflight tests, operator action tests, lint, diff checks,
  and backend tests pass.
