## Purpose

P0.203 made device preflight prefer `final_resource_fill_guide.first_blocker`. The preflight item still uses the shared sanitizer, but existing unsafe regression coverage only exercised the older `requiredInputs.first` fallback path.

## Scope

- Add a Swift contract test for unsafe `FinalResourceFillGuideFirstBlocker.command` and `detail`.
- Keep the existing production redaction path unchanged unless the regression test exposes a gap.
- Add mobile final launch readiness source gates requiring the new contract test.
- Add ProjectChecks coverage so local mobile project checks fail if the first blocker redaction contract is removed.

## Acceptance

- Backend mobile final launch readiness exposes `contract_final_resource_fill_guide_first_blocker_redaction`.
- Swift contract tests prove first blocker detail redacts provider keys, bearer tokens, local paths, local capture URLs, checkout text, and `api_key` snippets.
