# P0.558 External Ledger Live Consent Route Design

## Context

The final external action ledger is one of the main handoff sources for the final iPhone demo. It already separates live provider cost actions from safe local writes and device actions, but the top-level `operator_actions` list can still contain direct `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 ...` commands.

The detailed `live_provider_costs` action group should continue to show the exact Meshy/OpenAI/configured acceptance commands and cost-consent metadata. The top-level operator list should be safer: it should send the operator through the configured preflight and evidence bundle gate before any live provider run.

## Design

P0.558 adds a final normalization step to `final_external_action_ledger._operator_actions()`. The step replaces any top-level action containing `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 ` with:

```text
make final-configured-preflight; rerun make configured-live-evidence-bundle
```

The helper preserves ordering and dedupes the configured gate when multiple live provider actions are present. It intentionally does not mutate the nested `live_provider_costs` actions, because those entries are the audit trail that explains what will call Meshy/OpenAI and why cost consent is required.

## Acceptance

- A focused RED/GREEN unit test proves `_operator_actions()` collapses multiple direct live provider top-level actions into one configured gate action.
- The same test proves no top-level external ledger operator action contains `PMF_ALLOW_LIVE_PROVIDER_CALLS`.
- A refreshed `services/backend/.local/final-external-action-ledger.json` has zero top-level direct live provider actions and one configured gate action.
- Existing external ledger grouping, first blocker, print handoff, and iOS deploy handoff behavior remains covered by existing tests.

## Safety

This phase changes reporting only. It does not execute commands, call providers, write backend env files, write iOS deploy config, touch Keychain, or change Xcode/signing state.
