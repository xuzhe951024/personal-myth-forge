# P0.38 Print Quote Handoff Design

## Context

The product direction includes turning the generated game asset into a printable
real-world relic and eventually quoting/fulfilling through third-party print
services. The current backend already returns a `PrintCandidate`, but the demo
has no quote contract, no print-provider configuration switch, and no final
acceptance proof that the print handoff path is usable.

P0.38 adds the first quote/handoff layer while keeping live third-party calls
out of default verification.

## Goal

Add a backend print quote contract and final acceptance check that proves a
generated print candidate can become a safe, user-approved quote handoff without
requiring provider keys or leaking secrets.

## Non-Goals

- Do not place real Treatstock, Sculpteo, Shapeways, or Materialise orders.
- Do not call third-party print APIs by default.
- Do not claim real print-provider readiness before a live quote adapter exists.
- Do not store provider keys in mobile code or committed files.
- Do not implement mesh repair, printability analysis, or operator review UI in
  this iteration.

## Design

Extend the domain model with a small quote contract:

- `PrintQuoteRequest`: print candidate plus user-facing print preferences.
- `PrintQuote`: quote/handoff result with provider, status, price, currency,
  lead-time estimate, approval requirement, and notes.

Extend `LocalPrintProvider` with `create_print_quote(...)`. The local provider
returns a deterministic quote stub for demo acceptance:

- provider: `local_stub`
- status: `draft_quote`
- currency: `USD`
- estimated price in cents
- estimated production and shipping days
- no checkout URL
- user approval required

Add `POST /v1/print-quotes` to the FastAPI app. The endpoint accepts a
`PrintQuoteRequest`, calls the print provider, and returns a safe `PrintQuote`.
It must not return provider keys, raw source media, local filesystem paths, or
checkout/payment identifiers.

Add backend settings and readiness fields:

- `PRINT_PROVIDER`, default `local`
- `TREATSTOCK_API_KEY`
- `SCULPTEO_API_KEY`

For P0.38, `PRINT_PROVIDER=local` is demo-ready but not real-provider-ready.
`PRINT_PROVIDER=treatstock` or `sculpteo` reports missing configuration if the
matching key is absent. If a key is present, readiness should report
`not_implemented` rather than `ready`, because live quote adapters are not part
of this iteration. This avoids overstating real fulfillment readiness.

## Final Acceptance

Add one in-process final acceptance check:

- `id: "print_quote_acceptance"`
- label: `Print quote acceptance`
- default behavior: local/no-key

The check creates a deterministic local generated asset, derives a
`PrintCandidate`, creates a quote with the local print provider, and reports:

- print candidate format
- quote status
- provider
- currency
- estimated price cents
- user approval requirement
- safety flags

After P0.38, local quick final acceptance should have four passing checks:

1. provider handoff readiness,
2. demo acceptance,
3. capture-to-3D acceptance,
4. print quote acceptance.

The same two machine-local blockers remain: iOS deploy preflight and Xcode
license/build gate.

## Acceptance Criteria

- `POST /v1/print-quotes` returns a deterministic local quote for a print
  candidate.
- `LocalPrintProvider` can create a quote without provider keys.
- Provider readiness exposes print provider key slots without leaking key
  values or claiming unimplemented live adapters are ready.
- `final-acceptance --profile quick --provider-mode local` includes
  `print_quote_acceptance`.
- Local final acceptance quick reports `passed=4`, `blocked=2`, `failed=0`.
- Reports and API responses contain no provider secrets, raw media payloads,
  local file paths, or payment/checkout identifiers.
