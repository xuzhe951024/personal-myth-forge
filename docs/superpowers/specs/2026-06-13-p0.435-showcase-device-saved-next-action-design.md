# P0.435 Final Showcase Device Saved Next Action Design

## Context

The final showcase readiness report is still blocked by `ios_deployable`. The
safe local handoff already exposes a copy-pasteable first operator action:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

The same report also exposes `device_action_bundle.first_action`, and the iPhone
Final Launch summary already knows how to render `saved_next_action` on device
action rows. The gap is that `final_showcase_readiness.device_action_bundle`
does not yet carry `saved_next_action`, so the final showcase device row loses
the richer "write config, then rerun validation" instruction even though it is
available in `operator_actions`.

## Decision

Add a sanitized `saved_next_action` to
`final_showcase_readiness.device_action_bundle.first_action` and to the matching
entry in `device_action_bundle.actions`. The iPhone Final Launch summary renders
the `actions` rows, while downstream reports also read `first_action`; both must
carry the same handoff. The payload uses the promoted first device action's
identity and metadata, but the command is the copy-pasteable operator action:

```json
{
  "id": "run_mobile_deploy_preflight",
  "label": "Run mobile deploy preflight",
  "status": "blocked",
  "command": "DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight",
  "detail": "Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable",
  "source": "operator_actions",
  "validation_command": "make mobile-deploy-preflight"
}
```

This keeps the device bundle's primary command stable while preserving the full
next handoff for mobile display.

## Scope

Included:

- Add a backend regression proving `final_showcase_readiness.device_action_bundle.first_action.saved_next_action`
  and the matching action row carry the full operator action and validation
  detail.
- Update `final_showcase_readiness.py` to attach the saved action after
  operator actions are computed.
- Add Swift contract coverage proving the existing iPhone summary renders this
  field for final showcase device rows.
- Update the roadmap.

Not included:

- No new provider calls, real Meshy/OpenAI/Treatstock evidence, Xcode signing,
  Keychain writes, or paid actions.
- No change to the primary device action command or first blocker ordering.
- No new mobile model fields; P0.433 already added `savedNextAction`.
