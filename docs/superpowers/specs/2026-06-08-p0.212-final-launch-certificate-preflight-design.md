# P0.212 Final Launch Certificate Preflight Design

## Purpose

P0.212 makes the existing iOS device launch certificate visible in the same final-demo-launch contract that powers the mobile Device Preflight. The certificate already exists as a backend CLI report, but the iPhone preflight currently sees it only indirectly through launch rehearsal/final showcase summaries.

## Scope

- Embed a sanitized `ios_device_launch_certificate` report in `final_demo_launch_report`.
- Add Swift Codable models for the certificate summary needed by mobile.
- Add a required Device Preflight item with id `ios_device_launch_certificate` and label `Launch Certificate`.
- Surface missing, blocked, partial, manual/live-consent, ready, operator action, first gate, and safety details.
- Add backend and Swift source gates so the contract stays wired.

## Behavior

- Missing certificate in the final launch payload maps to waiting with detail `iOS device launch certificate has not loaded.`
- `ready` maps to ready and shows summary, mode, first gate, and safety flags.
- `blocked`, `missing`, or `partial` maps to blocked/waiting as appropriate and shows the first operator action plus first non-ready required gate.
- `manual`/`live` non-required gates stay visible as detail, but only required gates decide the preflight ready state through the certificate status reported by the backend.
- Unsafe strings from actions, commands, notes, local paths, provider tokens, checkout/payment URLs, and local capture URIs are redacted by existing Device Preflight sanitization.

## Boundaries

P0.212 does not run Xcode, accept Apple licenses, mutate signing/keychain state, write backend `.env`, write iOS deploy config, or call live providers. It only reports the certificate that backend code can already build.

## Acceptance

- Backend `final_demo_launch_report` includes `ios_device_launch_certificate`.
- Swift decodes the new report from final launch payload fixtures.
- Device Preflight has a required `Launch Certificate` item after `Launch Rehearsal` and before `Final Closure`.
- Contract tests cover missing, blocked, ready, live-consent/manual detail, and redaction.
- `make final-rehearsal-local` remains no-key and no-global-mutation.
