# P0.337 iOS Writer Action Lists Design

## Context

P0.335 and P0.336 made the iOS deploy first blocker and final-showcase handoff point to the safe project-local writer command:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

Current `.local` evidence still shows the older manual action inside some top-level operator action lists:

```text
provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
```

The old action is still valid in lower-level contexts, but when the writer command is already present in a final operator list, showing both creates avoidable ambiguity for the physical iPhone handoff.

## Goal

When final demo launch and iOS device launch rehearsal reports already include the iOS deploy writer action, their top-level `operator_actions` should drop the superseded manual Team ID action.

## Non-Goals

- Do not write `Deployment.local.xcconfig`.
- Do not run Xcode, signing, Keychain, provider calls, or paid print actions.
- Do not remove the lower-level preflight diagnostic string `Missing DEVELOPMENT_TEAM in Deployment.local.xcconfig`.
- Do not hide final resource actions such as `provide DEVELOPMENT_TEAM in final-resources.env`.

## Design

Add a shared operator-action helper that removes the validated manual Team ID action only when the validated writer action is also present in the same action list. The helper compares command roots before any detail suffix, so detail-suffixed writer rows can still coexist with the plain writer row while the older manual Team ID row is suppressed.

Use the helper in:

- `final_showcase_readiness`, replacing its local writer preference rule.
- `final_demo_launch`, after existing validation-aware dedupe.
- `ios_device_launch_rehearsal_readiness`, after existing validation-aware normalization.

## Acceptance Criteria

- `final-demo-launch-local.json.operator_actions` starts with the writer command and does not include the older manual Team ID deploy action when the writer command is present.
- `final-demo-launch-configured.json.operator_actions` follows the same rule.
- `ios-device-launch-rehearsal.json.operator_actions` follows the same rule.
- Existing lower-level iOS deploy config actions and final resource `DEVELOPMENT_TEAM` actions remain available where they are distinct and actionable.
- Focused backend tests, lint, and full backend tests pass.
