# P0.467 Safe Report Make Wrappers Design

## Context

Several final handoff reports intentionally return exit code `2` when they successfully write a usable JSON report whose status is blocked. Existing local wrapper scripts accept that code for final acceptance, iOS deploy runbook, final local refresh, and iOS device rehearsal. Two important final handoff Make targets still call the CLI directly:

- `make final-external-action-ledger`
- `make final-launch-closure-packet`

Both write their `.local/*.json` output and then fail Make on expected blocked state. That interrupts safe local evidence refresh even though the report was produced.

## Decision

Add project-local wrapper scripts for those two report targets. The scripts should:

- run the existing CLI command with the same `--repo-root` and `--output`
- accept exit code `0` or `2`
- print the accepted status and written output path
- fail on any other exit code

Update the Make targets to call the wrappers. This matches the existing safe local rehearsal script pattern and does not change CLI semantics.

## Scope

- Add two shell scripts under `services/backend/scripts/`.
- Add focused script tests using the existing fake `uv` harness.
- Update Make target dry-run tests.
- Update roadmap.

## Out Of Scope

- No live provider calls.
- No Xcode signing, Keychain, or global machine mutation.
- No change to report status calculation.
- No change to final resource or deploy config writing.

## Acceptance

- Wrapper tests prove exit code `2` is accepted and exit code `1` fails.
- `make -n final-external-action-ledger` and `make -n final-launch-closure-packet` show wrapper scripts.
- Wrapper scripts are executable because Make runs them directly.
- Focused final rehearsal script tests pass.
- `make backend-lint` and `git diff --check` pass.
