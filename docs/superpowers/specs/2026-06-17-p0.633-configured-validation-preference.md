# P0.633 Configured Validation Preference Spec

## Goal

Keep final configured evidence handoff actions on the `final-configured-evidence-plan`
validation path when granular live provider evidence actions are also present.

## Problem

P0.632 made `live-provider-evidence` expose granular Meshy 3D and OpenAI NPC actions.
That source report correctly validates itself with `make live-provider-evidence`, but
higher-level final handoff reports should keep using
`make final-configured-evidence-plan` as the validation command. Without extra
normalization, configured bundle and final local refresh can expose duplicate 3D/NPC
actions with both validation suffixes or prefer the source-report suffix.

## Acceptance

- `configured-live-evidence-bundle.operator_actions` keeps the configured plan 3D/NPC
  actions and removes matching `live-provider-evidence` variants.
- `final-local-report-refresh.operator_actions` prefers
  `rerun make final-configured-evidence-plan` over `rerun make live-provider-evidence`
  for the same granular configured live action roots.
- Source `live-provider-evidence` still validates itself with `make live-provider-evidence`.
- No live providers are called and no secrets or global/Xcode state are written.
