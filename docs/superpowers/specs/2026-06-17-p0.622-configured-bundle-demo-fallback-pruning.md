# P0.622 Configured Bundle Demo Fallback Pruning Spec

## Context

`configured-live-evidence-bundle` is the source report for configured Meshy/OpenAI evidence handoff. When configured evidence is blocked on live provider consent, the report should show the provider evaluation consent sequence without mixing in demo-launch fallback commands.

After P0.621, downstream reports prune these fallbacks, but the configured bundle can still expose:

```text
make final-demo-launch-configured; rerun make final-configured-evidence-plan
```

This happens because the fallback detector only matches exact action strings and does not compare command roots after `; rerun ...`.

## Requirements

- Preserve configured evidence consent actions for 3D, NPC, and configured final acceptance.
- Drop configured demo-launch fallback actions when configured evidence consent actions are present.
- Treat validation-suffixed fallback actions the same as bare fallback roots.
- Do not call live providers, write secrets, run Xcode signing, or mutate global state.

## Acceptance

- A focused test proves validation-suffixed `make final-demo-launch-configured` is pruned.
- Existing configured live evidence bundle tests still pass.
- Safe `.local/configured-live-evidence-bundle.json` refresh no longer lists the configured demo-launch fallback when blocked on live provider consent.
