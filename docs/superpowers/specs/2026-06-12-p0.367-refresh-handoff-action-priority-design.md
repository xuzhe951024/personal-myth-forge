# P0.367 Final Local Report Handoff Action Priority Design

## Goal

Make `final-local-report-refresh` preserve the same final-demo handoff priorities as `final-showcase-readiness`: after the immediate first blocker and showcase device action, the top-level `operator_actions` should surface live provider handoff and Treatstock print quote handoff before generic resource refresh steps consume the 12-action cap.

## Current Evidence

- `final-showcase-readiness.json` already starts with the iOS deploy writer action, `make provider-handoff`, and the Treatstock configured quote handoff.
- `final-local-report-refresh.json` starts with final resource key actions, iOS deploy config, and provider handoff, but the Treatstock print quote handoff is outside the top 12 actions even though the `print_fulfillment_readiness` step has the correct command.
- The final local refresh report is a safe, repo-local handoff entry point. It should not hide one of the three remaining external handoff lanes: iOS deploy, provider keys, and print quote evidence.

## Behavior

`final-local-report-refresh` should:

- Keep `next_action` and `showcase_next_action` first, preserving the current first-blocker semantics.
- Promote provider handoff actions such as `make provider-handoff; rerun make live-provider-evidence` after those first actions.
- Promote configured print handoff actions such as `after explicit Treatstock cost consent, save a sanitized services/backend/.local/print-quote-configured.json from POST /v1/print-quotes; rerun make print-fulfillment-readiness` after provider handoff.
- Deduplicate and normalize actions before applying the 12-action cap.

## Non-Goals

- Do not run live Meshy/OpenAI/Treatstock calls.
- Do not write API keys, signing values, or device-specific secrets.
- Do not change which step is the first blocker.
- Do not make `final-local-report-refresh` treat blocked external setup as ready.

## Acceptance

- A focused test proves the print handoff survives the 12-action cap and appears before generic final resource guidance.
- Existing `final_local_report_refresh` tests remain green.
- Safe refresh still exits blocked when external provider, print, or iOS evidence is missing.
