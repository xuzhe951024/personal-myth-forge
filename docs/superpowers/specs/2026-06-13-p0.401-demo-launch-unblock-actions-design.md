# P0.401 Demo Launch Unblock Actions Design

## Context

The final iPhone showcase remains blocked by external iOS evidence and provider
credentials. `final-demo-launch-local` is one of the main operator handoff
reports for the physical iPhone demo.

Its top-level `operator_actions` already starts with concrete actions:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
make final-resource-apply-preview; rerun make live-provider-evidence
after explicit Treatstock cost consent, save a sanitized services/backend/.local/print-quote-configured.json from POST /v1/print-quotes; rerun make print-fulfillment-readiness
```

The same list still ends with vague fallback actions:

```text
unblock apply_final_resources: make final-resource-apply-preview
unblock mobile_deploy_preflight: make mobile-deploy-preflight
```

Those are less useful than the concrete child actions and can make the final
handoff look less deterministic.

## Goal

Remove vague `unblock ...` fallback roots from `final-demo-launch` top-level
operator actions when equivalent concrete actions are already available.

## Scope

- Change only `myth_forge_api.final_demo_launch` action projection and its tests.
- Preserve concrete preview, iOS deploy writer, provider, and print actions.
- Preserve command lists and nested report data.
- Do not change resource writers, Xcode/signing, live provider calls, or print
  quote behavior.

## Design

Add a final-demo-launch action filter after normalization and dedupe:

1. Drop `unblock apply_final_resources: make final-resource-apply-preview` from
   top-level `operator_actions`.
2. Drop `unblock mobile_deploy_preflight: make mobile-deploy-preflight` from
   top-level `operator_actions`.
3. Keep `make final-resource-apply-preview` or validation-aware variants.
4. Keep the concrete mobile deploy writer action and other
   `Deployment.local.xcconfig` actions.
5. If the original action list requested apply preview, preserve a top-level
   preview action root after dropping targeted fallback strings.
6. Keep any non-targeted `unblock` strings unchanged if future reports introduce
   a distinct action that is not covered by this phase.

## Tests

Update `services/backend/tests/test_final_demo_launch.py`:

- Add assertions to `test_local_final_demo_launch_operator_actions_do_not_skip_apply_preview`
  that both vague unblock strings are absent.
- Add a direct `_dedupe_operator_actions` regression test proving targeted
  unblock strings are removed while concrete preview and mobile writer actions
  remain in the list.

Focused verification:

```bash

```

Report verification:

