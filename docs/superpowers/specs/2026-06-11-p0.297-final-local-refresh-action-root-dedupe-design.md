# P0.297 Final Local Refresh Action Root Dedupe Design

## Goal

Keep `final-local-report-refresh` operator actions focused by removing
detail-suffixed duplicates when an equivalent bare command root already appears
in the same top-level handoff checklist.

## Current Context

The refreshed local handoff report can currently emit both of these actions:

```text
start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight
start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight | PMF_BACKEND_BASE_URL must be iPhone-reachable
```

Both point to the same executable next step. Keeping both in the top-level
operator checklist makes the final iPhone handoff look larger than it is and
weakens the first-action signal.

## Design

Extend only the `final_local_report_refresh._operator_actions()` dedupe path.

The helper should:

- normalize each raw step command through the existing `normalize_operator_action()`;
- keep exact de-duplication behavior;
- treat the text before ` | ` as the executable command root;
- if a bare root action exists, drop detail-suffixed actions with the same
  root;
- if no bare root action exists, keep the detail-suffixed action so blocker
  context is not lost.

This keeps P0.297 scoped to the final local refresh report. It does not change
lower-level reports, final showcase readiness, provider handling, iOS deploy
state, or external action semantics.

## Out of Scope

- No Xcode, signing, Keychain, iPhone deployment, provider calls, or paid print
  actions.
- No readiness status changes.
- No generic rewrite of operator action normalization.

## Verification

- RED/GREEN unit test for `_operator_actions()` proving a bare command root wins

- Focused `test_final_local_report_refresh.py` regression.
- `make final-local-report-refresh` still exits 2, while the generated

- Full lint, Swift checks, and backend tests before merge.
