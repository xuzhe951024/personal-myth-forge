# P0.216 Final Showcase Next Action Design

## Context

`make final-showcase-readiness` already summarizes whether the iOS-first final showcase is ready, partial, or blocked. It also emits a `first_blocker` and a long `operator_actions` list. The iPhone Final Launch panel renders those rows, but the next concrete action is inferred from unstructured text selection. This is enough for source coverage, but it makes the last handoff harder to verify and easier to regress.

## Goal

Add a structured, sanitized `next_action` to the final showcase readiness report and surface it on the iPhone Final Launch panel before the broader readiness action list.

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, Xcode, signing, keychain, or any live provider.
- Do not write `.env`, iOS deploy config, or global machine state.
- Do not change readiness semantics or mark resource-gated capabilities ready.

## Backend Contract

`final_showcase_readiness_report` gains:

```json
"next_action": {
  "id": "ios_deployable",
  "label": "iOS deployable",
  "status": "blocked",
  "classification": "ios_deploy_evidence",
  "command": "make ios-device-launch-rehearsal",
  "detail": "iOS deploy runbook and device launch rehearsal must both be ready.",
  "source": "first_blocker"
}
```

For ready reports, `next_action` is `null`. For non-ready reports, it is derived from `first_blocker` when available. If `first_blocker` is absent, it falls back to the first non-ready capability. All fields pass through the existing report sanitizer.

## Mobile Contract

Swift adds `FinalShowcaseReadinessNextAction` and decodes `final_showcase_readiness.next_action`. `FinalLaunchMobileSummary.showcaseReadinessRows` renders:

```text
Next action: ios_deployable blocked | make ios-device-launch-rehearsal | iOS deploy runbook and device launch rehearsal must both be ready.
```

The existing capability row remains as supporting context. The row is sanitized with the same path, secret, local media, checkout, and bearer-token redaction path used by current final launch summary rows.

## Source And Visual Gates

Source gates require the backend report key, Swift model, mobile summary row text, and a new visual artifact id. Visual regression adds one 390x844 static proof showing the final readiness next action, increasing the default artifact count from 23 to 24.
