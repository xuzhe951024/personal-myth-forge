# P0.436 Final Demo Device Saved Next Action Design

## Context

P0.435 preserved the copy-pasteable iOS device handoff inside
`final_showcase_readiness.device_action_bundle`. The generated local launch
report now has the richer nested action:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

The remaining gap is that `final_demo_launch.device_action_bundle` is a top-level
handoff for the iPhone Final Launch view, but it normalizes copied device action
rows without preserving `saved_next_action`. That means the nested readiness
source is correct while the top-level final demo handoff still loses the safe
"write config, then rerun validation" sequence.

## Decision

When `final_demo_launch.device_action_bundle` is sourced from
`final_showcase_readiness.device_action_bundle`, preserve a sanitized
`saved_next_action` on both:

- `device_action_bundle.first_action`
- the matching row in `device_action_bundle.actions`

The copied top-level action keeps its existing primary `command` and
`validation_command`; `saved_next_action` carries the longer operator sequence
for display and handoff.

## Scope

Included:

- Add backend regression coverage proving top-level final demo launch device
  actions preserve the nested final showcase `saved_next_action`.
- Update the final demo launch device action sanitizer to copy
  `saved_next_action`.
- Add Swift contract coverage for the iPhone Final Launch top-level final demo
  device action rendering.
- Update the roadmap.

Not included:

- No real Meshy, OpenAI, Treatstock, Xcode signing, Keychain, provider spend, or
  order actions.
- No change to first blocker ordering or the primary device action command.
- No new mobile model shape beyond the existing `savedNextAction` support.
