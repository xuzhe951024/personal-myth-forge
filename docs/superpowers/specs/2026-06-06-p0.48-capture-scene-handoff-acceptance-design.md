# P0.48 Capture Scene Handoff Acceptance Design

## Product Intent

P0.37 proved that guided scan images reach the 3D provider contract. P0.47
proved that local generated assets can be served back to iOS as backend HTTP
URLs. The missing proof is the integrated path that matters for the mobile
showcase:

1. Upload a guided scan capture.
2. Create a myth session from that capture.
3. Preserve multi-image generation provenance.
4. Return backend-served `game.glb` and `scene.dae` URLs that iOS can download.

P0.48 adds a source/local acceptance gate for that full path. It does not add
new product UI; it makes the current scan-to-3D-preview path harder to regress.

## Scope

Add `capture_scene_handoff_acceptance`:

- Use FastAPI `TestClient` against the real app.
- Use a temporary `LocalCaptureStore` patched into the app and restored after the
  run.
- POST `/v1/object-captures` with synthetic non-private JPEG/PNG bytes.
- POST `/v1/myth-sessions/from-capture`.
- Assert the returned session:
  - has `generation_provenance.input_mode == "multi_image"`
  - records two source images
  - returns `generated_asset.uri` as backend HTTP `game.glb`
  - returns `ios_scene_asset.uri` as backend HTTP `scene.dae`
  - has no raw media, `local-capture://`, local path, or provider secret in the
    report
- Download both returned generated asset URLs and confirm:
  - `game.glb` starts with `glTF`
  - `scene.dae` contains `COLLADA`

Wire this gate into quick/full final acceptance after `local_asset_handoff`.

## Expected Final Acceptance Change

Default quick final acceptance should move from:

- `passed=8, blocked=2, failed=0`

to:

- `passed=9, blocked=2, failed=0`

The remaining blockers stay local/operator-owned:

- iOS deploy config
- Apple SDK/Xcode license

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, Sculpteo, Apple services, or network
  endpoints.
- Do not persist real user photos or scan files.
- Do not change iOS UI.
- Do not configure signing or accept the Apple SDK license.
- Do not replace live Meshy quality validation.
