# P0.555 Print Auto Prerequisite Preference Design

## Context

The final print fulfillment path currently distinguishes two actions:

- A no-cost prerequisite that writes the ignored iOS deploy config with an iPhone-reachable backend URL, then reruns `make print-fulfillment-readiness`.
- A live provider action that allows `print-quote-configured`, which can call a third-party print provider.

Recent reports correctly expose the backend-auto prerequisite:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make print-fulfillment-readiness
```

The shared print action preference helper does not yet classify that backend-auto action as a print quote prerequisite. As a result, higher-level reports can keep the live print provider action when a safer prerequisite action is present.

## Goal

Teach the shared operator action helper to prefer the backend-auto print readiness prerequisite over direct live print quote calls.

## Scope

- Modify `services/backend/src/myth_forge_api/operator_actions.py`.
- Add focused tests in `services/backend/tests/test_operator_actions.py`.
- Refresh local final demo/report evidence if the generated reports change.
- Do not write secrets, call Treatstock/Sculpteo, run Xcode signing, or mutate global state.

## Design

Extend `CONFIGURED_PRINT_QUOTE_REQUEST_PREREQUISITE_ACTION_ROOTS` to include `MOBILE_WRITE_DEPLOY_CONFIG_AUTO_ACTION`. The existing `CONFIGURED_PRINT_QUOTE_REQUEST_PREREQUISITE_ACTIONS` tuple will then include:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make print-fulfillment-readiness
```

`prefer_guarded_print_quote_handoff_actions()` already prefers request prerequisites before configured print provider calls, so this keeps the implementation small and shared by final demo launch, final local report refresh, final showcase, and closure packet paths that use the helper.

## Testing

- Add a RED test proving `prefer_guarded_print_quote_handoff_actions()` chooses the backend-auto prerequisite over `PMF_ALLOW_PRINT_PROVIDER_CALLS=1 make print-quote-configured`.
- Run the focused operator action tests.
- Refresh local reports and verify `final-demo-launch-local` and `final-local-report-refresh` no longer retain the live print provider action when the backend-auto prerequisite is present.
- Run backend lint, full backend tests, and `git diff --check`.

## Safety

This change only affects emitted, sanitized operator instructions. It does not execute the instructions, write local ignored config, call live providers, place print orders, or expose secrets.
