# P0.201 Mobile Resource Blocker Readiness Design

## Purpose

P0.200 added iOS decoding and display for `final_resource_requirements.first_blocker`, but the backend mobile final launch readiness acceptance gate does not yet require that contract. Without the gate, a future change could remove the Swift model or row and still pass mobile readiness acceptance.

## Scope

- Add mobile readiness source requirements for `FinalResourceRequirementsFirstBlocker`.
- Require the optional `firstBlocker` field on `FinalResourceRequirementsReport`.
- Require `FinalLaunchMobileSummary` to source `report.finalResourceRequirements` and render `resourceRequirementFirstBlockerRow`.
- Require the Swift contract fixture/assertion that includes `provide MESHY_API_KEY in final-resources.env`.
- Keep runtime behavior unchanged.

## Acceptance

- `mobile_final_launch_readiness_acceptance` reports requirement IDs for the resource requirements first blocker model, field, summary source, summary row, and contract fixture.
- The minimal source fixture in `test_mobile_final_launch_readiness_acceptance.py` includes those tokens.
- Current repo source gates pass.
