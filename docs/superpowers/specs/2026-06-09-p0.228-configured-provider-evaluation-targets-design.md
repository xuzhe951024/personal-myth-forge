# P0.228 Configured Provider Evaluation Targets Design

## Context

The configured evidence plan, live provider evidence report, and external action
ledger currently list raw CLI commands for Meshy/OpenAI configured evaluations:

```text
cd services/backend && uv run python -m myth_forge_api.cli evaluate-3d --provider meshy --suite default-v0 --output .local/3d-evaluation-configured.json
cd services/backend && uv run python -m myth_forge_api.cli evaluate-npc --provider openai --suite default-v0 --tick-steps 2 --output .local/npc-evaluation-configured.json
```

Those commands are the core proof that real 3D generation and AI Agent NPC
runtime work after the operator provides keys. Keeping them as raw CLI text makes
the final runbook harder to follow and harder to validate consistently.

## Decision

Add first-class Make targets for configured provider evaluations:

- `make backend-evaluate-3d-configured`
- `make backend-evaluate-npc-configured`
- `make backend-evaluate-configured`

Then update configured evidence reports to reference those targets instead of
raw CLI commands. The targets may call live providers when run with real keys,
but this iteration only adds the entry points and source gates; tests use dry-run
and report generation only.

## Scope

- Add configured evaluation Make targets and output paths.
- Update final configured evidence plan, live provider evidence, and external
  action ledger command strings.
- Add Make dry-run/source-gate tests and Swift ProjectChecks coverage.
- Document the configured evaluation commands.

## Non-Goals

- Do not execute configured Meshy/OpenAI evaluations in verification.
- Do not require or store real provider keys.
- Do not change provider adapter behavior.
- Do not call Meshy, OpenAI, Treatstock, Sculpteo, Xcode, signing, keychain, or
  global machine settings.

## Acceptance

- `make -n backend-evaluate-3d-configured backend-evaluate-npc-configured
  backend-evaluate-configured` shows the expected configured output paths.
- Configured evidence reports list the new Make targets.
- Existing live provider evidence semantics and consent flags are preserved.
- Focused pytest, full backend tests, Swift contract tests, ProjectChecks, and
  `git diff --check` pass.
