# P0.501 Configured Preflight Backend URL Design

## Context

`final-configured-preflight.json` is the configured/live handoff entry point. It pulls actions from final resources, provider handoff, resource handoff, configured final launch, and the configured iOS deploy runbook.

The current report can emit both of these iOS backend URL actions:

- `set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL; rerun make mobile-deploy-preflight`
- `provide PMF_BACKEND_BASE_URL in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`

The first action is the current physical-iPhone contract. The second is a generic legacy action and weakens the final handoff because it does not tell the operator that loopback URLs are invalid for a real iPhone.

## Decision

Normalize `final_configured_preflight.operator_actions` so the LAN-reachable backend URL handoff is preferred over the generic Deployment.local.xcconfig backend URL action.

Use the existing shared `prefer_iphone_reachable_backend_url_handoff_actions()` helper instead of adding another report-specific rule. Keep the existing final resource validation, mobile deploy validation, deploy-writer preference, and unblock-fallback filtering behavior unchanged.

## Alternatives Considered

1. Patch only the fixture refresh output. This would not prevent the stale action from returning in future runs.
2. Add a local string filter inside `final_configured_preflight`. This would work, but it would duplicate shared operator-action logic.
3. Reuse the shared backend URL preference helper. This is the smallest durable fix and keeps normalization behavior consistent with other final handoff reports.

## Safety

This is report-only. It does not run Xcode, write signing config, touch Keychain, call Meshy/OpenAI/Treatstock, write ignored env/config files, or spend provider credits.

## Verification

TDD coverage must prove:

- `final_configured_preflight.operator_actions` contains the LAN-reachable backend URL action.
- The generic `provide PMF_BACKEND_BASE_URL in Deployment.local.xcconfig` action is absent after normalization.
- Existing configured preflight focused tests still pass.
- Refreshed `.local/final-configured-preflight.json` contains the normalized action list.
