# P0.627 Configured Evidence Action Prerequisites Spec

## Goal

Keep configured live-evidence handoff actions strictly executable in prerequisite order.

## Problem

After resources and providers are configured, the final configured evidence plan can
correctly identify Meshy 3D and OpenAI NPC evaluation as the first missing live
evidence. Its operator actions still include downstream configured final acceptance
and configured demo launch actions before the prerequisite 3D/NPC evidence exists.
Those downstream actions are valid later, but they make the current operator handoff
less precise.

## Acceptance

- With ready resources and missing configured 3D/NPC evidence, the final configured
  evidence plan operator actions include the Meshy 3D and OpenAI NPC consent commands.
- The same state does not surface configured final acceptance or configured demo
  launch operator actions.
- The configured live evidence bundle inherits the pruned action list.
- Once configured evidence exists, ready reports remain ready.
- The reports remain read-only and do not call live providers.
