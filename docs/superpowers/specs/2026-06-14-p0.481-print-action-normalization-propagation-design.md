P0.481 Print Action Normalization Propagation Design

Goal

Make every final handoff aggregation surface the guarded configured print quote handoff even when saved child reports still contain the old prose-only Treatstock instruction.

Context

P0.480 added `PMF_ALLOW_PRINT_PROVIDER_CALLS=1 make print-quote-configured` and taught the global operator-action normalizer how to translate the old prose action into this guarded target. The canonical `final-showcase-readiness` top-level actions now show only the guarded command, but other aggregation surfaces can still expose both the stale prose action and the guarded action when they read saved nested reports.

This matters because the final operator flow should not ask the user to manually save `.local/print-quote-configured.json` from `/v1/print-quotes` when the repo now provides a cost-consent guarded target that writes sanitized evidence.

Non-Goals

- Do not call Treatstock or any live print provider.
- Do not write provider keys, `.env`, Xcode signing, Keychain, or global state.
- Do not change the print quote API contract or readiness semantics.
- Do not mark print fulfillment ready without real configured quote evidence.

Approach

Add a shared helper in `operator_actions.py` that prefers the guarded print quote handoff root over legacy/source-prefixed variants. Use it after existing exact-dedupe and validation-aware normalization in final report aggregators that currently keep duplicate print handoff actions:

- `final_demo_launch`
- `final_local_report_refresh`
- `final_handoff_index`
- `ios_device_launch_rehearsal`
- `ios_device_launch_rehearsal_readiness`
- `ios_device_launch_certificate`

Acceptance Criteria

- A source-prefixed legacy action such as `final_demo_launch_local: after explicit Treatstock cost consent ...; rerun make print-fulfillment-readiness` is recognized as the guarded `PMF_ALLOW_PRINT_PROVIDER_CALLS=1 make print-quote-configured; rerun make print-fulfillment-readiness` handoff.
- When a top-level action list includes both legacy/source-prefixed prose and guarded target forms, only the canonical guarded target remains.
- Saved stale child reports no longer cause top-level `operator_actions` for final demo launch, local refresh, final handoff, iOS launch certificate, rehearsal readiness, rehearsal, or final showcase to contain the legacy prose instruction.
- Focused backend tests prove the normalization behavior without external calls.
- Safe local readiness refresh still reports blocked/partial until external resources are provided.

Safety

This phase is report shaping only. Verification may refresh ignored `.local` reports through safe wrappers. It must not set consent env vars for live provider calls, supply real keys, run Xcode signing, or place print orders.
