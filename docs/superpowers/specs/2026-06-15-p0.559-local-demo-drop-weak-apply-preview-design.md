# P0.559 Local Demo Drop Weak Apply Preview Design

## Context

The refreshed local final handoff reports still show both:

```text
make final-resource-fill-guide; rerun make final-resource-apply-preview; rerun make provider-handoff; rerun make live-provider-evidence
```

and the weaker standalone:

```text
make final-resource-apply-preview
```

The standalone preview command is safe, but once the complete provider handoff chain is present it becomes redundant in the top-level operator list. Keeping both makes the final iPhone/operator handoff look less deterministic and can make the next action sequence feel broader than it is.

## Design

P0.559 keeps the complete provider handoff chain as the canonical action for provider/key progress in local final demo reports. It drops the standalone `make final-resource-apply-preview` only when the complete provider chain is present.

This applies to:

- `final_demo_launch` top-level operator actions.
- `final_local_report_refresh` top-level operator actions.

The rule does not remove `make final-resource-apply-preview` when it is the only safe resource action, or when the apply preview report itself is blocked and no complete provider chain is available.

## Acceptance

- A focused RED/GREEN test proves `final_demo_launch._dedupe_operator_actions()` removes bare `make final-resource-apply-preview` when the complete provider chain is present.
- A focused RED/GREEN test proves `final_local_report_refresh._dedupe_operator_actions()` does the same.
- Refreshed `.local/final-demo-launch-local.json` and `.local/final-local-report-refresh.json` contain the complete provider chain and no bare top-level `make final-resource-apply-preview`.
- Existing tests still prove apply preview is retained when it is the only actionable safe local write.

## Safety

This phase changes report action ordering/filtering only. It does not execute provider calls, write secrets, change `.local` resource files, touch Xcode signing, or mutate global state.
