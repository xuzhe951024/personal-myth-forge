# P0.550 Print Backend Auto Handoff Design

## Context

The final showcase objective still requires a printable `print_asset` handoff through a third-party quote path. The local print proof is available, but `print_fulfillment_readiness` can still block on a generic instruction to set `PMF_BACKEND_BASE_URL` manually when the local demo session exists and the iOS deploy config contains a loopback or missing backend URL.

Other final readiness reports already route iOS backend URL setup through the ignored deploy config writer:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto
```

## Decision

When the configured print quote request is blocked only because the print handoff needs an iPhone-reachable backend URL, `print_fulfillment_readiness` should point at the same deploy config auto writer. Its validation remains `make print-fulfillment-readiness` because this report owns the printable handoff gate.

The expected top-level operator action becomes:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make print-fulfillment-readiness
```

## Scope

This phase changes only command generation and readiness evidence for the print fulfillment report. It does not call Meshy, OpenAI, Treatstock, Xcode, Keychain, or any paid provider. It does not write ignored local configs during tests.

## Acceptance

- The focused print readiness test proves the backend URL blocker now returns the deploy auto writer command.
- The generated `.local/print-fulfillment-readiness.json` uses that command for the configured quote request blocker.
- The final showcase readiness print capability receives the improved action after refresh.
- Backend lint and tests pass.
