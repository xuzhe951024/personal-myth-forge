# P0.368 iOS Rehearsal Handoff Action Priority Design

## Goal

Make the iOS device launch rehearsal report preserve all three remaining final-demo handoff lanes near the top of its operator actions: physical iPhone deploy, live provider evidence, and Treatstock print quote evidence.

## Current Evidence

- `final-showcase-readiness.json`, `final-demo-launch-local.json`, and `final-local-report-refresh.json` already promote provider and Treatstock print handoff actions.
- `ios-device-launch-rehearsal.json` currently starts with the iOS deploy writer and Xcode/resource key actions, but omits `make provider-handoff; rerun make live-provider-evidence` and the Treatstock configured quote action from its top-level `operator_actions`.
- The iOS rehearsal report is the last safe Mac-side rehearsal artifact before physical-device validation, so hiding provider and print handoff actions weakens the final handoff.

## Behavior

`ios_device_launch_rehearsal._operator_actions(...)` should:

- Keep the concrete iOS deploy writer action first when present.
- Promote provider handoff actions such as `make provider-handoff; rerun make live-provider-evidence` immediately after the first iOS action.
- Promote configured print quote handoff actions such as `after explicit Treatstock cost consent, save a sanitized services/backend/.local/print-quote-configured.json from POST /v1/print-quotes; rerun make print-fulfillment-readiness` after provider handoff.
- Keep existing normalization, validation-command retention, source-prefix stripping, and iOS writer preference behavior.

## Non-Goals

- Do not run Xcode, signing, live provider calls, or Treatstock quote/order actions.
- Do not change the first blocker or next action semantics.
- Do not mark iOS deploy, provider, or print fulfillment as ready without external evidence.

## Acceptance

- A focused RED/GREEN test proves provider and print handoff actions survive crowded rehearsal action lists and appear before generic final-resource actions.
- Existing iOS device launch rehearsal tests remain green.
- Adjacent final handoff tests still pass.
- Safe refresh of `ios-device-launch-rehearsal.json` shows provider and print handoff actions near the top while the report remains blocked on external evidence.
