# P0.227 Configured Demo Launch Evidence Design

## Context

The final handoff flow already has `final-demo-launch-local` as a Make target
that writes `services/backend/.local/final-demo-launch-local.json`. The
configured counterpart is still exposed mainly as a raw CLI command in several
reports:

```text
cd services/backend && uv run python -m myth_forge_api.cli final-demo-launch --mode configured --repo-root [repo-root] --output .local/final-demo-launch-configured.json
```

That configured report is a required evidence slot for live provider evidence
and configured launch handoff, but it is not refreshed by
`final-local-report-refresh`.

## Decision

Add a `final-demo-launch-configured` Make target that writes
`services/backend/.local/final-demo-launch-configured.json`, and include the
configured report in `final-local-report-refresh`. This creates a stable local
evidence file without applying resources or calling live providers.

## Scope

- Add `make final-demo-launch-configured`.
- Add `final_demo_launch_configured` to the local report refresh steps.
- Add tests/source gates for the Make target, output path, and refresh file.
- Document the inspection command.

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, Sculpteo, or Apple services.
- Do not run configured final acceptance or live provider evaluations.
- Do not apply final resources or write backend/iOS config.
- Do not run backend servers, Xcode, signing, keychain, or global settings.

## Acceptance

- `make final-demo-launch-configured` writes the configured launch JSON and may
  exit `2` while required resources are missing.
- `make final-local-report-refresh-local` refreshes
  `services/backend/.local/final-demo-launch-configured.json`.
- Focused pytest and Swift ProjectChecks pass.
- Full backend and Swift regressions pass before merge.
