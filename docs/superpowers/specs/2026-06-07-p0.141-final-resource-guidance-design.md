# P0.141 Final Resource Guidance Design

## Context

P0.139 introduced `make final-resource-init` as the guarded way to initialize
`services/backend/.local/final-resources.env`. P0.140 normalized stale nested
operator actions in final showcase readiness.

Two current operator surfaces still point at the old manual copy flow:

- `services/backend/scripts/apply_final_resources.sh` help and missing-file
  stderr.
- Current handoff command blocks in the root `README.md`.

Historical plans, specs, and verification artifacts can still mention the old
copy flow as part of their audit trail. The current operator path should not.

## Goal

Make current final-resource guidance consistently direct operators to:

```bash
make final-resource-init
```

before editing and validating `services/backend/.local/final-resources.env`.

## Non-Goals

- Do not edit historical plan/spec/verification artifacts.
- Do not change final resource parsing, validation, writer behavior, or key
  schema.
- Do not create, apply, or read real provider keys.
- Do not call providers, run Xcode, touch signing/keychain, or mutate global
  machine state.

## Approaches Considered

1. Update only README. This fixes docs but leaves the apply script's runtime
   error pointing at the old manual copy flow.
2. Update only the script. This fixes runtime behavior but leaves a first-time
   operator following root docs on the old path.
3. Update both current surfaces and add tests for both.

Approach 3 is the right scope. It is small, testable, and directly supports the
final resource handoff.

## Design

Update `apply_final_resources.sh`:

- `--help` text says to run `make final-resource-init` for the default resources
  file, then edit and validate it.
- Missing default resources file stderr says the same.
- It must not print the old
  `Copy services/backend/final-resources.env.example to ...` instruction.

Update root `README.md` current command blocks:

- Replace `mkdir -p services/backend/.local` plus `cp ...` with
  `make final-resource-init`.
- Keep the next steps unchanged: edit file, preflight, preview/apply, and run
  configured handoff commands.

## Testing

Extend `services/backend/tests/test_final_resource_apply_script.py`:

- Missing default resources file returns `2`, mentions `make final-resource-init`,
  and does not mention the old copy instruction.
- `--help` mentions `make final-resource-init` and does not mention the old copy
  instruction.
- Root `README.md` no longer contains the old two-line copy sequence and does
  contain `make final-resource-init`.

## Safety

The implementation changes only checked-in text and shell output. Tests run the
apply script against temporary repositories and never write real backend/iOS
configuration, call providers, or mutate global machine state.
