# P0.421 Mobile Evidence Device Bundles Design

## Context

The final iOS deploy blocker is now consistently surfaced by higher-level reports through device action bundles. The bottom-level evidence reports that directly inspect the physical-iPhone readiness path still expose only `checks`, `first_blocker`, `next_action`, and `operator_actions`:

- `mobile_deploy_preflight_evidence_report`
- `mobile_xcode_build_evidence_report`

Operators can infer what to do from those fields, but the direct evidence reports do not provide the same standardized `device_action_bundle` contract used by `ios_device_evidence_bundle`, final launch reports, and configured handoff reports.

## Goal

Add top-level `device_action_bundle` fields to the mobile deploy preflight and mobile Xcode build evidence reports.

## Scope

- Add `device_action_bundle` to `mobile_deploy_preflight_evidence_report`, derived from its current checks and next action.
- Add `device_action_bundle` to `mobile_xcode_build_evidence_report`, derived from its current checks and first blocker.
- Preserve existing `status`, `checks`, `first_blocker`, `next_action`, `operator_actions`, stdout/stderr redaction, and safety behavior.
- Add focused TDD coverage for blocked evidence paths.
- Refresh `services/backend/.local/mobile-deploy-preflight-evidence.json` and `services/backend/.local/mobile-xcode-build-evidence.json`.
- Add the P0.421 roadmap entry.

## Non-Goals

- Do not change the deploy preflight script or Xcode build gate script.
- Do not execute extra commands beyond the existing evidence commands.
- Do not run signing, Keychain writes, global Mac configuration, or physical iPhone deployment.
- Do not call Meshy, OpenAI, Treatstock, Sculpteo, or any live provider.
- Do not write backend `.env`, iOS deploy config, provider keys, or final resource values.
- Do not change higher-level `ios_device_evidence_bundle` behavior in this cut.

## Report Contract

`mobile_deploy_preflight_evidence_report.device_action_bundle` contains:

- `id`: `mobile_deploy_preflight_evidence_actions`
- `label`: `Mobile Deploy Preflight Evidence Actions`
- `source_report`: `mobile_deploy_preflight_evidence`
- `status`: the report status
- `actions`: one action per check, with command/detail/next action where blocked
- `first_action`: first non-ready action, or `None` when ready
- `summary`: action counts for ready, missing, blocked, manual, provider calls, global actions, and Xcode/signing
- `safety`: mirrors that the evidence command has run, does not call providers, does not mutate globals, does not write secrets, and does not run Xcode/signing

`mobile_xcode_build_evidence_report.device_action_bundle` contains:

- `id`: `mobile_xcode_build_evidence_actions`
- `label`: `Mobile Xcode Build Evidence Actions`
- `source_report`: `mobile_xcode_build_evidence`
- `status`: the report status
- `actions`: one action per check, with command/detail/next action where blocked
- `first_action`: first non-ready action, or `None` when ready
- `summary`: action counts for ready, missing, blocked, manual, provider calls, global actions, and Xcode/signing
- `safety`: mirrors that the evidence command has run, does not call providers, does not write secrets, and describes Xcode/build-gate activity with signing disabled
