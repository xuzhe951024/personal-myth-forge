# P0.101 Print Fulfillment Approval Receipt Design

## Goal

Make the iPhone demo show a safe, explicit approval state before any printable relic is handed to a third-party print provider or checkout flow.

## Product Behavior

- Add a mobile-core print fulfillment receipt beside the existing print quote review.
- The receipt has four statuses:
  - `waiting`: no myth session or no quote yet.
  - `needsApproval`: a quote exists and requires user approval before fulfillment.
  - `ready`: the quote is approved, or the quote does not require explicit user approval.
  - `blocked`: quote or candidate data contains unsafe provider/payment/local-path details after sanitization.
- Add an iPhone approval toggle in `PrintQuoteReviewView` after a quote is loaded.
- Reset approval whenever a new myth session is forged, restored state is cleared, or a new quote is requested.
- Keep checkout/payment identifiers withheld. The panel can say a provider handoff is ready, but it must not expose a checkout URL or place an order.

## UI Placement

`PrintQuoteReviewView` remains the print section. It will render:

1. existing print candidate summary
2. existing quote request button and quote summary
3. new `PrintFulfillmentReceiptView`
4. approval toggle when approval is possible

## Privacy And Safety

- Provider keys, checkout URLs, payment links, local filesystem paths, raw capture URIs, and raw personal context must be redacted.
- Approval is local app state only; it does not call a print provider.
- The receipt text must make clear that third-party checkout/order placement remains outside the app.

## Acceptance

- Swift contract tests cover waiting, needs-approval, ready, blocked, reset-sensitive wiring, and redaction.
- SwiftUI compile and project source gates require the receipt view, builder, approval binding, and Xcode membership.
- Backend iOS showcase acceptance requires the new mobile print fulfillment feature.
- A 390x844 visual regression image shows waiting, needs-approval, and ready states without text overlap or clipping.
