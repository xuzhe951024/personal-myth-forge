# P0.188 Final Refresh Device Evidence Reuse Design

## Problem

`make final-local-report-refresh` is intentionally safe: it must not run device, Xcode, signing, live provider, keychain, or global-mutating commands. Today its safe `final_acceptance` command runner always reports the mobile deploy preflight and Xcode build gates as skipped blockers. That is correct when no external evidence exists, but it is too conservative after an operator has already generated ready evidence with the dedicated commands.

The result is noisy handoff state: `final-acceptance-local.json` can keep blocking `mobile_deploy_preflight` and `mobile_xcode_build` even when `mobile-deploy-preflight-evidence.json` and `mobile-xcode-build-evidence.json` already prove those gates are ready.

## Scope

This slice changes only local report refresh behavior:

- Preserve existing `services/backend/.local/mobile-deploy-preflight-evidence.json` when it is valid JSON.
- Keep existing Xcode evidence preservation behavior.
- Reuse existing ready mobile deploy preflight evidence when simulating `make mobile-deploy-preflight` for `final_acceptance`.
- Reuse existing ready Xcode build evidence when simulating `make mobile-xcode-build` for `final_acceptance`.
- Keep skipped/blocked behavior when evidence is missing, invalid, or not ready.
- Keep `refresh_safety.mobile_gate_commands_executed=false` and `refresh_safety.xcode_or_signing_executed=false`.

## Non-Goals

- Do not run `make mobile-deploy-preflight` from `final-local-report-refresh` when a snapshot already exists.
- Do not run `make mobile-xcode-build` from `final-local-report-refresh`.
- Do not change final acceptance classification rules.
- Do not claim iOS deployability without ready evidence files.
- Do not add provider keys or run live provider calls.

## Acceptance Criteria

- A ready mobile deploy preflight evidence file survives `run_final_local_report_refresh`.
- A ready mobile deploy preflight evidence file makes the embedded `final_acceptance` mobile deploy command check pass.
- A ready mobile Xcode build evidence file makes the embedded `final_acceptance` Xcode command check pass.
- Missing evidence preserves current accepted blocked behavior.
- Existing safe refresh tests remain green.
- No report leaks local temp paths or provider secrets.
