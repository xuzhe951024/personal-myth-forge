# P0.546 Print Readiness Smoke Source Design

## Goal

Let print fulfillment readiness use the local final demo launch report's current `local_showcase_smoke` session evidence when preparing the auto print quote request handoff.

## Current Gap

`final-demo-launch-local.json` already embeds the local smoke report at top-level `local_showcase_smoke`. That report includes the generated myth session id and proves local downloadable game and print assets are available. `print_fulfillment_readiness` still looks only under `source_reports.local_showcase_smoke`, so it can report `missing_local_demo_session_for_print_quote_request` even when the final demo launch report has the session evidence it needs.

## Intended Behavior

- Read the session id from top-level `local_showcase_smoke.session.session_id`.
- Keep support for the older `source_reports.local_showcase_smoke.session.session_id` shape.
- When a session id is present but `Deployment.local.xcconfig` has no iPhone-reachable backend URL, keep the existing backend URL blocker.
- When both the session id and iPhone-reachable backend URL are present, build the auto local served asset URIs for `print-quote-request-configured`.
- Do not create provider calls, write `.env`, write deploy config, invoke Xcode, or place print orders.

## Acceptance

- A print fulfillment readiness test with top-level `local_showcase_smoke` no longer reports `missing_local_demo_session_for_print_quote_request`.
- Existing source-report-shaped fixture tests continue to pass.
- Refreshed local reports move the print fulfillment blocker from missing local session toward the real backend URL or quote request handoff state.
