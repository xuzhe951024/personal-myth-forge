# P0.292 Final Handoff Make Wrapper Normalization Design

## Goal

Normalize final handoff operator actions that wrap existing project Make targets with `run make ...`.

## Context

The final showcase is still blocked by external iOS/device/provider evidence, but the local handoff checklist is getting close to executable. Some nested actions still use wrapper phrasing:

```text
final_handoff_index: run make final-configured-preflight
final_handoff_index: run make final-demo-launch-configured
ios_device_launch_certificate: run make final-handoff-index
ios_device_launch_certificate: run make ios-deploy-runbook-local
```

All of these point at existing Makefile targets. The extra `run` word makes the final operator checklist inconsistent with other canonical rows such as `make provider-handoff` and `make final-resource-requirements`.

## Design

Keep normalization centralized in `myth_forge_api.operator_actions`.

Extend make-target action normalization so exact actions and source-prefixed actions normalize to canonical make targets:

- `run make final-configured-preflight` -> `make final-configured-preflight`
- `run make final-demo-launch-configured` -> `make final-demo-launch-configured`
- `run make final-demo-launch-local` -> `make final-demo-launch-local`
- `run make final-handoff-index` -> `make final-handoff-index`
- `run make ios-deploy-runbook-local` -> `make ios-deploy-runbook-local`

Source prefixes must be preserved:

```text
ios_device_launch_certificate: run make final-handoff-index
```

becomes:

```text
ios_device_launch_certificate: make final-handoff-index
```

Detail suffixes such as ` | stale` must remain attached after normalization.

## Scope

In scope:

- shared operator action normalization
- focused backend tests for direct, prefixed, CLI-derived, and final-showcase aggregated actions
- roadmap entry for P0.292
- `.local` refresh verification

Out of scope:

- changing safety-qualified commands such as live provider acceptance or provider cost consent
- changing `run make final-apply-resources to apply the filled resource bundle`
- running Xcode, signing, Keychain, device, or live provider actions
- writing API keys or deploy config

## Verification

This iteration is valid when:

- RED tests show wrapper actions still leak through
- GREEN tests show target wrappers normalize while preserving prefixes and detail suffixes
- refreshed `.local` final showcase operator actions no longer expose the targeted wrapper forms
- backend lint and tests pass
