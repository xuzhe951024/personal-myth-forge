# P0.202 Resource Handoff First Blockers Design

## Purpose

`final_resource_requirements_report` now exposes a structured `first_blocker`, but the adjacent resource handoff reports still require clients to infer the next action from arrays:

- `final_resource_fill_guide_report.required_inputs`
- `final_resource_apply_preview_report.write_targets[].blocked_by`

That weakens the last-mile resource handoff. The backend, final launch payload, and iPhone summary should all expose the same "next required input or apply target" without list inference.

## Scope

- Add `first_blocker` to `final_resource_fill_guide_report`.
- Add `first_blocker` to `final_resource_apply_preview_report`.
- Keep reports read-only and sanitized; no env writes, shell writers, Xcode, provider calls, or secrets.
- Decode both first blockers in iOS models.
- Render first blocker rows before derived list rows in `FinalLaunchMobileSummary`.
- Add source acceptance gates so future edits cannot remove these contracts silently.

## Backend Contract

`final_resource_fill_guide_report.first_blocker` is the first required input, with:

- `id`, `label`, `status`, `classification`
- `command`: its fill action, such as `fill MESHY_API_KEY in services/backend/.local/final-resources.env`
- `detail`: notes
- `domain`, `input_source`, `write_destination`, `validation_command`

`final_resource_apply_preview_report.first_blocker` is the first target that is missing or blocked, with:

- `id`, `label`, `status`, `classification`
- `command`: the target command, usually `make final-apply-resources`
- `detail`: blocked slot ids joined for operator readability
- `destination`, `writer`, `blocked_by`, `validation_command`

Ready reports set `first_blocker` to `null`.

## Acceptance

- Backend tests prove blocked reports expose a structured first blocker and ready reports emit `null`.
- Swift contract tests decode both first blockers and summary rows show their commands/details.
- `ios_showcase_acceptance`, `mobile_final_launch_readiness_acceptance`, and Swift ProjectChecks require the backend/source/mobile first blocker contracts.
