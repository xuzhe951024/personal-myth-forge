# P0.531 Mobile Print Request Handoff Design

## Context

P0.529 taught backend print fulfillment readiness to block on the ignored configured quote request file before asking the operator to run the cost-consent guarded Treatstock quote command. P0.530 preserved that request-preflight action through the final external ledger and closure packet.

The iOS final launch contract fixtures still use the older direct provider-call action:

```text
PMF_ALLOW_PRINT_PROVIDER_CALLS=1 make print-quote-configured
```

That is now only correct after `services/backend/.local/print-quote-request-configured.json` exists. When the request file is missing, the iPhone final launch summary should display the repo-local request-preflight action instead.

## Design

Update the mobile contract fixture defaults and assertions for print fulfillment readiness so a missing configured Treatstock request surfaces:

```text
prepare services/backend/.local/print-quote-request-configured.json; rerun make print-fulfillment-readiness
```

The decoded `PrintFulfillmentReadinessReport.nextAction` should retain the same structure:

- `id`: `configured_treatstock_quote`
- `source`: `first_blocker`
- `requires_cost_consent`: `true`
- `validation_command`: `make print-fulfillment-readiness`

Only the command string changes. The mobile summary builder already renders `next_action.command`, `operator_actions`, and validation rerun text generically, so production Swift code should not need special-case logic unless the tests expose stale hard-coded behavior.

## Scope

In scope:

- iOS final launch contract tests and fixtures.
- Roadmap entry for P0.531.
- Focused mobile contract verification.

Out of scope:

- Backend provider command semantics.
- Treatstock live calls or real quote generation.
- Xcode signing, device deployment, Keychain writes, or global environment mutation.
- Backend aggregator behavior already covered by P0.529 and P0.530.

## Acceptance

- Mobile final launch fixture with missing configured Treatstock quote request decodes the request-preflight command.
- Mobile final launch summary displays the request-preflight action and validation rerun.
- Mobile summary still redacts unsafe print fulfillment details.
- The old direct provider-call action is not shown for the missing-request fixture.
- Focused Swift contract tests pass.
