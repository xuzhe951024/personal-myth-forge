# P0.81 Mobile Generation Provenance Design

## Purpose

The iPhone showcase already renders a generated artifact preview and has access
to `GeneratedAssetProvenance`. The current provenance UI is private SwiftUI logic
inside `ArtifactSummaryView`, so the final demo cannot contract-test the exact
operator-facing text that proves scan/photo inputs reached 3D generation.

P0.81 makes the 3D generation route visible, richer, and testable on mobile.

## Scope

Add a mobile-core summary builder for generated asset provenance. It must show:

- the provider route, such as `local_stub` or Meshy endpoint path
- the input route, such as `text prompt`, `single image`, `multi image`, or
  `scan asset`
- selected image count versus available image count
- scan asset count when present
- whether raw source media is withheld
- the provider selection reason, sanitized for mobile display

The app still does not expose raw media, local capture URIs, provider secrets,
or payment links. It does not call Meshy, OpenAI, or print providers.

## Architecture

Create `ArtifactGenerationProvenanceSummary.swift` in
`PersonalMythForgeMobileCore`. The builder receives an optional
`GeneratedAssetProvenance` and returns a small immutable summary:

- `title`
- `routeLabel`
- `sourceSummary`
- `providerRoute`
- `selectionReason`
- `privacySummary`
- `isAvailable`

`ArtifactSummaryView` replaces its private ad hoc text formatting with this
builder and renders the same compact `Generation` card using the tested summary.
No backend model changes are needed because the required fields already exist in
`GeneratedAssetProvenance`.

## Data Flow

1. Backend returns `generated_asset.generation_provenance` with the myth session.
2. Swift model decodes the provenance into `GeneratedAssetProvenance`.
3. `ArtifactGenerationProvenanceSummaryBuilder.build(provenance:)` converts raw
   fields into mobile-safe display text.
4. `ArtifactSummaryView` renders the summary under the 3D preview.

For ARKit/Object Capture scan packages, the summary should show the multi-image
route plus scan asset count, because the provider still receives prepared images
and a scan asset reference.

## Safety

The builder sanitizes secret-like and local-path-like text in provider route and
selection reason. Unsafe text is replaced with `[withheld]`. It never returns raw
media or raw source identifiers.

## Tests And Acceptance

Add Swift contract tests for:

- multi-image guided scan provenance with `3/4 images`, max image count, and raw
  source withholding
- ARKit scan package provenance with scan asset count
- missing provenance state
- unsafe route/reason redaction

Add source gates so iOS showcase acceptance requires the summary builder,
`sourceAssetCount`, `selectionReason`, `privacySummary`, and the new contract
tests. Add a static 390x844 visual evidence page for the generation card.
