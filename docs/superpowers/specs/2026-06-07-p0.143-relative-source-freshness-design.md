# P0.143 Relative Source Freshness Design

## Context

The final showcase report can call readiness builders from `services/backend`
with `repo_root=../..`. The shared `git_product_source_metadata()` helper then
passes that relative path both as `cwd` and as `git -C`, so Git evaluates the
path twice and can report `git_unavailable` even when the project root is a
valid repository.

This weakens freshness gates for saved final-acceptance, visual-regression, and
iOS device launch rehearsal reports. It also makes stale local reports less
actionable during the iPhone handoff path.

## Decision

Normalize `repo_root` once inside `source_freshness.git_product_source_metadata`
before invoking Git. Use the resolved absolute root consistently for subprocess
`cwd` and `git -C`.

Keep all report-level freshness semantics unchanged:

- Missing saved reports still return `source_missing`.
- Real Git failures still return `git_unavailable`.
- Freshness comparison remains based on saved report mtime versus the latest
  product-source commit.
- No report builder should rewrite `.local` evidence or call providers.

## Alternatives Considered

1. Normalize every caller before passing `repo_root`.
   This would be easy to miss because multiple builders use the same helper.

2. Remove `cwd` and rely only on `git -C`.
   This is viable, but resolving once in the helper is clearer and protects all
   current callers.

3. Leave the helper unchanged and normalize only final showcase calls.
   This would not fix direct CLI usage of the readiness builders.

## Testing

Add focused tests in `tests/test_source_freshness.py`:

- Calling `git_product_source_metadata(Path("../repo"))` from a different cwd
  returns a revision and `git_product_sources` scope.
- A relative path to a non-repository still returns `None`.

Add an integration regression in `tests/test_ios_device_launch_rehearsal.py`:

- From a nested `services/backend` cwd, pass `repo_root=Path("../..")` to
  `build_ios_device_launch_rehearsal_readiness_report`.
- A saved report newer than the latest product-source commit reports
  `fresh_report`, not `git_unavailable`.

## Safety

The change is read-only. It does not run provider calls, does not write ignored
resource files, does not touch Xcode signing, and does not mutate global state.
