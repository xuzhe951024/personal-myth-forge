# P0.404 Configured Preflight Actions Design

## Context

The final goal still needs configured provider evidence for Meshy, OpenAI NPC
Agent, and print quote handoff. The top-level final showcase and local refresh
reports now prioritize concrete commands, but the source
`final-configured-preflight` report still includes legacy fallback actions:

```text
unblock provider_readiness: make provider-handoff
unblock configured_final_acceptance: make final-acceptance-configured
```

The same report already includes better concrete actions such as
`make provider-handoff`, `make live-provider-evidence`, and the explicit live
provider cost consent line. Keeping the old `unblock ...` strings makes the
configured provider handoff less deterministic.

## Goal

Remove synthetic `unblock ...` fallback actions from
`final-configured-preflight` operator actions while preserving concrete provider,
resource, iOS deploy, and cost-consent handoff commands.

## Scope

- Change only `myth_forge_api.final_configured_preflight` and focused tests.
- Preserve `commands`, nested reports, first blocker, next action, and safety
  fields.
- Keep concrete child actions from final resources, provider handoff, resource
  handoff, configured final launch, and configured iOS deploy runbook.
- Keep the explicit configured final acceptance cost-consent action.
- Do not run live providers, Xcode signing, Keychain writes, print orders, or
  resource writers.

## Design

Add a small configured-preflight action filter at the end of operator action
normalization:

1. Normalize and dedupe actions as today.
2. Prefer `make final-resource-apply-preview` over direct apply actions as
   today.
3. Drop only actions whose root starts with `unblock `.
4. Keep concrete make targets and consent actions, including
   `make provider-handoff`, `make live-provider-evidence`, and
   `approve live provider cost review before make final-acceptance-configured`.

This keeps the full component evidence intact while making the operator-facing
handoff list copy-pasteable.

## Tests

Update `services/backend/tests/test_final_configured_preflight.py`:

- Add a regression test proving blocked configured preflight actions do not
  contain `unblock provider_readiness` or `unblock configured_final_acceptance`.
- Assert concrete replacements remain present:
  - `make provider-handoff`
  - `make live-provider-evidence`
  - configured final acceptance cost consent action

Focused verification:

```bash
cd services/backend && uv run pytest tests/test_final_configured_preflight.py -q
```

Report verification:

```bash
make final-local-report-refresh
jq '.steps_by_id.final_configured_preflight.report_operator_actions' \
  services/backend/.local/final-local-report-refresh.json
```

Expected: the report remains blocked by missing external values, but configured
preflight operator actions contain no synthetic `unblock ...` fallbacks.
