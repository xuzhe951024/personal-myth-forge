# P0.304 iOS Rehearsal Action Normalization Design

## Goal

Make `ios-device-launch-rehearsal.json` top-level `operator_actions`
copy-pastable by stripping source-report prefixes from the operator checklist.

## Current Context

`final-showcase-readiness.json` identifies `ios_deployable` as the first blocker,
and its command is `make ios-device-launch-rehearsal`. That rehearsal report is
therefore the main Mac-side guide for the physical iPhone demo path.

The report currently preserves provenance in top-level actions:

```text
final_rehearsal_local: mobile_deploy_preflight_evidence: provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable
final_configured_preflight: provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight
ios_device_launch_certificate: provide iOS deploy config in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
```

That provenance is useful in `sequence` rows and nested source reports, but the
top-level `operator_actions` should be directly actionable:

```text
provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable
provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight
provide iOS deploy config in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
```

## Design

Add a rehearsal-specific normalization step at the boundary where top-level
`operator_actions` are returned from `ios_device_launch_rehearsal._operator_actions()`.
The step should:

- reuse existing `normalize_operator_action()`;
- preserve detail suffixes such as `| Missing DEVELOPMENT_TEAM`;
- recursively strip only known rehearsal source prefixes from the command root;
- normalize simple `run make ...` wrappers to bare `make ...`;
- keep `sequence`, `first_blocker.detail`, and nested source report actions
  unchanged so provenance remains available for debugging.

## Out of Scope

- No changes to deploy config files.
- No backend env writes.
- No Xcode, signing, Keychain, physical iPhone, live provider, or paid print
  actions.
- No changes to readiness reports that merely read an existing saved rehearsal
  file.

## Verification

- RED/GREEN coverage for local rehearsal prefixed actions becoming bare top-level
  actions while `sequence` still preserves provenance.
- RED/GREEN coverage for configured preflight and device certificate actions
  becoming bare top-level commands.
- Refresh safe local reports and `make ios-device-launch-rehearsal`; verify the
  saved JSON remains blocked on external/device setup but top-level actions no
  longer start with known source prefixes.
- Run focused rehearsal tests, operator-action tests, lint, and whitespace checks.
