# P0.129 Mobile External Action Ledger Design

## Problem

P0.128 embeds `final_external_action_ledger` in the backend final launch
payload, but the iPhone model layer does not decode it and the Final Launch
screen cannot show it. The operator can inspect the JSON locally, but the
mobile demo still misses the safest summary of what remains external: missing
resources, safe local writes, live provider cost actions, global Mac/Xcode
actions, and device runtime actions.

## Goals

- Decode `final_external_action_ledger` in `FinalDemoLaunchReport`.
- Add mobile summary rows for the ledger using the same string-row pattern as
  the existing Final Launch sections.
- Render a compact `External Actions` group in `FinalLaunchStatusView`.
- Redact unsafe text in ledger rows: provider keys, bearer tokens, local paths,
  file URLs, checkout/payment links, and raw local capture URIs.
- Add Swift contract tests for decoding, row generation, and redaction.
- Expand backend source acceptance so the iOS showcase cannot lose the mobile
  ledger wiring silently.

## Non-Goals

- Do not add buttons that run commands from the app.
- Do not execute shell commands, call live providers, run Xcode, mutate signing,
  or write keychain/global machine state.
- Do not redesign the Final Launch screen.
- Do not add backend report behavior; P0.128 already owns the backend ledger.

## Mobile Data Model

Add Codable models in `PMFModels.swift`:

- `FinalExternalActionLedgerSummary`
- `FinalExternalActionLedgerAction`
- `FinalExternalActionLedgerActionGroup`
- `FinalExternalActionLedgerSafety`
- `FinalExternalActionLedgerReport`

`FinalDemoLaunchReport` gains:

```swift
public var finalExternalActionLedger: FinalExternalActionLedgerReport?
```

with `CodingKeys.finalExternalActionLedger = "final_external_action_ledger"`.

## Mobile Summary

`FinalLaunchMobileSummary` gains:

```swift
public var externalActionLedgerRows: [String]
```

`FinalLaunchMobileSummaryBuilder.externalActionLedgerRows(from:)` returns:

- `"External actions <status>: groups <n>, blocked <n>, missing <n>, live <n>, manual <n>."`
- Up to three non-ready group rows:
  - `"<group_id>: <status> | actions <n>, missing <n>, blocked <n>, live <n>, manual <n>"`
- Safety rows:
  - `"Consent: global confirmation <true/false>, live cost consent <true/false>."`
  - `"Safety: commands_run=<true/false> global_mutation=<true/false> live_calls=<true/false>"`
- First operator sequence command, when present.

## UI

`FinalLaunchStatusView` adds:

```swift
groupedLines(title: "External Actions", lines: summary.externalActionLedgerRows)
```

near Mode and Live Evidence, because this ledger explains the action gates that
surround live provider and deploy readiness.

## Safety

This iteration is display-only. The app continues to receive sanitized metadata
and never receives provider secrets intentionally. The summary builder still
sanitizes every row defensively.
