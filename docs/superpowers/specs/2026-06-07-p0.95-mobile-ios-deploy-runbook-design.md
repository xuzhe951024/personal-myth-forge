# P0.95 Mobile iOS Deploy Runbook Design

## Context

P0.94 added a backend `ios_deploy_runbook_report`, CLI, Make target, and final
operator handoff step. The report consolidates final physical-device inputs,
Mac-side commands, and safety policy. It is embedded in `/v1/final-demo-launch`.

The iPhone client currently decodes final resources, final acceptance, NPC
evaluation, and final operator handoff. It does not decode the embedded iOS
deploy runbook, so the phone cannot show the detailed deploy input slots,
command sequence, or safety receipt. The user can only infer the runbook through
generic handoff rows.

## Design

Add Swift models for the P0.94 report:

- `IOSDeployRunbookReport`
- `IOSDeployRunbookInputSlot`
- `IOSDeployRunbookCommandStep`
- `IOSDeployRunbookSafety`

`FinalDemoLaunchReport` will gain optional `iosDeployRunbook` decoding from
`ios_deploy_runbook`.

`FinalLaunchMobileSummary` will add three row groups:

- `deployRunbookRows`: status and highest-priority input slots
- `deployRunbookCommandRows`: ordered Mac-side commands from the runbook
- `deployRunbookSafetyRows`: safety receipt showing no app command execution,
  provider calls, global mutation, secrets, raw media, payment links, or local
  absolute paths

`FinalLaunchStatusView` will render those row groups between `NPC Evaluation`
and `Next`, keeping the iPhone app read-only. It will not add buttons that run
commands.

## Status Rules

- Missing runbook: show `iOS deploy runbook has not loaded.`
- Ready/partial runbook: show the runbook status plus up to three relevant input
  slots. Prefer missing/blocked slots; otherwise show ready required slots.
- Blocked runbook: show missing/blocked slots and the first operator action.
- Safety rows always sanitize unsafe text and explicitly surface false values
  from the backend safety receipt.

## Tests

Swift contract tests will be written first:

- decode `ios_deploy_runbook` from a final launch payload
- build summary rows for missing, blocked, partial, and ready runbooks
- sanitize unsafe slot/action/command text
- verify the static Swift project checks require the runbook models, summary
  rows, view section, and contract tests

Backend source acceptance will be extended to require the same mobile source
strings so final acceptance catches accidental removal.

## Visual Regression

Add a 390x844 static visual showing the Final Launch panel with:

- iOS Deploy Runbook section
- input slot rows
- command rows
- safety receipt rows

## Non-Goals

- No new backend endpoint
- No command execution from iOS
- No Xcode, signing, simulator, device deployment, keychain, or global config
  mutation
- No Meshy, OpenAI, Treatstock, or other live provider calls
