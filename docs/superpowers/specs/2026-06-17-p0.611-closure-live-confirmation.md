# P0.611 Closure Live Confirmation

## Problem

`final-launch-closure-packet` can surface live provider or cost-consent blockers without also marking them as requiring explicit user confirmation. That weakens the final operator handoff because Meshy/OpenAI configured evidence commands can spend credits and must never look like ordinary local actions.

## Scope

- Normalize closure packet actions so any `requires_cost_consent` or `live_provider_call` action also has `requires_user_confirmation`.
- Preserve existing command routing, operator action ordering, and local/deferred print behavior.
- Do not run live provider commands, Xcode signing, Keychain writes, or third-party print calls.

## Acceptance

- A focused RED test proves configured live evidence blockers previously lacked `requires_user_confirmation`.
- The GREEN implementation marks the configured live blocker, next action, and action row as requiring user confirmation.
- Existing closure packet tests continue to pass.
