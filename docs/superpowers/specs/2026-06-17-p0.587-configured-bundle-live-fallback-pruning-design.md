# P0.587 Configured Bundle Live Fallback Pruning Design

## Context

`configured-live-evidence-bundle.json` aggregates the configured evidence plan and live provider evidence. When live provider evidence is blocked by the guarded consent path, the bundle correctly exposes:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make final-acceptance-configured; rerun make live-provider-evidence
```

However, the same top-level `operator_actions` can also include fallback commands copied from child reports, such as `make provider-handoff`, configured 3D/NPC evaluation targets, configured demo launch, and bare `make live-provider-evidence`.

## Decision

When the guarded configured acceptance command is present, make it the single live-provider entry point for the configured evidence bundle:

- Keep the guarded consent command.
- Drop provider handoff, configured 3D/NPC evaluation, configured final/demo acceptance, bare live-provider evidence, and review-consent fallback actions.
- Preserve unrelated resource, device, and local handoff actions.

## Acceptance

- A live provider consent next action prunes configured live-provider fallback operator actions.
- Existing configured bundle, final launch closure, and external action ledger focused tests remain green.
- Report generation still performs no live provider calls.
