# P0.194 Mobile First Blocker Readiness Design

## Purpose

P0.193 added `final_demo_launch_report.first_blocker` and iOS receipt support. The remaining gap is that `mobile_final_launch_readiness_acceptance` and Swift `PersonalMythForgeMobileProjectChecks` do not yet require that top-level blocker contract. This iteration makes the mobile final launch readiness gate protect the new field.

## Scope

- Add mobile readiness source requirements for `FinalDemoLaunchFirstBlocker`.
- Require `FinalDemoLaunchReport.firstBlocker` in Swift ProjectChecks.
- Require `FinalLaunchMobileSummaryBuilder` to read `report.firstBlocker`.
- Require the Swift contract test that exercises the top-level receipt.
- Keep endpoint behavior, provider calls, and local resource files unchanged.

## Acceptance

- `run_mobile_final_launch_readiness_acceptance` reports requirement ids for the model, field, summary row, and contract test.
- Missing minimal mobile source fails when it lacks those tokens.
- `PersonalMythForgeMobileProjectChecks` fails if the final demo launch first blocker model, field, or summary preference is removed.
- Full local safety remains unchanged: no live provider calls, no secrets, no raw media, no absolute paths in reports.
