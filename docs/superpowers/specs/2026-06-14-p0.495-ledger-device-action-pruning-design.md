# P0.495 Ledger Device Action Pruning Design

## Context

`final-external-action-ledger.json` is the broadest operator ledger for final demo launch handoff. It already exposes specific device handoff actions:

- `DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`
- `provide PRODUCT_BUNDLE_IDENTIFIER in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`
- `provide PMF_BACKEND_BASE_URL in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`
- `start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight`
- `accept the Xcode license outside Codex, then rerun make mobile-xcode-build-evidence`

The same top-level `operator_actions` list still includes bare `make ios-device-launch-rehearsal`. That target remains useful in `operator_sequence`, action rows, and validation fields, but it is weak as a top-level handoff step while the exact device prerequisites are still missing.

## Decision

Filter only `final_external_action_ledger.operator_actions`. Keep the full ledger `operator_sequence`, action rows, and validation commands unchanged.

The top-level ledger checklist should:

- Keep the concrete device writer and mobile preflight actions.
- Drop bare `make ios-device-launch-rehearsal` when concrete device actions already exist.
- Preserve bare `make ios-device-launch-rehearsal` if no specific device handoff is available.
- Preserve the first blocker ordering so the Apple Team ID writer remains first.

## Safety

This is report-only. It does not run Xcode, write signing settings, touch Keychain, start backend servers, call providers, write ignored env files, or perform print/provider calls.

## Verification

Focused tests must prove:

- The full ledger fixture no longer emits bare `make ios-device-launch-rehearsal` in top-level `operator_actions` when specific device handoffs exist.
- The same fixture keeps `make ios-device-launch-rehearsal` in `operator_sequence`.
- `_operator_actions()` still preserves bare rehearsal if no specific device handoff exists.
- Focused external ledger tests, operator action tests, lint, and refreshed local evidence pass.
