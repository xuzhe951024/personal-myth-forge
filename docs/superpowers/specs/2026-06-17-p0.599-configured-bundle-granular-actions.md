# P0.599 Configured Bundle Granular Actions

## Context

`final-configured-evidence-plan` now exposes guarded per-capability commands for
Meshy 3D, OpenAI NPC, and configured final acceptance. `configured-live-evidence-bundle`
still lets the broader live evidence action appear first when live evidence is
blocked:

`PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make final-acceptance-configured; rerun make live-provider-evidence`

That is safe, but it hides the more precise action sequence the operator needs first.

## Decision

When configured evidence plan guarded actions are available, the configured live
evidence bundle should prefer them over broader live-provider-evidence fallback
actions in top-level `operator_actions`.

## Acceptance

- `operator_actions` starts with guarded 3D, NPC, then configured final acceptance
  commands from `final-configured-evidence-plan`.
- The broader `final-acceptance-configured; rerun make live-provider-evidence`
  action is pruned when granular configured-plan actions are present.
- Existing blocker and `next_action` metadata remain unchanged.
- No live provider calls, external writes, or Xcode/global mutations are performed.
