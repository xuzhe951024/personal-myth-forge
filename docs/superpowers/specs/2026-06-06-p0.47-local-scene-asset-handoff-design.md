# P0.47 Local Scene Asset Handoff Design

## Product Intent

The mobile demo has a 3D preview surface and can download HTTP(S) SceneKit assets,
but the deterministic local 3D provider still emits `local://generated-assets/...`
URIs. That is useful metadata for backend tests, but an iPhone cannot download or
load it. Without Meshy keys, the local demo therefore falls back to a placeholder
scene instead of proving the generated-asset handoff.

P0.47 makes the local backend return device-reachable, SceneKit-loadable asset
URLs for local demo sessions. The goal is not to replace Meshy quality; it is to
prove that the iOS path can receive a generated asset, download it from the same
backend base URL, cache it, and hand it to SceneKit.

## Scope

Backend:

- Add a deterministic generated-asset route:
  - `GET /v1/generated-assets/{session_id}/game.glb`
  - `GET /v1/generated-assets/{session_id}/scene.dae`
- Return deterministic local GLB metadata and a small valid Collada scene with
  stable geometry, no user media, no local filesystem paths, and no provider
  secrets.
- Rewrite local stub `generated_asset` URLs in API responses from `local://...`
  to absolute request-base URLs, for example:
  - `http://192.168.1.10:8080/v1/generated-assets/myth_.../scene.dae`
- Keep Meshy/OpenAI provider outputs untouched.
- Store the rewritten API-visible session so backend history restore returns the
  same iPhone-reachable scene URL.

Local provider:

- Keep the primary local `game_asset` as GLB metadata.
- Change the local `ios_scene_asset` variant to `format=dae` because the backend
  can safely serve a deterministic SceneKit-loadable Collada scene without
  external tooling.

iOS:

- The existing `ArtifactAssetPreparer` already treats `dae` as SceneKit-loadable
  and downloads HTTP(S) assets. P0.47 only updates fixture expectations and adds
  a focused contract case proving remote DAE is cached as `cachedSceneReady`.

Acceptance:

- Add `local_asset_handoff_acceptance` to final acceptance.
- The check should create a local API session through `TestClient`, inspect the
  returned `ios_scene_asset` URL, download the DAE route, and confirm the report
  is safe.

## Non-Goals

- Do not generate high-fidelity local meshes.
- Do not run Meshy/OpenAI/Treatstock/Sculpteo.
- Do not install or invoke Xcode device deployment.
- Do not add a Unity runtime.
- Do not store raw capture images in generated assets.

## Expected Final Acceptance Change

Default quick final acceptance should move from:

- `passed=7, blocked=2, failed=0`

to:

- `passed=8, blocked=2, failed=0`

The two blocked checks remain local iOS deployment config and Apple SDK license.
