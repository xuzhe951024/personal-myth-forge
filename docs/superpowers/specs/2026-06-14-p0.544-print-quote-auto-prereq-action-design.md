# P0.544 Print Quote Auto Prerequisite Action Design

## Goal

Make the print quote request handoff actionable before Treatstock cost consent. When `PRINT_SOURCE_ASSET_URI=auto PRINT_CANDIDATE_URI=auto make print-quote-request-configured` cannot run, `print-fulfillment-readiness` should name the missing prerequisite first instead of recommending the failing auto command.

## Current Gap

`print-fulfillment-readiness` can report this operator action:

```text
PRINT_SOURCE_ASSET_URI=auto PRINT_CANDIDATE_URI=auto make print-quote-request-configured; rerun make print-fulfillment-readiness
```

The target is safe because it only writes an ignored local request file and does not call Treatstock. However, it still requires two local prerequisites:

- `final-demo-launch-local` must contain a local showcase smoke `session_id`.
- `apps/mobile/ios/Config/Deployment.local.xcconfig` must contain an iPhone-reachable `PMF_BACKEND_BASE_URL`.

When those are missing, the command fails before writing useful handoff evidence.

## Intended Behavior

- If the configured quote request file is missing and no local demo session is available, the readiness first blocker should command `make final-demo-launch-local`.
- If a local demo session exists but `PMF_BACKEND_BASE_URL` is missing or loopback, the readiness first blocker should command `set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL`.
- Only when both prerequisites are present should readiness recommend concrete generated asset and print candidate URLs.
- The CLI auto resolver should share the same iPhone-reachable URL rule and reject loopback URLs before writing a request file.
- No Treatstock quote, paid provider call, global mutation, Xcode signing, Keychain write, or backend env write is performed.

## Acceptance

- Focused RED/GREEN tests cover the missing-session prerequisite.
- Focused RED/GREEN tests cover the missing or loopback backend URL prerequisite.
- Focused RED/GREEN tests cover the ready case with concrete LAN URLs.
- A CLI test proves auto mode rejects loopback URLs.
- `make print-fulfillment-readiness` remains a safe report writer that exits 0 at the Make level while writing blocked JSON.
