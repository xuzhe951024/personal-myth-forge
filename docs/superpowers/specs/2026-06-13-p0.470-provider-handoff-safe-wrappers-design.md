# P0.470 Provider Handoff Safe Wrappers Design

## Goal

Make provider and resource handoff report targets refreshable in a no-key, no-device state. When these reports write a usable blocked JSON payload and return exit code 2, Make should accept that state and exit 0 so the final handoff evidence chain can be regenerated without external secrets.

## Current Evidence

Current final readiness is blocked by external inputs: Apple Team ID, iPhone-reachable backend URL, Meshy/OpenAI keys, Treatstock quote evidence, and Xcode license/signing state. Probe runs show these read-only report targets currently write blocked reports but make exits nonzero:

- `provider-handoff`
- `resource-handoff`
- `final-resources-preflight`
- `final-resource-requirements`
- `final-resource-apply-preview`
- `final-configured-preflight`
- `final-configured-evidence-plan`
- `configured-live-evidence-bundle`
- `live-provider-evidence`

## Design

Add one shell wrapper per report target under `services/backend/scripts/`. Each wrapper follows the established safe report pattern:

- create `services/backend/.local`
- run the existing CLI command from `services/backend`
- accept exit code 0 or 2
- print the accepted exit code and written `.local` JSON path
- preserve non-report failures such as exit code 1

Update the Makefile so the listed targets call wrappers directly. The CLI implementations remain unchanged and still return exit code 2 for blocked reports; only Make-level orchestration changes.

## Safety

The wrappers are read-only report refreshers. They do not apply resources, write `services/backend/.env`, write iOS deploy config, call live providers, run Xcode, touch Keychain, or place print orders. Commands that may spend provider credits remain outside this phase.

## Acceptance

- Focused tests prove each new wrapper accepts exit code 2 and rejects exit code 1.
- Dry-run Make tests prove each target uses its wrapper.
- Source acceptance includes wrapper requirements so future source gates can detect accidental regression.
- Running the wrapped Make targets on the current no-key setup exits 0 while writing blocked `.local` evidence.
