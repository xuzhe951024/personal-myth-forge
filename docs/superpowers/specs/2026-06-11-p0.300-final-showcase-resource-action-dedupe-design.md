# P0.300 Final Showcase Resource Action Dedupe Design

## Goal

Remove duplicate source-prefixed final-resource operator actions from the final
showcase readiness report when an equivalent canonical final-resource handoff
action is already present.

## Current Context

`final-showcase-readiness.json` can currently show both of these actions:

```text
final_rehearsal_local: ios_deploy_runbook_local: provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight
provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight
```

They point to the same ignored resource handoff file and the same validation
command. The source-prefixed action is useful while reading the nested source
report, but noisy in the top-level final-showcase operator checklist.

## Design

Extend final-showcase operator-action dedupe so it treats source-prefixed final
resource actions as duplicates when their bare root matches a non-prefixed
canonical final-resource validation action already present in the list.

The dedupe remains narrow:

- It only applies to source-prefixed actions already recognized by
  `FINAL_SHOWCASE_SOURCE_ACTION_PREFIXES`.
- It only applies to actions whose bare root starts with one of the known
  final-resource validation action roots.
- It preserves source-prefixed device, Xcode, handoff, and non-resource actions
  unless existing dedupe rules already handle them.

## Out of Scope

- No writes to `.local/final-resources.env`.
- No provider calls, live Meshy/OpenAI/Treatstock checks, Xcode signing, Keychain
  writes, or physical-device operations.
- No changes to nested source reports; only the top-level final-showcase action
  list is normalized.

## Verification

- RED/GREEN final-showcase test proving prefixed Meshy/OpenAI resource actions
  are removed when canonical actions are present.
- Focused final-showcase and operator-action tests.
- `make final-showcase-readiness` still exits 2 while external resources and
  device evidence are missing, but generated operator actions contain canonical
  Meshy/OpenAI resource handoffs once each and no source-prefixed duplicates.
- Final lint, Swift contract/project checks, and backend tests before merge.
