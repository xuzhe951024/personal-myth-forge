# P0.561 Showcase Print Readiness Action Design

## Context

The final objective includes a printable `print_asset` and third-party print quote handoff. The current `final-showcase-readiness.json` exposes the print fulfillment capability, but its top-level `operator_actions` can drop the local prerequisite:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make print-fulfillment-readiness
```

That command is safe and project-local. It writes ignored iOS deploy config through the existing auto writer and reruns the print readiness report. Without it, the showcase action list points clearly at iOS deploy and provider/key handoff, but print fulfillment is less directly actionable.

## Design

P0.561 keeps filtering generic validation-only print readiness actions, such as visual-regression refreshes that happen to rerun `make print-fulfillment-readiness`. It preserves configured print quote request prerequisites when their validation command is `make print-fulfillment-readiness`.

The preserved prerequisite roots are the existing configured print quote request prerequisite actions:

- `make final-demo-launch-local`
- `set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL`
- `DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto`

This phase does not surface unguarded live provider calls. The guarded `PMF_ALLOW_PRINT_PROVIDER_CALLS=1 make print-quote-configured` action remains controlled by the existing print quote preference logic.

## Acceptance

- A RED/GREEN test proves final showcase filtering preserves the backend-auto print readiness prerequisite.
- Existing coverage still proves generic validation-only print readiness actions are not promoted.
- Refreshed `services/backend/.local/final-showcase-readiness.json` includes:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make print-fulfillment-readiness
```

- The refreshed final showcase report still excludes:

```text
make visual-regression-local; rerun make print-fulfillment-readiness
```

## Safety

This phase changes report action filtering only. It does not execute provider calls, request print quotes, write secrets, touch Xcode signing, mutate Keychain, or perform external paid actions.
