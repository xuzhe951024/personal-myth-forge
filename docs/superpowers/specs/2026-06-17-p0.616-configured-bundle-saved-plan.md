# P0.616 Configured Bundle Saved Plan

## Problem

`make final-configured-evidence-plan` can write a configured plan that has already consumed the ignored local handoff/resource state and reached the live-provider consent step. Running `make configured-live-evidence-bundle` then recomputes that plan from process settings and can fall back to a weaker local-stub blocker. This makes the saved configured plan and configured evidence bundle disagree.

## Scope

- When no explicit settings are passed and live provider calls are not allowed, let `configured-live-evidence-bundle` reuse `services/backend/.local/final-configured-evidence-plan.json` if it is a valid final configured evidence plan report.
- Keep explicit `Settings` calls and `allow_live_provider_calls=True` on the existing realtime build path.
- Ignore missing, unreadable, or wrong-kind saved reports and fall back to realtime build.
- Do not run live provider commands or any external paid/provider action.

## Acceptance

- RED test proves the bundle ignores a saved configured plan and reports a weaker blocker.
- GREEN implementation makes the bundle inherit the saved configured plan's live-consent blocker.
- Focused, adjacent, lint, and backend regression gates remain green.
