# P0.309 Live Provider Evidence Action Normalization Design

## Goal

Normalize the top-level `operator_actions` emitted by `live-provider-evidence` so the final configured provider proof path gives operators canonical make targets instead of narrative `run make ...` or `rerun provider ...` wrappers.

## Context

The final iOS showcase is still blocked by external inputs and real-device evidence. The provider/key handoff path already has shared action normalization in `myth_forge_api.operator_actions`, and several final readiness reports use that helper. `live_provider_evidence.py` still emits two verbose action strings directly:

- `run make live-provider-evidence after configured provider evidence files are refreshed`
- `rerun provider handoff readiness: make provider-handoff`

Those strings are already supported by `normalize_operator_action`, but the live provider report does not call it before publishing top-level actions.

## Approach

Use the existing shared normalizer inside `live_provider_evidence._operator_actions()`.

The implementation keeps the evidence row `command` fields unchanged because those commands describe the source slot contract. Only top-level `operator_actions` become canonical operator steps:

- `make live-provider-evidence`
- `make provider-handoff`

The report remains read-only and safe: no provider calls, no key reads beyond existing evidence files, no writes except the ignored report output when the CLI is invoked.

## Data Flow

`build_live_provider_evidence_report()` builds evidence rows, chooses `status` and `first_blocker`, calls `_operator_actions()`, then sanitizes the whole report. `_operator_actions()` should create the current raw action hints, pass each through `normalize_operator_action`, and dedupe normalized values.

## Testing

Add a focused regression assertion to `tests/test_live_provider_evidence.py`:

- missing evidence reports should emit canonical top-level actions.
- raw verbose action strings must not appear in `operator_actions`.
- row-level `commands` and `evidence[*].command` remain unchanged.

Run the focused live provider and operator action tests for RED/GREEN, then run backend lint and the full backend test suite before merge.

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, Xcode, Keychain, or real provider APIs.
- Do not change configured evidence ordering or readiness classification.
- Do not modify ignored resource files with secrets.
