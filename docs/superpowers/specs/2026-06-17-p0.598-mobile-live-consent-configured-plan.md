# P0.598 Mobile Live Consent Configured Plan

## Context

The final configured evidence plan now carries guarded Meshy/OpenAI commands such as
`PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-3d-configured`. The iPhone
Final Showcase and Provider Handoff summaries can surface those commands, but the
dedicated Live Provider Consent panel still only shows provider readiness, final
resources, resource handoff, live evidence, and configured bundle status.

That leaves the consent panel one step behind the current final handoff flow.

## Decision

Add a configured evidence plan row to the iPhone Live Provider Consent summary. The
row should display the plan status and prefer guarded operator actions before falling
back to the next action or first non-ready step.

## Acceptance

- The Live Provider Consent summary includes a `configured_plan` row.
- The row can display
  `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-3d-configured; rerun make final-configured-evidence-plan`.
- Configured acceptance remains disabled until the configured evidence plan, live
  evidence, and configured bundle are all ready.
- Unsafe text continues through existing mobile redaction.
