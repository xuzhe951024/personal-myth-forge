# P0.127 iOS Deploy Apply Preview Gate Design

## Problem

The final resource flow now includes a safe dry-run:

```text
final-resource-requirements -> final-resources-preflight -> final-resource-apply-preview -> final-apply-resources
```

The iOS deploy runbook still jumps from `make final-resources-preflight` to
`make final-apply-resources`. That is inconsistent with the safer final
handoff path and weakens deploy readiness: the operator should be able to see
what backend and iOS config files will be written before any writer scripts run.

## Goals

- Add final resource apply preview as an explicit iOS deploy runbook input slot.
- Insert `make final-resource-apply-preview` into the runbook command sequence
  before `make final-apply-resources`.
- Keep the runbook read-only. It may inspect generated preview reports in
  process, but it must not run shell writer scripts, write `.env`, write
  `Deployment.local.xcconfig`, call providers, run Xcode, or mutate global
  state.
- Preserve existing ready/partial/blocked behavior:
  - missing preview blocks the runbook before apply
  - ready preview allows the runbook to proceed to apply/manual deploy gates
  - Xcode/signing remain manual/global gates outside this report
- Make the device launch certificate inherit the preview step through its
  compact `ios_deploy_runbook.command_sequence` summary.

## Non-Goals

- Do not implement real Xcode deployment.
- Do not accept Apple SDK licenses.
- Do not write signing settings or keychain state.
- Do not run `make final-apply-resources`.
- Do not require real API keys for local tests.

## Data Model

`ios_deploy_runbook_report.input_slots` gains:

```json
{
  "id": "final_resource_apply_preview",
  "label": "Final resource apply preview",
  "status": "missing",
  "required": true,
  "source": "services/backend/.local/final-resource-apply-preview.json",
  "operator_action": "run make final-resource-apply-preview before applying resources"
}
```

`ios_deploy_runbook_report.command_sequence` gains:

```json
{
  "id": "preview_final_resource_apply",
  "label": "Preview final resource apply",
  "status": "missing",
  "command": "make final-resource-apply-preview",
  "detail": "Preview backend .env and iOS deploy config writes without running writer scripts."
}
```

The existing `apply_final_resources` step remains after the preview step.

## Safety

The runbook continues to report:

```json
{
  "commands_run": false,
  "provider_calls": false,
  "global_mutation": false,
  "provider_secrets_in_report": false,
  "raw_media_in_report": false,
  "payment_links_in_report": false,
  "local_paths_in_report": false
}
```

The apply preview builder has its own safety flags for no `.env` write, no iOS
deploy config write, no shell writers, no live provider calls, no global
mutation, and no Xcode/signing mutation. The runbook only consumes the sanitized
status and summary.
