# P0.289 Provider Evidence Action Normalization Design

## Goal

Normalize provider handoff and live-provider evidence operator actions to their canonical make targets so final iPhone showcase handoff lists do not show duplicate command wordings.

## Context

The final showcase is still blocked by real external resources and device proof. Current local reports are useful, but the top-level operator actions still include both canonical and verbose variants:

```text
rerun provider handoff readiness: make provider-handoff
make provider-handoff
run make live-provider-evidence after configured provider evidence files are refreshed
make live-provider-evidence
```

The verbose variants are understandable, but they duplicate existing make targets and make the final operator checklist noisier.

## Design

Keep normalization centralized in `myth_forge_api.operator_actions`.

Add an exact canonical make-target replacement pass for known verbose make-target actions:

- `rerun provider handoff readiness: make provider-handoff` becomes `make provider-handoff`
- provider-handoff CLI actions that currently normalize to the verbose form also become `make provider-handoff`
- `run make live-provider-evidence after configured provider evidence files are refreshed` becomes `make live-provider-evidence`
- `run make live-provider-evidence to refresh live provider evidence after cost consent` becomes `make live-provider-evidence`

The existing detail suffix handling should remain unchanged, so ` | detail` text is preserved after the canonical command.

## Scope

In scope:

- shared operator action normalization
- focused tests for direct normalization and final-showcase report action aggregation
- roadmap entry for P0.289
- `.local` refresh verification

Out of scope:

- running live provider evidence
- provider API calls, cost-incurring calls, or key writes
- Treatstock quote execution
- Xcode, signing, Keychain, or device actions

## Verification

This iteration is valid when:

- RED tests show the verbose provider/live evidence actions currently leak through
- GREEN tests show they normalize to canonical make targets
- refreshed `.local` reports no longer expose those verbose variants in top-level operator actions
- backend lint and tests pass
