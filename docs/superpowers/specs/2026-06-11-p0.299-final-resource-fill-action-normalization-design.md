# P0.299 Final Resource Fill Action Normalization Design

## Goal

Normalize final resource fill-guide operator actions into the existing
validation-aware resource handoff phrasing so top-level handoff reports do not
show separate `fill` and `provide` actions for the same missing key.

## Current Context

`final-local-report-refresh.json` can currently include both actions:

```text
provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight
fill MESHY_API_KEY in services/backend/.local/final-resources.env; rerun make final-resources-preflight
```

Both refer to the same ignored resource handoff file. The first form is the
canonical final-resource validation action used across resource requirements,
preflight, and showcase reports. The second form comes from the fill guide and
is useful inside the fill-guide report, but it is noisy in aggregated operator
checklists.

## Design

Extend the shared `normalize_operator_action()` helper so it recognizes fill
guide commands of the form:

```text
fill <RESOURCE_ID> in services/backend/.local/final-resources.env
```

When `<RESOURCE_ID>` is one of the known final resource validation keys, it
should normalize to:

```text
provide <RESOURCE_ID> in final-resources.env; rerun make final-resources-preflight
```

Detail suffixes after ` | ` should be preserved. Unknown fill targets should be
left unchanged to avoid rewriting unrelated instructions.

This keeps lower-level fill-guide fields intact while allowing aggregators such
as final-local refresh to dedupe against the canonical resource action.

## Out of Scope

- No `.local/final-resources.env` writes.
- No provider calls, Xcode changes, signing, Keychain, device deployment, or
  paid print actions.
- No changes to final-resource-fill-guide report shape.
