# P0.252 Mobile Final Demo Launch Status Decode Design

## Context

The backend final demo launch report now exposes both `overall_status` and the common top-level `status` field. The iPhone-side Swift model still only decodes `overallStatus`, and the final launch summary uses `overallStatus` for its badge and title.

For the final iPhone showcase, mobile code should consume the same `{status, first_blocker, next_action}` shape that Mac-side handoff reports expose, while staying compatible with older saved reports that only include `overall_status`.

## Goal

Update the mobile core final demo launch model and summary builder so the iPhone final launch panel decodes `status` and uses it as the preferred launch status, falling back to `overallStatus` when older payloads omit `status`.

## Design

Add `status: String?` to `FinalDemoLaunchReport`. The field is optional so older saved local reports remain decodable.

In `FinalLaunchMobileSummaryBuilder`, compute a local `reportStatus`:

```swift
let reportStatus = report.status ?? report.overallStatus
```

Use `reportStatus` for:

- `FinalLaunchMobileSummary.overallStatus`
- the `Final launch ...` title
- the launch receipt status row

Leave the existing `overallStatus` field intact for backward compatibility and for code paths that still need the original value.

## Testing

Add Swift contract tests that:

- decode a payload containing both `status` and `overall_status`
- assert `FinalDemoLaunchReport.status == "blocked"` when the top-level field differs from `overall_status`
- assert the mobile summary badge/title/receipt use `status`
- assert a legacy payload without `status` still decodes and falls back to `overallStatus`

## Safety

This is a local Swift model and summary change only. It does not call live providers, run Xcode signing, write secrets, mutate global state, or perform real device deployment.

## Out of Scope

This iteration does not supply provider keys, Treatstock configuration, Apple signing values, or physical iPhone evidence, and it does not mark the full goal complete.
