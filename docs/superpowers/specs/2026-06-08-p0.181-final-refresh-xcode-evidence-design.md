# P0.181 Final Refresh Xcode Evidence Design

## Context

`make final-rehearsal-local` is the one-command local rehearsal for the final
demo handoff. It ends by running `final-local-report-refresh-local`, which safely
accepts blocked reports and writes a consolidated `.local` evidence set.

The refresh currently writes `mobile-deploy-preflight-evidence.json`, but it
does not write `mobile-xcode-build-evidence.json`. Downstream reports can use
`mobile-xcode-build-evidence.json` when it already exists, but a fresh machine
or clean `.local` directory can reach the final closure packet without an
explicit Xcode evidence file.

Running the real Xcode build gate inside final refresh is not acceptable:

- `xcodebuild` can touch Xcode/SDK state.
- Xcode license acceptance is a global, sudo-protected user action.
- The refresh safety contract says it does not execute Xcode/signing gates.

## Goal

Make `final-local-report-refresh` always write an Xcode evidence report slot
without executing Xcode, while preserving any existing real
`mobile-xcode-build-evidence.json` generated by `make mobile-xcode-build-evidence`.

## Design

Add a `mobile_xcode_build_evidence` step to
`services/backend/src/myth_forge_api/final_local_report_refresh.py`.

The step will:

1. Look for `services/backend/.local/mobile-xcode-build-evidence.json`.
2. If it exists and contains a JSON object, return that report unchanged.
3. If it is missing or unreadable, return a safe blocked report:
   - `kind`: `mobile_xcode_build_evidence_report`
   - `status`: `blocked`
   - `classification`: `xcode_build_gate_not_run_by_final_local_report_refresh`
   - `command`: `make mobile-xcode-build`
   - `checks[0].id`: `xcode_build_gate`
   - `operator_actions[0]`: `run make mobile-xcode-build-evidence outside final-local-report-refresh`
   - `safety.commands_run`: `false`
   - `safety.xcode_or_signing`: `false`
   - `safety.global_mutation`: `false`
   - `safety.writes_derived_data`: `false`

The step writes to the same canonical path:

```text
services/backend/.local/mobile-xcode-build-evidence.json
```

When real evidence already exists, final refresh preserves its classification
and detail. When no evidence exists, downstream reports see an explicit,
sanitized, non-executing Xcode evidence blocker instead of inferring from final
acceptance.

## Ordering

The new step runs after `mobile_deploy_preflight_evidence` and before
`ios_device_evidence_bundle`. This ensures the iOS device evidence bundle and
final launch closure packet have a canonical Xcode evidence source during a
fresh one-command rehearsal.

## Non-Goals

- Do not run `xcodebuild` inside final refresh.
- Do not accept the Xcode license.
- Do not change signing, keychain, or global Xcode state.
- Do not alter provider calls, live calls, backend `.env`, or iOS deploy config
  writers.
- Do not mark Xcode ready without real evidence.

## Acceptance

- `run_final_local_report_refresh()` writes
  `services/backend/.local/mobile-xcode-build-evidence.json` on a fresh fixture.
- The new step appears in `steps_by_id.mobile_xcode_build_evidence`.
- The step status is `blocked`, accepted as a blocked local report.
- The safe fallback report has `commands_run: false`, `xcode_or_signing: false`,
  `global_mutation: false`, and
  `classification: xcode_build_gate_not_run_by_final_local_report_refresh`.
- If a real `mobile-xcode-build-evidence.json` already exists, final refresh
  preserves it instead of replacing it.
- `ios_device_evidence_bundle` reads the canonical Xcode evidence file produced
  by final refresh.
- `final_launch_closure_packet` receives an explicit Xcode blocker from the
  evidence bundle.
- `make final-rehearsal-local` still exits `0` via the wrapper when the report
  is blocked.
- Backend lint/tests and Swift mobile checks still pass.

## Test Strategy

Use TDD:

1. Add failing backend tests that expect final refresh to write and order the
   Xcode evidence step.
2. Run the focused tests and watch them fail because the step is absent.
3. Add the safe snapshot step and fallback report builder.
4. Run focused tests, then final rehearsal, backend lint/tests, and Swift mobile
   checks.

## Self-Review

- Placeholder scan: no TBD/TODO/open placeholders.
- Safety check: fallback report explicitly says no commands/Xcode/signing/global
  mutation ran.
- Scope check: this is one backend report-refresh slice; no mobile UI or
  provider behavior changes.
- Ambiguity check: existing real evidence is preserved; missing evidence creates
  a blocked snapshot rather than executing Xcode.
