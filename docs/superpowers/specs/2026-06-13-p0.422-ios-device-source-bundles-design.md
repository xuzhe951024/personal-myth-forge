# P0.422 iOS Device Source Bundles Design

## Goal

Preserve bottom-level mobile evidence `device_action_bundle` payloads inside `ios_device_evidence_bundle.source_reports` so final iPhone handoff reports can trace from an iOS deploy blocker down to the exact device-side or Mac-side action that proves or unblocks it.

## Current State

`mobile_deploy_preflight_evidence_report` and `mobile_xcode_build_evidence_report` now expose first-class `device_action_bundle` objects. `ios_device_evidence_bundle_report` loads both saved reports, uses them to improve evidence slot detail, and exposes each as a source report summary. The source summary currently keeps only `kind`, `status`, and `summary`, so the newly available child action bundles are lost at this boundary.

## Approach

Extend `ios_device_evidence_bundle._source_summary()` to copy `device_action_bundle` when the source report provides one. This keeps the propagation local to the report boundary and avoids re-deriving device actions from partially summarized checks. The existing report sanitization remains the final safety boundary, so copied bundles still pass through the same redaction path as the rest of the report.

## Data Flow

```text
mobile-deploy-preflight-evidence.json
mobile-xcode-build-evidence.json
  -> _load_optional_report()
  -> _source_summary()
  -> ios_device_evidence_bundle.source_reports[*].device_action_bundle
```

## Safety

This iteration does not run Xcode, change signing, write provider env files, invoke live providers, or place print orders. It only preserves already-saved ignored local report fields and relies on the existing recursive sanitizer to remove secrets, local paths, payment URLs, and raw private context.

## Testing

Add a focused contract test that writes synthetic mobile deploy and Xcode evidence reports containing child `device_action_bundle` objects, builds the iOS device evidence bundle, and asserts that both child bundles are present under `source_reports` with ids, first actions, and safety flags intact. The test also asserts no synthetic secret or temporary path leaks through the copied source summaries.
