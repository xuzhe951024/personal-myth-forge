# P0.595 Showcase Configured Plan Kind

## Context

P0.594 wired final showcase readiness to a saved configured evidence plan, but the
test fixture used `final_configured_evidence_plan` while the actual generated report
uses `final_configured_evidence_plan_report`.

The mismatch makes final showcase fall back to the coarse live provider evidence
action even though the real configured plan contains a granular guarded next action.

## Decision

The final showcase loader should accept the real
`final_configured_evidence_plan_report` kind and summarize it under the existing
`final_configured_evidence_plan` evidence key.

## Acceptance

- The focused final showcase test uses the real configured evidence plan report kind.
- Game asset 3D, AI Agent NPC, and provider/key handoff rows use the granular
  configured 3D evaluation action when the saved plan is consent-required.
- Safe `.local` refresh shows final showcase first blocker command as
  `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-3d-configured; rerun make final-configured-evidence-plan`.
