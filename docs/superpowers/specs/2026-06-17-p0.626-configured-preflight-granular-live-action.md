# P0.626 Configured Preflight Granular Live Action Spec

## Goal

When final resources and provider handoff are ready, `make final-configured-preflight`
should point the operator at the same granular live-provider evidence commands used by
the final handoff path.

## Problem

The higher-level final handoff index now routes live evidence through:

- `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-3d-configured; rerun make final-configured-evidence-plan`
- `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-npc-configured; rerun make final-configured-evidence-plan`

`final-configured-preflight` can still surface broader legacy actions such as
`make live-provider-evidence` and the configured final-acceptance cost-review action.
That is safe, but it makes the final iOS showcase handoff less executable when the
repo is otherwise ready and only live provider evidence remains.

## Acceptance

- With Meshy/OpenAI/iOS resources configured and print provider left local,
  `build_final_configured_preflight_report` exits ready.
- Its operator actions include the granular configured 3D and NPC evaluation consent
  commands.
- Its operator actions do not include bare `make live-provider-evidence`.
- Its operator actions do not include the broad `make final-acceptance-configured`
  cost-review fallback.
- Missing-resource flows still surface resource-fill actions.
- No live provider call is made by the preflight report.
