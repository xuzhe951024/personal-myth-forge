P0.482 Closure Packet Print Handoff Design

Goal

Make the final launch closure packet present the same cost-consent guarded print quote handoff as the final showcase, final demo launch, final handoff index, and iOS rehearsal reports.

Context

P0.480 added `PMF_ALLOW_PRINT_PROVIDER_CALLS=1 make print-quote-configured`, and P0.481 propagated that guarded action through most final handoff aggregators. The saved `final-launch-closure-packet.json` still surfaces the weaker validation-only `make print-fulfillment-readiness` action near the top of `operator_actions`, even when the guarded print quote action is also available later in the same list.

That closure packet is meant to be the final operator handoff. It should not ask the operator to run a readiness check when the actual missing configured print evidence requires the cost-consent guarded quote target.

Non-Goals

- Do not call Treatstock or any print provider.
- Do not change print quote APIs, provider adapters, or quote evidence schemas.
- Do not mark print fulfillment ready without configured quote evidence.
- Do not modify Xcode signing, Keychain, global environment, or real provider resources.

Design

Add print-handoff shaping to `final_launch_closure_packet._operator_actions()` after existing normalization and apply-preview filtering. The shaping should:

- detect the guarded configured print quote action: `PMF_ALLOW_PRINT_PROVIDER_CALLS=1 make print-quote-configured; rerun make print-fulfillment-readiness`;
- remove validation-only `make print-fulfillment-readiness` actions when that guarded handoff exists;
- continue preserving unrelated resource, iOS deploy, provider, and live-provider consent actions;
- leave fallback `make print-fulfillment-readiness` available when no guarded print quote action exists.

Acceptance Criteria

- A closure packet built from default missing resources contains the guarded print quote handoff in top-level `operator_actions`.
- The same top-level action list does not contain bare `make print-fulfillment-readiness` when the guarded print quote handoff exists.
- Existing final showcase readiness status remains blocked until external iOS/provider resources are supplied.
- Focused tests prove the RED/GREEN behavior without external calls.
- Safe local evidence refresh writes ignored `.local` reports and still exits through safe wrappers.

Safety

This phase only changes report action prioritization/filtering. It reads local ignored evidence and writes ignored `.local` reports during verification. It must not set provider consent env vars, use real keys, run Xcode signing, or place print orders.
