# P0.302 Final Showcase iOS Config Action Normalization Design

## Goal

Make the final-showcase top-level iOS deploy config operator action directly
copy-pastable by removing its source-report prefix.

## Current Context

`final-showcase-readiness.json` currently includes:

```text
ios_device_launch_certificate: provide iOS deploy config in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
```

The action is useful, but the `ios_device_launch_certificate:` prefix is source
provenance, not part of the operator handoff. The top-level `operator_actions`
list should contain the executable handoff:

```text
provide iOS deploy config in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
```

## Design

Extend the final-showcase-specific action normalization helper added in P0.301.
After shared normalization and validation enrichment, strip known final-showcase
source prefixes when the bare root is the canonical iOS deploy config handoff.

This stays narrow:

- It only applies in `final_showcase_readiness`.
- It only strips source prefixes already recognized by
  `FINAL_SHOWCASE_SOURCE_ACTION_PREFIXES`.
- It only applies to the canonical iOS deploy config action root.
- It preserves other non-make source-prefixed actions unless existing rules
  already normalize or dedupe them.

## Out of Scope

- No writes to `Deployment.local.xcconfig`.
- No Xcode signing, build, Keychain, backend-device-demo, or physical iPhone
  operation.
- No provider calls or paid print actions.
- No changes to nested source report payloads.
