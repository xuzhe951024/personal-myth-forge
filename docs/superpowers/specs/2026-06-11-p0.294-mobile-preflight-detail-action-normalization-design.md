# P0.294 Mobile Preflight Detail Action Normalization Design

## Goal

Convert mobile deploy preflight evidence details that were promoted as operator
actions into concrete, executable iOS deploy handoff actions.

## Current Context

`final-showcase-readiness` is blocked by iOS deploy evidence. The first action is
already concrete:

```text
provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
```

However, nested reports can still promote a diagnostic detail as if it were an
action:

```text
final_rehearsal_local: mobile_deploy_preflight_evidence: Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable
```

That string is useful evidence, but it is not something an operator can execute
or copy into a handoff checklist.

## Design

Extend the centralized `myth_forge_api.operator_actions.normalize_operator_action()`
path with a detail-derived mobile preflight normalizer.

When an action contains `mobile_deploy_preflight_evidence:` followed by a known
missing configuration detail, normalize the command portion to the matching
handoff action and preserve the original detail after a ` | ` suffix.

Mapping rules:

- Details containing `DEVELOPMENT_TEAM` become
  `provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`.
- Details containing `PRODUCT_BUNDLE_IDENTIFIER` become
  `provide PRODUCT_BUNDLE_IDENTIFIER in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`.
- Details containing `PMF_BACKEND_BASE_URL` or `iPhone-reachable` become
  `set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL; rerun make mobile-deploy-preflight`.

If multiple details are present, use the first actionable detail in the order
above and keep the full diagnostic detail as the suffix. This preserves evidence
without returning multiple actions from a string normalizer.

## Scope

- Add RED/GREEN tests for plain and prefixed `mobile_deploy_preflight_evidence`
  detail actions.
- Keep the behavior in `operator_actions.py` so final showcase, rehearsal, and
  local report aggregators benefit consistently.
- Update exact test expectations where reports should no longer surface the raw
  detail as an action.

## Out of Scope

- No real Xcode build, iPhone deployment, signing configuration, Keychain access,
  provider call, or paid print action.
- No changes to the underlying blocker status. External resources are still
  required before final readiness can pass.

## Verification

- Focused RED/GREEN tests for `normalize_operator_action()`.
- Focused report tests for iOS device launch rehearsal, final showcase readiness,
  and final local report refresh.
- Refresh local readiness reports and scan `.local` outputs for the raw
  detail-only action string.
- Run lint, Swift contract checks, and backend tests before merge/push.
