# P0.547 Mobile Device Bundle Validation Design

## Goal

Make mobile deploy preflight device action bundle commands validation-aware, including the backend URL auto-writer child action.

## Current Gap

`mobile-deploy-preflight-evidence.json` already exposes a top-level next action and operator action that include:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

The same report's `device_action_bundle.actions[].command` for `backend_base_url` still contains the bare auto-writer command without the validation rerun. That makes the iPhone-visible child action less directly copy-pasteable than the top-level action.

## Intended Behavior

- Device action bundle commands should use the same validation-aware form as top-level operator actions when they use the auto deploy config writer.
- The `next_action.command` nested inside the bundle action should match the action command.
- Existing blocked checks, safety flags, and no-Xcode/no-provider behavior stay unchanged.
- No deploy config is written by the report builder; it remains read-only evidence.

## Acceptance

- `test_preflight_evidence_suggests_auto_writer_for_team_and_backend_url` requires the `backend_base_url` device action and nested next action to include `; rerun make mobile-deploy-preflight`.
- Refreshed `.local/mobile-deploy-preflight-evidence.json` shows validation-aware commands for both device bundle actions.
- Focused and full backend tests pass.
