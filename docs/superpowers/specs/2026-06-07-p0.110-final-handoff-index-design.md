# P0.110 Final Handoff Index Design

## Context

P0.108 added `make final-rehearsal-local`, which writes the local/no-key handoff
reports used by the iPhone final showcase. P0.109 added
`make final-configured-preflight`, which writes a read-only configured resource
packet for final API/key and iOS deploy review. The next gap is that the
operator still has to remember several report paths and commands when switching
from no-key rehearsal to final key-backed acceptance.

P0.110 adds one read-only index that explains the final handoff state across
both lanes without triggering provider calls or writing secrets.

## Goal

Add a project-root `make final-handoff-index` command that writes
`services/backend/.local/final-handoff-index.json` as the final operator index
for local rehearsal, configured preflight, launch reports, iOS deploy runbook,
and consent-gated configured final acceptance.

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, Sculpteo, or any live provider.
- Do not run `final-acceptance --provider-mode configured` automatically.
- Do not run `final-rehearsal-local`, `final-configured-preflight`, or
  `final-apply-resources` from inside the report builder.
- Do not write `services/backend/.env`,
  `apps/mobile/ios/Config/Deployment.local.xcconfig`, or provider keys.
- Do not start backend servers, run Xcode, accept Apple licenses, mutate
  signing/keychain state, or change global developer settings.
- Do not add a new iPhone endpoint in this slice. The iPhone still reads
  `/v1/final-demo-launch`.

## Report Contract

New report kind: `final_handoff_index_report`.

Top-level fields:

- `kind`: fixed report kind.
- `status`: `ready` only when local rehearsal evidence exists, configured
  preflight is ready, and no embedded required gate is missing or blocked.
  Otherwise `blocked` while still writing the report.
- `summary`: counts `ready`, `missing`, `blocked`, `manual`, `partial`, and
  `live` across the indexed lanes and command groups.
- `lanes`: ordered list of final handoff lanes:
  - `local_rehearsal`: local/no-key saved reports from P0.108.
  - `configured_preflight`: P0.109 resource/provider/iOS preflight packet.
  - `configured_launch`: configured final launch report command.
  - `device_deploy`: backend-device-demo, deploy preflight, and Xcode manual
    gate.
  - `live_acceptance`: configured final acceptance command, always marked
    `live` and `requires_consent=true` until the operator explicitly runs it.
- `source_reports`: sanitized metadata for saved JSON files:
  - `services/backend/.local/3d-evaluation-local.json`
  - `services/backend/.local/npc-evaluation-local.json`
  - `services/backend/.local/final-acceptance-local.json`
  - `services/backend/.local/final-demo-launch-local.json`
  - `services/backend/.local/ios-deploy-runbook-local.json`
  - `services/backend/.local/final-configured-preflight.json`
- `operator_sequence`: exact next-step command sequence with statuses,
  required/optional/live flags, and safety notes.
- `commands`: project-root commands including `make final-rehearsal-local`,
  `make final-configured-preflight`, `make final-handoff-index`,
  `make backend-device-demo`, `make mobile-deploy-preflight`, and the
  consent-gated configured final acceptance CLI.
- `safety`: false for provider calls, command execution, backend/iOS config
  writes, global mutation, Xcode/signing, keychain writes, provider secrets in
  report, raw media in report, local paths in report, and payment links.

Exit behavior:

- Exit `0` only when the handoff index is ready enough for the next configured
  operator step.
- Exit `2` when blockers remain but the JSON packet was written.
- Exit `1` remains reserved for unusable runtime errors.

## CLI And Make

New CLI:

```bash
cd services/backend
uv run python -m myth_forge_api.cli final-handoff-index \
  --repo-root ../.. \
  --output .local/final-handoff-index.json
```

New Make target:

```bash
make final-handoff-index
```

The Make target writes only the ignored JSON report. It should not depend on
other Make targets because those may run acceptance checks or local servers.

## Testing

- Unit tests for missing local reports, partial local reports, ready configured
  preflight fixture, command ordering, and redaction.
- CLI test for `final-handoff-index --output`.
- Make/source-gate tests requiring:
  - `final_handoff_index.py`
  - CLI command `final-handoff-index`
  - Make target `final-handoff-index`
  - output path `.local/final-handoff-index.json`
  - docs and source gates mentioning the final handoff index.
- Swift ProjectChecks source gate for the new Mac-side handoff command.
- Visual regression evidence: 390x844 final handoff index panel showing local
  rehearsal, configured preflight, deploy, live acceptance, and safety.

## Design Self-Review

- No placeholders or deferred requirements remain.
- Scope is a single Mac-side read-only report and Make target.
- The report does not duplicate existing report internals; it indexes and
  summarizes saved artifacts and safe commands.
- The design preserves the P0.109 choice not to add another mobile endpoint.
