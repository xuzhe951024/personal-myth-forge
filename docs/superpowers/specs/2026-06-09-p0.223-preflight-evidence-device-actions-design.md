# P0.223 Preflight Evidence Device Actions Design

## Context

P0.222 added a `device_action_bundle` to `final_showcase_readiness_report`, but each action currently inherits the overall `ios_deployable` status. The project already has a read-only `mobile_deploy_preflight_evidence_report` that captures the result of `make mobile-deploy-preflight`, including backend health and deploy config blockers. Final showcase readiness should use that evidence to mark the backend/preflight device actions independently from Xcode/signing and final rehearsal proof.

## Goal

Connect `services/backend/.local/mobile-deploy-preflight-evidence.json` to final showcase readiness so the iPhone Final Launch panel can show which device actions have actual preflight evidence.

## Non-Goals

- Do not run `make mobile-deploy-preflight`, `backend-device-demo`, Xcode, signing, or keychain commands inside the readiness builder.
- Do not mark iOS deployability ready solely from preflight evidence.
- Do not expose LAN URLs, local absolute paths, provider secrets, raw media, or payment links.
- Do not change the existing `mobile-deploy-preflight-evidence` command behavior.

## Contract

When `mobile-deploy-preflight-evidence.json` is missing, final showcase readiness embeds a sanitized missing evidence summary and keeps `start_backend_device_demo` and `run_mobile_deploy_preflight` blocked:

```json
{
  "evidence": {
    "mobile_deploy_preflight_evidence": {
      "kind": "mobile_deploy_preflight_evidence_report",
      "status": "missing"
    }
  },
  "device_action_bundle": {
    "actions": [
      {
        "id": "run_mobile_deploy_preflight",
        "status": "blocked",
        "evidence_status": "missing",
        "evidence_source": "services/backend/.local/mobile-deploy-preflight-evidence.json",
        "validation_command": "make mobile-deploy-preflight-evidence"
      }
    ]
  }
}
```

When the evidence report is ready, `start_backend_device_demo` and `run_mobile_deploy_preflight` become ready while `resolve_xcode_build_gate` and `run_ios_device_launch_rehearsal` remain tied to iOS deploy evidence:

```json
{
  "device_action_bundle": {
    "summary": {"actions": 4, "ready": 2, "blocked": 2},
    "first_action": {"id": "resolve_xcode_build_gate"},
    "actions": [
      {"id": "start_backend_device_demo", "status": "ready"},
      {"id": "run_mobile_deploy_preflight", "status": "ready"}
    ]
  }
}
```

The Swift Final Launch summary should render evidence status on each device action row, for example `evidence ready | make mobile-deploy-preflight-evidence`.

## Safety

The integration is read-only and only loads an ignored local JSON evidence file. Existing final showcase sanitization remains the final guard. Missing, malformed, or wrong-kind evidence is treated as blocked/missing, not as ready.

## Verification

- Backend RED/GREEN tests for missing and ready `mobile_deploy_preflight_evidence`.
- Swift RED/GREEN tests for optional `evidenceStatus`, `evidenceSource`, `evidenceDetail`, and `validationCommand` decode/render.
- Source acceptance gates and visual regression evidence for the mobile device action evidence row.
- Full backend lint/test, Swift checks, visual regression, final rehearsal, and whitespace checks before merge.
