# P0.466 External Ledger Device-First Actions Design

## Context

The final external action ledger is the remaining top-level handoff report that still starts with provider keys. The wider final showcase path now treats the local physical iPhone deploy action as the first blocker. The ledger already carries an iOS deploy runbook `device_action_bundle`, but its `first_blocker`, `next_action`, and first `operator_actions` row still come from the `resource_inputs` group because that group is listed first.

## Decision

When the iOS deploy runbook device bundle has a non-ready first action, promote that action to the external ledger's top-level blocker path:

- `first_blocker`
- `next_action`
- first row in `operator_actions`

Keep `action_groups` order unchanged. The resource input group still records Meshy/OpenAI/Treatstock/iOS values and remains blocked until the user provides keys/config. This phase changes the operator reading path, not the inventory semantics.

## Scope

- Add a regression test proving the ledger starts with the iPhone deploy writer action before provider key actions.
- Promote the device bundle first action into top-level blocker selection.
- Promote the same action into top-level operator actions with existing normalization.
- Preserve provider key actions after the promoted device action.

## Out Of Scope

- No live provider calls.
- No Xcode signing, Keychain, global macOS mutation, or real iPhone deployment.
- No changes to final resource apply behavior.
- No print quote/order action.

## Acceptance

- The new focused test fails before implementation and passes after implementation.
- Existing final external action ledger tests pass after updating old provider-first assertions to the new device-first contract.
- Focused downstream handoff tests still pass.
- `make backend-lint` and `git diff --check` pass.
