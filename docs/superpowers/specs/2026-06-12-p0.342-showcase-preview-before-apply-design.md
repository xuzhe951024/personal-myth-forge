# P0.342 Showcase Preview Before Apply Design

## Context

`final-showcase-readiness.json` is the top-level report used to decide what is
left before the iOS demo can be shown. It currently aggregates actions from
provider handoff, resource handoff, rehearsal, and capability rows.

The resource handoff has a read-only preview phase before the writer phase:

```text
make final-resource-apply-preview
make final-apply-resources
```

When final resources are still blocked, the top-level showcase report can list
both actions. That is confusing because the writer command should not be the
next visible handoff while apply preview remains blocked.

## Goal

When `final-showcase-readiness` already includes the apply-preview gate, remove
direct `make final-apply-resources` actions from the same top-level action list.

## Non-Goals

- Do not remove `make final-apply-resources` when no apply-preview gate is
  present.
- Do not change the make targets or writer scripts.
- Do not write backend `.env`, iOS deploy config, keychain state, signing
  state, provider state, or print order state.
- Do not call live Meshy, OpenAI, Treatstock, or Xcode signing commands.

## Design

Add a narrow post-dedupe filter in
`final_showcase_readiness._dedupe_operator_actions()`.

The filter detects an action root of:

```text
make final-resource-apply-preview
```

If present, it removes actions whose root is exactly:

```text
make final-apply-resources
```

Action roots are compared before any ` | detail` suffix and after existing
source-prefix stripping. Other final resource actions remain visible, including
`provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight`.

## Acceptance Criteria

- `final-showcase-readiness.json.operator_actions` includes
  `make final-resource-apply-preview` while final resources are blocked.
- The same top-level list does not include direct `make final-apply-resources`
  while the preview gate is present.
- Direct `make final-apply-resources` remains available from the helper when the
  preview gate is absent.
- Focused final showcase tests, operator action tests, lint, diff checks, and
  backend tests pass.
