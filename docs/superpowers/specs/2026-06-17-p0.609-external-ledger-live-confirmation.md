# P0.609 External Ledger Live Confirmation

## Context

The final external action ledger separates repo-local actions, global Mac/Xcode
actions, live provider calls, and device runtime actions. Live provider actions
already carry `requires_cost_consent` and `live_provider_call`, but the generated
action rows could still report `requires_user_confirmation: false`.

That weakens the final iPhone/operator handoff because the same action is both a
cost-bearing live provider call and machine-readable as not requiring confirmation.

## Decision

All final external action ledger live-provider/cost actions must mark
`requires_user_confirmation: true` whenever they are live provider calls or require
cost consent. The top-level first blocker and next action already copy this field
from the underlying action, so the fix should start at `_live_action`.

The iPhone final launch summary should also decode this metadata and show that
external actions include confirmation-required work.

## Acceptance

- `run_live_provider_evidence` reports `requires_user_confirmation: true`.
- A live provider first blocker and next action preserve
  `requires_user_confirmation: true`.
- iPhone external action ledger contract fixtures decode the live provider action
  as confirmation-required.
- iPhone External Actions summary rows show action confirmation and group
  confirmation counts.
- No Meshy, OpenAI, print-provider, Xcode signing, Keychain, or global machine
  action is run.
