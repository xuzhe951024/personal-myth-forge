# P0.499 Backend URL Handoff Packets Design

## Context

The final iPhone demo requires `PMF_BACKEND_BASE_URL` to point to a backend URL that the physical iPhone can reach on the local network. `localhost` and `127.0.0.1` are not valid for the device path.

P0.498 already pruned the final demo launch top-level `operator_actions` so that the iPhone-reachable LAN URL action is the visible handoff. Two later handoff reports still expose a weaker action:

- `final-external-action-ledger.json`
- `final-launch-closure-packet.json`

Both can still list `provide PMF_BACKEND_BASE_URL in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`. That text is technically incomplete for a physical iPhone handoff because it does not state the LAN reachability requirement.

## Decision

Add a shared operator-action helper that rewrites final top-level handoff actions from the generic backend URL deploy-config action to:

```text
set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL; rerun make mobile-deploy-preflight
```

The helper will be used by:

- `final_external_action_ledger._operator_actions()`
- `final_launch_closure_packet._operator_actions()`

It will not change resource requirement records, action groups, closure sections, or final demo launch checklist rows. Those still preserve their source-level contract and destination metadata.

## Safety

This is report-only. It does not write ignored config files, run Xcode, change signing state, touch Keychain, call Meshy/OpenAI/Treatstock, place print orders, or spend provider credits.

## Verification

TDD coverage must prove:

- The shared helper converts the generic validated backend URL config handoff into the iPhone-reachable LAN URL handoff.
- The shared helper collapses duplicate generic and LAN backend URL actions into one LAN action.
- The final external action ledger top-level `operator_actions` emits the LAN backend URL handoff and no generic backend URL handoff.
- The final launch closure packet top-level `operator_actions` emits the LAN backend URL handoff and no generic backend URL handoff.
- Existing tests continue to prove that source actions and destination metadata remain present.
