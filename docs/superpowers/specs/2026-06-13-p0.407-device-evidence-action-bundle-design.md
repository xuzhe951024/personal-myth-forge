# P0.407 Device Evidence Action Bundle Design

## Context

The final iPhone showcase remains blocked on physical iOS deployment evidence. `final-showcase-readiness.json` already exposes a `device_action_bundle`, and P0.406 added the same style of operator-visible bundle to `ios-deploy-runbook-local.json`. The lower-level `ios-device-evidence-bundle.json` still has `device_action_bundle: null`, even though this report is the canonical evidence package for backend health, mobile preflight, Xcode build, and device rehearsal.

## Goal

Expose a sanitized `device_action_bundle` directly in the iOS device evidence bundle report so the physical iPhone evidence package carries its own ordered device actions, first action, summary, and safety metadata.

## Scope

- Add `device_action_bundle` to `build_ios_device_evidence_bundle_report`.
- Derive actions from existing `evidence_slots`; do not introduce new checks.
- Preserve existing `status`, `first_blocker`, `next_action`, `operator_actions`, and ready-report behavior.
- Add focused tests for blocked and ready bundle output.
- Update the roadmap with P0.407.

## Non-Goals

- Do not run backend, Xcode, signing, Keychain, provider, or iPhone commands.
- Do not change final showcase readiness classification.
- Do not alter mobile app models in this iteration.
- Do not embed secrets, local absolute paths, raw media, or payment links.

## Design

The report will include a top-level object:

- `id`: `ios_device_evidence_actions`
- `label`: `iOS Device Evidence Actions`
- `status`: `ready` only when all evidence slots are ready; otherwise `blocked`
- `actions`: one action per existing evidence slot
- `first_action`: the first non-ready action, matching the current first blocker command
- `summary`: counts for actions, ready, missing, blocked, manual, global actions, provider calls, and Xcode/signing actions
- `safety`: read-only flags matching the report's no-mutation behavior

Each action will mirror slot metadata: `id`, `label`, `status`, `classification`, `command`, `detail`, `evidence_source`, `global_action`, `xcode_or_signing`, `provider_calls: false`, `manual`, and a `next_action` for blocked/manual rows. This keeps the evidence bundle self-contained while avoiding a second source of truth.
