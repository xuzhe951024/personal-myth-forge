# P0.196 Final Launch Digest First Blocker Design

## Purpose

P0.193-P0.195 made `final_demo_launch_report.first_blocker` canonical and surfaced it in the receipt and Device Preflight. The remaining iPhone digest views still choose final launch details from phase, acceptance, or handoff rows. That can make the Demo Script and Final Showcase summary describe a different first action than the backend receipt.

## Scope

- Make `DemoScriptBuilder.finalLaunchDetail` prefer the first `launchReceiptRows` entry that starts with `First blocker:`.
- Make `FinalShowcaseSummaryBuilder.finalLaunchStage` use the same preference.
- Keep current fallbacks for summaries that do not include launch receipt rows.
- Add Swift contract tests for both digest views.
- Add ProjectChecks source gates for the helper and contract tests.

## Acceptance

- A blocked final launch summary with a top-level blocker shows that blocker in Demo Script detail.
- The Final Showcase `final_launch` stage shows the same first blocker detail.
- Existing fallback summaries without receipt rows still render.
- Redaction remains covered by existing encoded-summary redaction tests.
