# P0.455 Demo Script External Actions Design

## Context

Final launch, device preflight, and final showcase now surface the final external action ledger, including the `backend-device-demo` handoff. The guided iPhone demo script and autopilot still only understand 3D evaluation, NPC evaluation, and final launch readiness. During a physical-device rehearsal this can hide the concrete external action chain until the operator opens another panel.

## Goal

Add a deterministic `external_actions` checkpoint to the mobile demo script and autopilot so the guided iPhone path shows whether external actions are ready, blocked, or waiting before the final launch step.

## Scope

- Add a demo script step with id `external_actions` and label `External Actions` when a `FinalLaunchMobileSummary` is available.
- Derive status from `FinalLaunchMobileSummary.externalActionLedgerRows`:
  - `complete` when the first row starts with `External actions ready:`.
  - `blocked` when rows mention blocked, missing, live, manual, backend-device-demo, or failure terms.
  - `waiting` when rows are absent or do not prove readiness.
- Use the first ledger row plus the first backend-device-demo/operator row in the detail when available.
- Make `DemoScript.nextAction` prefer external action blockers after 3D and NPC evaluations and before final launch.
- Make `ShowcaseAutopilotPlanner` stop on the `external_actions` step before final launch and return button title `Check Actions`.
- Preserve existing redaction behavior for secrets, local paths, bearer tokens, checkout URLs, and local capture URIs.

## Non-Goals

- No provider calls, key lookup, Xcode signing, or Treatstock order/quote actions.
- No UI layout changes; existing `DemoScriptView` renders dynamic script steps.
- No changes to backend readiness generation.

## Tests

- Contract test: blocked external actions create an `external_actions` step before `final_launch`, include the backend-device-demo handoff, and update `nextAction`.
- Contract test: ready external actions complete and still allow final launch readiness to complete the guided script.
- Contract test: autopilot blocks on external actions before final launch.
- Project check: source and contract tests explicitly reference the new step and autopilot handling.
