# P0.225 Final Resources Preflight Evidence Design

## Context

`make final-resources-preflight` validates the ignored
`services/backend/.local/final-resources.env` file before any final resource
apply step. The CLI already supports `--output`, but the Make target only
printed the report, leaving no stable `.local` artifact for the final handoff
chain to inspect after the command exits.

## Decision

The Make target writes
`services/backend/.local/final-resources-preflight.json` by passing
`--output .local/final-resources-preflight.json` to the existing CLI command.
This mirrors the other final handoff reports and keeps both ready and blocked
resource states reviewable after the command completes.

## Scope

- Add the output path to the `final-resources-preflight` Make target.
- Add resource template acceptance coverage for the output path.
- Add Swift ProjectChecks coverage so the mobile project gate also watches the
  Makefile contract.
- Document the generated artifact in the README.

## Non-Goals

- Do not write `services/backend/.env`.
- Do not write `apps/mobile/ios/Config/Deployment.local.xcconfig`.
- Do not call Meshy, OpenAI, Treatstock, or Sculpteo.
- Do not run Xcode, alter signing, touch keychain, or mutate global machine
  state.
- Do not include provider key values or absolute local paths in the report.

## Acceptance

- `make final-resources-preflight` creates
  `services/backend/.local/final-resources-preflight.json` even when required
  resources are missing and the command exits `2`.
- The JSON report remains sanitized and read-only.
- `uv run pytest tests/test_evaluation_make_targets.py
  tests/test_resource_template_acceptance.py -q` passes.
- `swift run --package-path apps/mobile/ios
  PersonalMythForgeMobileProjectChecks` passes.
