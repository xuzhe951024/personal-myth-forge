# P0.209 Device Preflight iOS Launch Rehearsal Design

## Purpose

`FinalDemoLaunchReport` already carries `ios_device_launch_rehearsal_readiness`, and the iPhone Final Launch panel renders it under `Launch Rehearsal`. The iPhone Device Preflight panel does not yet surface that readiness directly, so an operator can see the deploy runbook and device evidence but not the saved launch rehearsal freshness/checklist state in the same preflight list.

## Approach

Add a read-only `ios_device_launch_rehearsal_readiness` Device Preflight item labeled `Launch Rehearsal`, placed after `Device Evidence`. The item uses the existing Swift model `IOSDeviceLaunchRehearsalReadinessReport` from `FinalDemoLaunchReport`; it does not call backend commands, run Xcode, mutate deploy config, or inspect files on device.

## Behavior

- Missing final launch report or missing rehearsal readiness returns `waiting` with `iOS launch rehearsal readiness has not loaded.`
- `status == "ready"` returns `ready` and shows ready/blocked/partial counts, freshness classification, source path, and safety flags.
- `status == "missing"` returns `waiting` and shows the operator action to run `make ios-device-launch-rehearsal`.
- `status == "blocked"` or `status == "partial"` returns `blocked`, showing counts, freshness, first non-ready sequence row, first blocker or operator action, and safety flags.
- Unknown statuses return `waiting` with the summary and freshness.
- The item is added to the required-ready set so Device Preflight cannot claim ready until saved launch rehearsal evidence is ready.

## Safety

All detail strings pass through the existing `DevicePreflightSummaryBuilder.item` sanitizer. Unsafe provider keys, bearer tokens, local paths, local capture URLs, checkout text, and `api_key` snippets must be withheld from encoded preflight output.

## Acceptance

- `DevicePreflightSummaryBuilder` returns an `ios_device_launch_rehearsal_readiness` item.
- Missing, blocked, ready, stale, and redaction cases are covered by Swift contract tests.
- Backend mobile final launch readiness source requirements check the item, label, source field, required-ready set entry, and contract tests.
- `PersonalMythForgeMobileProjectChecks` checks the same source-level contract.
