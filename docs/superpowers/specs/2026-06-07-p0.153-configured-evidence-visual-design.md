# P0.153 Configured Evidence Visual Regression Design

## Problem

The iPhone Final Launch panel now renders `Configured Evidence` rows for the
configured evidence plan, including current consent and planned live-provider
consent needs. Backend and Swift contract tests prove the data path, but the
default visual regression manifest does not yet include this screen state.

Without a visual artifact, a later change could make the configured evidence
rows unreadable on a 390x844 iPhone viewport while still passing model and source
checks.

## Goal

Add a default visual regression artifact for the configured evidence handoff.
The artifact should show the iPhone Final Launch Status panel with the
`Configured Evidence` section, the blocked upstream resource state, the
`consent now 0, planned 3` row, the live/cost step counts, the consent flag, and
the safety row.

## Non-Goals

- Do not change runtime SwiftUI behavior.
- Do not call Meshy, OpenAI, Treatstock, or any live provider.
- Do not execute final configured evidence commands.
- Do not write backend `.env`, iOS deploy config, keychain, signing, or Xcode
  global state.
- Do not include provider secrets, raw media, local file paths, commerce links,
  or private LAN URLs in the visual artifact.

## Design

Extend `DEFAULT_VISUAL_ARTIFACTS` with a new
`p0.153_configured_evidence_visual` entry. It will point to:

- `docs/superpowers/verification/p0.153-configured-evidence-visual.html`
- `docs/superpowers/verification/assets/p0.153-configured-evidence-visual-390x844.png`
- `docs/superpowers/verification/2026-06-07-p0.153-configured-evidence-visual-regression.md`

Required text:

- `Configured Evidence`
- `Configured evidence blocked`
- `consent now 0, planned 3`
- `Live steps 3, cost steps 3`
- `--allow-live-provider-calls`
- `commands_run=false live_calls=false`

The HTML fixture is a static mobile-width proof of the existing Swift summary
copy. The PNG is rendered from that HTML using local Chrome with a temporary
profile and a 390x844 viewport. The default visual regression gate checks the
HTML text, PNG dimensions, notes file, and unsafe-text patterns.

## Testing

Use TDD:

- Update the default visual artifact contract test first and watch it fail.
- Add the manifest entry and artifact files.
- Run `make visual-regression` and verify it passes with one additional default
  artifact.
- Run backend full tests and the existing Swift checks to ensure the visual
  evidence change does not break runtime contracts.
