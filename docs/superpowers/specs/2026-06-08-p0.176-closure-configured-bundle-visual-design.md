# P0.176 Closure Configured Bundle Visual Design

## Context

P0.174 made the mobile `Closure Packet` panel always include the
`configured_evidence_bundle` row when the final launch closure packet contains
that section. P0.175 made `final-local-report-refresh` regenerate the closure
packet so the saved final handoff evidence does not go stale.

The checked-in visual regression artifact for the closure packet still only
requires the older `resource_inputs` / `provide MESHY_API_KEY` copy. That means
visual regression can pass even if the new configured bundle row disappears from
the closure packet UI.

## Goal

Update the closure packet visual regression coverage so the checked-in artifact
proves the configured evidence bundle row is visible in the final launch status
panel.

## Design

`DEFAULT_VISUAL_ARTIFACTS` will keep the existing
`p0.157_final_launch_closure_packet` artifact, but its `required_text` will also
require:

- `configured_evidence_bundle`
- `configured_live_evidence_bundle`
- `make configured-live-evidence-bundle`

The existing static HTML artifact
`docs/superpowers/verification/p0.157-final-launch-closure-packet.html` will be
updated to show a second closure packet line for the configured evidence bundle
gate. The visual notes file will list the same required copy so the handoff
explains why this artifact exists.

The PNG dimensions remain unchanged at `390x844`. This iteration does not change
mobile Swift UI code or regenerate provider/device evidence. It tightens the
visual artifact contract around a UI row already implemented in P0.174.

## Alternatives Considered

1. Add a new P0.176 visual artifact.
   Rejected because the product UI surface is still the same closure packet
   panel. Updating the existing artifact keeps the regression index focused.

2. Rely on Swift contract tests only.
   Rejected because the user explicitly asked for visual regression coverage.
   Contract tests prove data flow; this artifact proves the operator-facing
   panel text is represented in visual handoff evidence.

3. Change the actual mobile UI layout again.
   Rejected because P0.174 already surfaced the row. This iteration only closes
   the visual coverage gap.

## Acceptance

- `test_default_visual_artifacts_cover_full_showcase_flow` asserts the visual
  required text includes `configured_evidence_bundle`,
  `configured_live_evidence_bundle`, and `make configured-live-evidence-bundle`.
- `p0.157_final_launch_closure_packet.required_text` includes those strings.
- The checked-in closure packet HTML artifact contains those strings.
- The closure packet visual notes mention the configured evidence bundle row.
- `test_visual_regression_default_passes_checked_in_showcase_artifacts` passes.
- No mobile UI, provider adapter, resource writer, live API, Xcode, or global
  configuration behavior changes.

## Test Strategy

Use TDD:

1. Add required-text assertions to `services/backend/tests/test_visual_regression.py`.
2. Run `uv run pytest tests/test_visual_regression.py -q` and confirm failure.
3. Add the required strings to `visual_regression.py` and the checked-in HTML
   artifact.
4. Update the visual notes file.
5. Re-run visual regression tests, iOS showcase acceptance, backend lint, and
   full backend tests before merging.
