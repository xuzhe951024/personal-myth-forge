# P0.277 External Ledger iOS Validation Design

## Context

P0.275 and P0.276 normalized iOS deploy configuration actions in the final
demo launch, local refresh, apply preview, and final showcase readiness reports.
The ignored `final-external-action-ledger.json` still shows the source external
action ledger emitting `Deployment.local.xcconfig` resource inputs with older
validation commands:

- `DEVELOPMENT_TEAM` and `PRODUCT_BUNDLE_IDENTIFIER` rerun
  `make ios-device-launch-rehearsal`.
- `PMF_BACKEND_BASE_URL` reruns `make final-resources-preflight`.

Those commands are later than the safe first validation step or validate the
wrong surface. After editing `Deployment.local.xcconfig`, the operator should
rerun `make mobile-deploy-preflight`.

## Goal

The final external action ledger should make every missing iOS deploy config
resource action point at `make mobile-deploy-preflight`, while backend
`final-resources.env` actions continue to point at their existing validation
commands.

## Non-Goals

- Do not change resource requirement status, first blocker, or provider
  readiness semantics.
- Do not run writer scripts, edit ignored env/config files by hand, call live
  providers, trigger Xcode signing, touch Keychain, or place print orders.
- Do not broaden this into live Meshy/OpenAI/Treatstock evidence or physical
  iPhone deployment.
- Do not change mobile UI decoding in this phase.

## Design

Keep the fix inside the external ledger's action projection. Add a small helper
that maps resource requirements targeting
`apps/mobile/ios/Config/Deployment.local.xcconfig` to the mobile deploy
preflight validation command. `_resource_action()` will use that helper when it
sets the action `command`.

This preserves the final resource requirements report as the canonical manifest
of resource slots while making the external operator ledger use the safest
validation step for iOS deploy inputs.

The shared operator action helper should also recognize the literal
`provide PMF_BACKEND_BASE_URL in Deployment.local.xcconfig` action root so any
future source that passes that text through the helper gets the same mobile
preflight suffix.

## Acceptance

- RED/GREEN test proves `final-external-action-ledger.actions_by_id` maps
  `DEVELOPMENT_TEAM`, `PRODUCT_BUNDLE_IDENTIFIER`, and
  `PMF_BACKEND_BASE_URL` iOS deploy actions to
  `make mobile-deploy-preflight`.
- RED/GREEN test proves `operator_actions` include
  `; rerun make mobile-deploy-preflight` for those three local iOS config
  inputs and do not include the old rehearsal or final resources preflight
  variants.
- Existing backend provider resource actions continue to rerun
  `make final-resources-preflight`.
- Refreshed `.local/final-external-action-ledger.json` stays blocked while
  surfacing the mobile preflight command for iOS deploy config inputs.
- Safety flags remain read-only with no provider calls, writes, global
  mutation, Xcode/signing, or Keychain actions.
