# P0.541 Provider Fill Guide Aggregators Design

## Goal

Propagate the P0.540 provider fill-guide handoff into final readiness aggregators. When an operator sees a live-provider handoff chain, the chain should start with the ignored final resource fill guide before previewing provider resources, validating provider handoff, and refreshing live provider evidence.

## Current Problem

`provider-handoff` now begins blocked operator actions with:

```text
make final-resource-fill-guide; rerun make final-resource-apply-preview
```

But higher-level reports still surface the older complete provider chain:

```text
make final-resource-apply-preview; rerun make provider-handoff; rerun make live-provider-evidence
```

That skips the fill-guide step in the final operator view, even though the fill guide is the safest project-local path for API/key handoff.

## Design

Add shared provider chain normalization in `services/backend/src/myth_forge_api/operator_actions.py`:

- Define a canonical complete chain:
  `make final-resource-fill-guide; rerun make final-resource-apply-preview; rerun make provider-handoff; rerun make live-provider-evidence`
- Normalize the old complete chain into the canonical chain while preserving source prefixes and ` | ` detail suffixes.

Update the complete provider-chain constants in the final report aggregators to use the canonical chain:

- `final_showcase_readiness.py`
- `final_local_report_refresh.py`
- `final_external_action_ledger.py`
- `final_launch_closure_packet.py`

For final showcase capability rows, normalize `_operator_action_with_validation()` so child next-action metadata that still composes the old chain becomes the canonical chain.

## Safety

This phase changes report text only. It does not write secrets, call live providers, mutate Xcode settings, touch Keychain, start backend services, or create paid print/provider actions.

## Testing

Use TDD with focused tests:

- `test_operator_actions.py` should prove the old complete provider chain normalizes to the canonical fill-guide-first chain.
- `test_final_showcase_readiness.py` should prove partial live provider capability rows expose the canonical chain.

Then run the related aggregator tests, backend lint, whitespace check, and a safe local report refresh.
