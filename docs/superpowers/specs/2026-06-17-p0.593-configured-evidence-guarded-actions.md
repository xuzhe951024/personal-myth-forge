# P0.593 Configured Evidence Guarded Actions

## Context

The final showcase readiness is now blocked on live Meshy/OpenAI evidence. The project must not run live providers or spend credits unless the operator explicitly provides consent.

`final-configured-evidence-plan` already classifies live-provider steps as `consent_required`, but its operator actions only said to review live provider cost consent. That was safe but not directly executable during final handoff.

## Decision

When a configured evidence step is `consent_required` and may call a live provider, the report should expose the exact guarded command form:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 <make target>; rerun make final-configured-evidence-plan
```

The report builder remains read-only:

- no provider calls
- no commands run
- no backend env writes
- no iOS deploy config writes
- no Xcode or signing actions

## Acceptance

- `first_blocker.command` and `next_action.command` include `PMF_ALLOW_LIVE_PROVIDER_CALLS=1` for live-provider consent blockers.
- `operator_actions` list guarded commands for configured 3D, configured NPC Agent, and configured final acceptance steps.
- The report no longer emits vague `review live provider cost consent before ...` actions for those live provider steps.
