# P0.469 Device Handoff Safe Wrappers Design

## Context

The final showcase blocker remains the physical iPhone device path. Several device-handoff report targets successfully write local JSON evidence in the current blocked state, but Make still fails on the expected blocked exit code `2`:

- `make mobile-xcode-build-evidence`
- `make ios-device-evidence-bundle`
- `make ios-device-launch-certificate`
- `make final-handoff-index`

These reports are useful handoff evidence even while blocked by missing `DEVELOPMENT_TEAM`, iPhone-reachable `PMF_BACKEND_BASE_URL`, provider keys, or live evidence. Failing Make interrupts operator refresh workflows and makes the final handoff less executable.

## Decision

Add safe wrapper scripts for the four device-handoff report targets. Each wrapper will run the existing CLI command, keep the same output file, accept exit code `0` or `2`, and fail on any other exit code.

Update Make targets to call the wrappers. This preserves CLI semantics and only changes Make behavior for local evidence refresh.

## Scope

- Add wrapper scripts under `services/backend/scripts/`.
- Update the four Make targets.
- Add script behavior tests, dry-run tests, and executable-bit tests.
- Update iOS showcase source gates and fixtures so final acceptance requires the safe wrapper handoff.
- Update the roadmap.

## Out Of Scope

- No Xcode signing, build, Keychain, or real device mutation.
- No real provider calls or paid API usage.
- No change to report status classification.
- No change to generated `.local` schema.

## Acceptance

- Script tests prove exit code `2` is accepted and exit code `1` fails for all four wrappers.
- `make -n` for each target shows the wrapper.
- Wrapper scripts are executable.
- Actual Make targets write their `.local` JSON files and exit `0` in the current blocked state.
- Focused tests, lint, and `git diff --check` pass.
