P0.489 Final Demo Xcode Action Normalization Design

Goal

Normalize stale Apple SDK license operator actions in final demo launch reports to the canonical Xcode evidence handoff.

Context

`final-demo-launch-local.json` still contains:

`accept the Apple SDK license outside this repository before Xcode build`

Other final handoff reports already use the clearer, validation-aware action:

`accept the Xcode license outside Codex, then rerun make mobile-xcode-build-evidence`

The old action tells the operator what category of problem exists, but it does not name the evidence target that proves the issue has been resolved. The final iPhone demo flow needs every Mac-side action to include a validation command where possible.

Non-Goals

- Do not run Xcode, signing, Keychain, or real device checks.
- Do not accept the Xcode license or mutate global Mac state.
- Do not change final demo launch status, phase status, first blocker, or provider/print handoff behavior.
- Do not remove Xcode build commands from the command reference list.

Design

Add a normalization rule in the shared `operator_actions.normalize_operator_action()` path so legacy Apple SDK license text maps to the canonical Xcode evidence action. `final_demo_launch._dedupe_operator_actions()` already passes action strings through this shared normalizer, so the local final demo launch report will inherit the canonical action without bespoke final-demo logic.

Acceptance Criteria

- `final_demo_launch._dedupe_operator_actions()` maps the stale Apple SDK license action to the canonical Xcode evidence action.
- `final-demo-launch-local.json` no longer contains the stale Apple SDK license action after safe refresh.
- Existing final demo provider, print, deploy writer, backend-device-demo, and apply-preview behavior remains unchanged.
- Verification stays local and safe: no Xcode build, signing, Keychain writes, provider calls, or print provider calls.

Safety

This phase changes report action text normalization only. It may write ignored `.local` JSON through safe report builders. It must not touch real provider keys, run Xcode, mutate global Mac state, call live providers, or place print orders.
