# P0.490 Closure Device Action Normalization Design

## Context

The final launch closure packet is one of the highest-level operator handoff
reports for the iPhone showcase. Current local evidence shows the packet still
surfaces bare device commands such as `make backend-device-demo`,
`make mobile-deploy-preflight`, `make mobile-xcode-build`, and
`make ios-device-launch-rehearsal` in `operator_actions`.

Those commands are real, but weaker than the final handoff convention used by
the showcase and demo launch reports: each action should say what to do and
which evidence gate to rerun afterwards.

## Design

Keep the closure packet status model unchanged. Do not run Xcode, touch signing,
write Keychain state, call live providers, or place print orders.

Normalize only top-level `operator_actions` generated by
`myth_forge_api.final_launch_closure_packet._operator_actions()`:

- Backend device demo actions become
  `start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight`.
- Mobile deploy configuration actions keep the existing project-local writer:
  `DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`.
- Xcode build gate actions become
  `accept the Xcode license outside Codex, then rerun make mobile-xcode-build-evidence`.
- Bare `make ios-device-launch-rehearsal` remains allowed only when it is the
  actual top-level validation target and no more specific saved child action is
  available.

## Acceptance

- A RED/GREEN backend test proves a closure packet built from saved blocked
  backend/Xcode child evidence exposes validation-aware top-level actions.
- Existing closure packet tests still pass.
- `make final-launch-closure-packet` refreshes ignored `.local` evidence without
  old bare device/Xcode actions taking precedence over validation-aware handoff.
- Lint passes.

## Safety

This phase is read-only with respect to external systems. It only changes report
generation and tests. `.local` evidence refreshes are ignored by git.
