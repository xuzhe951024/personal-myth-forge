# P0.114 Rehearsal Final Launch Sync Design

## Context

P0.113 made `/v1/final-demo-launch` and the iPhone Final Launch panel read the
saved `services/backend/.local/ios-device-launch-rehearsal.json` report through
`ios_device_launch_rehearsal_readiness`.

The current `make ios-device-launch-rehearsal` wrapper refreshes
`final-demo-launch-local.json` before it writes
`ios-device-launch-rehearsal.json`. That means the one-command refresh can leave
the saved final launch report one step behind the saved rehearsal report unless
the operator manually runs final demo launch again.

## Approaches Considered

1. Reorder the wrapper so `final-demo-launch-local.json` is only written after
   the rehearsal report. This breaks the existing rehearsal builder's source set,
   which expects final demo launch to be part of the local report evidence.

2. Add a post-rehearsal sync step that reruns `final-demo-launch --mode local`
   after `ios-device-launch-rehearsal.json` has been written. This keeps the
   existing local rehearsal source set intact and makes the final saved mobile
   payload reflect the newest rehearsal readiness.

3. Add a new synced final launch report path. This avoids overwriting the
   existing report, but it adds another handoff artifact and another iPhone-facing
   concept without changing behavior.

P0.114 uses approach 2.

## Design

Update `services/backend/scripts/write_ios_device_launch_rehearsal.sh` so the
sequence becomes:

1. `make final-rehearsal-local`
2. `final-configured-preflight`
3. `final-handoff-index`
4. `ios-device-launch-certificate`
5. `ios-device-launch-rehearsal`
6. `final-demo-launch --mode local --output .local/final-demo-launch-local.json`

The final step is labeled `final launch rehearsal sync`. It accepts exit `0` and
`2` with the same `run_report_command` helper. It remains read-only and does not
call providers, apply resources, start servers, run deploy preflight, run Xcode,
write backend/iOS config, touch signing/keychain, accept Apple licenses, or
mutate global state.

The final stdout should mention both files:

- `services/backend/.local/ios-device-launch-rehearsal.json`
- `services/backend/.local/final-demo-launch-local.json`

## Tests

Backend script tests should prove:

- the wrapper accepts exit `2` from the post-rehearsal final launch sync
- the fake `uv` invocation order has `ios-device-launch-rehearsal` before the
  final `final-demo-launch --mode local`
- the dry-run source gate requires the sync command and output path

Source gates should require:

- `final launch rehearsal sync` label in the wrapper
- `final-demo-launch --mode local` after the rehearsal command
- the final launch local output path in the wrapper

Visual regression should show the refreshed evidence order as a small operator
sequence card.

## Non-Goals

- Do not change the final launch report schema.
- Do not add a new endpoint.
- Do not run live Meshy/OpenAI/Treatstock calls.
- Do not apply secrets or resources.
- Do not run Xcode, sign apps, touch keychain, accept Apple licenses, or change
  global developer state.
