# P0.496 Showcase Rehearsal Action Pruning Design

## Context

`final-showcase-readiness.json` is the final objective ledger for the iPhone showcase. Its top-level `operator_actions` already contains concrete, copy-pasteable handoff steps for the current blockers:

- writing `DEVELOPMENT_TEAM` into ignored iOS deploy config and rerunning `make mobile-deploy-preflight`
- running final resource apply preview before provider handoff and live evidence
- running the guarded print quote command with explicit consent
- accepting the Xcode license outside Codex before Xcode evidence
- providing Meshy/OpenAI/iOS deploy values in ignored local files

The same top-level list still includes bare `make final-rehearsal-local`. That command is useful as a canonical local rehearsal command and must remain in `commands`, but it is weak as a top-level handoff action while concrete blocker actions already exist.

## Decision

Filter only `final_showcase_readiness.operator_actions`.

The report should:

- Drop bare `make final-rehearsal-local` when any specific handoff action exists.
- Preserve bare `make final-rehearsal-local` when it is the only actionable item.
- Keep `make final-rehearsal-local` in `commands`.
- Keep `make final-showcase-readiness` filtered from top-level actions as before.
- Preserve first blocker ordering, especially the concrete iOS deploy writer action.

## Safety

This is report-only. It does not run Xcode, write signing config, touch Keychain, start servers, call Meshy/OpenAI/Treatstock, or change ignored local env files.

## Verification

TDD coverage must prove:

- The full final showcase fixture does not emit bare `make final-rehearsal-local` in top-level `operator_actions` when concrete handoffs exist.
- The same fixture still lists `make final-rehearsal-local` in `commands`.
- The filter still preserves bare `make final-rehearsal-local` when it is the only operator action.
- Focused final showcase tests, operator action tests, lint, and refreshed local evidence pass.
