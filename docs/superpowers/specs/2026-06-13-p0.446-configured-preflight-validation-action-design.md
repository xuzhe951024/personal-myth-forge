# P0.446 Configured Preflight Validation Action Design

## Goal

Make configured final preflight operator actions tell the operator which evidence report to refresh next.

## Context

The final showcase and local demo launch reports still include a bare action:

```text
make final-configured-preflight
```

That action is useful, but it stops short of the evidence needed by the final iOS showcase goal. After configured preflight, the operator needs configured live evidence to prove that live 3D/NPC/provider paths are ready.

## Behavior

Normalize `make final-configured-preflight` and `run make final-configured-preflight` to:

```text
make final-configured-preflight; rerun make configured-live-evidence-bundle
```

This should happen in the shared operator action normalizer so final showcase, final demo launch, and other aggregators inherit the same action shape.

Existing validation-aware actions must remain unchanged. The change must not run configured preflight, call live providers, write keys, or mutate deploy config.

## Safety

This is read-only command text shaping. It does not call Meshy, OpenAI, Treatstock, Xcode, Keychain, or any paid provider. It does not write `.env`, `.xcconfig`, payment links, quote files, or order artifacts.

## Tests

Update focused tests so:

- prefixed final showcase make target normalization emits the configured-live-evidence validation suffix
- nested final showcase operator actions promote the validated configured preflight action
- local final demo launch operator actions include the same validated configured preflight action when supplied through final showcase handoff actions
