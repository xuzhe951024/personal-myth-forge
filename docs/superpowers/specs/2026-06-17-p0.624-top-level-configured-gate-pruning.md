# P0.624 Top-Level Configured Gate Pruning Spec

## Context

The final showcase and final demo launch now point directly at the configured 3D/NPC live provider consent gates. Two top-level operator handoff reports still keep this fallback after the granular consent action:

```text
make final-configured-preflight; rerun make configured-live-evidence-bundle
```

That fallback is useful when raw live provider actions need to be routed through the configured evidence bundle. It is redundant once a granular configured evidence action is already the first blocker, for example:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-3d-configured; rerun make final-configured-evidence-plan
```

## Requirements

- Preserve configured bundle gate fallback for bare live provider actions.
- When a granular configured live provider consent action exists, remove the configured bundle gate fallback from:
  - `final-launch-closure-packet`
  - `final-external-action-ledger`
- Keep live provider cost and confirmation metadata untouched.
- Do not run live provider commands, write secrets, or mutate global developer state.

## Acceptance

- Focused tests cover closure packet and external action ledger pruning.
- Existing closure packet and external action ledger tests still pass.
- Safe local evidence refresh shows top-level operator actions start with the granular 3D consent action without the configured bundle fallback.
