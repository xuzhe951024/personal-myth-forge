# P0.400 Refresh Apply Preview Action Design

## Context

The final iPhone showcase remains blocked by external iOS evidence and provider
credentials. The local refresh report is one of the operator-facing handoff
reports used to decide the next safe action.

`final-local-report-refresh.operator_actions` still includes:

```text
rerun make final-resource-apply-preview before applying resources
```

That string is older than the current handoff style. It is copy-heavy, does not
match the project target name cleanly, and can be confused with an apply step.
The safer top-level action is the concrete preview command:

```text
make final-resource-apply-preview
```

The report must still suppress `make final-apply-resources` while preview is
blocked.

## Goal

Normalize the final local refresh top-level apply-preview operator action to
`make final-resource-apply-preview` while keeping real apply hidden until the
preview gate is ready.

## Scope

- Change only `myth_forge_api.final_local_report_refresh` action projection and
  its tests.
- Do not change `final_resource_apply_preview` source report semantics.
- Do not run or enable `make final-apply-resources`.
- Do not touch provider adapters, resource writers, Xcode/signing, key files, or
  print quote behavior.

## Design

Add a local normalization step inside final-local refresh operator-action
dedupe:

1. Convert the legacy action root
   `rerun make final-resource-apply-preview before applying resources` to
   `make final-resource-apply-preview` for top-level
   `final-local-report-refresh.operator_actions`.
2. Keep the existing preview-before-apply gate: when preview is present, remove
   `make final-apply-resources`.
3. Preserve validation-aware provider, print, iOS deploy, and Xcode actions.
4. Leave nested child reports unchanged so older report data remains traceable.

## Tests

- Change `test_final_local_report_refresh_operator_actions_gate_apply_behind_preview`
  to expect `make final-resource-apply-preview` in top-level actions and no
  legacy preview string.
- Change `test_final_local_report_refresh_operator_actions_hide_apply_when_preview_is_blocked`
  to expect only `make final-resource-apply-preview`.
- Preserve the assertion that `make final-apply-resources` is absent.
