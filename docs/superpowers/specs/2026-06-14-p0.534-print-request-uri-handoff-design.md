# P0.534 Print Request URI Handoff Design

## Context

`print-fulfillment-readiness` can already prove the local print quote path, the iPhone print receipt source, the visual receipt artifact, and the resource handoff. Its current first blocker is the missing ignored configured quote request file:

```bash
PRINT_SOURCE_ASSET_URI=https://... PRINT_CANDIDATE_URI=https://... make print-quote-request-configured
```

That command is structurally correct, but it still asks the operator to discover two asset URLs manually. The final iPhone showcase goal needs a clearer handoff from the local demo artifact to the guarded Treatstock quote request without calling Treatstock or writing secrets.

## Design

When the configured quote request is missing, `print-fulfillment-readiness` should try to build a more concrete writer command from existing local evidence:

- Read the latest local demo session id from `services/backend/.local/final-demo-launch-local.json`, using the embedded `source_reports.local_showcase_smoke.session.session_id`.
- Read the iPhone-reachable backend base URL from ignored `apps/mobile/ios/Config/Deployment.local.xcconfig`.
- If both values are present and the backend URL is HTTP(S), construct the standard local served asset URLs:
  - `<base>/v1/generated-assets/<session_id>/game.glb`
  - `<base>/v1/print-candidates/<session_id>/print.3mf`
- Use those URLs in the configured request writer command.
- If either value is missing or invalid, keep the existing placeholder command.

This is report-only guidance. It does not run shell writers, call Treatstock, write `.local/print-quote-request-configured.json`, start a backend, touch Xcode signing, or expose local filesystem paths.

## Data Flow

```text
final-demo-launch-local.json
  -> local smoke session id

Deployment.local.xcconfig
  -> PMF_BACKEND_BASE_URL

print_fulfillment_readiness
  -> concrete PRINT_SOURCE_ASSET_URI / PRINT_CANDIDATE_URI command
```

## Error Handling

Unreadable or absent evidence silently falls back to the current placeholder command. Malformed JSON, missing session id, missing backend URL, and non-HTTP backend URL must not make the readiness report fail for a new reason.

## Testing

Add focused tests in `services/backend/tests/test_print_fulfillment_readiness.py`:

- Missing configured quote request plus usable final demo and deploy config evidence yields a concrete writer command with local served asset URLs.
- Missing/invalid local evidence still yields the existing placeholder writer command.

The phase remains safe to verify with focused print fulfillment tests, backend lint, and `make print-fulfillment-readiness`.
