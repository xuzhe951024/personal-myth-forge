# P0.17 NPC Agent Runtime Trace Design

## Goal

Move NPC behavior from a director-only response toward the product decision of
high-autonomy NPCs plus a world arbitrator by adding an explicit, testable agent
runtime trace contract.

The demo should still render the existing `npc_reactions`, but each reaction can
now be backed by a per-NPC trace that records belief, intention, proposed action,
rationale, and confidence.

## Context

The current backend already has:

- `LocalNPCDirector`, a deterministic local response for three NPCs.
- `OpenAINPCDirector`, an optional structured-output provider behind
  `NPC_PROVIDER=openai`.
- `LocalWorldArbitrator`, which accepts or rejects visible NPC actions.

This proves text NPC generation, but the API does not yet expose the shape of an
agent loop. P0.17 adds that missing contract without bringing in voice, Unity,
or an external NPC platform.

## Product Behavior

`MythSession` gains:

- `npc_agent_runtime`: the runtime that produced the trace, such as
  `local_agent_runtime` or `openai_structured_runtime`.
- `npc_agent_traces`: one trace per NPC when available.

Each `NPCAgentTrace` includes:

- `npc_id`
- `name`
- `belief`
- `intention`
- `proposed_action`
- `rationale`
- `confidence`

`npc_reactions` remains the player-facing summary. `world_resolution` still
resolves concrete actions. The new traces are the first inspectable bridge
between "NPC thought" and "world action."

## Backend Design

Add `NPCAgentTrace` to the domain model. Extend `NPCDirectorResult` with:

- `agent_runtime: str`
- `agent_traces: list[NPCAgentTrace]`

Extend `MythSession` with:

- `npc_agent_runtime: str`
- `npc_agent_traces: list[NPCAgentTrace]`

The local runtime should generate traces first, then derive existing
`NPCReaction` objects from those traces. That keeps deterministic tests and
shows how later LLM/runtime output will flow.

The OpenAI director should request traces in the structured output. If an older
mock or provider output omits traces, it may synthesize conservative traces from
the returned reactions instead of failing a valid three-NPC response. Provider
secrets remain backend-only.

## Mobile Design

Add Swift `NPCAgentTrace` and backward-compatible decoding for the new
`MythSession` fields:

- old responses without trace fields decode as `npcAgentRuntime == ""` and
  `npcAgentTraces == []`
- new responses decode trace rows and runtime label

Update `NPCReactionsView` to show compact per-NPC agent trace rows when traces
are present. This is source/compile evidence only, not a simulator run.

## Out Of Scope

- OpenAI Agents SDK adoption.
- Voice NPCs.
- Unity village movement execution.
- Long-term memory or vector recall.
- Multi-turn runtime ticking.
- Raw personal data access. Only approved context capsules are passed through.

## Testing And Evidence

- Backend tests cover local trace generation, API JSON fields, pipeline
  serialization, OpenAI structured output trace parsing, and trace fallback from
  reactions.
- Swift tests cover new fixture decoding, old response decoding without traces,
  and source checks for the trace view.
- Static Browser evidence shows an iOS-first trace panel with NPC belief,
  intention, proposed action, and world arbitrator handoff.
