# P0.427 Mobile Closure Resource Source Design

## Goal

Make the iPhone Final Launch summary decode and display the final launch closure packet's `final_resource_requirements` source report so the mobile handoff can show the canonical resource/key blocker inside the closure evidence path.

## Current State

The backend closure packet now includes `source_reports.final_resource_requirements.device_action_bundle`. The Swift `FinalLaunchClosurePacketReport` model only decodes sections, commands, operator actions, and safety. `FinalLaunchMobileSummary.closurePacketRows` therefore cannot show that the closure packet itself contains the safe resource requirements action bundle.

## Approach

Add small Swift Codable models for closure packet source reports and a generic resource action bundle shape. Decode `sourceReports` on `FinalLaunchClosurePacketReport`. Add one closure row that summarizes `finalResourceRequirements.deviceActionBundle.firstAction` with the action id, command, and validation command. Keep the existing top-level `finalResourceRequirements` rows unchanged.

## Safety

This is mobile decode and presentation only. It does not write backend `.env`, write iOS deploy config, run Xcode, call providers, or expose raw resource values. All displayed text passes through the existing `sanitize()` path.

## Testing

Add Swift contract coverage for decoding `source_reports.final_resource_requirements.device_action_bundle` from the final launch fixture and for rendering it in `closurePacketRows` without leaking synthetic secrets, local paths, or private URL schemes.
