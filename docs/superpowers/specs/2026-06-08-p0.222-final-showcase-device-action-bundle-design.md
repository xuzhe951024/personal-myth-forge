# P0.222 Final Showcase Device Action Bundle Design

## Context

`final-showcase-readiness` already aggregates source acceptance, local smoke, live provider evidence, resource handoff, print fulfillment, visual regression, and iOS deploy evidence. The first remaining blocker for the complete showcase is still iOS deployability: the operator must start the device backend, run the mobile deploy preflight, resolve signing or Xcode gates, and then rerun the iOS device rehearsal. Today those steps are present only as a long flat `operator_actions` list, which is hard to inspect from the mobile Final Launch panel.

## Goal

Add a structured `device_action_bundle` to `final_showcase_readiness_report` and surface it in the iPhone Final Launch summary. The bundle should make the final iOS-device demo path obvious without running any commands or mutating local/global state.

## Non-Goals

- Do not run `backend-device-demo`, `mobile-deploy-preflight`, Xcode, signing, keychain, or provider calls.
- Do not write `Deployment.local.xcconfig`, backend env files, generated media, or checkout/payment links.
- Do not mark iOS device launch evidence ready without the existing runbook and rehearsal reports.
- Do not replace `operator_actions`; keep the existing flat list for backward compatibility.

## Contract

`final_showcase_readiness_report` includes:

```json
{
  "device_action_bundle": {
    "id": "ios_device_manual_actions",
    "label": "iOS Device Manual Actions",
    "status": "blocked",
    "summary": {
      "actions": 4,
      "ready": 0,
      "manual": 4,
      "blocked": 4,
      "xcode_or_signing": 1,
      "global_actions": 0,
      "provider_calls": 0
    },
    "first_action": {
      "id": "start_backend_device_demo",
      "label": "Start backend device demo",
      "status": "blocked",
      "command": "make backend-device-demo"
    },
    "actions": [
      {
        "id": "start_backend_device_demo",
        "label": "Start backend device demo",
        "status": "blocked",
        "classification": "manual_backend_required",
        "command": "make backend-device-demo",
        "detail": "Start the LAN-reachable backend before running iPhone preflight.",
        "source": "final_showcase_readiness",
        "blocks": ["ios_deployable", "functional_regression"],
        "manual": true,
        "global_action": false,
        "provider_calls": false,
        "xcode_or_signing": false
      }
    ],
    "safety": {
      "commands_run": false,
      "global_mutation": false,
      "provider_calls": false,
      "xcode_or_signing": false
    }
  }
}
```

The mobile summary adds a compact `Device actions` row, the first action command, and the safety flags. Unsafe text must be sanitized by the existing mobile summary sanitizer.

## Safety

The bundle is a read-only action manifest. It can describe Xcode/signing and local server actions, but it must make clear that commands were not executed. It must not include absolute paths, secrets, raw private context, raw media, payment links, or provider payloads.

## Verification

- RED/GREEN backend tests for the bundle shape, ordering, summary counts, first action, and safety flags.
- RED/GREEN Swift contract tests for Codable decode and Final Launch summary rendering.
- Source acceptance gates for the backend report builder, Swift model, Swift summary, and contract tests.
- One static 390x844 visual regression artifact showing the new mobile summary panel.
