# P0.604 Top-Level Live Action Order

## Context

`final-configured-evidence-plan` and `configured-live-evidence-bundle` now agree that
the next live provider step is configured 3D evaluation. The top-level closure packet
and external action ledger can still surface an older final-acceptance/live-evidence
action first, which makes the operator handoff less precise before any live Meshy or
OpenAI call is approved.

## Decision

Top-level handoff reports should preserve the configured bundle's concrete first live
provider action when it includes a validation rerun, instead of replacing it with a
generic configured-live-evidence or final-acceptance fallback.

## Acceptance

- `final-launch-closure-packet` first blocker and operator actions preserve
  `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-3d-configured; rerun make
  final-configured-evidence-plan`.
- `final-external-action-ledger` uses the configured live evidence bundle's first
  operator action before stale live-provider-evidence fallbacks.
- The implementation remains read-only and does not call Meshy, OpenAI, print
  providers, Xcode signing, or global machine actions.
- Existing device, resource, print, and local rehearsal handoff behavior is unchanged.
