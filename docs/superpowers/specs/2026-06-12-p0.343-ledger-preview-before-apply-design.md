# P0.343 External Ledger Preview Before Apply Design

## Context

`final-external-action-ledger.json` is the operator-facing ledger for actions
that require user input, cost consent, local writes, or global machine changes.
It currently lists the read-only resource apply preview and the direct resource
apply writer in the same top-level `operator_actions` list when final resources
are still blocked:

```text
make final-resource-apply-preview
make final-apply-resources
```

The direct apply command writes ignored backend and iOS local config files. It
should remain documented in `actions_by_id` and `operator_sequence`, but it
should not be presented as a next operator action before the preview gate is
ready.

## Goal

When the external action ledger already lists the apply-preview gate, remove
direct `make final-apply-resources` from `operator_actions`.

## Non-Goals

- Do not remove the `apply_final_resources` action from `actions_by_id`.
- Do not remove `make final-apply-resources` from `operator_sequence`.
- Do not change make targets or writer scripts.
- Do not write backend `.env`, iOS deploy config, Keychain state, signing
  state, provider state, or print order state.
- Do not call live Meshy, OpenAI, Treatstock, or Xcode signing commands.

## Design

Add two narrow changes in `final_external_action_ledger._operator_actions()`.

First, when `preview_final_resource_apply` is `missing` or `blocked`, include
its read-only command in `operator_actions` even though it does not require a
secret or user-filled value. This makes the preview gate explicit in the ledger.

Second, add a post-dedupe filter.

The filter detects the action root:

```text
make final-resource-apply-preview
```

If present, it removes exact direct apply roots:

```text
make final-apply-resources
```

Other resource input actions stay visible, including required key/config fill
commands. The ledger still records the apply writer as a blocked action in the
safe local writes group so the complete sequence remains auditable.

## Acceptance Criteria

- Missing-resource ledger output includes `make final-resource-apply-preview`.
- Missing-resource ledger `operator_actions` does not include
  `make final-apply-resources`.
- `actions_by_id["apply_final_resources"]` still exists with command
  `make final-apply-resources`.
- `operator_sequence` still includes `make final-apply-resources`.
- Focused ledger tests, operator action tests, lint, diff checks, and backend
  tests pass.
