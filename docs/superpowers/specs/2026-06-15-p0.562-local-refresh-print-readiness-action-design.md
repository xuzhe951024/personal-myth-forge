# P0.562 Local Refresh Print Readiness Action Design

## Context

P0.561 fixed `final-showcase-readiness` so its top-level `operator_actions` retains the backend-auto print fulfillment prerequisite:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make print-fulfillment-readiness
```

`final-local-report-refresh` already regenerates `final-showcase-readiness.json`, but its own top-level `operator_actions` still omits that print prerequisite. This makes the consolidated refresh report less useful as the one command packet to inspect before the user provides external resources.

## Decision

Extend `final-local-report-refresh` step action selection so it can promote relevant `report_operator_actions` from `final_showcase_readiness`, not only from `final_demo_launch_local`. Preserve multiple validation-aware actions for the same command root when their validation targets differ; for example, the deploy writer can legitimately ask the operator to rerun both `make mobile-deploy-preflight` and `make print-fulfillment-readiness`.

The promoted action set remains narrow:

- provider handoff actions
- print handoff/readiness actions
- device handoff actions

Existing action normalization, dedupe, priority sorting, and top-12 truncation continue to apply.

## Scope

In scope:

- Add a regression test proving `final_showcase_readiness.report_operator_actions` can supply the backend-auto print readiness prerequisite.
- Update `_selected_step_report_actions()` and validation-aware action dedupe in `final_local_report_refresh.py`.
- Refresh local readiness evidence and roadmap notes.

Out of scope:

- Running real Meshy/OpenAI/Treatstock calls.
- Modifying iOS signing, Xcode, Keychain, or global machine state.
- Changing final showcase readiness behavior already covered by P0.561.

## Acceptance

- The new focused test fails before implementation and passes after implementation.
- `final-local-report-refresh` preserves the backend-auto print readiness prerequisite when the regenerated `final_showcase_readiness` child report exposes it.
- `make final-local-report-refresh` still writes `services/backend/.local/final-local-report-refresh.json` through the safe wrapper; the exact print blocker may be earlier in the resource chain when ignored local resource files are absent.
- The report still preserves existing deploy, backend-device-demo, backend URL, provider handoff, and Xcode actions.
- Backend lint and backend test suite pass with only the known Starlette/httpx deprecation warning.
