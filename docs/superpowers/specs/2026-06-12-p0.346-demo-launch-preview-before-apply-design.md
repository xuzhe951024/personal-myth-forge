# P0.346 Demo Launch Preview Before Apply Design

## Context

`final-demo-launch-configured.json` is a configured-mode operator report for the
final iPhone showcase path. When final resources are missing, its first blocker,
next action, and top-level action list currently point directly at:

```text
make final-apply-resources
```

That command is still the reviewed writer after resource values are filled, but
configured launch should route the operator through the read-only preview first:

```text
make final-resource-apply-preview
```

## Goal

Make configured final demo launch prefer the read-only final resource apply
preview before direct final resource writer actions whenever the apply phase is
not ready.

## Non-Goals

- Do not remove `make final-apply-resources` from `commands`.
- Do not change local-mode first-blocker behavior.
- Do not change resource writer scripts or resource templates.
- Do not execute live provider calls, Xcode signing, Keychain writes, resource
  writers, backend `.env` writes, iOS deploy config writes, or print orders.

## Design

Add local command constants in `final_demo_launch.py` for:

```text
make final-resource-apply-preview
make final-apply-resources
```

When `_launch_phases()` builds the `apply_final_resources` phase, use
`make final-resource-apply-preview` as the phase command only for configured
mode while the phase status is not `ready`. Keep `make final-apply-resources`
for local mode and for ready configured reports.

Because first blocker, next action, and operator actions derive from launch
phases, this moves the configured missing-resource handoff to the preview
without changing the commands list.

## Acceptance Criteria

- Configured final demo launch missing-resource path reports
  `apply_final_resources.command == "make final-resource-apply-preview"`.
- Configured first blocker and next action use
  `make final-resource-apply-preview`.
- Configured top-level `operator_actions` includes the preview command and does
  not include bare `make final-apply-resources`.
- Configured `commands` still includes `make final-apply-resources`.
- Local mode still exposes `make final-apply-resources` for the continuable
  apply phase.
- Focused final demo launch tests, lint, diff checks, and backend tests pass.
