# P0.451 External Ledger Backend Handoff Design

## Context

The final external action ledger is the consolidated report for things Codex must not do automatically: secrets, local ignored writes, live provider costs, global Mac/Xcode actions, and physical-device runtime actions. It already declares `make backend-device-demo` in `OPERATOR_SEQUENCE` and receives a device action bundle from the iOS deploy runbook, but its top-level `operator_actions` currently skip the backend device server step.

That leaves the final external handoff less copy-pasteable than the certificate and rehearsal reports. A physical iPhone demo requires the operator to start the LAN backend before running mobile deploy preflight.

## Goal

Make `final_external_action_ledger_report.operator_actions` include the validated backend-device-demo handoff:

```text
start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight
```

## Approaches Considered

1. Add the handoff as an explicit `operator_action` on the `start_backend_device_demo` action.
   - Pro: small, direct, and keeps the action owned by the `device_runtime_actions` group.
   - Con: requires `_operator_actions()` to include manual non-global operator actions.

2. Append a hard-coded backend action in `_operator_actions()`.
   - Pro: minimal logic.
   - Con: hides the relationship between the operator action and the device runtime action.

3. Pull the action from `device_action_bundle`.
   - Pro: mirrors iOS runbook evidence.
   - Con: larger change because the current top-level operator action builder only receives action groups.

Recommended approach: option 1. It preserves the existing action group architecture and avoids coupling `_operator_actions()` to report-level bundles.

## Design

Add a `BACKEND_DEVICE_DEMO_VALIDATED_ACTION` constant to `final_external_action_ledger.py` with the canonical validated handoff. Attach it as `operator_action` to `_device_runtime_actions()` for `start_backend_device_demo`.

Extend `_operator_actions()` so manual actions with a concrete `operator_action` are included even when they are not global/Xcode confirmation actions. The existing global/manual branch stays in place for actions that require user confirmation.

The report must still avoid running commands. This phase only changes strings in a read-only report.

## Testing

Add a focused regression test in `tests/test_final_external_action_ledger.py` that asserts:

- `actions_by_id["start_backend_device_demo"].operator_action` is the complete validated handoff.
- The validated backend handoff appears in `report["operator_actions"]`.
- The bare `make backend-device-demo` action is not emitted as a top-level operator action.

Verification commands:

```bash
cd services/backend
uv run pytest tests/test_final_external_action_ledger.py::test_external_action_ledger_operator_actions_include_backend_device_demo_handoff -q
uv run pytest tests/test_final_external_action_ledger.py -q
make backend-lint
git diff --check
```

No provider key, Xcode signing, backend server, live provider call, Keychain write, or print order is performed.
