# P0.476 Configured Final Demo Launch Safe Wrapper Design

## Context

The final showcase remains blocked by external iOS and provider resources, which is expected without `DEVELOPMENT_TEAM`, an iPhone-reachable backend URL, provider keys, and cost consent. Several canonical Make targets already use project-local safe wrappers so a blocked readiness report can be written to `services/backend/.local/` while Make exits 0. `final-demo-launch-configured` still calls the CLI directly and can return exit 2 after writing a usable blocked report.

## Decision

Add a safe wrapper for `final-demo-launch-configured` that mirrors the existing report-writer scripts:

- run `myth_forge_api.cli final-demo-launch --mode configured --repo-root ../.. --output .local/final-demo-launch-configured.json`;
- accept exit code 0 and exit code 2 as successful Make execution because both indicate a usable report path;
- keep any other exit code failing so command or runtime breakage is still visible;
- keep all output under ignored `.local` files and avoid provider calls, Xcode signing, Keychain writes, or global mutation.

## Scope

This phase only changes the execution wrapper, source gates, and tests for the configured final demo launch report. It does not resolve the external blockers, provide secrets, run live Meshy/OpenAI/Treatstock calls, or perform iOS signing.

## Files

- `services/backend/scripts/write_final_demo_launch_configured.sh`: new safe wrapper.
- `Makefile`: route `final-demo-launch-configured` through the wrapper.
- `services/backend/tests/test_final_rehearsal_scripts.py`: RED/GREEN wrapper behavior and dry-run coverage.
- `services/backend/src/myth_forge_api/ios_showcase_acceptance.py`: source gate requires the wrapper.
- `services/backend/tests/test_ios_showcase_acceptance.py`: fixture and source-gate assertions.
- `apps/mobile/ios/Sources/PersonalMythForgeMobileProjectChecks/main.swift`: Swift static check reads the wrapper for configured output details.
- `services/backend/tests/test_evaluation_make_targets.py`: dry-run expectations account for wrapper output.
- `services/backend/tests/test_resource_template_acceptance.py`: acceptance fixture includes the wrapper target.
- `docs/iteration-roadmap.md`: record P0.476.

## Acceptance

- Focused tests first fail on the missing wrapper and Make target routing.
- After implementation, focused backend tests pass.
- Swift ProjectChecks pass.
- `make final-demo-launch-configured` exits 0 when the configured report is blocked but usable.
- `make final-showcase-readiness` still reports blocked on real external conditions, with the same first iOS blocker.
