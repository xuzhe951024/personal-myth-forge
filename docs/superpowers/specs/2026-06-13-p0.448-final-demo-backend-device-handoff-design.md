# P0.448 Final Demo Backend Device Handoff Design

## Goal

Make the local final demo launch report explicitly show the backend-device-demo handoff when the physical iPhone deploy preflight is blocked.

## Context

The final objective requires a real iPhone to reach the FastAPI backend over LAN before the iOS showcase can be verified. `final-demo-launch-local` already lists `make backend-device-demo` in its command catalog, but its top-level operator actions currently only tell the operator to write deploy config and rerun `make mobile-deploy-preflight`.

That leaves out the concrete backend launch step needed before a physical-device preflight can pass.

## Behavior

When `build_final_demo_launch_report(mode="local")` has a `mobile_deploy_preflight` phase with status `blocked` or `missing`, the report should include:

```text
start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight
```

The action should appear in the local operator checklist and top-level operator actions after normal dedupe. It should not appear merely because the command catalog contains `make backend-device-demo`.

## Safety

This only changes report text. It must not start the backend, run iOS preflight, call Meshy/OpenAI/Treatstock, write provider keys, touch Xcode signing, write Keychain entries, or create print order/payment artifacts.

## Alternatives Considered

- Put the action only in the command catalog: already true, but too indirect for final handoff.
- Put the action in final showcase readiness only: already helps the global readiness ledger, but `final-demo-launch-local` remains incomplete as a standalone launch report.
- Add the validated backend-device-demo action to local final demo operator actions when mobile preflight is blocked: chosen because it is explicit, safe, and matches existing action normalization.

## Tests

Add a focused test that builds the local final demo report with missing iOS deploy config and asserts that `operator_actions` includes the validated backend-device-demo handoff, not a bare `make backend-device-demo`.
