# P0.361 Mobile Final Showcase Print Action Design

## Context

The backend final showcase readiness report now exposes validation-aware
operator actions for live provider handoff and print fulfillment handoff.
The iPhone `FinalLaunchMobileSummary` already renders selected final showcase
operator actions, but its priority filter focuses on final handoff, live
provider, final resource, and final showcase refresh actions.

The print fulfillment handoff action is later in the backend action list:

```text
after explicit Treatstock cost consent, save a sanitized services/backend/.local/print-quote-configured.json from POST /v1/print-quotes; rerun make print-fulfillment-readiness
```

Because the mobile summary only selects the first action plus known priority
fragments, the iPhone can miss the print handoff action even though print
fulfillment is part of the final demo objective.

## Goal

Make the iPhone Final Launch summary prioritize final showcase print fulfillment
handoff actions alongside live provider and resource handoff actions.

## Approach

Update `FinalLaunchMobileSummary.showcaseReadinessActionRows` so actions
containing print fulfillment handoff markers are selected as priority actions.
Use existing sanitized backend text; the mobile app must not execute commands,
call Treatstock, create orders, or store provider keys.

Priority markers to include:

- `treatstock`
- `print-fulfillment`
- `print_fulfillment`

Increase the bounded display count from 6 to 7 so the existing priority rows
stay visible while adding the print handoff row.

## Non-Goals

- Do not alter backend readiness status or command generation.
- Do not run Treatstock, Meshy, OpenAI, Xcode signing, Keychain writes, or
  global configuration.
- Do not add command execution to the iPhone app.

## Testing

Extend the Swift mobile core contract test
`testFinalLaunchMobileSummaryShowsPriorityFinalShowcaseActions` with a
validation-aware Treatstock print action placed after the existing priority
actions. The test should assert that the print action is visible and unrelated
extra actions remain hidden.
