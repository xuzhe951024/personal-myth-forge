# P0.497 Local Refresh Rehearsal Action Pruning Design

## Context

`final-local-report-refresh.json` is the safe local report-refresh aggregate. It already surfaces concrete next actions from the refreshed reports:

- write `DEVELOPMENT_TEAM` into ignored iOS deploy config and rerun `make mobile-deploy-preflight`
- run final resource apply preview before provider handoff and live evidence
- run the guarded print quote command with explicit consent
- provide Meshy/OpenAI values in ignored `final-resources.env`
- start the backend device demo before physical-device preflight
- accept the Xcode license outside Codex before Xcode evidence

The same top-level `operator_actions` still includes `make final-rehearsal-local`, including validation-aware variants such as `make final-rehearsal-local; rerun make ios-device-launch-rehearsal`. That command remains useful as a canonical aggregate command, but it is redundant as a top-level handoff action while concrete child handoff actions are already present.

## Decision

Filter only `final_local_report_refresh.operator_actions`.

The report should:

- Drop actions whose command root is `make final-rehearsal-local` when any more specific operator action exists.
- Preserve `make final-rehearsal-local` when it is the only actionable item.
- Keep all concrete device, provider, print, Xcode, and resource actions.
- Preserve ordering so the first blocker action remains first.
- Leave step commands, report execution, local evidence writing, and safe blocked-report behavior unchanged.

## Safety

This is report-only. It does not run Xcode signing, write Keychain entries, call Meshy/OpenAI/Treatstock, start backend servers, write ignored env files, or trigger print/provider costs.

## Verification

TDD coverage must prove:

- `_operator_actions()` drops `make final-rehearsal-local` command-root actions when a concrete handoff action exists.
- `_operator_actions()` preserves `make final-rehearsal-local` when it is the only action.
- Refreshed `final-local-report-refresh.json` no longer emits `make final-rehearsal-local` command-root actions in top-level `operator_actions` while current concrete handoff actions remain.
- Focused local-refresh tests, operator action tests, lint, and refreshed local evidence pass.
