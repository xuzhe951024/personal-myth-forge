# P0.441 Showcase Provider Handoff Action Design

## Context

`services/backend/.local/final-showcase-readiness.json` currently includes
`make provider-handoff` in `operator_actions`. That action is part of the
provider/key handoff path for moving from local stubs toward live Meshy/OpenAI
evidence. The iPhone Final Launch showcase readiness summary filters the action
list down to high-priority rows, but the priority filter does not explicitly
recognize `provider-handoff`.

This can hide a useful Mac-side command from the mobile final launch panel even
though the backend report already exposes it safely and without secrets.

## Design

Treat provider handoff actions as high-priority final showcase actions in the
iPhone Final Launch summary. The summary should preserve action strings that
contain:

- `provider-handoff`
- `provider_handoff`

The filtered action list currently caps at 7 rows. With provider handoff added,
the representative final-showcase action set has 8 useful rows, so raise the cap
to 8. This preserves the existing print quote handoff row instead of trading one
external handoff for another.

This is intentionally display-only. The mobile app still does not execute shell
commands, write provider keys, call live providers, or mutate Xcode/global
state.

## Scope

- Modify only `FinalLaunchMobileSummary.showcaseReadinessActionRows`.
- Add Swift contract coverage that proves `make provider-handoff` remains
  visible when mixed with other final showcase actions while the print handoff
  row remains visible.
- Do not change backend report generation, provider adapters, resource files,
  print quote behavior, or iOS signing behavior.

## Acceptance

- A contract test fails before implementation because `make provider-handoff`
  is omitted from showcase readiness rows.
- The same test passes after adding provider handoff fragments to the priority
  filter.
- Existing mobile project checks, Swift build, backend lint, and whitespace
  checks pass.
