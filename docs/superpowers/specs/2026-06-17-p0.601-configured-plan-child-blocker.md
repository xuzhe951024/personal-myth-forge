# P0.601 Configured Plan Child Blocker

## Context

P0.600 lets `final-configured-preflight` surface concrete child blockers from the
configured iOS deploy runbook. `final-configured-evidence-plan` still compresses that
source into a generic `final_configured_preflight` blocked step, which hides the
actual `mobile_deploy_preflight` command and detail.

## Decision

When a source report has a child blocker, the final configured evidence plan should
copy the child blocker id, command, detail, and validation command into the blocked
step and first blocker.

## Acceptance

- The `final_configured_preflight` step records `source_blocker_id`,
  `source_blocker_command`, `source_blocker_detail`, and `source_validation_command`.
- The plan first blocker uses the concrete source command and detail.
- The first operator action reruns the source validation and then the configured
  evidence plan.
- No live provider calls, API key writes, print orders, or Xcode/global actions are
  introduced.
