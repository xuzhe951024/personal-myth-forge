# P0.312 Final Handoff Live Consent Action Design

## Goal

Make configured final acceptance handoff actions read as explicit live-provider cost-consent gates instead of ordinary `run make ...` commands.

## Context

The final iOS showcase requires real Meshy/OpenAI/print-provider evidence before it can be marked complete. Those actions can spend provider credits and must only run after explicit operator consent. The project already centralizes the configured final acceptance command in `configured_acceptance_command.py`, but the current operator action says:

`run make final-acceptance-configured only after live provider cost review and --allow-live-provider-calls consent`

That text includes the right warning, but the leading `run` makes the action look like a normal command in final handoff lists. The safer handoff should foreground consent first while keeping the make target and `--allow-live-provider-calls` flag visible.

## Approach

Change `CONFIGURED_FINAL_ACCEPTANCE_COST_REVIEW_ACTION` to:

`approve live provider cost review before make final-acceptance-configured; --allow-live-provider-calls consent required`

Also update `normalize_operator_action()` so older saved reports that still contain the previous wording normalize to the new consent-first action. This keeps refreshed reports clean and protects aggregators that ingest stale `.local` JSON.

## Data Flow

The shared constant is used by:

- configured final preflight
- final handoff index
- final operator handoff
- iOS deploy runbook and device launch rehearsal/certificate paths

Changing the shared constant updates newly generated reports. Adding a normalizer replacement updates stale saved actions when report aggregators pass them through `normalize_operator_action`.

## Testing

Use TDD:

- Update configured acceptance command handoff tests to expect the consent-first action.
- Add operator action coverage that the old `run make final-acceptance-configured...` text normalizes to the new consent-first action.
- Run focused final handoff, configured preflight, final demo launch, operator handoff, and operator action tests.
- Refresh `final-handoff-index.json` and verify the saved top-level action no longer starts with `run make final-acceptance-configured`.

## Non-Goals

- Do not run `make final-acceptance-configured`.
- Do not pass `--allow-live-provider-calls`.
- Do not call Meshy, OpenAI, Treatstock, Xcode, Keychain, or any paid provider.
- Do not change acceptance command targets or provider readiness classifications.
