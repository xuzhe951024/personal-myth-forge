# P0.50 NPC Ritual Scene Design

## Product Intent

The iOS app can already capture an object, create a myth session, preview a
generated 3D asset, run NPC ticks, and show a final demo script. The weak part
of the final showcase is spatial storytelling: NPCs still mostly exist as text
below the artifact. P0.50 makes the smallest useful step toward "NPCs act around
the artifact" by rendering deterministic NPC markers around the 3D relic in the
SceneKit preview.

## Product Design Brief

- Product: Personal Myth Forge iOS app.
- Screen/component: existing `3D Artifact Preview`.
- Visual source: current SwiftUI/SceneKit app styling; no external Figma or
  saved Product Design context is available.
- Interactivity: full working source path, compiled by SwiftPM. No throwaway
  static prototype.

## Scope

Add a mobile-core `NPCRitualSceneBuilder` that converts existing typed state
into a safe spatial scene model:

- `NPCRitualScene`
- `NPCRitualActor`
- `NPCRitualStance`

Inputs:

- `MythSession`
- optional latest `NPCAgentTick`

Output:

- exactly three actor entries when NPC data is present
- deterministic positions around the artifact
- name, emotion, stance, action, and short label
- visible world changes copied from the latest tick when present, otherwise
  from the session world resolution
- no raw media, local file paths, checkout URLs, or provider secrets

Add the scene model to the iOS `Artifact3DPreviewView`:

- `ForgeRootView` passes `latestNPCTick`.
- Fallback SceneKit scene draws three simple NPC markers around the relic.
- The view shows a short `NPC ritual` line under the 3D scene.
- If a downloaded SceneKit asset loads successfully, keep using it as the base
  scene and add the NPC overlay nodes to it.

Add `npc_ritual_scene` to `ios_showcase_acceptance` so final acceptance proves
the checked-in mobile source includes the spatial NPC scene path.

## Behavior

- The initial session uses `session.npcAgentTraces` and `session.npcReactions`.
- The latest tick overrides the initial NPC data when present.
- Stance is derived from accepted/rejected actions:
  - accepted action -> `acting`
  - rejected action -> `debating`
  - no action but emotional output -> `watching`
- Positions are fixed and deterministic:
  - actor 0: left/front
  - actor 1: right/front
  - actor 2: center/back
- Unsafe text is replaced with `[withheld]`.

## Non-Goals

- No Unity integration.
- No animation engine.
- No new backend schema.
- No Meshy/OpenAI/print provider calls.
- No Apple SDK license or signing changes.
- No real device claim.

## Tests

Swift core contract tests must prove:

- initial myth session creates three actor nodes
- latest NPC tick overrides initial actor state
- accepted/rejected world actions affect stance
- unsafe text is redacted

Swift app/source checks must prove:

- `Artifact3DPreviewView` consumes `NPCRitualSceneBuilder`
- `ForgeRootView` passes `latestNPCTick`
- the Xcode project includes any new app source file if one is added

Backend source acceptance must prove:

- `ios_showcase_acceptance` includes `npc_ritual_scene`
- final acceptance test stubs use `ios_showcase_acceptance` internal
  `passed=11`, `failed=0`
