# P0.355 Showcase Bare iOS Writer Action Design

## Goal

Make the first `final-showcase-readiness` operator action copy-pasteable when the iOS deploy writer is the selected blocker action.

## Context

The final iOS showcase goal is currently blocked by missing device deployment inputs: `DEVELOPMENT_TEAM` and an iPhone-reachable `PMF_BACKEND_BASE_URL`. The report already preserves that detail in `first_blocker.detail` and `next_action.detail`.

The remaining usability gap is in `operator_actions[0]`: when the same iOS deploy writer command appears both as a bare command and with a `| Missing DEVELOPMENT_TEAM; ...` detail suffix, the final showcase dedupe step keeps the detail-bearing string:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable
```

That string is useful as explanatory text but is less useful as an operator command.

## Design

For the final showcase readiness report only, duplicate iOS deploy writer actions with the same command root should prefer the bare command:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

The explanatory detail remains available through the structured blocker fields:

- `first_blocker.detail`
- `next_action.detail`
- saved mobile preflight evidence

This keeps the top-level action list executable without deleting diagnostic context.

## Scope

In scope:

- Update `services/backend/src/myth_forge_api/final_showcase_readiness.py`.
- Update focused tests in `services/backend/tests/test_final_showcase_readiness.py`.
- Add this spec, an implementation plan, and a roadmap entry.
- Refresh safe ignored `.local` readiness evidence after tests.

Out of scope:

- Changing the shared `operator_actions.py` preference behavior.
- Running real Meshy, OpenAI, Treatstock, Xcode signing, Keychain, or paid provider actions.
- Modifying iOS app source or deployment config.

## Acceptance Criteria

- `_dedupe_operator_actions([bare_writer, detailed_writer])` returns `[bare_writer]` for the final showcase report.
- `first_blocker.detail` and `next_action.detail` still include the missing `DEVELOPMENT_TEAM` and iPhone backend URL detail.
- `services/backend/.local/final-showcase-readiness.json` remains `blocked`, but its first operator action is the bare validated writer command.
- Focused backend tests, lint, and diff checks pass.
