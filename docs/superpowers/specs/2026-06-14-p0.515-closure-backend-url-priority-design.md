# P0.515 Closure Backend URL Priority Design

## Context

The final local iPhone handoff chain is now consistent in the main readiness reports:

1. `DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`
2. `start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight`
3. `set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL; rerun make mobile-deploy-preflight`

`final-launch-closure-packet` still lists provider/key handoff actions before the backend URL action. That makes the final closure packet less executable than the lower-level reports because the operator should finish the local physical-iPhone connectivity chain before Meshy/OpenAI/Treatstock work.

## Decision

P0.515 updates only `final_launch_closure_packet` operator action ordering.

When all three local device actions are present, the closure packet must list them first in this order:

1. Deploy config writer
2. Backend device demo
3. iPhone-reachable backend URL

Provider keys, provider handoff, print quote consent, and Xcode actions remain in the packet, but must follow the three local device actions when those actions are available.

## Boundaries

- No real provider calls.
- No Xcode signing, Keychain writes, global mutation, or paid print/provider action.
- No changes to resource collection semantics, readiness status, report sanitization, or closure packet sections.
- No new secrets or local config values are written.

## Implementation Shape

Reuse the closure packet action ordering pipeline. Extend the existing backend-device-demo prioritizer so it also moves the canonical backend URL action immediately after backend-device-demo when both actions are present.

The helper should be idempotent:

- If backend URL is absent, keep the list unchanged.
- If backend-device-demo is absent, keep the list unchanged.
- If backend URL is already immediately after backend-device-demo, keep the list unchanged.
- If backend URL appears before backend-device-demo, move it after backend-device-demo without duplicating it.
