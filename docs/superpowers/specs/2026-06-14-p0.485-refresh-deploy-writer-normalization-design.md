P0.485 Final Local Refresh Deploy Writer Normalization Design

Goal

Make `final-local-report-refresh` prefer the project-local iOS deploy writer command over stale manual Team ID edit actions in its top-level operator actions.

Context

The final external action ledger, final launch closure packet, final demo launch, and final showcase readiness now surface:

`DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`

as the first concrete iOS device action. The refreshed final local report still includes the older action:

`provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`

This makes the final Mac-side refresh handoff less consistent. The project-local writer is safer because it writes ignored config through a controlled Make target and keeps validation attached.

Non-Goals

- Do not set a real Apple Team ID or write `Deployment.local.xcconfig`.
- Do not run Xcode, signing, Keychain, live providers, or print-provider calls.
- Do not remove independent bundle id or backend URL actions.
- Do not change mobile deploy preflight semantics or iOS deploy readiness status.

Design

Import and apply `prefer_project_local_ios_deploy_handoff_actions()` inside `final_local_report_refresh._dedupe_operator_actions()` after existing validation, provider-chain, backend-demo, and print handoff normalization.

This preserves the current action ordering while letting the shared helper remove stale Team ID edit actions only when the writer command is already present.

Acceptance Criteria

- `final_local_report_refresh._operator_actions()` keeps the validated writer command when both writer and manual Team ID edit actions are present.
- The stale `provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight` action is absent when the writer command exists.
- Bundle id and backend URL deploy actions remain present when still required.
- Existing provider chain, print quote, apply-preview, backend demo, and first-blocker prioritization tests continue to pass.
- Safe local refresh evidence remains blocked on missing external iOS/provider resources and does not perform external actions.

Safety

This phase changes report action normalization only. Verification may write ignored `.local` JSON reports through safe Make targets. It must not mutate global state, write real secrets, call providers, run signing, or place print orders.
