# P0.365 Mobile Showcase Print Handoff Filter Design

## Context

Backend P0.363/P0.364 narrowed print handoff action selection so validation-only actions like this are not promoted:

```text
make visual-regression-local; rerun make print-fulfillment-readiness
```

The iOS `FinalLaunchMobileSummary` still uses broad priority fragments that include `print-fulfillment` and `print_fulfillment`. If the app receives stale or edge-case final showcase payloads, it can still promote a validation-only print readiness action and hide the real Treatstock quote handoff from the compact mobile summary.

## Decision

Mirror the backend semantic boundary in the mobile summary layer:

- Prioritize real print quote handoff actions that mention `Treatstock`, `print-quote`, `print_quote`, or `/v1/print-quotes`.
- Do not prioritize generic `print-fulfillment-readiness` validation actions unless they also contain the real quote handoff markers.
- Keep existing live provider, resource, final showcase, and iOS handoff prioritization unchanged.

## Scope

In scope:

- Add a Swift contract regression to prove a validation-only print readiness action is hidden while the Treatstock handoff remains visible.
- Update `FinalLaunchMobileSummary.showcaseReadinessActionRows`.
- Add a P0.365 roadmap entry.

Out of scope:

- Changing backend report generation.
- Changing print fulfillment readiness behavior.
- Running provider calls, placing Treatstock orders, changing Xcode signing, or touching global state.

## Testing

Use TDD:

1. Add the validation-only action to the existing mobile summary priority test and confirm RED.
2. Narrow mobile print action prioritization.
3. Run the Swift contract test, backend focused tests that own the source reports, lint, and diff checks.

## Safety

This phase only edits tracked source, tests, and docs. It does not write secrets, run live provider calls, place print orders, mutate global settings, touch Keychain, or perform signing.
