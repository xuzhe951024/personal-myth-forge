# P0.67 ARKit Scan Generation Acceptance Design

## Context

The mobile demo now has iOS-side ARKit/Object Capture scan package code and
backend API tests that prove `arkit_scan` uploads can include a scan asset plus
reference images. Final acceptance still has a gap: the default end-to-end
report proves guided-scan images reach the 3D provider, but it does not prove
that an ARKit scan package is carried through backend capture storage,
source-building, myth-session generation, and provider request construction.

That matters for the iOS-first product direction because Object Capture is the
most native path from real-world object scanning to game-world artifact
generation.

## Goal

Add a deterministic final-acceptance gate that proves an ARKit/Object Capture
scan package can become a backend myth session and a 3D provider generation
request without leaking raw media, local paths, or secrets.

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, or any live provider by default.
- Do not perform real iPhone capture, Xcode signing, or Apple SDK license work.
- Do not expose raw image bytes, data URIs, capture URIs, local paths, or API
  keys in reports.
- Do not replace the existing FastAPI upload API tests; this gate verifies final
  acceptance coverage.

## Selected Approach

Create a new in-process backend runner:

- `run_arkit_scan_generation_acceptance(...)`
- synthetic local `arkit_scan` capture with one GLB scan asset and two
  reference images,
- same `LocalCaptureStore` and `build_capture_generation_sources(...)` path used
  by capture-backed myth-session creation,
- recording wrapper around `LocalThreeDProvider`,
- normal `create_demo_myth_session(...)` pipeline,
- sanitized report with counts, content types, provenance, and safety flags.

This is deliberately narrower than a FastAPI multipart flow because API tests
already cover the upload endpoint. The acceptance runner proves the final demo
claim that Object Capture packages can feed 3D generation while staying fast,
offline, and safe for every local run.

## Report Shape

The successful report includes:

- `kind: "arkit_scan_generation_acceptance_report"`
- `status: "succeeded"`
- `summary: {"passed": 6, "failed": 0}`
- `capture.capture_mode: "arkit_scan"`
- `capture.reference_image_count: 2`
- `capture.scan_asset_count: 1`
- `generation_request.source_image_count: 2`
- `generation_request.source_asset_count: 1`
- `generation_request.source_asset_content_types: ["model/gltf-binary"]`
- `generation_provenance.input_mode: "multi_image"`
- `generation_provenance.source_asset_count: 1`
- `generation_provenance.raw_sources_included: false`
- `safety.raw_media_in_report: false`
- `safety.local_paths_in_report: false`
- `safety.provider_secrets_in_report: false`

Failure reports use the same `kind`, set `status: "failed"`, include an error
message, and pass through the same sanitizer.

## Final Acceptance Integration

Add `arkit_scan_generation_acceptance` as an in-process final acceptance check
after `capture_3d_acceptance` and before `print_quote_acceptance`. It runs in
both quick and full profiles. In local mode it should pass without provider
keys. In configured mode it still remains local/offline unless a future live
ARKit-to-Meshy gate is added behind explicit consent.

Expected quick local summary after P0.67 on this machine:

- `passed: 12`
- `blocked: 2`
- `failed: 0`
- `skipped: 0`

The two known blockers remain local iOS deploy configuration and Apple SDK
license acceptance.

## Testing

Use TDD:

1. Add failing tests for the standalone runner proving ARKit scan source assets,
   reference images, provider request counts, generation provenance, and report
   sanitization.
2. Add failing tests for `run_final_acceptance(...)` proving the new check is
   wired into the quick/full check order and changes summary counts.
3. Implement minimal runner and final acceptance wiring.
4. Update documentation and visual evidence.
5. Verify with backend lint/test, SwiftPM project checks, SwiftPM core contract
   tests, quick final acceptance, and screenshot inspection.

## Acceptance Criteria

- A standalone `arkit_scan_generation_acceptance` runner succeeds locally.
- The runner proves one scan asset and two reference images reach the 3D
  generation request.
- Generated asset provenance records `multi_image` input and
  `source_asset_count: 1`.
- Final acceptance includes `arkit_scan_generation_acceptance` after
  `capture_3d_acceptance`.
- Reports remain sanitized and contain no raw data URI, `local-capture://`, temp
  paths, repository paths, provider keys, or bearer tokens.
- Existing backend and mobile contract checks remain passing.
