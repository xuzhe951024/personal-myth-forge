# P0.344 Resource Handoff Preview Before Apply Design

## Context

`resource-handoff.json` is the direct evidence report for provider/key handoff.
When provider selection is incomplete, it currently exposes the writer command
as a top-level operator action:

```text
make final-apply-resources
```

The writer is useful after review because it writes ignored local backend and
iOS config files, but it should not be the first visible handoff action while
the final resource preview has not been reviewed.

## Goal

When resource handoff is blocked by missing provider selections or keys, show
the read-only preview command in `operator_actions`:

```text
make final-resource-apply-preview
```

Keep the writer command documented in `commands`.

## Non-Goals

- Do not change writer scripts or make targets.
- Do not remove `make final-apply-resources` from `commands`.
- Do not write backend `.env`, iOS deploy config, Keychain state, signing
  state, provider state, or print order state.
- Do not call live Meshy, OpenAI, Treatstock, or Xcode signing commands.

## Design

Add a module constant for the preview target in `resource_handoff.py`.

When `_operator_actions()` detects missing provider selection, append:

```text
make final-resource-apply-preview
```

instead of `make final-apply-resources`.

Existing validation helpers still add explicit `rerun` commands for specific
resource inputs such as `MESHY_API_KEY`, `OPENAI_API_KEY`, and iOS deploy
config fields.

## Acceptance Criteria

- Missing-resource `resource-handoff.json.operator_actions` includes
  `make final-resource-apply-preview`.
- The same top-level list does not include direct `make final-apply-resources`
  while provider selection is incomplete.
- `commands` still includes `make final-apply-resources`.
- Focused resource handoff tests, operator action tests, lint, diff checks, and
  backend tests pass.
