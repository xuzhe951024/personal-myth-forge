# P0.229 Configured Handoff Make Commands Design

## Context

P0.228 added explicit configured provider evaluation targets for the live Meshy/OpenAI evidence path. The remaining configured handoff reports still expose a few raw `uv run python -m myth_forge_api.cli ...` commands for provider handoff and configured final demo launch. Those commands work, but they make the final operator path less consistent because most handoff steps now use `make ...` targets.

## Goal

Normalize configured handoff report commands to Make targets so the final resource/key handoff is easier to follow and less error-prone:

- Provider readiness should surface `make provider-handoff`.
- Configured final launch should surface `make final-demo-launch-configured`.
- Local final launch command slots that are meant for operators should surface `make final-demo-launch-local`.
- Existing safety semantics stay unchanged: reports remain read-only and do not run live providers.

## Scope

This iteration only changes command strings and tests. It does not add new provider behavior, does not run live provider calls, does not write ignored resource files, and does not change iOS signing or Xcode state.

## Architecture

Add a small backend constants module for final handoff Make commands, then reuse those constants across report builders. This keeps command wording consistent between `final-configured-evidence-plan`, `live-provider-evidence`, `final-demo-launch`, `final-handoff-index`, and `ios-device-launch-certificate`.

The iOS Swift contract fixtures should be updated to expect the same Make targets, and project checks should guard the public Make targets so future iterations cannot silently regress the final operator path.

## Testing

Use TDD:

1. Add failing backend tests asserting the affected reports expose Make targets.
2. Add failing Swift contract/project checks for the same target names.
3. Implement the command constants and report wiring.
4. Verify focused backend tests, Swift checks, full backend lint/test, and dry-run Make target output.

## Self-Review

- No placeholders remain.
- The scope is limited to report command normalization.
- The design does not weaken live-provider consent gates.
- All changed commands already have Makefile targets.
