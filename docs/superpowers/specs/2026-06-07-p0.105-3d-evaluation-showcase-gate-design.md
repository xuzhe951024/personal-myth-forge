# P0.105 3D Evaluation Showcase Gate Design

## Goal

Make `three_d_evaluation_readiness` a first-class final demo gate in the iPhone showcase path, not only in the Final Launch status panel.

## Problem

P0.104 added backend and mobile launch support for local 3D provider evaluation readiness. The Final Launch view can now show whether the saved 20-prompt-style evaluation report is missing, ready, or blocked. The higher-level showcase surfaces still skip that signal:

- `FinalShowcaseSummary` appends NPC evaluation, operator handoff, and launch stages, but not 3D evaluation.
- `DemoScript` guides the operator from print quote to NPC evaluation and launch, but not 3D evaluation.
- `ShowcaseAutopilotPlanner` can wait/block on NPC evaluation, but it does not stop on missing or failed 3D evaluation.

That creates a demo risk: an operator can believe the final scripted loop is ready even when the 3D provider quality gate has never been reviewed.

## Scope

In scope:

- Add a `three_d_evaluation` stage to `FinalShowcaseSummary` when `finalLaunchSummary` is available.
- Add a `three_d_evaluation` step to `DemoScript`, ordered before `npc_evaluation`.
- Add `ShowcaseAutopilotPlanner` waiting and blocked handling for the 3D evaluation step.
- Keep details privacy-safe by reusing existing sanitizers.
- Update Swift ProjectChecks and backend iOS showcase acceptance source gates.
- Add visual verification artifacts for the final showcase/demo script gate.

Out of scope:

- Running Meshy, Tripo, or any external 3D provider.
- Changing the backend evaluation report schema.
- Adding a new iOS view; existing stage/step renderers should pick up the new item.
- Changing final acceptance external blockers such as Xcode license or device loopback config.

## Product Behavior

When `FinalLaunchMobileSummary.threeDEvaluationRows` reports ready:

- Final Showcase includes `3D Evaluation` with `ready`.
- Demo Script includes `3D Evaluation` with `complete`.
- Autopilot continues to NPC evaluation and final launch.

When the 3D evaluation is missing/not loaded:

- Final Showcase overall status becomes `waiting`.
- Demo Script next action asks the operator to load 3D evaluation readiness.
- Autopilot returns a non-executable waiting plan with button title `Check 3D Eval`.

When the 3D evaluation is blocked/failed:

- Final Showcase overall status becomes `needsAttention`.
- Demo Script next action asks the operator to review 3D evaluation blockers.
- Autopilot returns a non-executable blocked plan with button title `Check 3D Eval`.

## Data Contract

Use the existing `FinalLaunchMobileSummary` contract:

- `threeDEvaluationRows`
- `npcEvaluationRows`
- `handoffRows`
- `overallStatus`

No new API fields are needed.

## Acceptance

- Swift contract tests cover ready, missing, blocked, ordering, and redaction.
- Swift ProjectChecks require the new stage/step/planner source references and tests.
- Backend `ios_showcase_acceptance` requires the same source references in its fixture.
- Full backend lint/test and Swift contract/project/app compile gates pass.
- Final acceptance quick remains allowed to exit 2 only for known external blockers.
