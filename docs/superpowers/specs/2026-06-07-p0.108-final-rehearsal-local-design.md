# P0.108 Final Rehearsal Local Design

## Goal

Give the operator one project-root local rehearsal command that prepares every saved JSON report needed by Final Launch and the iOS deploy runbook without changing global machine state or calling live providers.

## Context

P0.107 added `make backend-evaluate-local`, which writes the local 3D and NPC Agent evaluation reports. The next remaining local handoff gap is `services/backend/.local/final-acceptance-local.json`: Final Launch reads that canonical file, but the project root has no Make target that writes it safely. Running final acceptance directly often exits `2` on this machine because physical iPhone deploy config and Xcode license are external blockers. That exit code should still be accepted when the purpose is to save the report for read-only launch surfaces.

`make final-demo-launch` also previews the launch report but does not write the default local report path. The iOS runbook target writes its JSON but exits `2` when expected device blockers exist, which makes it awkward to combine in a rehearsal command.

## Scope

Add:

- `services/backend/scripts/write_final_acceptance_local.sh`
- `services/backend/scripts/write_ios_deploy_runbook_local.sh`
- `Makefile` target `final-acceptance-local`
- `Makefile` target `ios-deploy-runbook-local`
- `Makefile` target `final-rehearsal-local`

Update `make final-demo-launch` to write `services/backend/.local/final-demo-launch-local.json`.

`final-rehearsal-local` will run:

1. `backend-evaluate-local`
2. `final-acceptance-local`
3. `final-demo-launch`
4. `ios-deploy-runbook-local`

The two wrapper scripts accept CLI exit codes `0` and `2` because both produce useful saved reports. Any other exit code fails the wrapper. The scripts are project-local and may only write ignored `.local` JSON files.

## Non-Goals

- Do not accept the Xcode license or run any global Xcode/signing configuration.
- Do not start the backend server.
- Do not call Meshy, OpenAI, Treatstock, or Sculpteo.
- Do not change the final acceptance or runbook report schemas.
- Do not make iOS execute Mac-side commands.

## Behavior

Running:

```bash
make final-rehearsal-local
```

should leave these ignored files present:

- `services/backend/.local/3d-evaluation-local.json`
- `services/backend/.local/npc-evaluation-local.json`
- `services/backend/.local/final-acceptance-local.json`
- `services/backend/.local/final-demo-launch-local.json`
- `services/backend/.local/ios-deploy-runbook-local.json`

On the current machine, final acceptance and iOS runbook may describe the known physical-device blockers, but the command should still complete when those reports are successfully written. The operator can then inspect the iPhone-facing report from the API or the saved JSON without reassembling five commands by hand.
