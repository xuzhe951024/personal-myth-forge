# P0.350 Ledger Resource Action Command Design

## Context

`final-external-action-ledger.json` is the final categorized list of external
actions: missing resource inputs, safe local writes, live provider/cost actions,
global Mac/Xcode actions, and iPhone runtime actions.

The resource-input group already emits useful operator actions such as:

```text
provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight
```

However the top-level `first_blocker.command` and `next_action.command` for that
same action currently surface only the validation command:

```text
make final-resources-preflight
```

That is less actionable for final handoff. The next action should say what value
the operator must provide, while `validation_command` should say how to verify
the change afterwards.

## Goal

Make `final-external-action-ledger` expose concrete resource-input commands in
top-level `first_blocker` and `next_action` while keeping validation commands
separate and unchanged.

## Non-Goals

- Do not change the ledger action schema.
- Do not change `operator_actions`; they already include validation suffixes.
- Do not execute provider calls, resource writers, Xcode/signing commands,
  backend `.env` writes, iOS deploy config writes, or print orders.
- Do not change final resource parsing, secret redaction, or destination
  sanitization.

## Design

Add a blocker-command projection layer inside
`final_external_action_ledger._blocker_from_action()`:

- For `resource_inputs` actions that require user input and are `missing` or
  `blocked`, derive the command from the existing user-input action text and
  strip the validation suffix.
- For all other actions, keep the existing `action["command"]` behavior.
- Continue to compute `validation_command` from the original action command, so
  resource blockers still validate with `make final-resources-preflight` or
  `make mobile-deploy-preflight`.

This keeps the full operator action intact while making the top-level next
action directly executable as an instruction.

## Acceptance Criteria

- Missing Meshy ledger `first_blocker.command` is
  `provide MESHY_API_KEY in final-resources.env`.
- The corresponding `next_action.command` matches the first blocker command.
- `validation_command` remains `make final-resources-preflight`.
- Existing operator actions still include
  `provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight`.
- Ledger tests, closure-packet integration tests, lint, diff checks, and refreshed
  local ledger JSON pass.
