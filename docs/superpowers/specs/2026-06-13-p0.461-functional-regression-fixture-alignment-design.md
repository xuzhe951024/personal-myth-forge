# P0.461 Functional Regression Fixture Alignment Design

## Context

`make backend-test` fails on `main` before the P0.460 change is merged. The failures are concentrated in tests whose fixtures still assume older readiness behavior:

- `final-configured-preflight` actions now preserve the `configured-live-evidence-bundle` rerun handoff.
- Visual regression readiness now requires the saved report to cover the current `DEFAULT_VISUAL_ARTIFACTS` inventory.
- Print fulfillment readiness now treats the visual print receipt evidence as a blocking handoff input.

These failures are not caused by P0.460, but they block a clean functional regression gate for the final showcase goal.

## Decision

Align the stale tests and local fixtures with the current behavior instead of weakening production gates:

- Update configured preflight operator-action assertions to expect the normalized `configured-live-evidence-bundle` handoff where actions are normalized.
- Update visual regression fixtures to use the current default artifact inventory rather than a single legacy artifact.
- Update print fulfillment fixtures so the visual print receipt gate has current visual evidence when the test is about Treatstock quote behavior.

## Acceptance Criteria

- The 12 focused failures reproduced from `main` pass.
- P0.460 focused tests still pass.
- `make backend-test`, `make backend-lint`, and `git diff --check` pass on the development branch.

## Safety

This phase only changes tests and documentation. It does not run external providers, write global config, touch Xcode signing, write secrets, or place print orders.
