# P0.625 Handoff Index Granular Live Consent Spec

## Context

Most final handoff reports now point to the concrete configured 3D live provider consent action:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-3d-configured; rerun make final-configured-evidence-plan
```

`final-handoff-index` still exposes the older broad live acceptance action:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make final-acceptance-configured; rerun make live-provider-evidence
```

This makes the handoff index less useful as an execution index because it does not point to the current first configured evidence blocker.

## Requirements

- Prefer the saved final demo launch granular configured live consent action when available.
- Keep live provider actions out of the local rehearsal lane.
- Fall back to the broad configured final acceptance action when no granular saved action is available.
- Do not execute live providers, write secrets, or mutate global developer state.

## Acceptance

- A focused build-level test proves the handoff index uses the granular saved consent action and not the broad fallback.
- Existing final handoff index tests still pass.
- Safe `make final-handoff-index` refresh writes the granular action into `.local/final-handoff-index.json`.
