# P0.594 Showcase Granular Live Action

## Context

The final configured evidence plan now exposes guarded live-provider commands for the
next missing configured evidence file. The final showcase readiness report still
points live Meshy/OpenAI blockers at the coarser configured final acceptance command.

That leaves the operator with a broader command than the actual first configured
evidence blocker. After provider keys are supplied, the handoff should start at the
specific missing evidence step.

## Decision

`final-showcase-readiness` should read the saved
`services/backend/.local/final-configured-evidence-plan.json` report and prefer its
`next_action` when live provider evidence is not ready.

The preferred command keeps the existing consent gate:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-3d-configured; rerun make final-configured-evidence-plan
```

The report builder remains read-only:

- no live provider calls
- no backend env writes
- no iOS deploy config writes
- no global Xcode or signing changes

## Acceptance

- Game asset 3D, AI Agent NPC, and provider/key handoff rows prefer the configured
  evidence plan's guarded next action over the coarser live provider evidence action.
- The final showcase evidence section summarizes the configured evidence plan.
- Operator actions retain the granular configured evidence action and prune the
  coarser live-provider acceptance fallback when both are present.
