# P0.512 Final Demo Launch Backend URL Priority Design

## Context

`final-showcase-readiness` now surfaces the operator path for the iOS showcase as:

1. write the ignored iOS deploy config with `DEVELOPMENT_TEAM`
2. start `backend-device-demo`
3. set `PMF_BACKEND_BASE_URL` to an iPhone-reachable LAN URL

Fresh `make final-demo-launch-local` output still skips the backend URL action and jumps from `backend-device-demo` to provider and print handoffs. That weakens the final iPhone handoff because the local demo launch report is one of the most visible operator surfaces.

## Decision

Update final demo launch operator action prioritization so local mode keeps the iPhone device chain together:

1. concrete next action, usually `DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`
2. `start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight`
3. `set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL; rerun make mobile-deploy-preflight`
4. provider and print handoff actions
5. lower-priority follow-up actions within the existing limit

The sorter should only promote the LAN backend URL action when it is present. It must preserve existing behavior that prefers the LAN action over the generic `provide PMF_BACKEND_BASE_URL in Deployment.local.xcconfig` action.

The embedded iOS device launch certificate must still keep deploy/config handoff actions before backend URL actions when it consumes final demo launch child actions. The final demo launch report can move backend URL after backend-device-demo, but the certificate must not make backend URL the first action when Team ID or bundle identifier configuration is still missing.

## Scope

In scope:

- Add a final demo launch regression test for backend URL priority after backend-device-demo.
- Update final demo launch operator action sorting.
- Add a certificate child-action regression so deploy/config actions stay ahead of backend URL actions.
- Refresh local evidence after merge.

Out of scope:

- Real Xcode signing.
- Real iPhone device checks.
- Meshy/OpenAI/Treatstock calls or cost-incurring provider actions.
- Changing downstream reports unless they naturally inherit the refreshed final demo launch output.

## Validation

- Focused RED test must fail before implementation.
- Focused test must pass after implementation.

- `tests/test_ios_device_launch_certificate.py` must pass.
- `make backend-lint` and `make backend-test` must pass before merge.
- `make final-demo-launch-local` must show deploy writer, backend-device-demo, then backend URL as the first three operator actions.
