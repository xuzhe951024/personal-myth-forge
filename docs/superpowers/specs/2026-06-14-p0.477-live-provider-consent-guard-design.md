# P0.477 Live Provider Consent Guard Design

## Context

Configured evidence reports already mark Meshy/OpenAI live steps with `requires_live_provider_consent`, `may_call_live_provider`, and `cost_risk`. The remaining gap is execution-level protection: `backend-evaluate-3d-configured`, `backend-evaluate-npc-configured`, and `final-acceptance-configured` can be invoked directly from Make. If real keys are present, those targets may spend provider credits without an explicit local consent flag.

## Decision

Add a small POSIX shell guard that blocks live provider Make targets unless `PMF_ALLOW_LIVE_PROVIDER_CALLS=1` is present in the command environment. The guard prints a concrete consent command and exits before any provider CLI command runs.

Protected targets:

- `make backend-evaluate-3d-configured`
- `make backend-evaluate-npc-configured`
- `make backend-evaluate-configured`
- `make final-acceptance-configured`

The existing report-layer consent flag remains `--allow-live-provider-calls`; the Make-layer flag is `PMF_ALLOW_LIVE_PROVIDER_CALLS=1`, matching the mobile contract language already present in the app.

## Scope

This phase does not run Meshy, OpenAI, Treatstock, Xcode signing, Keychain, or global commands. It only adds a local execution guard and source-gate coverage. It does not change provider adapters or configured evidence report semantics.

## Files

- `services/backend/scripts/require_live_provider_consent.sh`: new guard.
- `services/backend/scripts/write_final_acceptance_configured.sh`: guard before configured final acceptance.
- `Makefile`: guard configured evaluation targets before CLI calls.
- `services/backend/tests/test_final_rehearsal_scripts.py`: RED/GREEN tests for the guard and configured acceptance wrapper.
- `services/backend/tests/test_evaluation_make_targets.py`: dry-run coverage for guarded configured evaluation targets.
- `services/backend/src/myth_forge_api/ios_showcase_acceptance.py`: source gate requires execution-level consent guard.
- `services/backend/tests/test_ios_showcase_acceptance.py`: source-gate assertions and fixture updates.
- `apps/mobile/ios/Sources/PersonalMythForgeMobileProjectChecks/main.swift`: Swift static check requires the guard.
- `docs/iteration-roadmap.md`: record P0.477.

## Acceptance

- Without `PMF_ALLOW_LIVE_PROVIDER_CALLS=1`, guarded targets fail before the fake provider command runs.
- With `PMF_ALLOW_LIVE_PROVIDER_CALLS=1`, guarded targets proceed to the existing command path.
- Focused pytest and Swift ProjectChecks pass.
- `make backend-evaluate-3d-configured` returns a consent error locally without provider calls.
- Final readiness remains blocked only on real external resources and device evidence.
