# P0.474 ProjectChecks Handoff Wrapper Design

## Context

Several final handoff Make targets now use safe wrapper scripts that accept blocked report exit code `2` while writing ignored local JSON evidence. That is the correct behavior for the final handoff because missing provider keys or device evidence should block readiness without making safe report generation fail.

The Swift `PersonalMythForgeMobileProjectChecks` executable still expects old raw Makefile command text for configured preflight, final handoff index, and iOS device launch certificate. Running it from the repo root fails before the rest of the project source checks can complete.

## Decision

Update ProjectChecks to validate wrapper paths and output evidence instead of old raw CLI Makefile commands for wrapper-backed handoff targets. Keep backend CLI parser/source requirements in backend acceptance because the wrappers still call the CLI internally.

## Scope

- Replace stale ProjectChecks Makefile assertions with checks for:
  - `services/backend/scripts/write_final_configured_preflight.sh`
  - `services/backend/scripts/write_final_handoff_index.sh`
  - `services/backend/scripts/write_ios_device_launch_certificate.sh`
  - wrapper script output paths for `final-resources-preflight` and `resource-handoff`
- Add backend source-gate coverage so configured preflight, final handoff index, and iOS device launch certificate features require ProjectChecks to mention their wrapper scripts.
- Update complete test fixtures to include the ProjectChecks wrapper text.
- Add P0.474 to the roadmap.

## Non-Goals

- Do not change provider handoff semantics.
- Do not run real Meshy, OpenAI, Treatstock, Xcode signing, Keychain, or device actions.
- Do not change the safe wrapper implementation unless tests prove it is wrong.

## Verification

- RED: `swift run --package-path apps/mobile/ios PersonalMythForgeMobileProjectChecks` fails with old CLI expectations.
- RED: new source-gate tests fail until ProjectChecks wrapper text is required by the acceptance manifest.
- GREEN: Swift ProjectChecks runs successfully from repo root.
- GREEN: focused backend acceptance tests and lint pass.
