# P0.511 Final Showcase Backend URL Priority Design

## Goal

Make `final-showcase-readiness` place the iPhone-reachable backend URL handoff immediately after `backend-device-demo`, before provider, print, Xcode, and provider-key handoffs.

## Current Behavior

The top-level final showcase readiness report is still blocked on the physical iPhone path, which is expected. Its current operator order is less precise:

1. write `DEVELOPMENT_TEAM`
2. start `backend-device-demo`
3. provider handoff
4. print handoff
5. Xcode or later device/resource actions
6. set iPhone-reachable `PMF_BACKEND_BASE_URL`

For a real iPhone demo, the backend URL must be supplied before `make mobile-deploy-preflight` can prove device reachability. It should therefore stay directly behind the backend server startup action.

## Desired Behavior

When `operator_actions` begins with the concrete deploy writer handoff, final showcase readiness should order actions as:

1. deploy writer
2. `backend-device-demo`
3. iPhone-reachable backend URL
4. provider and print handoffs
5. remaining Xcode, key, and configured evidence handoffs

When the first action is not the deploy writer handoff, preserve the previous conservative ordering so unrelated blocked states do not promote device-specific actions.

## Scope

In scope:

- Add a focused regression test to `services/backend/tests/test_final_showcase_readiness.py`.
- Update `services/backend/src/myth_forge_api/final_showcase_readiness.py`.
- Add a P0.511 roadmap entry.

Out of scope:

- Real provider calls, print quotes, Xcode signing, Keychain writes, or paid actions.
- Changing the first blocker. `DEVELOPMENT_TEAM` remains the first blocker until the operator supplies local iOS deploy values.
- Changing downstream reports such as final demo launch or local refresh; they can be aligned in later small phases if still stale after this change.

## Verification

4. Refresh `make final-showcase-readiness` on `main` and confirm action 0 is deploy writer, action 1 is `backend-device-demo`, and action 2 is the backend URL handoff.
