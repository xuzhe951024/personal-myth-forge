# P0.503 Final Showcase Backend Demo Action Design

## Context

The final iPhone showcase handoff is currently blocked by local device setup, not by missing repo code. The saved lower-level handoff reports already tell the operator to start the local backend device demo before running physical-device checks:

`start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight`

The top-level `final-showcase-readiness.json` omits this action even though `final-demo-launch-local.json`, `final-local-report-refresh.json`, `final-external-action-ledger.json`, and `final-launch-closure-packet.json` include it. That makes the final readiness report less executable than the child reports and can leave the operator with Team ID/backend URL steps before the local backend service has an explicit launch handoff.

## Scope

This iteration only changes the final showcase readiness operator-action aggregation. It does not start servers, write secrets, modify Xcode signing, call providers, or place print orders.

## Design

`final_showcase_readiness` should treat the backend-device-demo handoff as a device-priority action. When the action appears in child report operator actions, the final showcase top-level `operator_actions` list should preserve exactly one normalized command:

`start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight`

The action should remain behind the concrete first blocker, but ahead of lower-priority provider/key and configured-live follow-up actions when needed to avoid report-limit loss. Existing provider and print handoff priority behavior remains unchanged.

## Testing

Add a regression test that mirrors the current saved evidence shape: local proof ready enough for final showcase aggregation, mobile deploy preflight blocked by `DEVELOPMENT_TEAM` and iPhone-reachable `PMF_BACKEND_BASE_URL`, and iOS launch rehearsal exposing backend-device-demo from child reports. The test must fail before implementation because the top-level final showcase action list omits the backend-device-demo handoff.
