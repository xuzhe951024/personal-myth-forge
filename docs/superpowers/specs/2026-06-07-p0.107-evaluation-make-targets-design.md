# P0.107 Evaluation Make Targets Design

## Goal

Give the final demo operator one project-root command path for producing the saved local 3D and NPC Agent evaluation reports required by Final Launch, Final Showcase, Final Operator Handoff, Demo Script, Autopilot, and the iOS deploy runbook.

## Context

The backend already has safe local CLI commands:

- `evaluate-3d --provider local --suite default-v0 --output .local/3d-evaluation-local.json`
- `evaluate-npc --provider local --suite default-v0 --tick-steps 2 --output .local/npc-evaluation-local.json`

P0.104 through P0.106 made those saved reports visible as readiness gates, but the project root `Makefile` does not expose a one-command operator path. The runbook and handoff still point at long backend CLI commands. That is easy to mistype when switching between local rehearsal and the later real-provider handoff.

## Scope

Add Make targets:

- `backend-evaluate-3d`: writes `services/backend/.local/3d-evaluation-local.json` with the no-key local 20-case 3D suite.
- `backend-evaluate-npc`: writes `services/backend/.local/npc-evaluation-local.json` with the no-key local NPC Agent suite.
- `backend-evaluate-local`: runs both targets in order.

Update the read-only readiness command constants and operator actions to point at those Make targets. The readiness builders still only inspect saved JSON files; they must not run commands or call providers.

Update docs, source gates, tests, and visual evidence so the Make command path remains part of the accepted final iPhone demo workflow.

## Non-Goals

- Do not add live Meshy/OpenAI evaluation Make targets yet.
- Do not make API endpoints or iOS code execute evaluation commands.
- Do not change the saved evaluation report schemas.
- Do not auto-run provider calls during `final-demo-launch`, `ios-deploy-runbook`, or `final-acceptance`.
- Do not resolve Apple signing, Xcode license, or physical device deployment blockers.

## Behavior

When no saved reports exist, 3D and NPC readiness operator actions should say to run the Make targets from the project root. Blocker command fields should use the same Make targets. The iOS deploy runbook should inherit these actions without exposing local absolute paths or secrets.

`make backend-evaluate-local` should run 3D first, then NPC Agent evaluation, because 3D quality is the primary product demo gate and NPC evaluation uses deterministic local session outputs.

## Testing

- Add a backend test that dry-runs the new Make targets and asserts their output paths and CLI flags.
- Update readiness tests to assert Make-target command constants and operator actions.
- Update iOS deploy runbook tests to assert Make-target actions.
- Update iOS showcase source acceptance so `Makefile` must contain all three targets and the readiness modules must reference them.
- Keep existing local backend, Swift contract, project source, compile, final acceptance, and diff checks.
