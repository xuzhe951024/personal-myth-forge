# P0.445 Final Local Provider Chain Design

## Goal

Make `final-local-report-refresh` prefer the complete provider handoff chain in its operator action list.

## Context

P0.444 cleaned `final-showcase-readiness` and `final-demo-launch-local` so they keep:

```text
make final-resource-apply-preview; rerun make provider-handoff; rerun make live-provider-evidence
```

The aggregate `final-local-report-refresh` report still emits the weaker:

```text
make final-resource-apply-preview; rerun make live-provider-evidence
```

That weak action skips the explicit provider handoff validation step when an operator follows the top-level local refresh report.

## Behavior

When `final_local_report_refresh._dedupe_operator_actions()` sees the complete provider chain, it must keep that chain and remove weaker provider duplicates:

- `make final-resource-apply-preview; rerun make live-provider-evidence`
- `make provider-handoff`
- `make provider-handoff; rerun make live-provider-evidence`

The report must still preserve the first concrete resource and iOS deployment actions, print quote handoff actions, and unrelated blocked-step actions.

## Safety

This is read-only report shaping. It must not call Meshy, OpenAI, Treatstock, Xcode, Keychain, or any live provider. It must not write secrets, deploy config, payment artifacts, or order artifacts.

## Tests

Add a focused operator-action unit test that combines the complete provider chain, weaker provider actions, and a print handoff. The expected output keeps the complete provider chain and the print handoff, without weaker provider duplicates.
