# P0.498 Demo Backend URL Action Pruning Design

## Context

`final-demo-launch-local.json` is the iPhone-visible launch report. Its top-level `operator_actions` can currently contain both:

- `provide PMF_BACKEND_BASE_URL in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`
- `set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL; rerun make mobile-deploy-preflight`

For the final physical iPhone demo, the second action is the more precise handoff. The iPhone cannot use `localhost` or `127.0.0.1`; the operator must supply a LAN-reachable backend URL. Keeping both actions makes the checklist noisier and can hide the real acceptance requirement.

## Decision

Filter only final demo launch top-level `operator_actions`.

When both backend URL actions exist, keep the iPhone-reachable LAN URL action and drop the generic `provide PMF_BACKEND_BASE_URL in Deployment.local.xcconfig` action. Preserve the generic action when it is the only backend URL action.

The change must not alter:

- final resource requirements
- ignored config writers
- iOS deploy preflight execution
- provider/key handoff actions
- print quote consent actions
- Xcode/global action gates

## Safety

This is report-only. It does not run Xcode, write signing config, touch Keychain, start servers, write ignored config/env files, call Meshy/OpenAI/Treatstock, or trigger provider/print costs.

## Verification

TDD coverage must prove:

- `_dedupe_operator_actions()` drops generic backend URL config handoff when the LAN URL handoff is present.
- `_dedupe_operator_actions()` preserves the generic backend URL config handoff when it is the only backend URL action.
- Refreshed `final-demo-launch-local.json` no longer emits both backend URL actions together.
- Focused final demo tests, operator action tests, lint, and refreshed local evidence pass.
