# P0.537 Closure Print Auto Action Design

## Context

P0.536 made the configured print quote request writer accept `PRINT_SOURCE_ASSET_URI=auto PRINT_CANDIDATE_URI=auto make print-quote-request-configured`. Current `final-showcase-readiness` and `final-local-report-refresh` evidence now surface that concrete handoff. The `final-launch-closure-packet` still keeps a bare `make print-fulfillment-readiness` action ahead of the concrete print quote request action, which makes the final closure packet less executable than the canonical readiness report.

## Goal

Make the final launch closure packet prefer the concrete auto print quote request handoff over the bare print fulfillment validation command whenever both are present.

## Non-Goals

- Do not call Treatstock or any live print provider.
- Do not create quote files, place orders, or write provider credentials.
- Do not change iOS deploy, Xcode, Keychain, or Apple signing state.
- Do not claim print fulfillment is ready without configured quote request and quote evidence.

## Design

The closure packet already normalizes operator actions through `_prefer_guarded_print_quote_action`. That helper drops `make print-fulfillment-readiness` when it sees a more specific print quote request or guarded quote action. It still recognizes the old `https://...` placeholder request action, so the new `auto` request action does not trigger the same pruning.

This phase updates the closure packet's configured print quote request constant to:

```text
PRINT_SOURCE_ASSET_URI=auto PRINT_CANDIDATE_URI=auto make print-quote-request-configured; rerun make print-fulfillment-readiness
```

The same helper will then remove the bare print readiness action when the auto request action exists. Reports that only have the bare validation command will still keep it.

## Testing

Focused tests will cover:

- The closure packet operator action normalizer drops `make print-fulfillment-readiness` when the auto request handoff is present.
- The existing request-before-provider-quote behavior now uses the auto request handoff.
- Closure packet focused regression still preserves device and backend action ordering.

This phase only changes report text and prioritization; no live provider or device execution is required.
