# P0.630 Handoff Index Live Action Completeness Spec

## Goal

Make `final-handoff-index` preserve both configured live evidence consent actions
when the final demo launch report exposes Meshy 3D and OpenAI NPC evidence steps.

## Problem

Most top-level reports now list the required configured live evidence steps:

- `backend-evaluate-3d-configured`
- `backend-evaluate-npc-configured`

The final handoff index still collapses the live acceptance lane to only the first
granular action. Operators who start from the handoff index can therefore miss the
NPC Agent live evidence step.

## Acceptance

- `final-handoff-index.operator_actions` includes both configured 3D and NPC live
  provider consent actions when the saved final demo launch report contains both.
- The broad configured final acceptance action remains hidden while granular 3D/NPC
  prerequisite actions exist.
- Local rehearsal lane actions still do not copy live provider actions into local
  handoff rows.
- No live providers are called and no secrets or global device/Xcode state are
  written.
