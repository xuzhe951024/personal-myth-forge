# P0.618 Local Refresh Configured Bundle Saved Plan

## Problem

`final-local-report-refresh` writes `final-configured-evidence-plan.json` with the current configured live-provider consent blocker, then refreshes `configured-live-evidence-bundle.json` with explicit `LOCAL_SETTINGS`. That explicit settings argument prevents `configured-live-evidence-bundle` from reusing the saved configured evidence plan, so the bundle can fall back to a weaker local-stub provider handoff blocker.

## Scope

- Let the local report refresh configured bundle step reuse the saved configured evidence plan when live provider calls are not allowed.
- Preserve the no-live-call safety contract: the refresh may read ignored local handoff files and write ignored reports, but must not call Meshy/OpenAI/Treatstock, run Xcode signing, or mutate global state.
- Keep local print fulfillment and explicitly local-only steps on local settings where they need that isolation.

## Acceptance

- RED test proves the default local refresh configured bundle step ignores a saved live-consent configured evidence plan.
- GREEN implementation makes the configured bundle step surface the saved `three_d_evaluation_configured` consent blocker.
- Focused configured-bundle/local-refresh tests, lint, and backend regression gates remain green.
