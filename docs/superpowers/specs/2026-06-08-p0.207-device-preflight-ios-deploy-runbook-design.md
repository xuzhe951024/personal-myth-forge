# P0.207 Device Preflight iOS Deploy Runbook Design

## Purpose

The backend final launch report already includes `ios_deploy_runbook`, and the mobile final launch summary renders runbook rows. `DevicePreflightSummaryBuilder` currently stops before the deploy runbook, so the iPhone preflight can say resource handoff is ready without showing the next iOS deployment checkpoint.

## Scope

- Add an `ios_deploy_runbook` device preflight item labeled `Deploy Runbook`.
- Show waiting when the final launch report or runbook report is missing.
- Show blocked when the runbook status is blocked or partial and has a non-ready command step or operator action.
- Show ready when the runbook status is `ready`.
- Include the item in the required ready set after resource apply preview.
- Reuse the existing device preflight sanitizer; do not execute Xcode, write config, call providers, or mutate global machine state.
- Add Swift contract tests, ProjectChecks gates, and backend mobile readiness source requirements.

## Acceptance

- `DevicePreflightSummaryBuilder` returns an `ios_deploy_runbook` item.
- Blocked reports surface the first blocked command step, command, notes, and first operator action.
- Ready reports mark the item ready and include mode, required slot count, command step count, and safety flags.
- Encoded preflight output redacts provider keys, bearer tokens, local paths, local capture URLs, checkout text, and `api_key` snippets from runbook fields.
- Mobile final launch readiness requires the item, label, source, contract tests, and required-ready set entry.
