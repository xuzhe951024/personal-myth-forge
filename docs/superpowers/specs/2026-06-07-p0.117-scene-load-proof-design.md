# P0.117 Scene Load Proof Design

## Context

P0.116 made final handoff evidence freshness visible on iPhone. The mobile app already downloads a preferred `ios_scene_asset` variant and tries to hand a local DAE/USDZ/SCN/USD file to SceneKit. The remaining demo risk is that the UI can report a cached SceneKit-ready file even if `SCNScene(url:)` cannot parse it on device.

## Goal

Make the iPhone 3D preview report a verifiable SceneKit load proof: waiting, loaded, or failed. The proof should be visible in the app, covered by Swift contract tests, and included in the source-only iOS showcase acceptance gate.

## User-Facing Behavior

- Before a myth session exists, the preview still shows the placeholder artifact.
- While asset preparation is pending, the handoff actions still show "Preparing generated asset."
- When a SceneKit-loadable file is downloaded or selected locally, the app validates the file with a SceneKit loading step before treating the preview as openable.
- If SceneKit parses the file, the preview shows a load-proof row such as `SceneKit load proof: Loaded`.
- If SceneKit cannot parse the file, the preview remains on the fallback proxy scene and shows an attention state with a retry action.
- GLB/GLTF remains a conversion-required state. This iteration does not add GLB-to-USDZ conversion.

## Architecture

Add a small mobile-core proof model that is independent of SwiftUI:

- `ArtifactSceneLoadProof` describes the current proof status, title, detail, and whether the preview may open a parsed scene.
- `ArtifactSceneLoadProofBuilder` maps `PreparedArtifactAsset` plus an optional scene-load error into a sanitized proof.
- `PreparedArtifactAssetStatus` gets one new status for parse failure so handoff actions can distinguish download failure from SceneKit load failure.

The SwiftUI preview owns the actual SceneKit parser because `SCNScene(url:)` is an app-layer concern. `Artifact3DPreviewView` should update the prepared asset to `sceneLoadFailed` if the parser throws, then render the placeholder scene plus the proof row.

## Data Flow

1. Backend returns `generated_asset.variants` with a preferred `ios_scene_asset`.
2. `ArtifactAssetPreparer` downloads or selects the asset and returns a `PreparedArtifactAsset`.
3. `Artifact3DPreviewView` attempts `SCNScene(url:)` only when `preparedAsset.sceneURL` is present.
4. On success, the view stores a loaded proof and renders the parsed scene with the NPC ritual overlay.
5. On failure, the view stores a failed proof, clears `sceneURL` through the prepared-asset status, and renders the fallback proxy scene.
6. `ArtifactHandoffActionBuilder` maps `sceneLoadFailed` to an enabled retry action.

## Error Handling And Privacy

- SceneKit parser errors are reduced to a short sanitized string. Local file paths, `file://` URLs, home-directory paths, API keys, bearer tokens, and checkout URLs must not reach visible summaries or JSON-encoded contract test output.
- Parse failure is separate from download failure. Download failure means the file never reached the cache; parse failure means the cache/local file exists but SceneKit could not load it.
- Retrying reruns the same asset preparation path and SceneKit load attempt.

## Testing

- Add Swift contract tests for:
  - waiting proof before preparation.
  - loaded proof for a cached scene asset.
  - conversion proof for cached GLB/GLTF.
  - parse-failed proof sanitization.
  - handoff actions exposing retry after parse failure.
- Add source acceptance checks requiring the app view and mobile core to reference the load-proof builder/status.
- Run:
  - `swift run --package-path apps/mobile/ios PersonalMythForgeMobileCoreContractTests`
  - `swift run --package-path apps/mobile/ios PersonalMythForgeMobileProjectChecks`
  - `make backend-test`
  - `make backend-lint`

## Non-Goals

- No live Meshy/OpenAI/Treatstock calls.
- No Unity integration.
- No GLB-to-USDZ conversion pipeline.
- No global Xcode configuration, signing, or local machine state changes.
