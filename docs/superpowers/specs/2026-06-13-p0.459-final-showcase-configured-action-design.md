# P0.459 Final Showcase Configured Action Design

## Context

`test_final_showcase_readiness_normalizes_legacy_final_resource_copy_action` fails on `main`. The final showcase operator action list includes `make final-handoff-index`, but drops the configured preflight action sourced from the iOS device launch rehearsal report.

The final iPhone handoff needs the configured evidence lane to remain visible so the operator can move from local rehearsal evidence to configured provider/resource evidence without guessing which command comes next.

## Root Cause

`_selected_report_operator_actions()` treats iOS rehearsal reports specially. It takes the first few actions and then keeps priority commands such as `make final-configured-preflight`.

However `_report_operator_actions()` normalizes prefixed make actions before selection. `final_handoff_index: make final-configured-preflight` becomes `make final-configured-preflight; rerun make configured-live-evidence-bundle`, so the exact priority-target check no longer matches and the action is dropped.

The existing exact assertion for bare `make final-configured-preflight` is also stale. The current command contract keeps the configured-live-evidence validation suffix.

## Decision

Update the iOS rehearsal priority predicate to compare the command root before `; rerun ...`.

Acceptance behavior:

- `make final-configured-preflight; rerun make configured-live-evidence-bundle` survives the iOS rehearsal report action limit.
- `make final-handoff-index` remains present.
- legacy `final-resources.env.example` copy text remains redacted/normalized to `run make final-resource-init`.
- the top-level final showcase action list keeps the validation-aware configured preflight command, not the stale bare assertion.

## Safety

The change is local to report action selection. It does not run providers, write `.env`, mutate Xcode signing, touch Keychain, or make network calls.
