# P0.539 Normalize Legacy Print Request Auto Design

## Context

P0.536 through P0.538 moved current print quote request handoff actions to:

```text
PRINT_SOURCE_ASSET_URI=auto PRINT_CANDIDATE_URI=auto make print-quote-request-configured
```

The shared `operator_actions` module still recognizes older saved reports that contain the legacy placeholder form:

```text
PRINT_SOURCE_ASSET_URI=https://... PRINT_CANDIDATE_URI=https://... make print-quote-request-configured
```

Recognition is enough when both legacy and auto actions appear together, but a saved report with only the legacy placeholder can still pass that outdated command through final handoff lists.

## Goal

Normalize legacy configured print quote request placeholder actions to the current `auto` request handoff, including validated and source-prefixed variants.

## Non-Goals

- Do not remove legacy placeholder recognition.
- Do not write configured quote request files.
- Do not call Treatstock or any live provider.
- Do not change readiness status without real configured quote evidence.

## Design

Add a small normalization step in `myth_forge_api.operator_actions.normalize_operator_action` before the general legacy/prose/make-target normalizers. The step will:

- Convert the bare legacy placeholder request action to the canonical auto request action.
- Convert the validated legacy action ending in `; rerun make print-fulfillment-readiness` to the validated canonical auto action.
- Preserve source prefixes such as `final_demo_launch_local: ` while replacing only the command portion.
- Preserve detail suffixes after ` | ` because `normalize_operator_action` already splits those before command normalization.

The existing request preference helper will continue to use the canonical auto action as the preferred request, while legacy saved input becomes canonical earlier in the pipeline.

## Testing

Focused tests will cover:

- Bare legacy placeholder request normalizes to the auto request.
- Validated legacy placeholder request normalizes to the auto request plus validation rerun.
- Source-prefixed validated legacy request keeps the prefix and normalizes the command.
- The existing request-before-provider behavior still passes.

This phase is pure string normalization. It does not execute provider calls, device commands, global writes, or Xcode steps.
