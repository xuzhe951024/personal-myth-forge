# P0.631 Handoff Index Action Command Spec

## Goal

Ensure `final-handoff-index.operator_actions` contains executable commands instead
of plain blocker detail text.

## Problem

When local rehearsal is blocked only by a live provider evidence detail, the handoff
index can promote the lane `detail` string as the first operator action. That yields
an entry such as `Local 3D proof is ready; live Meshy evidence still needs consent.`
which is useful context but not an actionable command.

## Acceptance

- Local rehearsal fallback operator actions use `make final-rehearsal-local` when no
  concrete nested operator action exists.
- Pure blocker detail text is not promoted into `operator_actions`.
- Configured 3D and NPC live evidence actions remain present in the final handoff
  index.
- No live providers are called and no secrets or global/Xcode state are written.
