# P0.270 iOS Deploy Runbook Resource Actions Design

## Context

The iOS device launch rehearsal now preserves validation-aware final resource
actions, but its upstream `ios-deploy-runbook-local.json` still emits bare
resource handoff commands. The final showcase readiness report also had a local
copy of the final-resource action whitelist, so new iOS resource actions could
still surface without an immediate validation command.

## Goal

The iOS deploy runbook and final showcase readiness report should add the
validation command to every known `final-resources.env` handoff action while
preserving existing action order, selection limits, and status behavior.

## Non-Goals

- Do not write or apply local config.
- Do not call live providers, Xcode, Keychain, or signing tools.
- Do not change deploy readiness status, action prioritization, or command
  sequencing.

## Acceptance

- RED/GREEN tests prove missing final resources in the iOS deploy runbook and
  selected final-showcase nested actions show
  `; rerun make final-resources-preflight`.
- Existing deploy runbook, iOS rehearsal, and final showcase readiness tests
  still pass.
- Refreshed local reports show no bare final-resource actions in
  `ios-deploy-runbook-local.json`, `ios-device-launch-rehearsal.json`, or
  `final-showcase-readiness.json`.
