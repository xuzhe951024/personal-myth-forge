# P0.293 Mobile Deploy Action Normalization Design

## Goal

Normalize the remaining `run make mobile-deploy-preflight after backend is running`
operator action into the existing concrete iPhone device handoff:

```text
start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight
```

This keeps final handoff reports executable without starting Xcode signing, device
deployment, real provider calls, or paid print actions.

## Current Context

`final-showcase-readiness` is still blocked by iOS deploy evidence because the
operator must provide `DEVELOPMENT_TEAM` and an iPhone-reachable backend URL.
Most recent iterations normalized resource, provider, Xcode, and final handoff
actions, but saved or nested iOS device reports can still surface the older
mobile preflight action string.

The shared `myth_forge_api.operator_actions.normalize_operator_action()` helper
already owns legacy action rewrites and is used by the reports that aggregate
saved child report actions. P0.293 should extend that central helper rather than
add report-specific string patches.

## Scope

- Add one normalization rule for the residual mobile deploy preflight wording.
- Preserve report/source prefixes such as `final_handoff_index:` or
  `ios_device_launch_certificate:`.
- Preserve detail suffixes after ` | `.
- Update backend tests that currently expect the old wording.
- Update the mobile contract fixture only if the changed report payload reaches
  iOS-facing decoded models.

## Out of Scope

- No Xcode signing or global Xcode state changes.
- No real iPhone launch, provider calls, Meshy/OpenAI/Treatstock spending, or
  real print order placement.
- No broader operator-action wording rewrite beyond this one residual action.

## Verification

- RED/GREEN focused tests for `normalize_operator_action()`.
- Focused backend tests for final handoff, iOS device certificate/evidence, and
  final showcase readiness.
- Mobile Swift contract tests if fixture text changes.
- `make final-local-report-refresh` may remain blocked by external resources,
  but generated reports must not contain the residual old action string.
