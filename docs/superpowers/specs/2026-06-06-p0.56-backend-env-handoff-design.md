# P0.56 Backend Env Handoff Design

## Product Intent

P0.55 added a safe writer for the ignored iOS deployment config. The backend has
the same handoff need for final provider resources: the repo already has
`.env.example`, provider readiness reports, and `resource-handoff`, but the
operator still has to manually edit `services/backend/.env`.

P0.56 adds a project-local backend env writer so final Meshy/OpenAI keys can be
supplied explicitly and stored only in the ignored backend `.env` file. This
makes the final API/key handoff more repeatable without calling Meshy, OpenAI,
print providers, Xcode, keychain, Apple services, or any global machine state.

## Behavior

Add `services/backend/scripts/write_backend_env.sh`.

The script writes `services/backend/.env` with the backend runtime slots needed
for configured provider acceptance:

- `MYTH_FORGE_ENV=local`
- `MYTH_FORGE_API_HOST=127.0.0.1`
- `MYTH_FORGE_API_PORT=8080`
- `THREE_D_PROVIDER=meshy`
- `MESHY_API_BASE_URL=https://api.meshy.ai`
- `MESHY_POLL_INTERVAL_SECONDS=5`
- `MESHY_MAX_WAIT_SECONDS=600`
- `MESHY_API_KEY=<provided>`
- `NPC_PROVIDER=openai`
- `OPENAI_NPC_MODEL=gpt-4.1-mini`
- `OPENAI_API_BASE_URL=<optional>`
- `OPENAI_API_KEY=<provided>`
- `PRINT_PROVIDER=local`
- `TREATSTOCK_API_KEY=<optional>`
- `SCULPTEO_API_KEY=<optional>`
- `CAPTURE_STORAGE_DIR=<optional>`
- `MYTH_SESSION_STORAGE_DIR=<optional>`

Inputs come from environment variables by default:

```sh
MESHY_API_KEY=... \
OPENAI_API_KEY=... \
services/backend/scripts/write_backend_env.sh
```

Equivalent flags are supported:

```sh
services/backend/scripts/write_backend_env.sh \
  --meshy-key ... \
  --openai-key ...
```

The script fails without writing when:

- `MESHY_API_KEY` is missing.
- `OPENAI_API_KEY` is missing.
- a value contains a line break.
- `services/backend/.env` is tracked by git.
- an unknown flag is passed.

On success it writes through a temporary file and atomic move, then sets file
mode `600` when `chmod` is available. Output is concise and redacted:

```text
services/backend/.env written.
THREE_D_PROVIDER: meshy
MESHY_API_KEY: configured (redacted)
NPC_PROVIDER: openai
OPENAI_API_KEY: configured (redacted)
PRINT_PROVIDER: local
Next: cd services/backend && uv run python -m myth_forge_api.cli provider-handoff --require-core-real
```

The script must never print secret values to stdout/stderr.

## Make Target

Add:

```sh
make backend-write-provider-env
```

The target invokes the script. Users can supply values through env vars:

```sh
MESHY_API_KEY=... \
OPENAI_API_KEY=... \
make backend-write-provider-env
```

This target writes local backend config only. It does not validate provider
network access or spend provider credits; existing provider readiness,
`provider-handoff`, and final acceptance commands remain the validation path.

## Acceptance

Add pytest coverage that runs the real script inside a temporary git repo:

- missing Meshy key exits `2`, writes no `.env`, and does not print secret-like
  values.
- missing OpenAI key exits `2`, writes no `.env`, and does not print secret-like
  values.
- unknown flag exits `2`, writes no `.env`.
- tracked `services/backend/.env` exits `1` and refuses to overwrite it.
- env input writes the expected provider selections and keys.
- flag input overrides env input.
- stdout/stderr redact provided key values.
- a written `.env` can be parsed by `load_settings` and yields Meshy/OpenAI
  configured settings.

Extend resource template acceptance so it recognizes the writer script and Make
target as the active backend-local resource handoff lane. Extend Swift project
source checks or backend tests to prove the script source contains required
keys, redaction output, tracked-file protection, and no banned global/provider
commands.

## Docs and Evidence

Update README, mobile README where relevant, and roadmap with the new two-step
backend resource handoff:

1. `make backend-write-provider-env`
2. provider readiness or final configured acceptance command

Create static visual evidence showing:

- user-owned Meshy/OpenAI keys enter the writer.
- ignored `services/backend/.env` is produced.
- logs redact secrets.
- provider validation remains explicit and opt-in.
- no global machine or live provider calls run during writing.

## Non-Goals

- No live Meshy/OpenAI/Treatstock/Sculpteo calls.
- No provider key creation or retrieval.
- No iOS config writing; P0.55 owns that lane.
- No Xcode, signing, keychain, Apple developer portal, or global shell profile
  mutation.
- No committed `.env` or secret values.
