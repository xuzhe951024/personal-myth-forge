# P0.540 Provider Handoff Fill Guide Design

## Goal

Make the provider/key handoff more executable when real Meshy/OpenAI providers are not configured. `provider-handoff` should point the operator at the project-local ignored final resource fill guide before asking them to preview or validate provider settings.

## Current Problem

`provider-handoff` already reports that backend provider secrets stay out of the repo and that missing values belong in `services/backend/.local/final-resources.env`. Its `operator_actions` still include bare prose such as `provide MESHY_API_KEY in final-resources.env; rerun make final-resources-preflight`.

That prose is safe, but it is less actionable than the existing repo-local handoff path:

- `make final-resource-fill-guide` writes ignored JSON/Markdown guidance.
- `make final-resource-apply-preview` previews the safe local apply step.
- `make provider-handoff` validates whether the configured real providers are ready.

## Design

Update only `services/backend/src/myth_forge_api/provider_handoff.py`.

When `provider-handoff` is blocked by missing real provider resources, `operator_actions` should:

1. Start with `make final-resource-fill-guide; rerun make final-resource-apply-preview`.
2. Preserve validation-aware missing-resource actions for each required backend-only value.
3. Keep `make provider-handoff` as the final validation command.

`first_blocker` and `next_action` remain unchanged for this phase. They still identify the first missing value or local provider stub so downstream reports keep their existing blocker semantics.

## Safety

This phase does not write secrets, call Meshy/OpenAI/Treatstock, modify Xcode signing, write Keychain values, or mutate global state. It only changes generated report action strings and tests.

## Testing

Use TDD in `services/backend/tests/test_provider_handoff.py`.

Add a failing test that asserts a blocked default provider handoff report begins with:

```text
make final-resource-fill-guide; rerun make final-resource-apply-preview
```

Then update the implementation and run the focused provider handoff tests. Before merging, run lint and `git diff --check`.
