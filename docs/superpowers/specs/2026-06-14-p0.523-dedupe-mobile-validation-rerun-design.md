# P0.523 Dedupe Mobile Validation Rerun Design

## Context

After P0.522, iOS deploy runbook device actions include the mobile deploy preflight rerun. Some operator action inputs already carried that rerun and then passed through validation helpers again, producing strings such as:

```bash
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight; rerun make mobile-deploy-preflight
```

That duplicated instruction is safe but confusing in final launch handoff checklists.

## Goal

Normalize mobile deploy validation action strings so each command contains at most one `; rerun make mobile-deploy-preflight` segment.

## Scope

- Update the shared `add_mobile_deploy_validation_command()` helper so duplicated mobile preflight reruns collapse to one.
- Preserve detail suffixes after ` | `.
- Preserve other validation chains, such as provider or print validation reruns.
- Verify iOS deploy runbook operator actions and final showcase operator actions no longer surface duplicated mobile preflight reruns.

## Non-Goals

- Do not change real device, Xcode, signing, provider, or print behavior.
- Do not execute external commands other than local report generation/tests.
- Do not remove distinct validation steps such as `rerun make provider-handoff` or `rerun make live-provider-evidence`.

## Acceptance Criteria

- `add_mobile_deploy_validation_command()` collapses duplicated mobile deploy reruns.
- iOS deploy runbook operator actions do not include duplicated mobile deploy reruns.
- Refreshed final showcase readiness operator actions do not include duplicated mobile deploy reruns.
- Backend lint and tests pass before merge.
