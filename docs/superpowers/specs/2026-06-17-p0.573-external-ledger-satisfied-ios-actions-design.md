# P0.573 External Ledger Satisfied iOS Actions Design

## Context

After refreshing `make ios-device-launch-rehearsal`, the local device evidence
can show `ios_deployable` as ready. The final external action ledger still
lists Apple SDK license and Xcode signing/build actions as manual blockers
because `global_machine_actions` are unconditional.

That makes the final handoff less accurate: the operator has already produced
repo-local evidence that the iOS deploy runbook is ready, so the ledger should
not keep asking for the same Apple/Xcode global actions as first blockers.

## Design

`final_external_action_ledger` should pass the current iOS deploy runbook report
into the global machine action builder.

When the iOS deploy runbook status is `ready`, the global machine actions become
ready evidence rows:

- `accept_apple_sdk_license`
- `configure_apple_signing`
- `run_xcode_build_gate`

Ready global rows keep their labels, commands, and safety classification, but
they no longer require user confirmation and no longer emit operator actions.
The physical iPhone runtime step remains represented by device runtime actions,
so this does not claim that a live iPhone has been opened unless device evidence
already proves it.

When the iOS deploy runbook is missing, blocked, manual, or partial, behavior is
unchanged.

## Safety

This phase only changes report interpretation. It does not run Xcode, accept
licenses, write Keychain items, change signing settings, call providers, or
place print orders.

## Testing

Add a regression test that stubs a ready iOS deploy runbook while resources are
configured. The final external action ledger should mark `global_machine_actions`
ready, remove Xcode/Apple manual operator actions, and move the first blocker to
the remaining live provider evidence handoff.
