# P0.592 iOS Rehearsal Runbook Convergence

## Context

`make ios-device-launch-rehearsal` regenerates the final local rehearsal, then builds the handoff index, certificate, rehearsal, readiness, and final demo launch reports.

The local rehearsal target refreshes `final-demo-launch-local.json` after `ios-deploy-runbook-local.json`. That can leave the runbook evidence one generation behind the final launch report. The next handoff index then sees stale device-loop blockers and marks `final_handoff_index` blocked, even when a manual rerun of `make ios-deploy-runbook-local` followed by `make final-handoff-index` converges to ready.

## Decision

After `final-rehearsal-local` completes, `write_ios_device_launch_rehearsal.sh` must refresh `ios-deploy-runbook-local` once more before generating the configured preflight, final handoff index, certificate, and rehearsal reports.

This keeps the evidence chain safe and local:

- no provider calls
- no key writes
- no Xcode signing
- no long-running backend server start
- writes only ignored `.local` JSON reports

## Acceptance

- The script test proves `write_ios_device_launch_rehearsal.sh` runs `ios-deploy-runbook-local` after `final-rehearsal-local`.
- The script test proves the final launch sync still happens after rehearsal readiness.
- Running `make ios-device-launch-rehearsal` should no longer leave `final_handoff_index` as the first rehearsal readiness blocker when the runbook can converge locally.
