# P0.412 Final Local Device Bundle Design

## Goal

Expose a top-level `device_action_bundle` in `final-local-report-refresh` so the safe refresh report carries the current iPhone handoff actions without requiring consumers to inspect nested demo or showcase reports.

## Current Context

`final-showcase-readiness` and `final-demo-launch-local` already emit safe device action bundles. `final-local-report-refresh` runs both reports and writes them into `.local`, but its own top-level contract only exposes `first_blocker`, `next_action`, `showcase_next_action`, `steps`, and `operator_actions`. This leaves the post-refresh summary less useful for mobile handoff automation and checklist rendering.

## Design

`run_final_local_report_refresh()` will derive `device_action_bundle` from the already-built step reports. It will prefer `final_demo_launch_local.device_action_bundle`, then fall back to `final_showcase_readiness.device_action_bundle`, then `ios_device_evidence_bundle.device_action_bundle`, because those are the most direct device handoff sources already refreshed by this command.

The top-level bundle will use:

- `id`: `final_local_report_refresh_device_actions`
- `label`: `Final Local Report Refresh Device Actions`
- `source_report`: the step id that supplied the bundle
- `status`: normalized source status
- `actions`: sanitized action copies with only report-safe fields
- `first_action`: copied from the source bundle when present, otherwise the first non-ready action
- `summary`: counts for actions, ready, missing, blocked, manual, provider calls, global actions, and Xcode/signing actions
- `safety`: read-only flags showing no commands, live providers, signing, keychain, backend env writes, or iOS deploy config writes were performed

If no source bundle exists, the report will still expose a blocked empty bundle with `source_report: "missing"` and `summary.missing: 1`.

## Safety

The change is metadata-only. It must not run Xcode, signing, Keychain, live provider calls, provider spend, print ordering, or resource-apply writers. It reuses the existing report sanitizer so local paths and secrets remain redacted.

## Testing

Add focused tests in `services/backend/tests/test_final_local_report_refresh.py` proving:

- the blocked safe refresh report exposes `device_action_bundle`
- the bundle source is `final_demo_launch_local`
- the first action promotes the saved deploy preflight child command
- summary counts include one Xcode/signing action and zero provider calls/global actions
- the serialized report does not leak temp paths or secrets

Run the focused file, refresh the real local report, then run lint and backend tests.
