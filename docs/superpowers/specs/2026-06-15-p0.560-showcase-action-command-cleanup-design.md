# P0.560 Showcase Action Command Cleanup Design

## Context

`services/backend/.local/final-showcase-readiness.json` currently surfaces a top-level operator action like:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable
```

The suffix after `|` is useful evidence detail, but the `operator_actions` list is meant to be copy-pasteable command guidance. Other final reports already expose the same action without the detail suffix.

## Design

P0.560 keeps `next_action.detail`, capability detail, and device evidence detail intact, but normalizes final showcase top-level `operator_actions` so the iOS deploy writer action is an executable command only.

The cleanup applies to the `mobile-write-deploy-config-auto` action root when a detail suffix is present. It does not remove detail fields from structured report objects, and it does not alter provider, print, Xcode, or resource handoff commands.

## Acceptance

- A RED/GREEN test proves `_dedupe_operator_actions()` strips a detail suffix from the iOS deploy writer command.
- Existing saved-next-action tests still prove detail can be recovered from structured device evidence or a legacy suffixed operator action.
- After refreshing the safe local report set, `services/backend/.local/final-showcase-readiness.json` has no `operator_actions` entry containing ` | Missing DEVELOPMENT_TEAM`.
- The first final showcase operator action remains:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

## Safety

This phase changes report string normalization only. It does not write provider secrets, run live provider calls, touch Xcode signing, mutate Keychain, or perform external paid actions.
