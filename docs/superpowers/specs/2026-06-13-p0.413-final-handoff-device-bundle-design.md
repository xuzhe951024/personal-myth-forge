# P0.413 Final Handoff Device Bundle Design

## Goal

Expose a top-level `device_action_bundle` in `final-handoff-index` so the final cross-lane handoff index carries the current iPhone device actions directly.

## Current Context

`final-demo-launch-local`, `final-showcase-readiness`, `ios-device-evidence-bundle`, `ios-deploy-runbook-local`, `ios-device-launch-certificate`, `final-launch-closure-packet`, and `final-local-report-refresh` already expose device action bundles. `final-handoff-index` reads `final-demo-launch-local.json` as a source report and promotes selected operator actions, but it currently drops the nested device bundle. This leaves `final-handoff-index.json` without the machine-readable device handoff that downstream summaries and operators need.

## Design

`build_final_handoff_index_report()` will copy a safe device bundle from the freshest local sources. The preferred source order is:

1. `final_demo_launch_local.device_action_bundle`
2. `ios_deploy_runbook_local.device_action_bundle`

The top-level bundle will use:

- `id`: `final_handoff_index_device_actions`
- `label`: `Final Handoff Index Device Actions`
- `source_report`: the source id that supplied the bundle
- `status`: normalized source status
- `actions`: bounded, report-safe action objects
- `first_action`: copied from the source when present, otherwise the first non-ready action
- `summary`: action, ready, missing, blocked, manual, partial, live, provider, global, and Xcode/signing counts
- `safety`: read-only flags showing this report did not run commands, providers, signing, Keychain, backend env writes, or iOS deploy config writes

If no source has a device bundle, the report will emit a blocked empty bundle with `source_report: "missing"` and `summary.missing: 1`.

## Safety

This is metadata propagation only. It must not run Xcode, signing, Keychain, live provider calls, print ordering, resource application, or any external paid action. Sanitization must continue to remove temp paths and secrets.

## Testing

Add a focused test proving a final demo launch source with a nested device action bundle produces top-level `final_handoff_index_device_actions`, preserves the promoted preflight first action, counts Xcode/signing and provider flags correctly, and does not leak temp paths or secrets. Run the focused file, refresh the real `.local/final-handoff-index.json`, then run lint and backend tests.
