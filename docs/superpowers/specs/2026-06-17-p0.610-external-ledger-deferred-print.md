# P0.610 External Ledger Deferred Print

## Context

`print-fulfillment-readiness` can mark configured Treatstock quote/request checks as
`deferred` while `PRINT_PROVIDER=local`. That matches the current demo policy:
local printable 3MF handoff is enough before attempting a live third-party print
provider quote.

The final external action ledger still used a hard-coded
`refresh_configured_treatstock_quote` action whenever print fulfillment was not
ready. When the actual blocker was source acceptance, visual regression, or local
resource handoff, the ledger pointed operators at the wrong print-provider action.

## Decision

The external action ledger print action should be derived from the
`print-fulfillment-readiness` first blocker or next action:

- Use the concrete blocker id in the action id, e.g. `refresh_visual_print_receipt`.
- Use the concrete blocker label, command, detail, and classification.
- Keep `operator_action` as the validation-aware print readiness action.
- Continue to treat live print-provider quote actions as cost-gated live actions
  when the print readiness report supplies a provider-call operator action.

## Acceptance

- A visual print receipt blocker becomes `refresh_visual_print_receipt`.
- A source acceptance blocker becomes `refresh_ios_print_fulfillment_source`.
- Deferred local-mode Treatstock quote checks do not appear as the external ledger
  first print blocker.
- No Meshy, OpenAI, print-provider, Xcode signing, Keychain, or global machine
  action is run.
