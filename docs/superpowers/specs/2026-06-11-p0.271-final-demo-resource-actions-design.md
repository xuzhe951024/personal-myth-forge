# P0.271 Final Demo Resource Actions Design

## Context

P0.270 made iOS deploy runbook, rehearsal, and final showcase readiness resource
actions validation-aware. The local final demo launch report can still surface a
selected `final-resources.env` action with an unrelated launch-level rerun
command, such as `make ios-device-launch-rehearsal`, because its action dedupe
keeps any existing `; rerun` suffix.

## Goal

Final demo launch operator actions should route every selected
`final-resources.env` handoff through `make final-resources-preflight`, even when
the upstream action already contains a different rerun command.

## Non-Goals

- Do not apply or write final resources.
- Do not run live providers, Xcode, signing, Keychain, or paid print actions.
- Do not change final demo launch status, phase ordering, or first-blocker
  prioritization.

## Design

The shared `operator_actions` helper remains the single source of truth for
final resource validation. It should canonicalize known `final-resources.env`
actions by replacing any existing rerun suffix with
`make final-resources-preflight`.

`final_demo_launch` should apply this helper to the assembled top-level operator
actions before final dedupe and truncation. This preserves existing action
ordering while ensuring the visible handoff path always validates local resources
before moving to launch-level checks.

## Acceptance

- A RED/GREEN test proves local final demo launch replaces
  `provide DEVELOPMENT_TEAM in final-resources.env; rerun make ios-device-launch-rehearsal`
  with `provide DEVELOPMENT_TEAM in final-resources.env; rerun make final-resources-preflight`.
- Existing final demo launch tests still pass.
- Refreshed `final-demo-launch-local.json` has no selected
  `final-resources.env` operator action that reruns a non-resource validation
  command.
- Safety flags remain read-only and do not show provider, Xcode, signing,
  Keychain, global, or env/config writes.
