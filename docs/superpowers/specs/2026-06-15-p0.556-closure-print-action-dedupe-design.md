# P0.556 Closure Print Action Dedupe Design

## Context

The final launch closure packet now prefers the backend-auto print readiness handoff over direct live print quote actions:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make print-fulfillment-readiness
```

After P0.555, refreshed closure packet evidence can include that same action twice. The duplication happens because a bare `make print-fulfillment-readiness` action is replaced with the backend-auto handoff after earlier de-duplication, while another source report already contributed the same backend-auto handoff.

## Goal

Keep the final launch closure packet top-level operator actions free of duplicate backend-auto print readiness handoff actions.

## Scope

- Modify only final launch closure packet action normalization and its focused tests.
- Preserve first blocker, source report, provider handoff, and iOS deploy handoff semantics.
- Do not execute provider calls, place print orders, write keys, or mutate Xcode/global state.

## Design

Update `_prefer_print_fulfillment_backend_handoff()` so replacing a bare print readiness target cannot create a duplicate action. The helper should keep the first occurrence of `PRINT_FULFILLMENT_BACKEND_AUTO_VALIDATED_ACTION` and drop later duplicates, regardless of whether the duplicate came from replacement or from an existing source report action.

This keeps the behavior local to closure packet formatting, where the duplicate is created.

## Testing

- Add a RED test for `_operator_actions()` that passes both the backend-auto print handoff and bare `make print-fulfillment-readiness`, then asserts the backend-auto handoff appears exactly once.
- Run focused closure packet tests.
- Refresh `services/backend/.local/final-launch-closure-packet.json` and assert the backend-auto print handoff appears once.
- Run backend lint, full backend tests, and `git diff --check`.

## Safety

This is a report formatting change only. It does not run commands, expose secrets, call paid providers, or change fulfillment semantics.
