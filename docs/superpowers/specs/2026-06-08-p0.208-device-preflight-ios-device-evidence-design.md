# P0.208 Device Preflight iOS Device Evidence Design

## Purpose

The backend final launch report already includes `ios_device_evidence_bundle`, and the mobile final launch summary renders evidence rows. `DevicePreflightSummaryBuilder` now includes the deploy runbook but still does not show whether the concrete iPhone evidence bundle is ready.

## Scope

- Add an `ios_device_evidence_bundle` device preflight item labeled `Device Evidence`.
- Show waiting when the final launch report or evidence bundle is missing.
- Show blocked when the bundle status is blocked and has an incomplete evidence slot or operator action.
- Show ready when the bundle status is `ready`.
- Include the item in the required ready set after the deploy runbook.
- Reuse the existing device preflight sanitizer; do not execute Xcode, run deploy checks, write config, or mutate global machine state.
- Add Swift contract tests, ProjectChecks gates, and backend mobile readiness source requirements.

## Acceptance

- `DevicePreflightSummaryBuilder` returns an `ios_device_evidence_bundle` item.
- Blocked reports surface the first incomplete evidence slot, command, detail, classification, and first operator action.
- Ready reports mark the item ready and include ready/blocked/required/global counts and safety flags.
- Encoded preflight output redacts provider keys, bearer tokens, local paths, local capture URLs, checkout text, and `api_key` snippets from evidence fields.
- Mobile final launch readiness requires the item, label, source, contract tests, and required-ready set entry.
