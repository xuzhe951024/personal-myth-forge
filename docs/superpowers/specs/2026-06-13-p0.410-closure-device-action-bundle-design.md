# P0.410 Final Closure Device Action Bundle Design

## Context

The final launch closure packet is intended to be the final operator handoff package for resources, safe local writes, device evidence, live provider consent, configured evidence, and final acceptance. It already includes a `device_evidence` section derived from `ios_device_evidence_bundle`, but it does not expose a top-level `device_action_bundle`.

Other final reports now expose device bundles directly: final showcase readiness, iOS deploy runbook, iOS device evidence bundle, and iOS device launch certificate. The closure packet should provide the same machine-readable iPhone action surface so a final handoff reader does not need to dig into nested sections.

## Goal

Add a sanitized, read-only `device_action_bundle` to `final-launch-closure-packet.json`, derived from the existing iOS device evidence bundle report.

## Scope

- Add top-level `device_action_bundle` to `build_final_launch_closure_packet_report`.
- Derive actions from `ios_device_evidence_bundle.device_action_bundle` when present.
- Preserve existing closure packet sections, first blocker, next action, operator actions, source reports, and status behavior.
- Add blocked and ready focused tests.
- Update the roadmap with P0.410.

## Non-Goals

- Do not run backend, Xcode, signing, Keychain, provider, iPhone, or paid-print commands.
- Do not change the closure packet's first blocker ordering.
- Do not change final showcase, final handoff index, or iOS evidence report behavior.
- Do not write backend env, iOS deploy config, or provider secrets.
- Do not embed secrets, local absolute paths, raw media, or payment links.

## Design

The closure packet report will include:

- `id`: `final_launch_closure_device_actions`
- `label`: `Final Launch Closure Device Actions`
- `status`: copied from the iOS device evidence bundle action status
- `actions`: sanitized compact copies of source device actions
- `first_action`: sanitized compact copy of source bundle first action
- `summary`: action counts for ready, missing, blocked, manual, provider calls, global actions, and Xcode/signing
- `safety`: read-only no-mutation flags
- `source_report`: `ios_device_evidence_bundle`

Each copied action will retain `id`, `label`, `status`, `classification`, `command`, `detail`, `evidence_source`, `manual`, `provider_calls`, `global_action`, `xcode_or_signing`, and `next_action` when present. The closure packet remains a read-only aggregator; the source report remains the source of truth for device slot classification.

If the source report is missing a bundle, the closure packet will derive a fallback bundle from `evidence_slots` so the top-level field is stable.
