# P0.142 Rehearsal Action Normalization Design

## Context

P0.139 introduced `make final-resource-init`. P0.140 normalized stale
final-resource copy guidance inside `final_showcase_readiness`. P0.141 aligned
current README and apply-script guidance.

The iOS device launch rehearsal path still has two action aggregation layers
that can surface older saved report actions:

- `ios_device_launch_rehearsal` reads nested saved reports and prefixes their
  operator actions.
- `ios_device_launch_rehearsal_readiness` reads the saved rehearsal report and
  exposes its bounded operator actions to `/v1/final-demo-launch` and the iPhone
  Final Launch panel.

If either saved layer contains the old copy instruction, the phone can still
show outdated final-resource setup guidance.

## Goal

Normalize legacy final-resource copy guidance across the rehearsal path so all
current Mac and iPhone launch surfaces point to:

```text
run make final-resource-init
```

## Non-Goals

- Do not rewrite ignored `.local` reports.
- Do not refresh saved evidence automatically.
- Do not change rehearsal freshness, status, or action bounding semantics.
- Do not call providers, apply resources, run Xcode, write signing/keychain
  state, or mutate global machine state.

## Design

Create `services/backend/src/myth_forge_api/operator_actions.py` with:

```python
LEGACY_FINAL_RESOURCE_COPY_MARKERS = (
    "services/backend/final-resources.env.example",
    "services/backend/.local/final-resources.env",
)
NORMALIZED_FINAL_RESOURCE_INIT_ACTION = "run make final-resource-init"

def normalize_operator_action(action: str) -> str:
    if all(marker in action for marker in LEGACY_FINAL_RESOURCE_COPY_MARKERS):
        return NORMALIZED_FINAL_RESOURCE_INIT_ACTION
    return action
```

Use the helper in:

- `final_showcase_readiness._report_operator_actions`
- `ios_device_launch_rehearsal._bounded_operator_actions`
- `ios_device_launch_rehearsal_readiness._operator_actions`

Normalization happens before dedupe and action limits. Prefixes already added by
the rehearsal report are intentionally removed for this specific legacy action,
because the actionable next step is now a project-root command and dedupe should
collapse equivalent legacy sources into one `run make final-resource-init`.

## Testing

Add RED tests for:

- Saved blocked rehearsal source action is normalized while priority actions are
  preserved.
- Saved rehearsal readiness action list is normalized and does not leak
  `services/backend/final-resources.env.example`.
- Existing final showcase normalization still passes through the shared helper.

Then implement the helper and wire the three call sites.

## Safety

This change only transforms strings in in-memory reports. It does not write
config files, does not edit `.local` reports, and does not execute commands.
