# P0.341 Apply Preview Before Apply Design

## Context

The final resource handoff has two intentionally separate phases:

```text
make final-resource-apply-preview
make final-apply-resources
```

The preview phase is read-only and reports which ignored local files would be written. The apply phase writes project-local ignored files such as backend `.env` and iOS deploy config. The user explicitly requires API/key resources to stay out of the repo and to remain a local handoff that they provide.

`final-resource-apply-preview.json` already reports a safe action when required values are missing:

```text
rerun make final-resource-apply-preview before applying resources
```

However, `final-local-report-refresh.json.operator_actions` can still include `make final-apply-resources` in the same blocked action list. That makes the final handoff less clear because it exposes the writer command before the preview is ready.

## Goal

When `final-local-report-refresh` already exposes a blocked apply-preview action, suppress direct `make final-apply-resources` operator actions from the same top-level list.

## Non-Goals

- Do not remove `make final-apply-resources` from reports whose apply preview is ready or absent.
- Do not remove final resource fill actions such as `provide MESHY_API_KEY in final-resources.env`.
- Do not change the actual `make final-apply-resources` target, scripts, or writer behavior.
- Do not write backend `.env`, iOS deploy config, Keychain values, or any real provider/order state.

## Design

Add two narrow checks in `final_local_report_refresh.py`.

First, when `_run_step()` summarizes the `final_resource_apply_preview` step as
blocked, promote the step action as:

```text
rerun make final-resource-apply-preview before applying resources
```

This prevents the preview step's nested first blocker from surfacing the writer
command before the preview itself is rerun.

Second, add a post-dedupe filter in `_dedupe_operator_actions()`.

The filter should detect this action root:

```text
rerun make final-resource-apply-preview before applying resources
```

If that root exists, remove actions whose root is exactly:

```text
make final-apply-resources
```

Roots are compared before any ` | detail` suffix. Other resource actions remain unchanged so the operator still sees what must be filled before preview/apply can proceed.

## Acceptance Criteria

- `final-local-report-refresh.json.operator_actions` does not include `make final-apply-resources` while a preview-before-apply action is present.
- `make final-apply-resources` remains present when no preview-before-apply action exists.
- Existing provider selector normalization still maps provider setup actions to the apply target when preview gating is absent.
- Focused tests, lint, diff checks, and backend tests pass.
