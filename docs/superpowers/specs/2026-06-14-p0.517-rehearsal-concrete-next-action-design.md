# P0.517 iOS Rehearsal Concrete Next Action Design

## Context

The final handoff reports now promote the local iPhone device chain before provider/key work:

1. `DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`
2. `start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight`
3. `set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL; rerun make mobile-deploy-preflight`

`ios-device-launch-rehearsal` already lists that chain in `operator_actions`, and `next_action.command` already uses the first concrete operator action. Its `first_blocker.command` still points to the broader `make final-rehearsal-local`, and `next_action.validation_command` still points back to `make ios-device-launch-rehearsal`.

That makes the final iOS rehearsal packet less actionable than the reports feeding it. The top-level blocker and validation should guide the operator to the same concrete device command that appears first in `operator_actions`.

## Decision

P0.517 updates only `ios_device_launch_rehearsal` top-level action selection.

When a blocked sequence step has concrete `operator_actions`, the report should use the first normalized top-level operator action as:

- `first_blocker.command`
- `next_action.command`

When that promoted command includes `; rerun make ...`, `next_action.validation_command` should use the rerun target. For the current device blocker:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

the validation command is:

```text
make mobile-deploy-preflight
```

Missing-source fallback remains unchanged:

```text
first_blocker.command = make final-rehearsal-local
next_action.validation_command = make ios-device-launch-rehearsal
```

## Boundaries

- No provider calls.
- No Xcode signing, Keychain writes, global mutation, or paid print/provider action.
- No changes to sequence status, source report shape, operator action ordering, or safety fields.
- No changes to Make wrappers except evidence commands already used for verification.
