# P0.224 Xcode Evidence Device Action Design

## Context

P0.223 connected `mobile-deploy-preflight-evidence.json` to the final showcase device action bundle. The next blocked iOS device action is `resolve_xcode_build_gate`, but today it still shows a generic Xcode/signing instruction even when `services/backend/.local/mobile-xcode-build-evidence.json` already contains the exact blocker, such as an unaccepted Apple SDK license or missing Xcode installation.

## Goal

Connect `services/backend/.local/mobile-xcode-build-evidence.json` to `final_showcase_readiness_report.device_action_bundle.actions["resolve_xcode_build_gate"]` so the iPhone Final Launch panel can show the Xcode evidence status, source, detail, and validation command.

## Non-Goals

- Do not run `make mobile-xcode-build`, `make mobile-xcode-build-evidence`, Xcode, signing, simulator, keychain, or Apple license commands inside the readiness builder.
- Do not accept Apple licenses or change global Xcode state.
- Do not mark the final rehearsal ready solely from Xcode build evidence.
- Do not expose local absolute paths, signing identities, provider secrets, raw media, payment links, LAN URLs, or private context.

## Contract

When the Xcode evidence file is missing, final showcase readiness keeps `resolve_xcode_build_gate` blocked and embeds a safe missing evidence summary:

```json
{
  "evidence": {
    "mobile_xcode_build_evidence": {
      "kind": "mobile_xcode_build_evidence_report",
      "status": "missing"
    }
  },
  "device_action_bundle": {
    "actions": [
      {
        "id": "resolve_xcode_build_gate",
        "status": "blocked",
        "evidence_status": "missing",
        "evidence_source": "services/backend/.local/mobile-xcode-build-evidence.json",
        "validation_command": "make mobile-xcode-build-evidence"
      }
    ]
  }
}
```

When the Xcode evidence file is blocked, the action remains blocked but shows the saved detail, for example `Apple SDK license agreement is not accepted.` and the operator action `accept the Xcode license outside Codex, then rerun make mobile-xcode-build-evidence`.

When both preflight and Xcode evidence are ready, `start_backend_device_demo`, `run_mobile_deploy_preflight`, and `resolve_xcode_build_gate` become ready while `run_ios_device_launch_rehearsal` remains blocked until launch rehearsal evidence is refreshed.

## Mobile Summary

The Swift model already decodes optional device action evidence fields. P0.224 extends the contract fixture so the Final Launch summary proves the Xcode row can render `evidence blocked | make mobile-xcode-build-evidence` without needing new model fields.

## Safety

The integration is read-only and loads only an ignored local JSON evidence file. Missing, malformed, or wrong-kind evidence is treated as missing/blocked. The report remains sanitized by the existing final showcase readiness and mobile summary redaction paths.

## Verification

- Backend RED/GREEN tests for missing, blocked, and ready `mobile_xcode_build_evidence`.
- Swift RED/GREEN tests that the Xcode device action evidence decodes and renders.
- Source acceptance gates and visual regression evidence for the Xcode evidence device action row.
- Full backend lint/test and Swift checks before merge.
