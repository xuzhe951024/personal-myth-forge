# P0.291 Final Resource Requirements Action Normalization Design

## Goal

Normalize the final resource requirements operator action to the canonical make target used elsewhere in the final iPhone handoff.

## Context

The final demo launch report still surfaces this operator action:

```text
run make final-resource-requirements after filling resources
```

The action is understandable, but the project already exposes `make final-resource-requirements` as the canonical command in Makefile, backend reports, and mobile command rows. Keeping both wordings creates noise in the final operator checklist and makes the key handoff path look less directly executable.

## Design

Keep normalization centralized in `myth_forge_api.operator_actions`.

Add a make-target replacement so:

- `run make final-resource-requirements after filling resources`

becomes:

- `make final-resource-requirements`

The mapping should preserve existing detail suffixes such as ` | resource file ready`, matching the current operator action normalization pattern.

Backend reports that already route actions through `normalize_operator_action()` should automatically expose the canonical command. Mobile fixtures that model final resource requirements operator actions should use the same canonical command.

## Scope

In scope:

- shared operator action normalization
- backend tests for direct normalization and final demo launch operator actions
- final local refresh verification
- mobile contract fixture update if it still embeds the old wording
- roadmap entry for P0.291

Out of scope:

- writing `final-resources.env`
- applying resources
- real Meshy/OpenAI/Treatstock calls
- changing required resource semantics
- iOS signing, Keychain, or Xcode execution

## Verification

This iteration is valid when:

- RED tests show the old final-resource-requirements wording leaks through
- GREEN tests show it normalizes to `make final-resource-requirements`
- refreshed `.local` launch/readiness reports no longer expose the old wording
- backend lint and tests pass
