# P0.405 Demo Launch Configured Unblocks Design

## Context

The final configured demo launch report is a source handoff artifact for the final iPhone showcase path. Earlier iterations cleaned stale `unblock ...` fallback actions from the aggregate configured preflight and iOS runbook outputs, but `services/backend/.local/final-demo-launch-configured.json` can still expose two abstract fallback rows:

- `unblock provider_readiness: make provider-handoff`
- `unblock configured_final_acceptance: make final-acceptance-configured`

Those rows duplicate more concrete handoff actions that are already present in the same operator list, such as provider key handoff, live provider evidence refresh, Treatstock quote evidence, and mobile deploy preflight actions.

## Goal

Remove stale configured final demo launch `unblock ...` fallback rows while keeping concrete provider, print, resource, and iOS deployment handoff commands intact.

## Scope

- Extend the existing targeted fallback filter in `services/backend/src/myth_forge_api/final_demo_launch.py`.
- Add focused regression coverage in `services/backend/tests/test_final_demo_launch.py`.
- Refresh the project roadmap entry for this phase.

## Non-Goals

- Do not change final showcase readiness capability classification.
- Do not run real Meshy, OpenAI, Treatstock, Xcode signing, or iPhone deployment actions.
- Do not rewrite launch report schemas or reorder unrelated operator actions.

## Design

The implementation reuses `FINAL_DEMO_TARGETED_UNBLOCK_ACTION_ROOTS` and `_drop_targeted_unblock_fallbacks()`. Add the two configured launch fallback roots to that allowlist-style removal set. This keeps the filter narrow and avoids hiding unknown future blockers.

The regression test will assert that configured launch top-level `operator_actions` no longer contain any `unblock ...` rows for provider readiness or configured final acceptance, while retaining the concrete actions that make those blockers actionable.

- Confirm the focused test passes after implementation.
- Run `tests/test_final_demo_launch.py`.
- Refresh local reports and verify `final-demo-launch-configured.json` top-level operator actions contain no `unblock ...` rows.
- Run backend lint and backend test before merging.
