# P0.414 External Ledger Device Bundle Design

## Goal

Expose a top-level `device_action_bundle` in `final-external-action-ledger` so the external action ledger directly carries the current iPhone handoff actions.

## Current Context

Most final handoff reports now expose machine-readable device actions. `final-external-action-ledger` remains the exception: it includes device runtime actions in `action_groups` and already builds `ios_deploy_runbook`, but it does not expose the runbook's structured `device_action_bundle`. This makes the external action ledger less useful as a single safe operator ledger for iPhone readiness.

## Design

`build_final_external_action_ledger_report()` will derive `device_action_bundle` from `ios_deploy_runbook.device_action_bundle`. The bundle will be copied into a report-safe top-level shape:

- `id`: `final_external_action_ledger_device_actions`
- `label`: `Final External Action Ledger Device Actions`
- `source_report`: `ios_deploy_runbook`
- `status`: normalized source status
- `actions`: bounded safe action objects
- `first_action`: copied from the source when present, otherwise the first non-ready action
- `summary`: action, ready, missing, blocked, manual, partial, live, provider, global, and Xcode/signing counts
- `safety`: flags showing no commands, providers, signing, Keychain, backend env writes, or iOS deploy config writes were performed

If the runbook does not expose a device bundle, the report will emit a blocked empty bundle with `source_report: "missing"` and `summary.missing: 1`.

## Safety

This change is metadata-only. It must not run Xcode, signing, Keychain, live providers, print ordering, resource apply writers, or any paid action. The existing report sanitizer continues to remove local paths and secrets.

## Testing

Add a focused test proving the ledger exposes `final_external_action_ledger_device_actions`, inherits `ios_deploy_runbook` as the source, preserves the first deploy config handoff action, counts Xcode/signing and provider flags, and does not leak temp paths or secrets. Run the focused file, refresh `.local/final-external-action-ledger.json`, then run lint and backend tests.
