# P0.552 Print Fulfillment Next Action Design

## Context

The final showcase still requires a printable `print_asset` handoff and a third-party quote/review path. `print-fulfillment-readiness` already emits `operator_actions` that rerun the print readiness target, but its direct `next_action.command` keeps only the raw blocker command.

For example, the current report can expose:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto
```

The command writes the ignored deploy config, but it does not refresh the print fulfillment evidence that the final ledger checks. The validation command exists in a separate field, which is easy for direct report readers and mobile rows to miss.

## Decision

Keep `first_blocker.command` as the raw blocker command for provenance. Make `next_action.command` validation-aware by appending `; rerun make print-fulfillment-readiness` when the command does not already include it.

Examples:

```text
make final-demo-launch-local; rerun make print-fulfillment-readiness
```

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make print-fulfillment-readiness
```

```text
PMF_ALLOW_PRINT_PROVIDER_CALLS=1 make print-quote-configured; rerun make print-fulfillment-readiness
```

Cost-bearing print provider actions still carry `requires_cost_consent: true` in `next_action`.

## Scope

This phase only changes local report command shaping. It does not call Treatstock, Meshy, OpenAI, Xcode, Keychain, or any paid provider. It does not write secrets or mutate global state.

## Acceptance

- Print fulfillment `next_action.command` includes the print readiness rerun for missing local demo session, backend URL, auto quote request, and configured quote blockers.
- `first_blocker.command` stays raw for provenance.
- `operator_actions` keep the same validation-aware commands.
- Refreshed `.local/print-fulfillment-readiness.json` contains the validation-aware `next_action.command`.
- Focused tests, lint, diff hygiene, and backend tests pass.
