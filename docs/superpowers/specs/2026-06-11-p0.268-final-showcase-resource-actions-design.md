# P0.268 Final Showcase Resource Actions Design

## Context

`final-showcase-readiness` is the top-level C-path readiness ledger for the
iPhone demo. Its current saved report can still surface nested resource actions
such as `final_rehearsal_local: ios_deploy_runbook_local: provide
MESHY_API_KEY in final-resources.env` without a validation command, even though
lower-level final resource reports now provide `; rerun make
final-resources-preflight`.

## Goal

Final showcase operator actions should preserve nested source prefixes while
making known final-resource actions validation-aware. The operator should see
what value to provide and the exact command that validates it.

## Non-Goals

- Do not write API keys or deploy configuration.
- Do not call Meshy, OpenAI, Treatstock, Xcode, Keychain, or any live provider.
- Do not change final-showcase blocker ordering or capability status.

## Acceptance

- A focused RED/GREEN test proves prefixed Meshy/OpenAI resource actions gain
  `; rerun make final-resources-preflight`.
- The refreshed `final-showcase-readiness.json` does not include bare prefixed
  Meshy/OpenAI resource actions in the first operator actions.
- Focused final showcase tests, backend lint, and diff checks pass.
