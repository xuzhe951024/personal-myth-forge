# P0.253 Mobile Deploy Preflight Next Action Design

## Context

The current final showcase blocker is the physical iPhone deploy path. The saved
`mobile-deploy-preflight-evidence.json` report already records `checks` and
`operator_actions`, but it does not expose a top-level `first_blocker` or
`next_action`. Downstream reports can summarize the checks, but the evidence
report itself cannot directly tell the operator or iPhone UI which single action
comes next.

## Goal

Add a sanitized top-level `first_blocker` and `next_action` to the mobile deploy
preflight evidence report. The fields should point to the first blocked check and
the matching operator action, while ready reports should expose `null` for both
fields.

## Design

Keep the report local and read-only with no provider calls, signing, Keychain
writes, or global mutation. Reuse the existing check parsing and operator action
mapping:

- `first_blocker` is the first check whose status is not `ready`.
- `next_action.command` uses the matching operator action when available, so the
  report distinguishes "provide DEVELOPMENT_TEAM" from "start backend-device-demo".
- `next_action.validation_command` is `make mobile-deploy-preflight`, so callers
  keep the command needed to re-run the evidence gate.
- `next_action.detail` summarizes all blocked check details, preserving parallel
  missing slots such as Team ID, bundle ID, and backend URL.
- Ready reports keep `first_blocker: null` and `next_action: null`.

## Testing

Add focused backend tests proving:

- missing local config produces `first_blocker.id == "development_team"` and a
  next action to provide `DEVELOPMENT_TEAM`.
- backend health failure produces `first_blocker.id == "backend_health"` and a
  next action to start `backend-device-demo`.
- ready output leaves both top-level fields absent/null.

## Safety

The change only restructures sanitized local evidence output. It does not run
Xcode, write deployment config, use external API keys, call live providers, place
print orders, or mutate global machine state.

## Out of Scope

This iteration does not provide `DEVELOPMENT_TEAM`, set a real
`PMF_BACKEND_BASE_URL`, run a physical iPhone deployment, or mark the full goal
complete.
