# P0.620 Local Refresh Granular Configured Consent

## Problem

`final-local-report-refresh` turns any configured evidence consent blocker into the broad `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make final-acceptance-configured; rerun make live-provider-evidence` action. The current configured evidence plan already knows the more specific first missing live action, such as `backend-evaluate-3d-configured`. The broader action makes the operator handoff less precise than final showcase, configured bundle, and closure packet.

## Scope

- Preserve the configured evidence plan's specific live action in `final-local-report-refresh`.
- Add the explicit `PMF_ALLOW_LIVE_PROVIDER_CALLS=1` guard when a report action is live/provider-cost gated but its command lacks the guard.
- Preserve live/cost/user-confirmation metadata and the source validation command.
- Do not run live providers, Xcode signing, or global mutations.

## Acceptance

- RED test proves `final-local-report-refresh` rewrites a granular configured 3D consent blocker to the broad configured final acceptance action.
- GREEN implementation keeps `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-3d-configured; rerun make final-configured-evidence-plan`.
- Focused local refresh tests, lint, and backend regression gates remain green.
