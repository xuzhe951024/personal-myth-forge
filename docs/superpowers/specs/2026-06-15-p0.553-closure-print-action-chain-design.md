# P0.553 Closure Print Action Chain Design

## Context

The final showcase readiness, live provider evidence, and print fulfillment readiness reports now expose validation-aware next actions. The final launch closure packet still can surface a bare `make print-fulfillment-readiness` operator action, which is weaker than the current print handoff path because it does not remind the operator to prepare an iPhone-reachable backend URL or rerun the readiness report after the prerequisite action.

## Goal

Make the final launch closure packet operator actions prefer a concrete print fulfillment readiness chain over a bare validation target.

## Scope

- Keep all work local and source-only.
- Do not write provider keys, call live providers, mutate Xcode state, or place print orders.
- Preserve existing first blocker behavior for iOS deploy evidence.
- Keep provider handoff actions on the existing complete chain:
  `make final-resource-fill-guide; rerun make final-resource-apply-preview; rerun make provider-handoff; rerun make live-provider-evidence`.

## Design

The closure packet already builds operator actions from compact section actions, then applies normalization helpers. Add a closure-specific preference step after generic print/provider normalization so that a remaining bare `make print-fulfillment-readiness` is replaced when a stronger print prerequisite chain is available.

Preferred chain:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID PMF_BACKEND_BASE_URL=auto make mobile-write-deploy-config-auto; rerun make print-fulfillment-readiness
```

Fallback chain:

```text
set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL; rerun make print-fulfillment-readiness
```

The first chain matches the current ignored iOS deploy config writer and helps physical iPhone demos because the print quote request depends on the same `PMF_BACKEND_BASE_URL` slot.

## Testing

- Add a regression test in `services/backend/tests/test_final_launch_closure_packet.py` proving the packet no longer includes bare `make print-fulfillment-readiness`.
- Assert the replacement action includes `rerun make print-fulfillment-readiness`.
- Run the focused closure packet tests, then backend lint and full backend tests before merging.

## Safety

This phase only changes emitted instructions in sanitized reports. It does not execute those instructions, write secrets, call Meshy/OpenAI/Treatstock, or touch Xcode signing/global state.
