# P0.220 Final Resource Auto Backend URL Design

## Context

P0.219 added `PMF_BACKEND_BASE_URL=auto` support to the iOS deploy config writer and a Make target for the physical-device path. The unified final resource file is the intended last-mile handoff for API keys, iOS signing values, and backend URL configuration. That path should explicitly support the same `auto` backend URL mode instead of leaving it as an undocumented string that happens to pass validation.

## Goal

Make `PMF_BACKEND_BASE_URL=auto` a first-class value in final resource preflight, requirements, apply preview, and final apply tests.

## Non-Goals

- Do not auto-detect the LAN IP during read-only report builders.
- Do not start the backend, run deploy preflight, run Xcode, change signing, touch keychain, call providers, or write global state.
- Do not expose resolved private IPs in read-only final resource reports.
- Do not mark iOS device evidence ready without the actual deploy preflight and Xcode/device proof.

## Contract

When `services/backend/.local/final-resources.env` contains:

```env
PMF_BACKEND_BASE_URL=auto
```

`final-resources-preflight` treats the slot as ready and emits:

```json
{
  "id": "PMF_BACKEND_BASE_URL",
  "status": "ready",
  "configured": true,
  "classification": "apply_time_auto_url",
  "resolution_mode": "apply_time_auto",
  "apply_note": "Resolved by write_deploy_local_config.sh during final-apply-resources."
}
```

`final-resource-requirements` and `final-resource-apply-preview` carry the same classification and resolution mode for the backend URL row/slot. `final-apply-resources` accepts `auto`, delegates resolution to `write_deploy_local_config.sh`, and still blocks loopback or placeholder literal URLs.

## Safety

All report builders remain read-only. `PMF_BACKEND_AUTO_HOST` is only used in tests to make shell resolution deterministic. Provider secrets remain redacted, and read-only reports must not include the resolved LAN IP.

## Verification

- RED/GREEN backend report tests for preflight, requirements, and apply preview.
- RED/GREEN shell apply test proving `auto` writes the resolved iOS local config through the existing writer.
- Focused final resource regression.
- Full backend lint/test and local final rehearsal before merge.
