# P0.416 Configured Evidence Device Bundle Design

## Context

`final_configured_evidence_plan_report` is the configured live-provider plan that sequences resource fill, apply preview, configured preflight, provider handoff, Meshy 3D evaluation, OpenAI NPC evaluation, configured acceptance, final demo launch, and live evidence refresh. It already builds `final_configured_preflight`, and P0.415 made that nested preflight expose a sanitized `device_action_bundle`.

The saved `final-configured-evidence-plan.json` still lacks a top-level `device_action_bundle`. Operators can see the configured resource and live-provider blocker, but they must inspect nested source reports to see the physical-iPhone action sequence. That weakens the final iOS handoff packet.

## Goal

Expose a sanitized top-level `device_action_bundle` in `final_configured_evidence_plan_report`, sourced from `final_configured_preflight.device_action_bundle`.

## Scope

- Add `device_action_bundle` to `build_final_configured_evidence_plan_report`.
- Source it from the already-built `configured_preflight` report.
- Preserve all existing `first_blocker`, `next_action`, step status, and live-provider consent behavior.
- Add focused TDD coverage for the blocked plan path.
- Refresh `services/backend/.local/final-configured-evidence-plan.json` and verify the new field appears.
- Add the P0.416 roadmap entry.

## Non-Goals

- Do not run or apply final resources.
- Do not call Meshy, OpenAI, Treatstock, Sculpteo, or any live provider.
- Do not run Xcode, signing, Keychain, global Mac configuration, or real iPhone deployment.
- Do not write backend `.env`, iOS deploy config, or provider resource files.
- Do not change configured evidence command ordering or consent gating.

## Report Contract

`final_configured_evidence_plan_report.device_action_bundle` will contain:

- `id`: `final_configured_evidence_plan_device_actions`
- `label`: `Final Configured Evidence Plan Device Actions`
- `source_report`: `final_configured_preflight`
- `status`: normalized from the source bundle status
- `actions`: sanitized copies of source preflight device actions
- `first_action`: sanitized source first action when present, otherwise first non-ready copied action
- `summary`: counts for actions, ready, missing, blocked, manual, partial, live, provider calls, global actions, and Xcode/signing actions
- `safety`: all false for command execution, provider calls, global mutation, Keychain writes, backend env writes, iOS deploy config writes, and Xcode/signing execution from the report builder

If the nested configured preflight bundle is missing, the plan returns a blocked empty bundle with `source_report: "missing"` and `summary.missing == 1`.

## Testing

Use TDD:

1. Run the existing focused baseline for `tests/test_final_configured_evidence_plan.py`.
2. Add a failing test that expects `result.report["device_action_bundle"]` in the blocked plan path.
3. Confirm RED with `KeyError: 'device_action_bundle'`.
4. Implement the top-level bundle copy helper.
5. Confirm GREEN with the new focused test and then the full focused file.
6. Refresh `make final-configured-evidence-plan`; exit `2` is expected while resources are missing, but the saved JSON must include the device action bundle.
7. Run backend lint and backend tests before merging.
