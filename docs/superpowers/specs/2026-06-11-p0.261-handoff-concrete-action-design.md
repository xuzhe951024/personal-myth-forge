# P0.261 Final Handoff Concrete Action Design

## Goal

Make the final handoff index show a concrete, operator-executable first action for the local rehearsal lane when the saved local final demo launch report already exposes a structured `next_action`.

## Context

`final-demo-launch-local.json` now reports:

- `first_blocker.command`: `make mobile-deploy-preflight`
- `next_action.command`: `provide DEVELOPMENT_TEAM in Deployment.local.xcconfig`
- `next_action.validation_command`: `make mobile-deploy-preflight`

`final-handoff-index.json` still turns the local rehearsal blocker into a long detail string such as `final_demo_launch_local: physical iPhone backend health gate...`. That is useful context, but it is not the next action the operator should do first.

## Scope

This iteration only changes `final_handoff_index` aggregation behavior:

- Keep the local rehearsal lane status and blocker detail unchanged.
- Prefer a saved source report's `next_action.command` when building the local rehearsal operator action.
- Include `validation_command` in the action text when present, using the established `rerun <command>` wording.
- Fall back to existing detail-based behavior when no structured `next_action` exists.
- Do not touch real provider calls, resource application, Xcode signing, Keychain, or any global machine state.

## Data Flow

1. `_source_report()` reads saved `.local/*.json` reports.
2. It extracts the existing saved status/detail fields.
3. It also extracts a bounded concrete operator action from `next_action.command`.
4. `_source_detail()` continues to expose blocker details for lane detail and first blocker context.
5. `_operator_actions()` uses the local rehearsal lane's concrete action when available, otherwise keeps the existing fallback.

## Testing

Add a focused regression test in `services/backend/tests/test_final_handoff_index.py` where a saved `final-demo-launch-local.json` contains both a long `first_blocker.detail` and a concrete `next_action`. The test must first fail because the current implementation returns the long detail as `operator_actions[0]`. After implementation, the test should pass with:

`final_demo_launch_local: provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`

## Self-Review

- No placeholders or deferred behavior.
- The scope is a single aggregation behavior change.
- Existing first blocker detail remains available for diagnosis.
- The design does not require external credentials, paid APIs, or Xcode state changes.
