# P0.563 Print Provider Quote Deferred Design

## Context

The final product goal still includes a printable `print_asset` and a third-party print quote or fulfillment handoff. The current demo strategy now defers the external print provider while the rest of the iOS, 3D, and NPC paths stabilize: for v0 local showcase review, generating and handing off a printable file is enough.

`print-fulfillment-readiness` still treats configured Treatstock quote request and quote evidence as required even when `PRINT_PROVIDER=local`. This makes the final readiness ledger report a print blocker after the local 3MF handoff is already ready, and it keeps surfacing Treatstock operator actions that the user explicitly wants to defer.

## Decision

Make `PRINT_PROVIDER=local` mean:

- local print acceptance, iOS source acceptance, visual print receipt, and print resource handoff are required;
- configured Treatstock quote request and quote checks remain visible but become `deferred`;
- deferred provider quote checks do not affect `status`, `first_blocker`, `next_action`, or `operator_actions`;
- the report can return `ready` once the local printable file handoff is ready.

Keep `PRINT_PROVIDER=treatstock` behavior unchanged:

- configured quote request evidence is required;
- configured quote evidence is required;
- missing or unsafe Treatstock quote evidence still blocks or marks the report partial;
- cost-producing quote commands keep their explicit consent path.

## Scope

In scope:

- Update focused backend tests for local-vs-Treatstock print readiness.
- Update `print_fulfillment_readiness.py` report semantics for deferred provider quote checks.
- Update final showcase readiness copy/coverage so a local-ready print handoff is not described as configured Treatstock evidence.
- Add a roadmap note for P0.563.

Out of scope:

- Calling Treatstock, Sculpteo, Meshy, OpenAI, Xcode, signing, Keychain, or physical-device flows.
- Removing Treatstock adapter code or configured quote handoff commands.
- Changing the final goal definition; the full goal still requires real configured provider evidence before completion.

## Acceptance

- A RED test proves `PRINT_PROVIDER=local` can produce a ready print fulfillment report without configured Treatstock quote evidence.
- Existing Treatstock-mode tests still prove configured quote request and quote evidence are required for `PRINT_PROVIDER=treatstock`.
- Final showcase readiness marks print fulfillment ready when the child print report is local-ready.
- The report still exposes deferred configured provider checks for operator visibility without promoting their commands.
- Focused backend tests, backend lint, backend test suite, and whitespace diff checks pass.
