# P0.619 iOS Rehearsal Final Local Refresh

## Problem

`write_ios_device_launch_rehearsal.sh` runs `final-local-report-refresh` as part of `make final-rehearsal-local` before it writes the latest iOS device launch rehearsal and rehearsal readiness reports. After the script completes, `final-local-report-refresh.json` can still describe a stale device/backend blocker even though `final-showcase-readiness.json` has moved on to the live 3D provider consent blocker.

## Scope

- Refresh `final-local-report-refresh.json` again after the iOS rehearsal, rehearsal readiness, and local final-demo launch sync have been written.
- Keep the operation safe: no live provider calls, no Xcode signing, no global state mutation, only ignored `.local` evidence writes.
- Preserve existing safe wrapper behavior that accepts usable blocked reports with exit code 2.

## Acceptance

- RED test proves the iOS rehearsal wrapper does not refresh final-local-report-refresh after final-demo launch sync.
- GREEN implementation appends a post-rehearsal final-local-report-refresh step.
- Focused script tests, lint, and backend regression gates remain green.
