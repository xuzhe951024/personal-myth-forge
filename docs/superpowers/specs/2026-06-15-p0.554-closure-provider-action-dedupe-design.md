# P0.554 Closure Provider Action Dedupe Design

## Context

The final launch closure packet is the operator-facing handoff that gathers device, provider, visual, print, and final acceptance actions. P0.551 made live provider evidence use the full validation-aware chain:

```text
make final-resource-fill-guide; rerun make final-resource-apply-preview; rerun make provider-handoff; rerun make live-provider-evidence
```

After P0.553, the closure packet still can emit this full chain and a weaker duplicate:

```text
make final-resource-fill-guide; rerun make final-resource-apply-preview; rerun make provider-handoff
```

The weaker action is not wrong as a source-level validation target, but it is weaker as a final operator action because it stops before the evidence report that proves Meshy/OpenAI readiness.

## Goal

When the final launch closure packet has the full provider live evidence chain, remove the shorter provider handoff chain from top-level operator actions.

## Scope

- Modify only final launch closure packet action normalization and its tests.
- Keep first blocker and source report details unchanged.
- Do not call Meshy, OpenAI, Treatstock, Xcode, or any paid/live provider.
- Do not write secrets or checked-in local config.

## Design

Extend the closure packet weak provider action set to include:

```text
make final-resource-fill-guide; rerun make final-resource-apply-preview; rerun make provider-handoff
```

The existing `_prefer_complete_provider_handoff_chain()` already removes actions listed in `WEAK_PROVIDER_HANDOFF_OPERATOR_ACTIONS` whenever `COMPLETE_PROVIDER_HANDOFF_OPERATOR_ACTION` is present. Adding the missing variant keeps the implementation local and follows the existing normalization pattern.

## Testing

- Add a regression test to `services/backend/tests/test_final_launch_closure_packet.py` that asserts the closure packet does not include the shorter provider handoff chain when the full live-provider chain is present.
- Verify the test fails before the weak-action set is updated.
- Run the focused closure packet tests, backend lint, full backend tests, and `git diff --check`.

## Safety

This phase only changes sanitized report instructions. It does not run external providers, mutate global Mac/Xcode state, place print orders, or expose keys.
