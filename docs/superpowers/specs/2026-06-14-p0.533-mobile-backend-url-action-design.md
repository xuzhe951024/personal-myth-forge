# P0.533 Mobile Backend URL Action Design

## Context

`mobile-deploy-preflight-evidence` parses `make mobile-deploy-preflight` output and turns blocked checks into device actions. When both `DEVELOPMENT_TEAM` and `PMF_BACKEND_BASE_URL` are blocked, the top-level next action already uses the safe ignored config writer:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

The per-check `backend_base_url` device action can still fall back to the raw error detail:

```text
PMF_BACKEND_BASE_URL must be iPhone-reachable
```

That is not executable, so downstream launch bundles show one actionable first step and one weak backend URL row.

## Design

Keep the existing writer and safety behavior. Do not write `Deployment.local.xcconfig` during report generation.

When `mobile-deploy-preflight-evidence` has a blocked `backend_base_url` check and the raw actions include the auto deploy-config writer, the backend URL device action should use the same writer command root:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto
```

The action still carries `validation_command: make mobile-deploy-preflight`, so consumers know how to verify it. The top-level `next_action` and operator action remain validation-aware with `; rerun make mobile-deploy-preflight`.

## Scope

In scope:

- `mobile_deploy_preflight_evidence` device action command selection.
- Focused backend tests for the backend URL device action.
- Roadmap/spec/plan.

Out of scope:

- Running or modifying Xcode signing.
- Writing real `Deployment.local.xcconfig`.
- Changing `write_deploy_local_config.sh`.
- Changing final provider/key handoffs.

## Acceptance

- A blocked `backend_base_url` device action no longer uses the raw error text as its command when the auto writer is available.
- The backend URL action uses `DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto`.
- Safety flags remain `writes_ios_deploy_config: false` for evidence generation.
- Focused backend tests pass.
