# P0.418 Final Handoff Configured Device Actions Design

## Context

`final_handoff_index_report` is the final Mac-side index that gathers local rehearsal evidence and configured-provider preflight evidence for the physical iPhone handoff. The configured preflight report now exposes a sanitized top-level `device_action_bundle`, but `final_handoff_index_report.source_reports["final_configured_preflight"]` only includes status, kind, command, and freshness metadata from the computed report.

That means the handoff index can show the configured preflight lane is blocked, but the computed source row does not carry the iPhone action bundle that already exists upstream. Operators reviewing the index have to jump into the child report to see the configured iPhone action sequence.

## Goal

Expose `final_configured_preflight.device_action_bundle` inside the computed `final_handoff_index_report.source_reports` entry for `final_configured_preflight`.

## Scope

- Copy the already-computed configured preflight `device_action_bundle` into the configured source report summary when it exists.
- Preserve the existing top-level handoff index device bundle preference. Local demo launch remains the preferred top-level source when available.
- Preserve existing lane status, first blocker, next action, operator actions, and command sequence behavior.
- Add focused TDD coverage to `tests/test_final_handoff_index.py`.
- Refresh `services/backend/.local/final-handoff-index.json` and verify the configured source row contains the bundle.
- Add the P0.418 roadmap entry.

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, Sculpteo, or any live provider.
- Do not run configured final acceptance, live 3D evaluation, live NPC evaluation, or print quote calls.
- Do not run Xcode, signing, Keychain, global Mac configuration, or physical iPhone deployment.
- Do not write backend `.env`, iOS deploy config, provider keys, or final resource values.
- Do not change the handoff index top-level bundle source preference.

## Report Contract

`final_handoff_index_report.source_reports` entry with `id == "final_configured_preflight"` will include `device_action_bundle` when the computed configured preflight report contains one.

The copied bundle keeps the upstream configured preflight contract:

- `id`: `final_configured_preflight_device_actions`
- `source_report`: `configured_ios_deploy_runbook`
- `status`: normalized source status
- `actions`: sanitized device actions from the configured iOS deploy runbook
- `first_action`: the first configured iPhone handoff action, including validation command
- `summary`: action counts for ready, missing, blocked, manual, live, provider calls, global actions, and Xcode/signing actions
- `safety`: no commands run, no provider calls, no global mutation, no Keychain writes, no backend env writes, no iOS deploy config writes, and no Xcode/signing execution from the report builder

If the configured preflight report ever lacks a bundle, the handoff index leaves the computed source row unchanged rather than synthesizing a fallback there. The existing top-level handoff index fallback bundle still handles missing top-level sources.
