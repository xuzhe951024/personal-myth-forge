# P0.296 Final Showcase Device Action Dedupe Design

## Goal

Keep the final showcase top-level operator checklist focused by dropping
source-prefixed duplicates for device actions when the same executable action is
already present as a bare command.

## Current Context

After P0.295, the final showcase checklist no longer repeats
`mobile_deploy_preflight_evidence` details. It can still repeat device actions
from saved reports:

```text
final_handoff_index: start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight
ios_device_launch_certificate: start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight
start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight
```

Only the bare action is needed for the top-level operator handoff. The prefixed
versions remain useful in lower-level reports, but they add noise to the final
checklist.

## Design

Extend the final-showcase-only dedupe helper in
`final_showcase_readiness.py`.

The helper should drop source-prefixed actions only when:

- The action root is one of the known device handoff roots:
  - `start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight`
  - `provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`
- A bare action with the same root exists anywhere in the same final showcase
  checklist.

The helper must keep:

- Provider/resource source-prefixed actions such as `final_rehearsal_local:
  ios_deploy_runbook_local: provide MESHY_API_KEY...`.
- `ios_device_launch_certificate: provide iOS deploy config in
  Deployment.local.xcconfig; rerun make mobile-deploy-preflight`, because it is
  a distinct aggregate action rather than a duplicate of the specific team id
  or backend server action.
- Lower-level report payloads and evidence details.

## Out of Scope

- No Xcode signing, Keychain mutation, iPhone deployment, provider calls, or
  paid print actions.
- No readiness status changes.
- No broad source-prefix stripping.

## Verification

- RED/GREEN final showcase test proving duplicate backend-device-demo source
  prefixes are removed while iOS deploy config source prefixes remain.
- Focused final showcase regression tests.
- Refresh local reports and confirm `final-showcase-readiness.json` keeps the
  bare backend-device-demo action once and removes the source-prefixed duplicate.
- Full lint, Swift checks, and backend tests before merge.
