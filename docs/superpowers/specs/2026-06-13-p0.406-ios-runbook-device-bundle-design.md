# P0.406 iOS Runbook Device Bundle Design

## Context

The final showcase readiness report already exposes `device_action_bundle`, which is the clearest iPhone-deployment evidence path for the final demo. The iOS deploy runbook is the lower-level report an operator naturally opens when the first blocker is `ios_deployable`, but it currently has no device action bundle. Its `first_blocker` can point at missing final provider resources, while the actual physical-iPhone blocker still needs an explicit Apple Team ID writer, iPhone-reachable backend URL preflight, Xcode evidence, and device rehearsal sequence.

## Goal

Add a sanitized `device_action_bundle` to the iOS deploy runbook so the physical iPhone deployment path is visible directly in the runbook without changing secrets, signing, Keychain, or provider state.

## Scope

- Add a top-level `device_action_bundle` field to `build_ios_deploy_runbook_report`.
- Derive the bundle from existing runbook input slots and command sequence.
- Cover missing deploy config, backend URL, Xcode build gate, and device rehearsal handoff.
- Add focused regression tests for the bundle structure and safety flags.
- Update the roadmap with P0.406.

## Non-Goals

- Do not change the existing `first_blocker` ordering.
- Do not run Xcode, sign the app, start the backend, call providers, or write ignored config.
- Do not expose secrets, raw local paths, payment links, or provider credentials.
- Do not duplicate the full final showcase readiness implementation.

## Design

The runbook will build a compact bundle with:

- `id`: `ios_deploy_runbook_device_actions`
- `label`: `iOS Deploy Runbook Device Actions`
- `status`: combined status from device-facing actions
- `actions`: ordered manual actions for deploy config, backend device demo, mobile preflight, Xcode build gate, and device rehearsal
- `first_action`: the first blocked/manual action
- `summary`: action counts by status, manual, global, provider, and xcode/signing
- `safety`: all commands are report-only and do not mutate global state from inside the report

The first action should prefer the concrete deploy writer when Apple Team ID is missing:

`DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto`

This keeps the operator handoff aligned with the current final showcase first blocker while preserving the runbook's existing `first_blocker` field for resource readiness diagnostics.
