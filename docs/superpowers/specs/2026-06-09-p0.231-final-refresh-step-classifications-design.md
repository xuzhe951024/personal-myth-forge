# P0.231 Final Refresh Step Classifications Design

## Context

`final-local-report-refresh` is the safe one-command local evidence refresh path. It writes all relevant ignored `.local` reports without live provider calls, Xcode signing, keychain writes, or resource application. Its step rows currently include `blocker_hint`, but the most useful blocker fields are nested. Operator-facing readers that list step rows must know to inspect the nested shape to explain why a step is blocked.

## Goal

Promote blocker hint fields onto each refresh step:

- `classification`
- `command`
- `detail`

Keep `blocker_hint` unchanged for compatibility. Ready steps should expose empty strings for these fields, so every step has a stable shape.

## Scope

This only changes the JSON report shape for `final-local-report-refresh`. It does not change status logic, command execution, provider behavior, resource writes, Xcode behavior, or iOS deployment behavior.

## Architecture

Extend `_run_step` to compute `blocker_hint` once, then copy sanitized `classification`, `command`, and `detail` strings into the step row. Do the same for unexpected failed steps, using `classification="step_failed"`.

## Testing

Use TDD:

1. Add failing tests that assert blocked refresh steps expose top-level blocker fields.
2. Add failing tests that ready steps expose stable empty strings.
3. Implement minimal report-shape changes.
4. Verify focused backend tests, full backend lint/test, Swift checks, and refreshed local reports.
