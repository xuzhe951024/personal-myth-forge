# P0.358 Showcase Print Handoff Action Design

## Goal

Make the `print_fulfillment` capability in `final-showcase-readiness` expose the concrete configured print handoff action, validation command, and cost-consent requirement.

## Context

The print fulfillment readiness report already distinguishes local print proof from configured third-party fulfillment. When local proof is ready but a configured Treatstock quote is missing, the child report emits:

```text
after explicit Treatstock cost consent, save a sanitized services/backend/.local/print-quote-configured.json from POST /v1/print-quotes
validation: make print-fulfillment-readiness
requires_cost_consent: true
```

The final showcase `print_fulfillment` capability currently collapses this to:

```text
make print-fulfillment-readiness
```

That hides the actual operator handoff and the cost-consent gate from the final showcase capability table.

## Design

For `_print_fulfillment_capability`:

- Ready reports keep the existing ready command.
- Missing/incomplete reports keep the existing `make print-fulfillment-readiness` command.
- Partial or blocked reports with `next_action.command` promote that command to the capability row.
- If the child next action has `validation_command`, include it on the capability row.
- If the child next action has `requires_cost_consent`, include it on the capability row.

This preserves the existing print readiness report as the source of truth and only makes the final showcase summary more actionable.

## Scope

In scope:

- Update `services/backend/src/myth_forge_api/final_showcase_readiness.py`.
- Extend focused tests in `services/backend/tests/test_final_showcase_readiness.py`.
- Add this spec, an implementation plan, and a roadmap entry.
- Refresh safe ignored `.local` evidence after verification.

Out of scope:

- Calling Treatstock, Meshy, OpenAI, Xcode, Keychain, or any paid provider.
- Writing real quote evidence or real API keys.
- Changing print fulfillment readiness semantics.

## Acceptance Criteria

- Partial `print_fulfillment` capability exposes the configured Treatstock quote handoff command.
- The same row exposes `validation_command: make print-fulfillment-readiness`.
- The same row exposes `requires_cost_consent: true`.
- Ready and missing print fulfillment paths keep existing behavior.
- Focused backend tests, lint, and diff checks pass.
