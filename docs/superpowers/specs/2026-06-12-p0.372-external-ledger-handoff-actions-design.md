# P0.372 External Ledger Handoff Actions Design

## Goal

Make `final-external-action-ledger` expose concrete provider and Treatstock print handoff actions instead of generic live-cost approval strings.

## Current Evidence

- `final-showcase-readiness`, `final-demo-launch-local`, `final-handoff-index`, `ios-device-launch-certificate`, and `ios-device-launch-rehearsal` all surface:
  - `make provider-handoff; rerun make live-provider-evidence`
  - `after explicit Treatstock cost consent, save a sanitized services/backend/.local/print-quote-configured.json from POST /v1/print-quotes; rerun make print-fulfillment-readiness`
- `final-external-action-ledger.json` still says `approve live provider cost before make live-provider-evidence` and `approve live provider cost before make print-fulfillment-readiness`.
- The external action ledger is the safest place for final resource/key/cost handoff, so it should use the same concrete action language as the rest of the final chain.

## Behavior

`final_external_action_ledger` should:

- Keep missing resource inputs first.
- For `run_live_provider_evidence`, emit `make provider-handoff; rerun make live-provider-evidence`.
- For `refresh_configured_treatstock_quote`, emit the configured Treatstock quote handoff from `print_fulfillment_readiness.next_action`, including validation.
- Keep cost-consent metadata on the action rows.
- Preserve apply-preview-before-apply filtering, iOS deploy validation commands, and safety guarantees.

## Non-Goals

- Do not run live provider calls or Treatstock requests.
- Do not write keys or local resource files.
- Do not mark live actions ready.
- Do not change first blocker or resource input ordering.

## Acceptance

- A focused RED/GREEN test proves the external ledger top-level actions contain the concrete provider and Treatstock handoff strings.
- Existing external action ledger tests remain green.
- Adjacent final showcase/demo tests remain green.
- Refreshed `.local/final-external-action-ledger.json` shows concrete handoff actions while staying blocked on missing resources.
