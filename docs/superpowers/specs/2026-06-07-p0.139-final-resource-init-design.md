# P0.139 Final Resource Init Design

## Context

The final showcase readiness flow now reports the real remaining blockers:
provider keys, print provider resources, iOS deploy values, and live provider
evidence. The resource interface exists, but the first operator action is still
a manual copy:

```text
copy services/backend/final-resources.env.example to services/backend/.local/final-resources.env
```

That is easy to mistype and keeps the "final resource handoff" less executable
than the rest of the Make-based launch path.

## Goal

Add a safe `make final-resource-init` command that initializes the ignored final
resource intake file from the checked-in template. This makes the API/key/iOS
resource interface concrete for final handoff without requiring real keys now.

## Behavior

Create `services/backend/scripts/init_final_resources.sh`.

The script must:

- copy `services/backend/final-resources.env.example` to
  `services/backend/.local/final-resources.env` when the destination is missing
- create `services/backend/.local` if needed
- refuse to overwrite an existing destination
- refuse to proceed if the destination is tracked by git
- refuse to proceed if the template is missing
- print only paths and next commands, never secret values
- avoid backend `.env`, iOS xcconfig, keychain, Xcode, signing, provider calls,
  payment calls, or global configuration

Add a root Make target:

```make
final-resource-init:
	cd services/backend && sh scripts/init_final_resources.sh
```

## Readiness Integration

Update operator actions that currently tell the operator to copy the template by
hand. Prefer:

```text
run make final-resource-init
```

Keep the detailed destination path in the output/report for review, but route
the primary action through the Make target.

## Safety

This command writes one ignored repo-local file only. It does not fill secrets,
does not mutate global state, and does not run provider or Xcode commands.

## Testing

Use TDD:

1. Script test fails first because `init_final_resources.sh` and the Make target
   do not exist.
2. Script test proves missing destination is created from the template and stdout
   recommends `make final-resources-preflight`.
3. Script test proves existing destination is not overwritten.
4. Script test proves a tracked destination is refused.
5. Readiness tests prove missing-resource operator actions include
   `make final-resource-init`.
6. Source acceptance checks prove the new command is part of the source gate.

Run backend lint/test and mobile Swift checks before merge.
