# P0.269 iOS Rehearsal Resource Actions Design

## Context

The top-level final showcase report now normalizes nested resource actions, but
the saved `ios-device-launch-rehearsal.json` still contains bare resource
handoff actions from `ios-deploy-runbook-local` and `final-configured-preflight`.
That makes the physical-device rehearsal source less actionable than the
reports that consume it.

## Goal

The iOS device launch rehearsal report should retain source prefixes while
adding the validation command for known final-resource handoff actions.

## Non-Goals

- Do not write provider keys, backend env files, or iOS deploy config.
- Do not run Meshy, OpenAI, Treatstock, Xcode, Keychain, or live provider calls.
- Do not change readiness status, blocker ordering, or external action policy.

## Acceptance

- A RED/GREEN test proves generated rehearsal actions include `; rerun make
  final-resources-preflight` for prefixed Meshy/OpenAI/deploy resource actions.
- A saved rehearsal readiness test proves the normalized actions survive through
  `build_ios_device_launch_rehearsal_readiness_report`.
- Focused rehearsal tests, lint, report refresh evidence, and diff checks pass.
