# P0.267b Final Refresh Resource Action Dedupe Design

## Context

P0.267 made final demo launch operator actions prefer resource handoff entries
that include the validation command. A main-workspace refresh still surfaced a
bare `provide MESHY_API_KEY in final-resources.env` action in the first eight
final-local report actions because `final-local-report-refresh` re-aggregates
step commands after the launch report.

## Goal

When a validation-aware action exists, the final local refresh report should not
surface the exact bare root action in its operator action list.

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, Xcode, Keychain, or any live provider.
- Do not write secrets or mutate global environment.
- Do not change blocker ordering beyond removing exact bare roots that already
  have a `; rerun ...` validation-aware equivalent.

## Acceptance

- A focused unit test proves `_operator_actions` drops a bare resource root when
  a validation-aware variant is present.
- The refreshed `final-local-report-refresh.json` has the Meshy validation-aware
  action and no bare Meshy action in the first eight operator actions.
- Focused refresh tests and backend lint pass.
