# P0.590 External Ledger Live Device Action Pruning Design

## Goal

Keep the final external action ledger's top-level operator handoff focused on live Meshy/OpenAI consent when live provider evidence is the first blocker.

## Context

`final-showcase-readiness`, `final-demo-launch-local`, and `final-local-report-refresh` now all surface the guarded live-provider command as the only top-level action when local evidence is otherwise ready:

```text
PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make final-acceptance-configured; rerun make live-provider-evidence
```

`final-external-action-ledger` still listed a device runtime fallback after that live action:

```text
start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight
```

That device action remains useful when iPhone runtime evidence is the blocker, but it is not part of resolving a live Meshy/OpenAI evidence blocker.

## Design

Add a narrow final pruning step in `myth_forge_api.final_external_action_ledger`:

- detect when the first blocker is live-provider related
- remove device-specific operator fallbacks from the top-level operator action list
- preserve configured live evidence validation actions
- preserve existing device action behavior when device readiness is the first blocker

The structured `action_groups` and `actions_by_id` remain unchanged, so the ledger still records device runtime actions for audit and UI consumers.

## Scope

In scope:

- external action ledger top-level operator action pruning
- focused RED/GREEN coverage for live-first plus backend-device-demo fallback
- full external ledger regression tests
- roadmap entry for P0.590

Out of scope:

- live provider calls
- iOS signing, Xcode, or physical device actions
- changing action group membership
- changing normal device-first action ordering

## Verification

This phase is valid when:

- RED proves a live-first ledger still includes `backend-device-demo`
- GREEN proves live-first top-level actions omit device fallbacks
- existing device-first tests still show backend-device-demo where appropriate
- refreshed `.local/final-external-action-ledger.json` keeps the live provider command first without backend-device-demo in top-level operator actions
