# P0.529 Print Quote Request Preflight Design

## Context

`make print-quote-configured` is the guarded Treatstock quote handoff for the final showcase. It requires an ignored local input file at `services/backend/.local/print-quote-request-configured.json`, then writes sanitized quote evidence to `services/backend/.local/print-quote-configured.json`.

Current `print-fulfillment-readiness` reports the missing output quote as the next action, but it does not distinguish a missing or invalid request file from a missing quote result. That leaves the operator with a command that can still fail before reaching the provider guard.

## Decision

Add a read-only configured quote request preflight check to `print_fulfillment_readiness`.

The check should:

- inspect only `services/backend/.local/print-quote-request-configured.json`;
- never call Treatstock or any provider;
- validate that the file is JSON and can satisfy the existing `PrintQuoteRequest` contract;
- surface a blocked first action when the request file is missing or invalid;
- keep configured quote evidence as the next partial action once the request preflight is ready.

## Non-Goals

- Do not generate a request file automatically.
- Do not add API keys or provider credentials.
- Do not run real Treatstock calls.
- Do not change checkout/order placement behavior.

## Acceptance

- Focused tests prove missing request file blocks before the provider quote command.
- Focused tests prove a valid request file restores the existing configured quote partial handoff.
- `print-fulfillment-readiness` still reports provider calls as false and cost consent required only for the guarded configured quote action.
