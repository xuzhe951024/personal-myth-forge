# P0.331 Existing Xcode Evidence Action Normalization Design

## Context

P0.330 added `first_blocker` and `next_action` to newly generated
`mobile_xcode_build_evidence_report` payloads. The main workspace still had an
older ignored `.local/mobile-xcode-build-evidence.json` report, and
`final-local-report-refresh` preserves existing reports without rewriting them.

That preservation is correct because it avoids running Xcode, but stale reports
without action fields make the current handoff harder for later sessions to
consume.

## Decision

When `final-local-report-refresh` finds an existing mobile Xcode evidence
report, normalize its schema before returning it:

- Ready reports get `first_blocker: null` and `next_action: null`.
- Blocked reports promote the first check into `first_blocker`.
- The blocker command reuses the existing first `operator_actions` entry when
  present.
- Missing validation command is `make mobile-xcode-build-evidence`.
- `next_action` mirrors `first_blocker` with `source: "first_blocker"`.

This keeps the local evidence current without invoking Xcode or changing any
recorded safety fields.

## Safety

This phase does not run Xcode, accept Apple SDK licenses, modify signing
configuration, write Keychain, call providers, or place secrets in repo files.

## Acceptance

- Existing blocked mobile Xcode evidence is preserved and gains action fields.
- Existing ready mobile Xcode evidence is preserved and gains null action
  fields.
- Synthetic safe Xcode evidence behavior from P0.330 remains unchanged.
