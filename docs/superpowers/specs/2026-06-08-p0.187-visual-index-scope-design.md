# P0.187 Visual Index Scope Design

## Goal

Keep the top-level iOS showcase/source acceptance gates aligned with the current visual regression scope after P0.186 added the configured acceptance command visual proof.

## Current Behavior

The default visual regression manifest now checks 20 iPhone-sized artifacts and includes `p0.186_configured_acceptance_command_visual`. The lower-level visual regression tests cover that artifact directly. The higher-level iOS showcase acceptance and mobile ProjectChecks still only require the older visual index strings and README copy still says the visual gate checks ten static artifacts. That means final source acceptance can pass even if the operator-facing documentation and broad source gate do not mention the new configured acceptance command proof.

## Approach

Add one narrow source-acceptance slice that treats P0.186 as part of the final visual index contract:

- README must describe the visual gate as 20 static 390x844 iPhone artifacts.
- README must explicitly mention the configured final acceptance command visual proof.
- `ios_showcase_acceptance.py` must require the P0.186 artifact id in the visual index gate.
- `PersonalMythForgeMobileProjectChecks` must require the same README and manifest strings.

No runtime behavior changes are needed. This is a scope freshness guard so final reviewers see the same visual coverage in docs, backend acceptance, and mobile project checks.

## Safety

This iteration is read-only at runtime. It does not run configured final acceptance, call live providers, write secrets, start servers, run Xcode signing, or mutate global state.

## Acceptance

- Red tests prove the current top-level gates do not require the P0.186 artifact/README copy.
- Green tests pass after updating README and source gates.
- Existing visual regression still reports 20 passed artifacts.
- Backend lint/test and Swift project checks remain green.
