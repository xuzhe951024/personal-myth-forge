# P0.516 Handoff Index Concrete Next Action Design

## Context

After P0.511 through P0.515, the main final launch reports list the local iPhone execution chain before provider/key work:

1. `DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`
2. `start backend-device-demo before device checks: make backend-device-demo; rerun make mobile-deploy-preflight`
3. `set PMF_BACKEND_BASE_URL to an iPhone-reachable LAN URL; rerun make mobile-deploy-preflight`

`final-handoff-index` already exposes that chain in `operator_actions`, but its `first_blocker` and `next_action` still point to the broader lane command `make final-rehearsal-local`. That is less useful for the final operator handoff because the top-level next action should be copy-pasteable and aligned with the concrete device blocker.

## Decision

P0.516 updates only `final_handoff_index` blocker command selection.

When a blocked required lane has a concrete `operator_action`, the report should use that action as:

- `first_blocker.command`
- `next_action.command`

For the local rehearsal lane, source prefixes such as `final_demo_launch_local:` must be stripped using the existing final handoff action normalization path. If no lane operator action exists, the report keeps the existing lane command fallback.

## Validation Command

If a promoted command includes `; rerun make ...`, `next_action.validation_command` should use the last rerun target. For example:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

gets:

```text
validation_command = make mobile-deploy-preflight
```

Missing-source fallback remains unchanged:

```text
first_blocker.command = make final-rehearsal-local
next_action.validation_command = make final-rehearsal-local
```

## Boundaries

- No real provider calls.
- No Xcode signing, Keychain writes, global mutation, or paid print/provider action.
- No change to lane status, source report freshness, operator action ordering, or safety fields.
- No local ignored config values are written by implementation.
