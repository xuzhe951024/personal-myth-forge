# P0.468 Device Evidence Safe Wrappers Design

## Context

The current final showcase blocker is physical iPhone deploy evidence. Two Make targets closest to that blocker successfully write local JSON evidence but still fail Make when the report status is expectedly blocked:

- `make mobile-deploy-preflight-evidence`
- `make final-showcase-readiness`

This makes local evidence refresh brittle. Operators need these report targets to behave like the existing safe final report wrappers: a blocked report is usable evidence, while unexpected command failures should still fail.

## Decision

Add project-local wrapper scripts for those two report targets. Each script will run the current CLI command, write the same `.local/*.json` output, accept exit code `0` or `2`, and fail on any other exit code.

Update the Make targets to call the wrappers. This keeps CLI semantics unchanged and only makes Make evidence refresh safe.

## Scope

- Add `services/backend/scripts/write_mobile_deploy_preflight_evidence.sh`.
- Add `services/backend/scripts/write_final_showcase_readiness.sh`.
- Update the two Make targets.
- Add focused script tests, dry-run tests, and executable-bit tests.
- Update source-gate expectations so final acceptance recognizes wrapper-backed evidence generation.
- Update the roadmap.

## Out Of Scope

- No real `DEVELOPMENT_TEAM` value.
- No Xcode signing, Keychain, or device build.
- No live provider calls or paid external API use.
- No change to report status calculation.

## Acceptance

- Wrapper tests prove exit code `2` is accepted and exit code `1` fails.
- `make -n mobile-deploy-preflight-evidence` and `make -n final-showcase-readiness` show wrapper scripts.
- Wrapper scripts are executable because Make runs them directly.
- Actual Make targets write `.local/mobile-deploy-preflight-evidence.json` and `.local/final-showcase-readiness.json` while exiting `0` in the current blocked state.
- Focused tests, `make backend-lint`, and `git diff --check` pass.
