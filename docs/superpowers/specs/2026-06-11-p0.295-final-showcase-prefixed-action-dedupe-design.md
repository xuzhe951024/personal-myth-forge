# P0.295 Final Showcase Prefixed Action Dedupe Design

## Goal

Remove the mobile-preflight-evidence source-prefixed duplicate operator action
from the top-level final showcase readiness checklist when the same executable
action already appears as a bare command.

## Current Context

`final-showcase-readiness` is still blocked by external iOS deploy resources.
Its top-level `operator_actions` now starts with concrete actions, but can still
include a duplicate source-prefixed variant:

```text
provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
final_rehearsal_local: mobile_deploy_preflight_evidence: provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable
```

The second line is useful evidence in the nested report, but as a top-level
operator action it repeats the first line and makes the iPhone launch checklist
look more like a log than a concise handoff.

## Design

Add final-showcase-only dedupe logic after report actions are normalized and
before the list is truncated.

The dedupe should:

- Keep the first bare action.
- Drop subsequent `final_rehearsal_local: mobile_deploy_preflight_evidence:`
  source-prefixed actions when their executable command root matches a bare
  action in the same checklist.
- Compare command roots before ` | ` detail suffixes.
- Preserve non-duplicate prefixed actions such as provider-specific resource
  slots and Xcode evidence commands.
- Leave nested source reports unchanged so diagnostics remain inspectable.

This belongs in `final_showcase_readiness.py`, not the shared
`operator_actions.py`, because source prefixes are useful in lower-level reports
and only become noisy in the final top-level checklist.

## Out of Scope

- No external resource writes, Xcode signing, real device deploys, provider
  calls, or paid print calls.
- No change to blocker ordering or readiness status.
- No broad rewrite of all operator action prefixes or provider/resource slot
  prefixes.

## Verification

- RED/GREEN test for final showcase top-level action dedupe.
- Focused final showcase and local report tests.
- `make final-local-report-refresh` remains externally blocked, but generated
  `final-showcase-readiness.json.operator_actions` should not contain the
  duplicate source-prefixed `DEVELOPMENT_TEAM` action when the bare action is
  present.
- Full lint, Swift checks, and backend test verification before merge.
