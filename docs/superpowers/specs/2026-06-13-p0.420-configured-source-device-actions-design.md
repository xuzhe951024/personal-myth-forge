# P0.420 Configured Source Device Actions Design

## Context

The configured/live handoff chain now exposes top-level `device_action_bundle` fields in:

- `final_configured_preflight_report`
- `final_configured_evidence_plan_report`
- `configured_live_evidence_bundle_report`

The next aggregate reports still summarize those child reports through local `_source_summary()` helpers that keep only basic status metadata. As a result:

- `final_configured_evidence_plan_report.source_reports.final_configured_preflight` drops the configured preflight device actions.
- `configured_live_evidence_bundle_report.source_reports.final_configured_evidence_plan` drops the configured evidence plan device actions.

The top-level bundles are available, but operators reading the configured source summaries cannot inspect the child iPhone handoff actions without opening the child reports.

## Goal

Preserve existing child `device_action_bundle` values inside configured-chain `source_reports` summaries.

## Scope

- Update `_source_summary()` in `final_configured_evidence_plan.py` to copy `report.device_action_bundle` when it exists.
- Update `_source_summary()` in `configured_live_evidence_bundle.py` to copy `report.device_action_bundle` when it exists.
- Add focused TDD coverage for both source-summary contracts.
- Refresh `services/backend/.local/final-configured-evidence-plan.json` and `services/backend/.local/configured-live-evidence-bundle.json`.
- Add the P0.420 roadmap entry.

## Non-Goals

- Do not synthesize a fallback bundle when a child report lacks one.
- Do not change top-level configured evidence plan or configured live evidence bundle `device_action_bundle` behavior.
- Do not change blocker selection, next actions, operator action ordering, live consent behavior, or evidence file readiness.
- Do not call Meshy, OpenAI, Treatstock, Sculpteo, or any live provider.
- Do not run Xcode, signing, Keychain, global Mac configuration, or physical iPhone deployment.
- Do not write backend `.env`, iOS deploy config, provider keys, or final resource values.

## Report Contract

For any child report passed through the configured-chain `_source_summary(report)`, the summary includes:

- `kind`
- `status`
- `summary`
- existing `first_blocker` where the helper already exposes it
- `device_action_bundle`, only when `report["device_action_bundle"]` is a dictionary

The copied bundle remains the child report's own sanitized bundle. The aggregating report does not rewrite the bundle id, source report, actions, first action, summary, or safety values.
