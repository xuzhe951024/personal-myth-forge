# P0.338 iOS Specific Deploy Actions Design

## Context

P0.334-P0.337 promoted the project-local iOS deploy writer command into the primary final handoff:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

Some final readiness reports still keep a generic action in the same top-level list:

```text
provide iOS deploy config in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
```

That generic action is useful when no more specific path is known, but it is noisy once the writer command is already present. In the final iPhone handoff it can look like a second required setup path instead of an older umbrella instruction.

## Goal

When an operator action list already contains the validated iOS deploy writer command, drop the generic validated iOS deploy config action from the same top-level list.

## Non-Goals

- Do not remove `provide iOS deploy config...` when the writer command is absent.
- Do not remove specific backend URL actions.
- Do not remove final-resources.env actions for `DEVELOPMENT_TEAM`, `PRODUCT_BUNDLE_IDENTIFIER`, or `PMF_BACKEND_BASE_URL`.
- Do not write any ignored deploy config, run Xcode, touch Keychain, call providers, or place print orders.

## Design

Extend the existing shared operator-action helper in `operator_actions.py`:

- Treat `DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight` as the concrete project-local iOS deploy handoff.
- If that concrete handoff exists in the same list, remove:
  - `provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`
  - `provide iOS deploy config in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`
- Compare command roots before any ` | detail` suffix, so the detail-bearing writer row still suppresses the generic action.

Existing readiness callers already use this helper in `final_showcase_readiness` and `ios_device_launch_rehearsal_readiness`. The full `ios_device_launch_rehearsal` report has its own top-level action aggregation and must also call the helper after stripping source prefixes.

## Acceptance Criteria

- `final-showcase-readiness.json.operator_actions` does not include the generic iOS deploy config action when the writer action is present.
- `ios-device-launch-rehearsal.json.operator_actions` follows the same rule.
- Generic iOS deploy config actions remain present when no writer action exists.
- Specific backend URL and final-resources actions remain present.
- Focused tests, lint, and full backend tests pass.
