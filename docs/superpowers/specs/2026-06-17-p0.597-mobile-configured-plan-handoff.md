# P0.597 Mobile Configured Plan Handoff

## Context

Backend final showcase readiness now exposes capability-specific configured live
actions. The iPhone Final Showcase summary already receives configured evidence plan
rows, but the Provider Handoff stage only considers live evidence, resource handoff,
apply preview, and configured bundle rows.

That can hide the concrete guarded command the operator should run next.

## Decision

The iPhone Provider Handoff stage should include configured evidence plan rows when
building its provider handoff detail. If live evidence is blocked and the configured
plan has a guarded live-provider command, the stage should surface that concrete
command.

## Acceptance

- Provider Handoff detail can show
  `PMF_ALLOW_LIVE_PROVIDER_CALLS=1 make backend-evaluate-3d-configured; rerun make final-configured-evidence-plan`.
- Ready live evidence still keeps the Provider Handoff stage ready.
- Unsafe text remains sanitized by the existing summary sanitization path.
