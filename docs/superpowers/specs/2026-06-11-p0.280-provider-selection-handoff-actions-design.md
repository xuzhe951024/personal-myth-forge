# P0.280 Provider Selection Handoff Actions Design

## Context

The final handoff reports now point backend API keys at the ignored
`final-resources.env` file and include `make final-resources-preflight`.
However, `resource_handoff.py` still emits provider selection actions as bare
manual text:

- `set THREE_D_PROVIDER=meshy`
- `set NPC_PROVIDER=openai`

Those actions can also appear through stale saved `.local` reports that final
showcase readiness aggregates. They are not actual final resource inputs; the
checked-in `final-resources.env` flow intentionally keeps only secrets and
deploy values there, then `make final-apply-resources` writes
`THREE_D_PROVIDER=meshy` and `NPC_PROVIDER=openai` into the ignored backend
`.env` through `write_backend_env.sh`.

## Goal

Replace bare provider selection operator actions with the existing safe apply
handoff: `run make final-apply-resources to apply the filled resource bundle`.

## Non-Goals

- Do not add `THREE_D_PROVIDER` or `NPC_PROVIDER` to `final-resources.env`.
- Do not change final resource preflight accepted keys.
- Do not write or modify ignored local resource/config files.
- Do not call Meshy, OpenAI, Treatstock, Xcode, Keychain, or any paid/global
  provider flow.

## Design

Add a provider-selection normalization rule in
`myth_forge_api.operator_actions.normalize_operator_action()` so any stale or
nested report action equal to `set THREE_D_PROVIDER=meshy` or
`set NPC_PROVIDER=openai` becomes
`run make final-apply-resources to apply the filled resource bundle`.

Update `resource_handoff._operator_actions()` to emit that apply action once
when either core provider selector is not ready. Keep backend key actions and
iOS deploy actions unchanged, including their validation commands. This avoids
asking the operator to manually edit provider variables while preserving the
current final resources contract.

## Acceptance

- RED/GREEN tests prove `resource_handoff` does not emit the bare provider
  selector actions and emits a single final-apply action when provider selectors
  are not ready.
- RED/GREEN tests prove `normalize_operator_action()` converts stale provider
  selector text into the final-apply action.
- Existing Meshy/OpenAI/Treatstock key actions still point to
  `final-resources.env` and retain `make final-resources-preflight`.
- Refreshed `.local/resource-handoff.json` and final showcase readiness no
  longer expose the bare provider selector actions.
