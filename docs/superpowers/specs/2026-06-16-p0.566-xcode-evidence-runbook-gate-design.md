# P0.566 Xcode Evidence Runbook Gate Design

## Problem

The local handoff reports can have a fresh `mobile-xcode-build-evidence.json` with `status=ready`, while `resource_handoff` and `ios_deploy_runbook` still model the Xcode build gate as manual. That keeps final showcase readiness in a partial iOS deploy state even after the safe Xcode evidence wrapper has proven the build gate.

## Goal

When saved Xcode build evidence is ready, downstream resource and iOS deploy reports should treat the Xcode gate as ready. When the evidence is missing or blocked, they should keep the current safe manual action that asks the operator to run `make mobile-xcode-build-evidence`.

## Approach

- Add a small saved-evidence reader for `services/backend/.local/mobile-xcode-build-evidence.json`.
- In `resource_handoff`, set `APPLE_SDK_LICENSE` to ready when the saved evidence is ready.
- In `ios_deploy_runbook`, set the `xcode_build_gate` command step to ready when the saved evidence is ready.
- Preserve the existing manual fallback when evidence is missing, malformed, or blocked.

## Safety

This phase only reads ignored local JSON evidence. It does not run Xcode, write signing state, call providers, or touch keychain/global machine state.

## Tests

Add failing tests that write ready and blocked Xcode evidence fixtures. Verify ready evidence removes stale manual Xcode actions, while blocked evidence still surfaces the blocked Xcode action.
