# P0.13 iOS App Compile Gate Design

## Problem

P0.12 wired real media inputs into the SwiftUI app shell, but the local
verification still relied on source-string checks for `apps/mobile/ios/App`.
That allowed an app-source compile blocker to slip through review before it was
caught manually. The final product needs iOS deployment, so the next step is to
make app-source compilation a regular local gate without changing global Xcode
settings.

The current machine has Command Line Tools selected, so `xcodebuild` cannot run
without a global developer-directory change. P0.13 therefore adds a SwiftPM
compile gate for the app source. It does not claim simulator, device, archive,
signing, or TestFlight readiness.

## Design

Add a SwiftPM executable target named `PersonalMythForgeMobileAppCompileCheck`
that points at `apps/mobile/ios/App`, excludes `Info.plist`, and depends on
`PersonalMythForgeMobileCore`. Building that product compiles the SwiftUI app
source under SwiftPM on the local macOS toolchain, catching undefined symbols,
signature drift between `ForgeRootView` and child views, and SwiftUI API
availability issues that source-string checks cannot catch.

Keep the existing Xcode project as the eventual iOS deployment artifact. The
SwiftPM app compile target is a local no-Xcode guard, not a replacement for the
Xcode app target.

## Required Source Adjustments

The temporary compile spike exposed two app-source compatibility issues:

- `textInputAutocapitalization(.never)` is not available in the macOS SwiftPM
  compile context.
- The two-argument SwiftUI `onChange` overload used by P0.12 requires macOS 14,
  while the package currently declares macOS 13.

P0.13 should fix these without changing iOS behavior:

- Wrap iOS-only text input capitalization behind a small `View` helper that is
  a no-op outside iOS.
- Use the one-argument `onChange(of:)` form that compiles against the package
  macOS 13 baseline while preserving the same state transitions.

## Verification

The new gate is:

```bash
swift build --package-path apps/mobile/ios --product PersonalMythForgeMobileAppCompileCheck
```

The full P0.13 verification set remains:

```bash
swift run --package-path apps/mobile/ios PersonalMythForgeMobileProjectChecks
swift run --package-path apps/mobile/ios PersonalMythForgeMobileCoreContractTests
swift build --package-path apps/mobile/ios --product PersonalMythForgeMobileAppCompileCheck
make backend-lint
make backend-test
```

The visual regression artifact should explain that this iteration strengthens
the iOS deployability gate but is still not a simulator or device screenshot.

## Non-Goals

- Do not run or require `xcode-select`.
- Do not run simulator, device, archive, signing, or deployment commands.
- Do not add real ARKit mesh capture or iOS 3D rendering in this iteration.
- Do not move provider API keys into the mobile app.
