P0.483 Deploy Writer Normalization Design

Goal

Make final operator handoff surfaces prefer the project-local iOS deploy writer command over stale manual Team ID edit actions and duplicate writer spellings.

Context

The current final evidence correctly identifies the first iOS blocker as:

`DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto`

However, final operator surfaces still include older or duplicate variants:

- `provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight`
- `run DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`

This weakens the final handoff because the operator sees both the safe project-local writer path and a stale direct-edit path for the same Team ID blocker.

Non-Goals

- Do not remove bundle id or backend URL actions; those can still be valid independent iOS deploy inputs.
- Do not run Xcode signing, write Keychain, call providers, or change real device state.
- Do not change deploy writer scripts or make targets.
- Do not mark iOS deploy ready without physical-device evidence.

Design

Use the existing shared `prefer_project_local_ios_deploy_handoff_actions()` in:

- `final_external_action_ledger._operator_actions()`
- `final_launch_closure_packet._operator_actions()`

Also teach `normalize_operator_action()` to canonicalize:

`run DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`

into:

`DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight`

The helper will then dedupe the writer action and remove the stale manual Team ID edit action while preserving bundle id, backend URL, provider, print, and Xcode handoff actions.

Acceptance Criteria

- `normalize_operator_action()` canonicalizes the `run DEVELOPMENT_TEAM=... make mobile-write-deploy-config-auto` variant.
- Final external action ledger top-level `operator_actions` start with the writer command and do not include the manual Team ID Deployment.local action when the writer command exists.
- Final launch closure packet top-level `operator_actions` start with the writer command and do not include either the manual Team ID Deployment.local action or the `run DEVELOPMENT_TEAM=...` writer duplicate.
- Bundle id and backend URL deploy actions remain present when still required.
- Safe local evidence refresh still reports blocked until external iOS/provider resources are supplied.

Safety

This phase changes only report action normalization and prioritization. Verification may write ignored `.local` JSON through safe wrappers. It must not set real `DEVELOPMENT_TEAM`, call providers, run Xcode signing, or write global state.
