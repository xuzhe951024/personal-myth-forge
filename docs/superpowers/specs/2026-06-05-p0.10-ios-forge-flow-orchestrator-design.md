# P0.10 iOS Forge Flow Orchestrator Design

## Context

P0.9 made iOS scan capture drafts and scan upload payloads verifiable, but the
mobile core still has no single operation that turns a draft into a ready myth
session. The SwiftUI app shell also still contains no-op button bodies, so the
mobile client cannot yet demonstrate the capture-to-myth backend loop.

The backend already supports `POST /v1/object-captures` and
`POST /v1/myth-sessions/from-capture`. The iOS core already has
`CaptureDraft`, `PersonalMythForgeAPIClient`, response models, and a reducer.
P0.10 connects those pieces without requiring real API keys, global Xcode
changes, or device camera access.

## Goal

Add a SwiftPM-tested forge flow orchestrator that validates a `CaptureDraft`,
uploads the capture, creates a myth session from the returned capture id, and
emits the same phase transitions the SwiftUI app needs for a clickable demo.

## Non-Goals

- Real camera, PhotosPicker, fileImporter, ARKit, LiDAR, or Object Capture
  runtime capture.
- Unity, RealityKit, or SceneKit 3D rendering.
- Meshy image-to-3D, multi-image-to-3D, or provider API-key integration.
- Persistent NPC memory or long-running OpenAI Agents SDK orchestration.

These remain required for the final objective, but P0.10 focuses on the mobile
backend loop that those future pieces will call.

## Approach

Create a small `ForgeFlowAPI` protocol in the mobile core with only the two
methods needed for the from-capture loop:

- `uploadObjectCapture(metadata:media:)`
- `createMythSessionFromCapture(captureId:context:)`

`PersonalMythForgeAPIClient` conforms to the protocol. Tests can use a fake API
without reaching the network.

Create `ForgeFlowService` with:

```swift
public func forge(
    draft: CaptureDraft,
    context: ContextCapsule,
    stateChanged: (ForgeFlowState) -> Void = { _ in }
) async -> ForgeFlowState
```

The service:

1. Validates the draft locally.
2. Emits `setObjectMetadata` and `setContextCapsule`.
3. Emits `beginUpload`.
4. Calls upload.
5. Emits `captureUploaded`.
6. Calls from-capture session creation.
7. Emits `sessionCreated`.

If draft validation fails, it returns `.failed(.invalidCaptureDraft(...))`
before any network call. If upload or session creation fails, it returns
`.failed(error)` after the last successful phase.

## SwiftUI Source Wiring

The Xcode app shell remains source-checked rather than simulator-compiled in
this environment. P0.10 changes the app source so the capture and forge buttons
are closures, and `ForgeRootView` owns a `ForgeFlowService` plus a small local
sample media draft. This does not make real media picking available; it removes
the no-op bodies and proves the intended button-to-service seam exists for the
future device iteration.

Project checks enforce:

- `CaptureFormView` exposes `chooseCapture` and `forgeMyth` closures.
- `ForgeRootView` constructs `ForgeFlowService`.
- `ForgeRootView` calls `forgeService.forge`.
- App source no longer contains the previous future no-op button comments.

## Backend Regression

Add a backend regression for an `arkit_scan` capture containing a GLB scan asset
and one reference image, followed by `POST /v1/myth-sessions/from-capture`. The
result must be ready and include both media refs in the object card symbolic
reading. This proves the server-side from-capture loop handles scan bundles as a
real session input.

## Validation

Local validation remains no-global-Xcode:

```bash
swift run --package-path apps/mobile/ios PersonalMythForgeMobileCoreContractTests
swift run --package-path apps/mobile/ios PersonalMythForgeMobileProjectChecks
make backend-lint
make backend-test
```

Visual evidence uses a static storyboard, not simulator evidence. It should show
the service-driven mobile loop states: draft ready, uploading capture, creating
session, ready with generated asset/NPC/world output.

## Acceptance Criteria

- Swift tests prove the orchestrator uploads first, creates a session second,
  and reaches `.ready(session)`.
- Swift tests prove upload failure stops before session creation.
- Swift tests prove invalid drafts fail before network calls.
- Project checks prove the app source is wired to service closures instead of
  no-op button bodies.
- Backend tests prove an ARKit scan bundle can create a ready myth session.
- Docs and visual evidence describe P0.10 as mobile flow orchestration, not real
  camera, ARKit, or 3D rendering.
