# P0.70 Final Resources Preflight Design

## Context

The final handoff now uses one ignored file:

`services/backend/.local/final-resources.env`

`make final-apply-resources` validates that file and writes the ignored backend
`.env` plus iOS `Deployment.local.xcconfig`. That is the correct final apply
step, but it is still a write path. Before the final resource handoff, the
operator should be able to inspect the file for missing keys, unknown keys,
loopback backend URLs, and Treatstock dependencies without writing any config.

## Goal

Add a read-only final resources preflight report and surface it in
`final-demo-launch`.

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, Sculpteo, Apple, or Xcode services.
- Do not write `services/backend/.env`.
- Do not write `apps/mobile/ios/Config/Deployment.local.xcconfig`.
- Do not replace `make final-apply-resources`; this only checks the resource
  file before apply.
- Do not expose raw key values in reports, logs, docs, screenshots, or test
  failure messages.

## Behavior

Create a backend module that reads a final resources file and returns a
sanitized JSON report:

- kind: `final_resources_preflight_report`
- status: `ready`, `blocked`, or `missing`
- default path label: `services/backend/.local/final-resources.env`
- items for each accepted resource key
- unknown key list by key name only
- required missing keys
- optional key status
- print-provider dependency status
- iPhone backend URL loopback status
- safety flags confirming no writes, no provider calls, and no secret values

Required keys:

- `MESHY_API_KEY`
- `OPENAI_API_KEY`
- `DEVELOPMENT_TEAM`
- `PRODUCT_BUNDLE_IDENTIFIER`
- `PMF_BACKEND_BASE_URL`

Conditional key:

- `TREATSTOCK_API_KEY` is required when `PRINT_PROVIDER=treatstock`.

Accepted optional keys:

- `OPENAI_API_BASE_URL`
- `PRINT_PROVIDER`
- `TREATSTOCK_API_BASE_URL`
- `SCULPTEO_API_KEY`
- `CAPTURE_STORAGE_DIR`
- `MYTH_SESSION_STORAGE_DIR`

`PRINT_PROVIDER` defaults to `local` when omitted. Supported values are
`local` and `treatstock`.

Add CLI:

```bash
cd services/backend
uv run python -m myth_forge_api.cli final-resources-preflight \
  --repo-root ../.. \
  --output .local/final-resources-preflight.json
```

Add Make target:

```bash
make final-resources-preflight
```

Exit codes:

- `0` when status is `ready`
- `2` when status is `blocked` or `missing`
- `1` only for unexpected CLI errors

Integrate with `final-demo-launch`:

- include `final_resources_preflight` as a top-level report object
- set `apply_final_resources` phase status from the preflight when the
  preflight is missing or blocked
- list `make final-resources-preflight` before `make final-apply-resources`
  in final launch commands
- keep the phase command as `make final-apply-resources`
- keep local deterministic final acceptance runnable even when resource
  preflight is missing or blocked

## Acceptance

- Missing default file reports `missing`, exits `2`, and leaks no local absolute path.
- Unknown key reports `blocked`, lists the key name, exits `2`, and leaks no key values.
- Loopback `PMF_BACKEND_BASE_URL` reports `blocked`.
- `PRINT_PROVIDER=treatstock` without `TREATSTOCK_API_KEY` reports `blocked`.
- A valid local-print resource file reports `ready`, exits `0`, and exposes only configured/redacted status.
- `final-demo-launch` includes the preflight report and reflects missing/blocked preflight in `apply_final_resources`.
- Existing apply script behavior remains unchanged.
