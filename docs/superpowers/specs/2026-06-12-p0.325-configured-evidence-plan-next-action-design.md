# P0.325 Final Configured Evidence Plan Next Action Design

## Context

`final-configured-evidence-plan.json` is the plan for collecting the real-provider evidence needed by the final iOS demo: final resources, provider handoff, configured Meshy 3D evaluation, OpenAI NPC Agent evaluation, configured acceptance, configured launch, and live provider evidence.

The report currently returns `status: "blocked"` with useful `steps`, but top-level `first_blocker` and `next_action` are absent. That makes downstream launch and mobile summaries infer the next action from step arrays instead of reading one machine-readable action.

## Decision

Add top-level blocker/action fields to `final_configured_evidence_plan_report`:

- `first_blocker` is the first step whose status is not `ready`, in priority order `blocked`, `consent_required`, `ready_to_run`.
- `next_action` copies `first_blocker` and adds `source: "first_blocker"`.
- both include the step `id`, `label`, `status`, `command`, `detail`, `blocked_by`, `requires_live_provider_consent`, `may_call_live_provider`, `cost_risk`, `repo_local_write`, `evidence_status`, `evidence_path`, and `validation_command: "make final-configured-evidence-plan"`.
- ready reports set both fields to `null`.

This is report-only. It does not apply resources, call Meshy/OpenAI/Treatstock, run Xcode, write Keychain, place orders, or mutate global state.

## iOS Handoff

`FinalConfiguredEvidencePlanReport` decodes optional `first_blocker` and `next_action` using the existing step model. `FinalLaunchMobileSummary` shows a `Next action:` row before the existing first non-ready step row.

The summary continues to sanitize displayed text.

## Acceptance

- Backend tests prove blocked plans expose `first_blocker` and `next_action`.
- Backend tests prove consent-required plans expose the first consent step as `next_action`.
- Backend tests prove ready plans set both fields to `null`.
- Swift contract tests prove final launch payloads decode `final_configured_evidence_plan.next_action`.
- Swift summary tests prove the iPhone final launch view shows the configured evidence next action.
- `make final-configured-evidence-plan` still exits `2` in the current local missing-resource state and only writes ignored JSON.
