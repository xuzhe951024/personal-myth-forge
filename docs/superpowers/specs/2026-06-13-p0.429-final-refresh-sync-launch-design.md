# P0.429 Final Refresh Sync Launch Design

## Context

`final-demo-launch-local.json` is the iPhone-visible launch report. P0.428 added backend and mobile support for the `final_local_report_refresh` source, but the saved local report can remain stale because `final-rehearsal-local` writes `final-demo-launch-local.json` before `final-local-report-refresh-local` writes `final-local-report-refresh.json`.

## Decision

After `final-local-report-refresh-local` accepts the refresh report exit code, the script will run a final local demo launch sync and rewrite `services/backend/.local/final-demo-launch-local.json`. This keeps the final iPhone launch handoff current without adding provider calls, Xcode signing, keychain writes, or repo-tracked secrets.

## Alternatives Considered

- Reorder `final-rehearsal-local` so refresh runs before demo launch. This helps the aggregate target but does not fix direct `make final-local-report-refresh-local` usage.
- Add a new `final-demo-launch-sync-local` target. This is explicit, but it adds another operator command when the existing refresh command can safely perform the sync.
- Modify `write_final_local_report_refresh.sh`. This is the selected approach because the script already treats blocked refresh reports as accepted local evidence and is the most direct place to sync the downstream iPhone report.

## Behavior

- The script still writes `services/backend/.local/final-local-report-refresh.json`.
- If the refresh command exits with `0` or `2`, the script runs `myth_forge_api.cli final-demo-launch --mode local --repo-root ../.. --output .local/final-demo-launch-local.json`.
- If the sync exits with `0` or `2`, the script exits `0` and prints that both reports were written.
- If the refresh command fails with any other exit code, the script does not run the sync and exits with that code.
- If the sync command fails with any exit code other than `0` or `2`, the script exits with the sync code.

## Safety

The change only writes ignored files under `services/backend/.local/`. It does not read or write API keys, does not call live providers, does not run Xcode, does not mutate global Mac state, and does not perform print orders.

## Tests

`tests/test_final_rehearsal_scripts.py` will assert that `final-local-report-refresh-local` now invokes the refresh command first and a local final demo launch sync second. Existing script tests continue to prove accepted blocked reports exit successfully.
