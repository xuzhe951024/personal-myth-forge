# P0.148 Final Local Report Refresh Design

## Problem

The final demo handoff depends on many `.local/*.json` evidence reports. Most report
commands correctly return exit code `2` when blocked, while still writing a useful
report. That is good for strict CI-style gates, but poor for operator refresh: a
single blocked report can stop a Make chain and leave downstream reports stale.

The current workaround is spread across shell scripts. Some scripts also call
broader gates that can touch mobile/Xcode checks, which is too risky for an
automatic refresh workflow.

## Goal

Add one repo-local refresh workflow that regenerates the safe local evidence set
in dependency order, accepts expected blocked states, and writes a summary report
that tells the operator what is current and what remains blocked.

## Non-Goals

- Do not apply final resources or write backend `.env`.
- Do not write iOS `Deployment.local.xcconfig`.
- Do not run live Meshy, OpenAI, Treatstock, Sculpteo, or payment-provider calls.
- Do not run Xcode, signing, keychain, device install, Apple license, or global
  machine configuration.
- Do not replace the strict readiness commands; keep them strict for direct gates.

## Design

Create a Python module `final_local_report_refresh.py` with
`run_final_local_report_refresh(repo_root=None)`. It writes only ignored
`services/backend/.local/*.json` reports.

The refresh sequence uses direct Python report builders where possible instead of
shelling out through Make. For local evaluation reports, it uses the existing
local providers and evaluation suites:

- final resource requirements
- final resource repair preview
- final resource apply preview
- local 3D evaluation
- local NPC Agent evaluation
- visual regression
- safe final acceptance local report with mobile/Xcode command checks suppressed
- local final demo launch
- iOS deploy runbook
- final configured preflight
- final handoff index
- iOS device launch certificate
- iOS device launch rehearsal
- live provider evidence readiness report
- print fulfillment readiness report
- final external action ledger
- final showcase readiness

The final acceptance step uses `run_final_acceptance(..., provider_mode="local")`
with an injected command runner. The runner does not execute
`mobile-deploy-preflight` or `mobile-xcode-build`; it returns blocked command
results that make the report current without mutating global machine state.

Each step records:

- `id`
- `label`
- `status`
- `exit_code`
- `output`
- `kind`
- `summary`
- `accepted_blocked`
- `writes_repo_local_report`

The workflow exits `0` only when every step is ready/passed. It exits `2` when
one or more steps are blocked but every report was refreshed successfully. It
exits `1` when a step fails unexpectedly before writing a usable report.

## CLI and Make

Add:

- CLI: `final-local-report-refresh --repo-root ../.. --output .local/final-local-report-refresh.json`
- Make: `make final-local-report-refresh`

The Make target may exit `2` while still writing a useful report. That mirrors
other readiness commands.

## Safety

The workflow is allowed to write only repo-local ignored report files under
`services/backend/.local`. It does not write secrets or local absolute paths into
reports.

The report safety block must say:

- `live_provider_calls=False`
- `global_mutation=False`
- `xcode_or_signing=False`
- `keychain_writes=False`
- `writes_backend_env=False`
- `writes_ios_deploy_config=False`
- `writes_repo_local_reports=True`
- `provider_secrets_in_report=False`
- `local_paths_in_report=False`

## Testing

Use TDD:

- Unit test that a missing final resource file refreshes all safe reports, exits
  `2`, writes the expected output files, and does not leak paths/secrets.
- Unit test that a runner exception marks the refresh failed with exit code `1`
  and keeps safety flags false.
- CLI test that writes `.local/final-local-report-refresh.json`.
- Make target test that the target exists and points to the CLI.
- Regression test that the final showcase report is written after the rehearsal
  reports, so it reads fresh local evidence.
