# P0.527 Live Capability Validation Design

## Goal

Make final showcase partial capability rows for live 3D generation, OpenAI NPC Agent evidence, and print quote handoff expose copy-pasteable validation-aware commands.

## Context

P0.526 aligned the `provider_key_handoff` capability command with its validation command. After refreshing local evidence, the remaining partial capability rows still split their command and validation across two fields:

- `game_asset_3d_generation.command` is `make final-resource-apply-preview; rerun make provider-handoff`
- `ai_agent_npc.command` is `make final-resource-apply-preview; rerun make provider-handoff`
- `print_fulfillment.command` is `PMF_ALLOW_PRINT_PROVIDER_CALLS=1 make print-quote-configured`

Their `validation_command` fields are correct, and top-level `operator_actions` already include the full chains. The capability rows should be equally executable because mobile and JSON consumers can surface those rows directly.

## Design

When a final showcase capability row is partial and has both a command and a validation command, compose the command with `_operator_action_with_validation()`. This produces:

- `make final-resource-apply-preview; rerun make provider-handoff; rerun make live-provider-evidence`
- `PMF_ALLOW_PRINT_PROVIDER_CALLS=1 make print-quote-configured; rerun make print-fulfillment-readiness`

The existing helper avoids duplicate rerun suffixes, so already-expanded commands remain stable.

## Safety

This change is report-only. It does not execute Meshy, OpenAI, Treatstock, Xcode, Keychain, or signing flows. Live provider and print quote actions still require their existing explicit consent environment variables.

## Tests

Update focused final showcase readiness tests so RED proves the capability rows and nested `next_action` commands are currently missing their validation suffixes. GREEN requires the row commands, nested next actions, validation fields, and operator action dedupe behavior to remain consistent.
