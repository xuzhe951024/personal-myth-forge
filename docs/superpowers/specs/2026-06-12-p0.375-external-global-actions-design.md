# P0.375 External Global Actions Design

## Goal

Make `final-external-action-ledger` global/Xcode operator actions concrete enough for final Mac-side handoff.

## Current Evidence

`services/backend/.local/final-external-action-ledger.json` currently emits generic actions:

- `confirm global/manual action before accept Apple SDK license manually on this Mac`
- `confirm global/manual action before configure Apple Team ID, bundle id, certificates, and device trust manually`
- `confirm global/manual action before make mobile-xcode-build`

Other final readiness reports already use clearer evidence-oriented handoff strings such as `accept the Xcode license outside Codex, then rerun make mobile-xcode-build-evidence`.

## Behavior

The external action ledger should:

- Keep global action rows marked as `manual`, `global`, `xcode_or_signing`, and `requires_user_confirmation`.
- Emit concrete top-level operator actions for Apple SDK license, Apple signing setup, and Xcode build gate.
- Point each global action back to `make mobile-xcode-build-evidence` as the validation command.
- Avoid running Xcode, accepting licenses, writing Keychain, changing signing, or mutating global machine state.

## Non-Goals

- Do not execute `xcodebuild`, open Xcode, accept licenses, install certificates, or touch Keychain.
- Do not mark iOS deploy, Xcode build, or signing ready.
- Do not change provider key, Meshy/OpenAI, Treatstock, or print behavior.
- Do not change first blocker ordering.

## Acceptance

- A RED/GREEN test proves `final-external-action-ledger.operator_actions` contains the concrete Xcode/global handoff strings.
- Existing external action ledger tests remain green.
- Adjacent final showcase/local refresh tests remain green.
- Refreshed `.local/final-external-action-ledger.json` still reports no global mutation and no Xcode/signing execution.
