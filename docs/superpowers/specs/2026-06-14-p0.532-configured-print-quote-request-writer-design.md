# P0.532 Configured Print Quote Request Writer Design

## Context

The final showcase now correctly blocks print fulfillment on the missing configured Treatstock quote request file:

```text
services/backend/.local/print-quote-request-configured.json
```

The handoff currently tells the operator to prepare that file manually. That is safe, but weak: the next external step depends on exact `PrintQuoteRequest` JSON shape before the guarded `make print-quote-configured` provider call can run.

## Design

Add a repo-local writer for the configured print quote request:

```text
PRINT_SOURCE_ASSET_URI=https://... PRINT_CANDIDATE_URI=https://... make print-quote-request-configured
```

The target writes only ignored local evidence:

```text
services/backend/.local/print-quote-request-configured.json
```

It does not read keys, call Treatstock, create checkout links, place orders, mutate Xcode state, or write tracked files. It only validates operator-provided asset URLs and serializes a backend-compatible `PrintQuoteRequest`.

The generated request uses:

- `print_candidate.provider`: `treatstock`
- `print_candidate.format`: default `3mf`, with only `stl`, `ply`, and `3mf` accepted
- `quantity`: default `1`
- `material`: default `standard_resin`
- `finish`: default `matte`
- `ship_to_country`: default `US`
- `requires_user_approval`: `true`

## Handoff Copy

Replace the missing-request action:

```text
prepare services/backend/.local/print-quote-request-configured.json; rerun make print-fulfillment-readiness
```

with the executable safe local handoff:

```text
PRINT_SOURCE_ASSET_URI=https://... PRINT_CANDIDATE_URI=https://... make print-quote-request-configured; rerun make print-fulfillment-readiness
```

When that request file exists and the configured quote is still missing, the next action remains the cost-consent guarded provider call:

```text
PMF_ALLOW_PRINT_PROVIDER_CALLS=1 make print-quote-configured; rerun make print-fulfillment-readiness
```

## Scope

In scope:

- CLI subcommand to write `PrintQuoteRequest` JSON.
- Make target and shell wrapper.
- Print fulfillment readiness command/action text.
- Operator action normalization for the new request writer target.
- Backend focused tests and mobile contract fixture updates where they assert final print handoff text.

Out of scope:

- Calling Treatstock.
- Generating or uploading a live print candidate.
- Writing provider keys or user resources into tracked files.
- Changing the downstream guarded quote command.

## Acceptance

- CLI writes a valid `PrintQuoteRequest` JSON from explicit source/candidate URLs.
- CLI rejects non-http(s) source or candidate URLs.
- Make dry run exposes `print-quote-request-configured` without the provider consent guard.
- Missing-request readiness action uses the executable writer target.
- Existing guarded provider-call behavior remains unchanged after the request exists.
- Focused backend and mobile contract tests pass.
