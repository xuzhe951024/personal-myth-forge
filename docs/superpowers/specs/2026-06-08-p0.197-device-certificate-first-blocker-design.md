# P0.197 Device Certificate First Blocker Design

## Purpose

The final demo launch report now exposes a canonical `first_blocker`, but `ios_device_launch_certificate` compresses that report into a nested `final_demo_launch` summary without carrying the blocker forward. The final device certificate is one of the last operator-facing artifacts before real iPhone deployment, so losing the blocker forces operators to inspect another JSON file.

## Scope

- Add `first_blocker` to `ios_device_launch_certificate_report.final_demo_launch`.
- Preserve only sanitized summary fields already emitted by `final_demo_launch_report.first_blocker`.
- Keep the certificate read-only; no new commands are executed.
- Add backend tests and iOS showcase source gates for this field.

## Acceptance

- A generated final demo launch report with `first_blocker` produces a device certificate whose `final_demo_launch.first_blocker` contains id, status, classification, command, detail, source, and source_id.
- Reports without a blocker keep `first_blocker` as `null`.
- Existing secret/path redaction still applies to the certificate JSON.
- `ios_showcase_acceptance` protects the source and test coverage.
