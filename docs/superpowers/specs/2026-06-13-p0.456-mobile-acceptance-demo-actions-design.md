# P0.456 Mobile Acceptance Demo Actions Design

## Context

P0.455 added an `external_actions` checkpoint to the iPhone demo script and showcase autopilot. The Swift contract tests and project checks now protect that behavior, but the backend `mobile_final_launch_readiness_acceptance` gate still does not require the DemoScript or ShowcaseAutopilot external-actions path. Final readiness evidence should cover the guided demo path, not only the static Swift project check.

## Goal

Extend mobile final launch readiness acceptance so it verifies the guided iPhone demo script and autopilot both include the final external action ledger handoff before final launch.

## Scope

- Add source requirements for `DemoScript.swift`:
  - `external_actions` step id.
  - `externalActionsStep` builder.
- Add source requirements for `ShowcaseAutopilot.swift`:
  - `script.step(id: "external_actions")`.
  - `Check Actions` operator button title.
- Add source requirements for contract tests:
  - `testDemoScriptShowsBlockedExternalActionsBeforeFinalLaunch`.
  - `testDemoScriptCompletesReadyExternalActionsBeforeFinalLaunch`.
  - `testShowcaseAutopilotBlocksOnExternalActions`.
- Update the backend acceptance test to assert these new requirement ids.

## Non-Goals

- No changes to Swift runtime behavior.
- No `.local` report refresh.
- No provider keys, Xcode signing, iPhone device execution, Treatstock calls, or paid/live actions.

## Verification

- `cd services/backend && uv run pytest tests/test_mobile_final_launch_readiness_acceptance.py -q`
- `git diff --check`
