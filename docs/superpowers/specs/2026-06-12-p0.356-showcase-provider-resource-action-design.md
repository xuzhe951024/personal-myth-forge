# P0.356 Showcase Provider Resource Action Design

## Goal

Make the `provider_key_handoff` capability in `final-showcase-readiness` point at the concrete local resource blocker when final provider/iOS resources are incomplete.

## Context

The final showcase report already includes evidence from `final_resources_preflight_report`. That child report has precise `next_action` values such as:

```text
provide MESHY_API_KEY in final-resources.env
validation: make final-resources-preflight
```

The parent `provider_key_handoff` capability currently collapses any local resource blocker to:

```text
make final-resource-apply-preview
```

That is useful after resource values exist, but it is less direct when the first blocker is a missing required value.

## Design

When local provider/key handoff is blocked:

- If `final_resources` is blocked or missing and has a child `next_action.command`, the capability uses that command.
- If that child action has `validation_command`, the capability includes it as `validation_command`.
- If `final_resources` is ready but apply preview or resource handoff is blocked, the capability keeps the existing apply preview handoff.
- Live provider evidence and configured evidence bundle partial paths keep their existing commands.

This keeps the final showcase capability table aligned with the authoritative child preflight report without changing live provider behavior or writing secrets.

## Scope

In scope:

- Update `services/backend/src/myth_forge_api/final_showcase_readiness.py`.
- Add focused tests in `services/backend/tests/test_final_showcase_readiness.py`.
- Add this spec, an implementation plan, and a roadmap entry.
- Refresh safe ignored `.local` reports after verification.

Out of scope:

- Writing `.local/final-resources.env` with real API keys.
- Running Meshy, OpenAI, Treatstock, Xcode signing, Keychain, or paid provider actions.
- Changing final resources preflight semantics.
- Changing live provider or configured evidence bundle commands.

## Acceptance Criteria

- When `final_resources_preflight_report.next_action` is `provide MESHY_API_KEY in final-resources.env`, `provider_key_handoff.command` matches it.
- `provider_key_handoff.validation_command` is `make final-resources-preflight`.
- The capability still reports `status: blocked` and `classification: provider_handoff_incomplete`.
- Existing partial paths still use `make live-provider-evidence` and `make configured-live-evidence-bundle`.
- Focused backend tests, lint, and diff checks pass.
