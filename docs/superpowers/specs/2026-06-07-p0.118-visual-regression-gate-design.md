# P0.118 Visual Regression Gate Design

## Context

P0.117 added a real SceneKit load-proof state to the iPhone artifact preview, but it did not add the matching mobile visual regression artifact. The repository already contains many 390x844 static HTML/PNG evidence files, yet there is no focused machine gate that checks whether a new visual artifact exists, has the expected dimensions, includes required visible copy, and avoids leaking provider secrets or local paths.

## Goal

Add a repeatable visual regression gate for the mobile showcase evidence, and use the SceneKit load-proof preview as the first required artifact in that gate.

## Approach Options

1. **Static artifact only:** add `p0.118-scene-load-proof.html` and a PNG. This is fast but weak; the next iteration could forget to update or verify the screenshot.
2. **Static artifact plus ad hoc checks:** add files and a single pytest. This proves this one artifact but does not create a reusable path.
3. **Manifest-backed visual gate:** add a small backend module, CLI command, Make target, tests, and the P0.118 artifact. This is the recommended approach because it creates a repeatable pattern for later iOS visual regressions without external services.

## User-Facing Artifact

Create a 390x844 mobile evidence page:

- `docs/superpowers/verification/p0.118-scene-load-proof.html`
- `docs/superpowers/verification/assets/p0.118-scene-load-proof-390x844.png`
- `docs/superpowers/verification/2026-06-07-p0.118-scene-load-proof-regression.md`

The visual should show:

- artifact preview with NPC ritual markers.
- `SceneKit load proof: Loaded`.
- `SceneKit load proof: Conversion needed`.
- `SceneKit load proof: Failed`.
- retry state for failed parsing.
- no full local paths, provider keys, checkout URLs, or raw media/data URIs.

## Backend Gate

Add `myth_forge_api.visual_regression` with a compact manifest:

- `id`
- HTML path
- PNG path
- expected width/height
- required HTML text tokens
- forbidden text regexes

The checker should:

- read the HTML and PNG from the repo root.
- parse PNG dimensions using the PNG header, not Pillow or browser APIs.
- fail if the HTML or PNG is missing.
- fail if PNG dimensions are not `390x844`.
- fail if required visible-copy tokens are missing.
- fail if HTML or regression notes contain known unsafe patterns such as `sk-`, `Bearer`, `file://`, `/Users/`, `/tmp/`, `data:image`, `checkout_url`, or payment checkout URLs.
- return a JSON-safe report.

Expose the checker through:

- `python -m myth_forge_api.cli visual-regression --repo-root ../..`
- `make visual-regression`

## Testing

Use TDD against the backend module:

- missing artifact returns exit `1` and a failed check.
- valid fixture with a minimal PNG header returns exit `0`.
- wrong dimensions fail with actual dimensions.
- missing required text fails.
- unsafe text is redacted from report output and causes failure.
- CLI writes the JSON report when `--output` is provided.

Run the standard validation:

- `uv run --directory services/backend pytest tests/test_visual_regression.py -q`
- `make visual-regression`
- `make backend-lint`
- `make backend-test`
- `swift run --package-path apps/mobile/ios PersonalMythForgeMobileCoreContractTests`
- `swift run --package-path apps/mobile/ios PersonalMythForgeMobileProjectChecks`
- `swift build --package-path apps/mobile/ios --product PersonalMythForgeMobileAppCompileCheck`

## Non-Goals

- No live iOS simulator or device screenshots.
- No global browser installation.
- No Playwright or Chrome downloads.
- No provider calls.
- No visual scoring of 3D asset quality.
