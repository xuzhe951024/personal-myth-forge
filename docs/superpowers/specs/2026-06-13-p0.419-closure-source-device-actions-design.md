# P0.419 Final Launch Closure Source Device Actions Design

## Context

`final_launch_closure_packet_report` is the final Mac-side closure packet that combines resource inputs, safe local writes, iPhone device evidence, final showcase readiness, configured live evidence, and final acceptance readiness. Its top-level `device_action_bundle` already exposes the iPhone device evidence actions, but `source_reports` currently summarize child reports with only `kind`, `status`, and `summary`.

Two child reports already expose sanitized top-level device action bundles:

- `ios_device_evidence_bundle_report.device_action_bundle`
- `configured_live_evidence_bundle_report.device_action_bundle`

The closure packet currently drops those bundles in `source_reports`, so operators reviewing the final packet cannot inspect the child iPhone action evidence from the source summaries.

## Goal

Preserve existing child `device_action_bundle` fields inside `final_launch_closure_packet_report.source_reports` summaries.

## Scope

- Update `_source_summary()` in `final_launch_closure_packet.py` to copy `report.device_action_bundle` when it exists.
- Cover both `ios_device_evidence_bundle` and `configured_live_evidence_bundle` source summaries through one focused test.
- Preserve the existing top-level closure `device_action_bundle`, section status, first blocker, next action, operator actions, commands, and safety behavior.
- Refresh `services/backend/.local/final-launch-closure-packet.json` and verify both source summaries carry device bundles.
- Add the P0.419 roadmap entry.

## Non-Goals

- Do not synthesize new device bundles when a child report lacks one.
- Do not change the top-level closure packet `device_action_bundle` source preference.
- Do not run live provider evaluations, configured final acceptance, or print quote calls.
- Do not call Meshy, OpenAI, Treatstock, Sculpteo, or any live provider.
- Do not run Xcode, signing, Keychain, global Mac configuration, or physical iPhone deployment.
- Do not write backend `.env`, iOS deploy config, provider keys, or final resource values.

## Report Contract

For any child report passed through `_source_summary(report)`, the summary includes:

- `kind`
- `status`
- `summary`
- `device_action_bundle`, only when `report["device_action_bundle"]` is a dictionary

The copied bundle remains the child report's own sanitized bundle. The closure packet does not rewrite the bundle id, source report, actions, first action, summary, or safety values.

## Testing

Use TDD:

1. Run the existing focused baseline for `tests/test_final_launch_closure_packet.py`.
2. Add a failing test that expects `source_reports["ios_device_evidence_bundle"].device_action_bundle` and `source_reports["configured_live_evidence_bundle"].device_action_bundle`.
3. Confirm RED with `KeyError: 'device_action_bundle'`.
4. Implement the minimal copy in `_source_summary()`.
5. Confirm GREEN with the new focused test and then the full focused file.
6. Refresh `make final-launch-closure-packet`; exit `2` is expected while resources/evidence are blocked, but the saved JSON must include both source device bundles.
7. Run backend lint and backend tests before merging.
