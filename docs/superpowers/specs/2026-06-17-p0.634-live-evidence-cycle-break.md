# P0.634 Live Evidence Cycle Break

## Problem

After real Meshy and OpenAI configured evidence succeeds, `final-showcase-readiness`
can still report `live_3d_provider_unproven`.

The cycle is:

- `final-showcase-readiness` reads `live-provider-evidence`.
- `live-provider-evidence` treats `final-demo-launch-configured` partial as a blocker.
- `final-demo-launch-configured` embeds `final-showcase-readiness`, so it remains partial while
  `live-provider-evidence` is partial.

## Scope

Break the cycle inside `live-provider-evidence`. The report should prove live provider
readiness from provider handoff, configured 3D evaluation, configured NPC evaluation, and
configured final acceptance. The configured final demo launch report may remain useful
context, but its partial status must not block live provider proof.

## Acceptance

- A partial `final-demo-launch-configured` report does not make `live-provider-evidence`
  partial when provider handoff, 3D evaluation, NPC evaluation, and final acceptance are ready.
- Existing missing/blocked provider, 3D, NPC, and acceptance behavior is unchanged.
- Final configured evidence can become ready once the configured launch report is regenerated
  after the cycle is broken.
