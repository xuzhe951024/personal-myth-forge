# P0.128 Final External Action Ledger Design

## Problem

The final showcase readiness report correctly says the demo is still blocked or
partial, but the remaining work is not one kind of work. Some actions are safe
repo-local writes, some require user-provided secrets, some can spend Meshy,
OpenAI, or print provider credits, and some would mutate global Mac/Xcode or
Apple signing state.

Without a first-class ledger, the final handoff is harder to operate safely:
the operator has to infer which commands are safe for Codex to run and which
must wait for explicit user confirmation.

## Goals

- Add a read-only `final_external_action_ledger_report`.
- Group final external actions by responsibility:
  - `resource_inputs`
  - `safe_local_writes`
  - `live_provider_costs`
  - `global_machine_actions`
  - `device_runtime_actions`
- Redact provider secrets and local absolute paths.
- Report explicit safety flags proving the ledger does not run commands, write
  `.env`, write iOS deploy config, call providers, run Xcode, write keychain
  state, or mutate global machine state.
- Expose the ledger through the backend CLI and `make final-external-action-ledger`.
- Embed the ledger in `final_demo_launch_report` so the iPhone final launch
  payload has the same external action context as local operator reports.
- Add source acceptance coverage so future iterations cannot remove the ledger
  wiring silently.

## Non-Goals

- Do not call Meshy, OpenAI, Treatstock, Sculpteo, or any live provider.
- Do not run `make final-apply-resources`.
- Do not write `services/backend/.env`.
- Do not write `apps/mobile/ios/Config/Deployment.local.xcconfig`.
- Do not accept Apple SDK licenses, change signing, install to device, or touch
  keychain state.
- Do not add mobile UI rendering for this ledger in P0.128; this iteration only
  embeds the payload for the existing final launch API.

## Status Rules

- `blocked` if any required resource input is missing or blocked, or if any
  global machine action is blocked/manual.
- `partial` if required resource inputs are present but live provider cost
  consent or manual device actions remain.
- `ready` only when all required external actions are ready and no manual/live
  consent remains.

## Safety

The builder only imports existing read-only report builders and static command
metadata. It may inspect missing/ready states, but it must not execute shell
commands or write files. All local filesystem paths are represented as repo
relative paths or sanitized before returning the report.
