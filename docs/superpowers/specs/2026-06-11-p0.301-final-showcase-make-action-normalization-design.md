# P0.301 Final Showcase Make Action Normalization Design

## Goal

Make final-showcase top-level operator actions use copy-pastable `make ...`
commands when the action is only a source-prefixed make target from a nested
report.

## Current Context

After local evidence refresh, `final-showcase-readiness.json` still includes
actions such as:

```text
final_handoff_index: make final-configured-preflight
ios_device_launch_certificate: make final-handoff-index
```

Those strings preserve provenance, but they are not commands an operator can
copy into a terminal. The same report already carries source report evidence,
so the top-level `operator_actions` list should prioritize executable handoff
commands.

## Design

Add a final-showcase-specific normalization step after shared operator-action
normalization and validation-aware enrichment. If an action has a recognized
final-showcase source prefix and the stripped bare root is a make target, return
the bare make target. Examples:

```text
final_handoff_index: run make final-configured-preflight -> make final-configured-preflight
final_handoff_index: make final-demo-launch-configured -> make final-demo-launch-configured
ios_device_launch_certificate: run make final-handoff-index -> make final-handoff-index
ios_device_launch_certificate: make ios-deploy-runbook-local -> make ios-deploy-runbook-local
```

The normalization stays narrow:

- It only applies inside `final_showcase_readiness`.
- It only strips prefixes already recognized by
  `FINAL_SHOWCASE_SOURCE_ACTION_PREFIXES`.
- It only strips when the bare root starts with `make `.
- Non-make actions keep their existing source context, including deploy config,
  Xcode, resource, print, and provider handoff details.

## Out of Scope

- No changes to nested source report payloads.
- No changes to actual Make targets.
- No `.local` secrets, provider calls, Xcode signing, Keychain writes, or
  physical-device operations.

## Verification

- RED/GREEN test for `_report_operator_actions()` proving source-prefixed make
  targets become bare make targets.
- RED/GREEN final-showcase report test proving top-level actions contain bare

- `make final-showcase-readiness` still exits 2 while external iOS/provider
  evidence is missing, but the refreshed action list contains bare make targets
  for configured preflight, configured demo launch, final handoff index, and iOS
  deploy runbook.
- Final lint, Swift contract/project checks, and backend tests before merge.
