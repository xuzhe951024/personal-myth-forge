# P0.428 Final Local Refresh Mobile Source Design

## Goal

Expose the safe final-local-report-refresh evidence through `/v1/final-demo-launch` and show it in the iPhone Final Launch summary.

## Current State

`services/backend/.local/final-local-report-refresh.json` already carries the canonical safe refresh status, first blocker, showcase next action, device action bundle, operator actions, and safety flags. The final demo launch report does not include that source, so the iPhone Final Launch view cannot show whether the Mac-side safe refresh packet is present or what it says to do next.

## Approach

Add a read-only `final_local_report_refresh` field to the backend final demo launch report. The builder reads the existing ignored `.local/final-local-report-refresh.json` snapshot and sanitizes it with the same final demo launch report sanitizer. If the file is missing or invalid, the field becomes a safe `missing` or `blocked` placeholder with `make final-local-report-refresh-local` as the operator action.

On iOS, add compact Codable models for the refresh report fields the summary needs: `status`, `firstBlocker`, `nextAction`, `showcaseNextAction`, `deviceActionBundle`, `summary`, `operatorActions`, and `safety`. Add `localRefreshRows` to `FinalLaunchMobileSummary` and render one status row plus the first actionable command, preferring `showcaseNextAction`, then `nextAction`, then `firstBlocker`, then the device bundle first action, then operator actions.

## Safety

This iteration is decode and read-only presentation. It does not run `make final-local-report-refresh-local`, write ignored resource files, call live providers, call Xcode, touch Keychain, or place print orders. Mobile text passes through the existing Final Launch sanitizer.

## Testing

Backend tests should prove `build_final_demo_launch_report()` embeds a saved `final_local_report_refresh` report and emits a safe missing placeholder when the ignored snapshot is absent. Swift contract tests should decode the new field from the final launch fixture and show sanitized refresh rows in `FinalLaunchMobileSummary`.
