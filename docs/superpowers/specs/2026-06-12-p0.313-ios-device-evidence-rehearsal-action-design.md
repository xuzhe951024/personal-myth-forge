# P0.313 iOS Device Evidence Rehearsal Action Design

## Goal

Normalize the iOS device evidence bundle's launch rehearsal operator action to the canonical make target so the final iPhone handoff is directly executable and consistent with other final evidence reports.

## Context

The final goal is blocked on real iPhone evidence, Xcode signing configuration, and an iPhone-reachable backend URL. The iOS device evidence bundle already preserves the right blocker details, but its top-level operator action can still start with `run make ios-device-launch-rehearsal`. Recent P0 phases removed `run make ...` wrappers from adjacent final handoff reports, so this bundle should follow the same convention.

## Scope

This iteration only changes local report text normalization and tests. It does not run Xcode, write signing config, call providers, place orders, write secrets, or mutate global machine state.

## Behavior

When an operator action starts with `run make ios-device-launch-rehearsal`, it should normalize to `make ios-device-launch-rehearsal`.

If the action includes a detail suffix after ` | `, the suffix must be preserved. For example:

```text
run make ios-device-launch-rehearsal | provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM
```

normalizes to:

```text
make ios-device-launch-rehearsal | provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM
```

When the launch rehearsal readiness report is missing and the evidence bundle falls back to the child report's first action as slot detail, that detail must be normalized too. The generated evidence bundle must not contain `run make ios-device-launch-rehearsal` anywhere in its `operator_actions`.

## Testing

Add a unit test for the shared operator action normalizer and tighten the iOS device evidence bundle tests so they reject stale `run make ios-device-launch-rehearsal` text in both blocked-detail and missing-report paths. Refresh the local evidence bundle and verify the generated ignored JSON has the canonical action.
