# P0.258 Final Showcase Functional Concrete Action Design

## Goal

Make the final showcase `functional_regression` capability surface the concrete final acceptance preflight action instead of raw command stderr.

## Current Gap

P0.257 enriched final acceptance readiness blockers with `next_action`, but final showcase still summarizes blocked simple reports with `blocker.detail`. For mobile deploy preflight failures this can include multiline `make` stderr instead of the specific handoff action.

## Requirements

- Keep capability status, command, evidence, and blocker ordering unchanged.
- When the first report blocker has `next_action.command`, capability detail uses that action.
- If `next_action.validation_command` exists, capability detail tells the operator to rerun it.
- Include the child action detail when present.
- Preserve existing fallback behavior when no `next_action` exists.
- Do not run Xcode, provider calls, signing, Keychain writes, or paid actions.

## Non-Goals

- No changes to final acceptance execution.
- No changes to iOS UI models.
- No attempt to satisfy missing external values.
