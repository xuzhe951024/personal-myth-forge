# P0.340 iOS Sequence Writer Actions Design

## Context

P0.337-P0.339 cleaned the top-level final showcase and iOS rehearsal operator actions so the project-local deploy writer is the primary physical-device handoff:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

The full iOS device launch rehearsal report still exposes nested `sequence[].operator_actions`. Those nested rows can still contain the older generic action:

```text
provide iOS deploy config in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
```

This matters because the iPhone launch UI and Mac-side handoff tools can expand sequence steps. If the expanded row shows the old generic deploy config action while another row already shows the writer command, the operator sees two setup paths for the same deploy slot.

## Goal

When an iOS device launch rehearsal sequence already contains the concrete deploy writer action, remove old generic/manual iOS deploy config actions from nested `sequence[].operator_actions`.

## Non-Goals

- Do not remove generic deploy config actions when no writer action exists anywhere in the same sequence.
- Do not remove backend URL actions.
- Do not remove final-resources.env actions for API keys, Team ID, bundle ID, or backend URL.
- Do not run Xcode, write deploy config, touch Keychain, call live providers, or place print orders.

## Design

Add a sequence-level normalization pass in `ios_device_launch_rehearsal.py` after sequence rows are assembled and before details are calculated.

The pass scans every nested `operator_actions` list in the sequence. It uses `_top_level_operator_action()` to compare each nested action by its normalized command root, ignoring source prefixes such as `ios_device_launch_certificate:`.

If any nested action root is:

```text
DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; rerun make mobile-deploy-preflight
```

then each nested row drops actions whose normalized root is one of:

```text
provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
provide iOS deploy config in Deployment.local.xcconfig; rerun make mobile-deploy-preflight
```

The row keeps its original source-prefixed writer action when present. It also dedupes duplicate writer roots inside the same row, preferring the first detail-bearing writer action.

## Acceptance Criteria

- `ios-device-launch-rehearsal.json.sequence[].operator_actions` does not expose the old generic iOS deploy config action when another sequence row already contains the writer action.
- The writer action remains visible in the sequence row that supplied it.
- Generic deploy config actions remain visible when no writer action exists anywhere in the sequence.
- Top-level operator action behavior from P0.337-P0.339 remains unchanged.
- Focused iOS rehearsal tests, lint, diff checks, and backend tests pass.
