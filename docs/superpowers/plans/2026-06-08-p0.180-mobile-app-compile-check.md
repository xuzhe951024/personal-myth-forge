# P0.180 Mobile App Compile Check Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the SwiftPM mobile app compile-check executable compile the SwiftUI app source and exit cleanly when run.

**Architecture:** Keep `PersonalMythForgeMobileAppCompileCheck` pointed at the app source folder, but exclude the real SwiftUI app entry point from that SwiftPM target. Add a compile-check-only `@main` guarded by `PMF_APP_COMPILE_CHECK`, then protect the package settings with `PersonalMythForgeMobileProjectChecks`.

**Tech Stack:** Swift Package Manager executable targets, SwiftUI, Swift static project checks, backend lint/tests.

- Modify `apps/mobile/ios/Package.swift`: exclude the real app entry point from the compile-check target and define `PMF_APP_COMPILE_CHECK`.
- Create `apps/mobile/ios/App/AppCompileCheckMain.swift`: compile-check-only entry point that references `ForgeRootView` and exits.
