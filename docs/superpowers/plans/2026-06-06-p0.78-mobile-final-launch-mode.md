# P0.78 Mobile Final Launch Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let final resources configure the iPhone final launch report mode and let the app safely switch between local and configured final launch readiness views.

**Architecture:** Extend the existing final resource handoff with `PMF_FINAL_LAUNCH_MODE`, validate it as `local` or `configured`, and write it to the ignored iOS deployment config. Add a Swift mobile-core mode enum, expose the value through `Info.plist`/`AppConfiguration`, and wire a segmented mode control in `ForgeRootView` that reloads `/v1/final-demo-launch` without executing commands or live-provider calls.

**Tech Stack:** POSIX shell scripts, FastAPI backend acceptance helpers, pytest, Swift 6, SwiftPM, SwiftUI, static HTML visual evidence.
