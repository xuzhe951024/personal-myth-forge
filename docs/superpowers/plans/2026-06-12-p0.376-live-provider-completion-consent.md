# P0.376 Live Provider Completion Consent Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Clarify final showcase readiness rows so safe immediate provider handoff actions are not confused with downstream live provider consent required to complete 3D/NPC evidence.

**Architecture:** Keep the existing `requires_live_provider_consent` field tied to the immediate `command`. Add an optional `completion_requires_live_provider_consent` field on final showcase capability rows when a partial capability is blocked by incomplete live provider evidence.

**Tech Stack:** Python 3.10, FastAPI backend support modules, pytest, project-local Make targets.
