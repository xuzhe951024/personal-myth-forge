# P0.61 NPC Agent Provider Acceptance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a dedicated source/config acceptance gate proving that NPCs have an OpenAI AI Agent provider path with structured output, backend-only keys, readiness/resource handoff coverage, and final acceptance nesting.

**Architecture:** Create `npc_agent_provider_acceptance.py` using the existing source-only acceptance pattern, add focused tests, inject the runner into `final_acceptance`, then update docs and static verification evidence. The gate performs repository source scans only and never calls OpenAI.

**Tech Stack:** Python 3.10, pytest, FastAPI backend package, static HTML/PNG verification evidence, SwiftPM smoke checks.
