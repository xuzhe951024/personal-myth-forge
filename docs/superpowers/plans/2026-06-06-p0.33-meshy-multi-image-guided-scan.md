# P0.33 Meshy Multi-Image Guided Scan Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Route Meshy-backed guided scans with multiple prepared reference
images through Meshy's Multi-Image to 3D endpoint while preserving single-image
and text fallbacks.

**Architecture:** Extend the existing `MeshyThreeDProvider` routing logic: text
requests still use text-to-3D, one supported image uses image-to-3D, and two or
more supported prepared JPEG/PNG images use multi-image-to-3D capped at four
images. Keep capture storage private and prove from-capture requests forward
multiple source images without leaking raw media.

**Tech Stack:** Python 3.10, pytest, httpx MockTransport, FastAPI TestClient,
SwiftPM regression checks, static HTML evidence.

## Acceptance Criteria

- Single-image and text-to-3D behavior remains intact.
- Capture API tests prove guided scan from-capture forwards multiple prepared
  images without leaking raw media or local paths.
- Existing final acceptance, provider readiness, NPC agent, mobile compile, and
  deploy gates remain intact.
