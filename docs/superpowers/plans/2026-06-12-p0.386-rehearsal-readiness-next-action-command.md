> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Keep nested `ios_device_launch_rehearsal_readiness.next_action.command` values copy-pasteable while preserving detailed operator text.

**Architecture:** `ios_device_launch_rehearsal_readiness._next_action()` already builds the structured next action from the first operator action. Add one helper that strips detail suffixes only for that structured command assignment.

**Tech Stack:** Python report builder, pytest.

```python
def test_ios_device_launch_rehearsal_readiness_next_action_command_strips_detail_suffix(
    tmp_path: Path,
) -> None:
    repo_root = tmp_path / "repo"
    local_dir = repo_root / "services/backend/.local"
    local_dir.mkdir(parents=True)
    _write_json(
        local_dir / "ios-device-launch-rehearsal.json",
        {
            "kind": "ios_device_launch_rehearsal_report",
            "status": "blocked",
            "summary": {"ready": 2, "missing": 0, "blocked": 1},
            "sequence": [
                {
                    "id": "final_rehearsal_local",
                    "label": "Local final rehearsal",
                    "status": "blocked",
                    "classification": "step_blocked",
                    "command": "make final-rehearsal-local",
                    "detail": (
                        "Missing DEVELOPMENT_TEAM; "
                        "PMF_BACKEND_BASE_URL must be iPhone-reachable"
                    ),
                }
            ],
            "operator_actions": [
                (
                    "DEVELOPMENT_TEAM=YOUR_TEAM_ID "
                    "make mobile-write-deploy-config-auto; "
                    "rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM; "
                    "PMF_BACKEND_BASE_URL must be iPhone-reachable"
                )
            ],
            "commands": ["make ios-device-launch-rehearsal"],
            "safety": {
                "provider_calls": False,
                "xcode_or_signing": False,
                "keychain_writes": False,
            },
        },
    )

    result = build_ios_device_launch_rehearsal_readiness_report(repo_root=repo_root)

    assert result.report["operator_actions"][0] == (
        "DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; "
        "rerun make mobile-deploy-preflight | Missing DEVELOPMENT_TEAM; "
        "PMF_BACKEND_BASE_URL must be iPhone-reachable"
    )
    assert result.report["next_action"]["command"] == (
        "DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; "
        "rerun make mobile-deploy-preflight"
    )
    assert result.report["next_action"]["detail"] == (
        "Missing DEVELOPMENT_TEAM; PMF_BACKEND_BASE_URL must be iPhone-reachable"
    )
```

### Task 2: Implementation

**Files:**
- Modify: `services/backend/src/myth_forge_api/ios_device_launch_rehearsal_readiness.py`

- [ ] **Step 1: Add a structured command helper**

Add near `_next_action()`:

```python
def _structured_next_action_command(action: str) -> str:
    command, _separator, _detail = action.partition(" | ")
    return command.strip()
```

- [ ] **Step 2: Use the helper in `_next_action()`**

Change:

```python
        next_action["command"] = operator_actions[0]
```

to:

```python
        next_action["command"] = _structured_next_action_command(operator_actions[0])
```
