# P0.599 Configured Bundle Granular Actions

## Goal

Make `configured-live-evidence-bundle` top-level operator actions prefer the granular
configured evidence plan consent commands over the broader live evidence fallback.

## Steps

1. Inspect current git state, readiness reports, docs, configured bundle code, and
   focused tests.
2. Add a RED backend test that expects granular configured plan actions to lead
   `configured-live-evidence-bundle.operator_actions`.
3. Update configured bundle action prioritization narrowly.
4. Run focused backend tests, lint, and whitespace checks.
5. Refresh safe local evidence reports, commit, merge, and push.

## Safety

This phase only changes report action ordering and documentation. It must not run
live Meshy/OpenAI calls, write secrets, place print orders, or touch Xcode signing.
