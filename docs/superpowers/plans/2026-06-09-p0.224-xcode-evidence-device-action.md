# P0.224 Xcode Evidence Device Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Connect saved Xcode build evidence to the final showcase iOS device action bundle without running Xcode or changing global Apple tooling state.

**Architecture:** `final_showcase_readiness.py` will load the ignored `.local/mobile-xcode-build-evidence.json` report, expose it in top-level `evidence`, and use it to mark `resolve_xcode_build_gate` independently from preflight and launch rehearsal evidence. Swift already has optional evidence fields on `FinalShowcaseDeviceAction`; the contract fixture and summary tests will prove Xcode evidence renders on the device action row.

**Tech Stack:** Python report builders and pytest, Swift Codable/contract tests, static 390x844 visual regression evidence.
