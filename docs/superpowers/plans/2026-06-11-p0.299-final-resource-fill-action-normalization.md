> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Normalize final resource fill-guide operator actions into canonical validation-aware `provide <KEY> in final-resources.env` actions so aggregated handoff reports do not show duplicate fill/provide instructions for the same key.

**Architecture:** Extend `myth_forge_api.operator_actions.normalize_operator_action()` with a narrowly scoped fill-guide resource normalizer before final-resource validation is applied. Existing aggregators already call this helper, so final-local refresh can dedupe the canonical action without changing lower-level fill-guide report payloads.

**Tech Stack:** Python 3.10, pytest, FastAPI backend support modules, SwiftPM contract checks for iOS-facing readiness payload stability.

Add this helper near `_normalize_unblock_action()`:

```python
def _normalize_final_resource_fill_action(action: str) -> str | None:
    prefix = "fill "
    suffix = f" in {FINAL_RESOURCE_FILL_TARGET}"
    if not action.startswith(prefix) or not action.endswith(suffix):
        return None
    resource_id = action[len(prefix) : -len(suffix)].strip()
    canonical = f"provide {resource_id} in final-resources.env"
    if canonical not in FINAL_RESOURCE_VALIDATION_ACTION_ROOTS:
        return None
    return add_final_resource_validation_command(canonical)
```
