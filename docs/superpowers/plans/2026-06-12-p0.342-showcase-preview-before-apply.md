> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Keep `final-showcase-readiness` from surfacing direct resource apply writer actions while the apply-preview gate is still present.

**Architecture:** Add constants and a post-dedupe filter in `final_showcase_readiness.py`. The filter removes exact `make final-apply-resources` roots only when `make final-resource-apply-preview` is already in the same action list.

**Tech Stack:** Python 3.10, FastAPI backend package, pytest, ruff.

- [x] **Step 3: Add helper**

Add this helper after `_dedupe_operator_actions()`:

```python
def _prefer_apply_preview_before_apply(actions: list[str]) -> list[str]:
    action_roots = {_operator_action_bare_root(action) for action in actions}
    if FINAL_RESOURCE_APPLY_PREVIEW_ACTION not in action_roots:
        return actions
    return [
        action
        for action in actions
        if _operator_action_bare_root(action) != FINAL_RESOURCE_APPLY_ACTION
    ]
```
