# P0.13 iOS App Compile Gate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a local SwiftPM compile gate for the SwiftUI app source so iOS app-shell regressions are caught before simulator or device work is available.

**Architecture:** Add a SwiftPM executable target over `apps/mobile/ios/App` that depends on `PersonalMythForgeMobileCore`, then make the app source compile in that no-Xcode context without changing iOS runtime behavior. Keep source-string project checks as structural guards, but make app source compilation the stronger gate.

**Tech Stack:** Swift 6, SwiftPM, SwiftUI, PhotosUI, UniformTypeIdentifiers, existing FastAPI backend unchanged.

---

## Environment Constraint

Do not run `xcode-select`, simulator builds, device builds, archive builds,
signing commands, or any global Xcode configuration. P0.13 verification is
SwiftPM builds/checks, backend tests, source checks, and static visual evidence
only.

- [x] **Step 3: Replace direct text input capitalization modifiers**

In `CaptureFormView.swift`, replace both direct modifiers:

```swift

    .mobileTextInputAutocapitalizationDisabled()
TextField("Materials", text: $materials)

```

Add this helper at the bottom of `CaptureFormView.swift`:

```swift
private extension View {
    @ViewBuilder
    func mobileTextInputAutocapitalizationDisabled() -> some View {
        #if os(iOS)
        self.textInputAutocapitalization(.never)
        #else
        self
        #endif
    }
}
```
