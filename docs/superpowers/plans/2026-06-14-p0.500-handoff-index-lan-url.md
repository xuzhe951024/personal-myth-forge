# P0.500 Handoff Index LAN URL Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `final-handoff-index.json` preserve the iPhone-reachable backend URL handoff in its top-level operator actions.

**Architecture:** Extend `final_handoff_index._local_lane_handoff_actions()` to select concrete device handoff actions from saved final demo launch reports, keep enough prioritized saved source actions to carry those handoffs forward, then update final handoff prioritization so device handoff actions fit within the six visible actions. The change is report-only and does not affect provider calls, config writers, or source report generation.

**Tech Stack:** Python 3.10, pytest, ruff, FastAPI backend package.

- [ ] **Step 1: Update prioritization**

Change `_prioritize_final_handoff_operator_actions()` so it extracts LAN URL actions before provider and print actions, and only promotes `backend-device-demo` together with the LAN URL action:

```python
    lan_backend_url_actions = [
        action for action in rest if _is_lan_backend_url_action(action)
    ]
    backend_device_demo_actions = [
        action for action in rest if _is_backend_device_demo_action(action)
    ]
    device_actions = [
        *lan_backend_url_actions,
        *(backend_device_demo_actions if lan_backend_url_actions else []),
    ]
    provider_actions = [
        action for action in rest if _is_provider_handoff_action(action)
    ]
    print_actions = [action for action in rest if _is_print_handoff_action(action)]
    priority_actions = set(device_actions + provider_actions + print_actions)
    remaining = [action for action in rest if action not in priority_actions]
    return first_actions + device_actions + provider_actions + print_actions + remaining
```
