# P0.564 Backend Test Dotenv Isolation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prevent backend tests from loading real ignored provider keys and iOS deploy handoff files while preserving normal backend `.env` and local demo behavior.

**Architecture:** Add a small skip switch in `myth_forge_api.config.load_settings()`: explicit `env` mappings and `PMF_SKIP_DOTENV=1` bypass `load_dotenv()`. Make `backend-test` and backend pytest set the skip switch so tests stay deterministic even in a checkout with real Meshy/OpenAI keys. Update script-test fixture copying so ignored `Deployment.local.xcconfig` is not copied into temporary repos.

**Tech Stack:** Python 3.10, python-dotenv, pytest, Make.

def test_load_settings_explicit_env_ignores_dotenv_file(tmp_path, monkeypatch) -> None:
    monkeypatch.chdir(tmp_path)
    (tmp_path / ".env").write_text(
        "THREE_D_PROVIDER=meshy\nMESHY_API_KEY=real-secret\n",

    assert settings.three_d_provider == "local"

def test_load_settings_skip_dotenv_env_flag_ignores_dotenv_file(tmp_path, monkeypatch) -> None:
    monkeypatch.chdir(tmp_path)
    (tmp_path / ".env").write_text(

        encoding="utf-8",

    monkeypatch.delenv("NPC_PROVIDER", raising=False)
    monkeypatch.delenv("OPENAI_API_KEY", raising=False)

    assert settings.npc_provider == "local"
    assert settings.openai_api_key is None

### Task 2: Config Implementation

**Files:**
- Modify: `services/backend/src/myth_forge_api/config.py`

- [ ] **Step 1: Add skip helper**

Add:

```python
def _should_load_dotenv(env: Mapping[str, str] | None) -> bool:
    if env is not None:
        return False
    return os.environ.get("PMF_SKIP_DOTENV", "").strip().lower() not in {
        "1",
        "true",
        "yes",
    }
```

Then replace `load_dotenv()` with:

```python
if _should_load_dotenv(env):
    load_dotenv()
```
