# P0.16 3D Asset Variant Contract Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development or superpowers:executing-plans to
> implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for
> tracking.

**Goal:** Add a backward-compatible 3D asset variant contract so the backend can
return both a primary GLB game asset and an iOS SceneKit-loadable USDZ variant,
then make the mobile core prefer that variant for preview preparation.

**Architecture:** Keep provider credentials on the backend. Extend the
FastAPI/Pydantic contract with asset variants, request GLB+USDZ from Meshy when
available, and update Swift models plus asset preparation selection logic.

**Tech Stack:** FastAPI, Pydantic, httpx, pytest, Swift 6, SwiftPM, Foundation.

---

## Environment Constraint

Do not run `xcode-select`, simulator builds, device builds, archive builds,
signing commands, or any global Xcode configuration. P0.16 verification is
SwiftPM builds/checks, backend tests, source checks, and static visual evidence
only.
