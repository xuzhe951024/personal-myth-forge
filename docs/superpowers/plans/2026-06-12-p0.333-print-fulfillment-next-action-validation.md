# Print Fulfillment Next Action Validation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add validation and cost-consent metadata to print fulfillment next
actions.

**Architecture:** Keep the existing readiness checks and operator actions. Add
metadata in the `next_action` helper based on the blocker id and
classification, so downstream reports can reason about Treatstock cost consent
without parsing prose.

**Tech Stack:** Python 3.10, pytest, FastAPI backend reports, ignored `.local`
handoff JSON.
