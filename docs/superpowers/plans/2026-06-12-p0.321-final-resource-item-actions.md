# P0.321 Final Resource Item Actions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make every missing or blocked final resource preflight item carry concrete operator action metadata and show those actions in the iPhone final launch resource checklist.

**Architecture:** Reuse the backend's existing item blocker helpers as the single source for item-level `command`, `detail`, and `validation_command`. Extend the Swift Codable model with optional fields and append those sanitized fields to existing checklist rows.

**Tech Stack:** Python FastAPI support modules, pytest, SwiftPM, Swift Codable contract tests.

- [ ] **Step 3: Implement item action enrichment**

In `final_resources_preflight.py`, make `_items(...)` build the existing raw items, then pass them through a helper:

```python
def _items(...) -> list[dict[str, Any]]:
    items = [
        ...
    ]
    return [_with_item_action(item) for item in items]
```

Add:

```python
def _with_item_action(item: dict[str, Any]) -> dict[str, Any]:
    if not (
        (item["status"] == "missing" and item["required"])
        or item["status"] == "blocked"
    ):
        return item
    blocker = _item_blocker(item)
    enriched = dict(item)
    enriched["command"] = blocker["command"]
    enriched["detail"] = blocker["detail"]
    enriched["validation_command"] = blocker["validation_command"]
    return enriched
```
