# P0.328 Resource Handoff Next Action Plan

## Goal

Make `resource_handoff_report` expose the same top-level next-action contract
as the other final handoff reports, without touching real provider resources.

## Steps

1. Add backend RED tests in `services/backend/tests/test_resource_handoff.py`:
   - blocked report includes `first_blocker` for `MESHY_API_KEY`;
   - `next_action` mirrors it with `source: "first_blocker"`;
   - ready report has `first_blocker is None` and `next_action is None`.
2. Add Swift RED coverage in
   `apps/mobile/ios/Sources/PersonalMythForgeMobileCoreContractTests/main.swift`:
   - decode optional `resource_report.next_action`;
   - summary includes `Next action: ...`.
3. Implement backend helpers in
   `services/backend/src/myth_forge_api/resource_handoff.py`.
4. Extend Swift `ResourceHandoffReport` with optional action fields and summary
   row formatting in `FinalLaunchMobileSummary.swift`.
5. Update `docs/iteration-roadmap.md` with P0.328.
6. Refresh ignored resource handoff evidence with `make resource-handoff` and
   confirm it remains blocked for missing local resources.
7. Verify focused tests, lint/full backend tests, Swift checks, and
   `git diff --check`.

## Commands

```bash
cd services/backend && uv run pytest tests/test_resource_handoff.py -q
swift run --package-path apps/mobile/ios PersonalMythForgeMobileCoreContractTests
make resource-handoff
make backend-lint

swift run --package-path apps/mobile/ios PersonalMythForgeMobileProjectChecks

```

## Safety

All commands are source/local-report checks. No live provider calls, real print
orders, signing changes, or global config mutations are part of this phase.
