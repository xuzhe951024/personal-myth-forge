# P0.165 Provider Handoff Evidence Refresh Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make provider handoff a first-class saved evidence artifact for final local and live-provider acceptance.

**Architecture:** Extend the existing provider handoff report with status metadata, expose a Make target that writes `.local/provider-handoff.json`, include the report in final local refresh, and improve live-provider evidence detail when core real providers are not ready. No live provider calls or resource writes are introduced.

**Tech Stack:** Python 3.10, pytest, Ruff, Make, existing backend report builders.

---
