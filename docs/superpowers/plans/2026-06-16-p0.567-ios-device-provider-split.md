# P0.567 iOS Device Provider Split Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Keep live provider evidence actions from becoming the primary iOS device deploy blocker when a concrete device action is available.

**Architecture:** Preserve existing report schemas and action normalization helpers. Add device-first filtering in the iOS device launch certificate and rehearsal readiness paths so provider/live actions remain visible but do not drive `next_action` for device gates.

**Tech Stack:** Python 3.10, FastAPI backend report builders, pytest, project Make targets.
