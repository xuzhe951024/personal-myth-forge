# P0.273 Resource Source Actions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make source-level final resource reports emit validation-aware top-level operator actions for known `final-resources.env` edits.

**Architecture:** Reuse `add_final_resource_validation_command()` in the three source report action aggregators before dedupe. This preserves status logic, blocker fields, and child report payloads while making top-level operator handoff consistent.

**Tech Stack:** Python, FastAPI backend report builders, pytest, ruff.
