# P0.223 Preflight Evidence Device Actions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Connect mobile deploy preflight evidence to final showcase device action rows without running device, Xcode, or provider commands.

**Architecture:** `final_showcase_readiness.py` will load the ignored `.local/mobile-deploy-preflight-evidence.json` report, expose its status in the top-level `evidence`, and use it to mark the backend/preflight device actions. Swift models will decode optional evidence fields and the Final Launch summary will render them next to the existing device actions.

**Tech Stack:** Python report builders and pytest, Swift Codable/contract tests, static 390x844 visual regression evidence.
