# P0.403 iOS Runbook Operator Actions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove stale synthetic `unblock ...` fallbacks from iOS deploy runbook operator actions while preserving concrete child handoff commands.

**Architecture:** Keep `command_sequence` as the full deployment runbook, but make `operator_actions` derive from source reports, input slots, and the manual Xcode gate. This prevents blocked apply/preflight steps from being shown as next actions before resource preview and device config are ready.

**Tech Stack:** Python 3.10, backend report builders, pytest.
