# P0.605 Top-Level Live Consent Metadata Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Surface live-provider and cost-consent flags on the top-level final handoff blocker and next-action payloads.

**Architecture:** Reuse the existing action metadata produced by configured live evidence and external action groups. Copy only explicit boolean fields from action objects into blockers, then let next-action generation preserve those fields.

**Tech Stack:** Python, FastAPI backend support modules, pytest, project Make targets.
