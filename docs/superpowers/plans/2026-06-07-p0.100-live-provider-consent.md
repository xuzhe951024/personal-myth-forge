# P0.100 Live Provider Consent Interface Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a read-only iPhone panel that summarizes real provider readiness, final resource handoff, and live-call consent before configured provider calls.

**Architecture:** Add a small mobile-core summary builder that consumes existing `ProviderReadinessResponse` and `FinalDemoLaunchReport` data. Render that summary in a focused SwiftUI view wired from `ForgeRootView`, then extend existing source gates and visual verification.

**Tech Stack:** Swift 6, SwiftUI, SwiftPM executable contract tests, FastAPI backend source-gate acceptance, static HTML visual regression.
