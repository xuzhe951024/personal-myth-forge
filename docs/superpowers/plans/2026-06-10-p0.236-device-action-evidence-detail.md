# P0.236 Device Action Evidence Detail Plan

## Scope

Improve the read-only final showcase `device_action_bundle` evidence detail fields so the first iOS device blocker tells the operator the most relevant local action.

## Steps

1. Baseline focused tests for `test_final_showcase_readiness.py`.
2. Add failing tests:
   - backend device action prefers backend URL or backend health details over signing details.
   - mobile deploy preflight action summarizes multiple blocker details.
3. Implement small helpers in `final_showcase_readiness.py`:
   - step-specific backend evidence detail selection.
   - bounded multi-check preflight evidence detail summary.
4. Run focused tests for final showcase readiness.
5. Run broader backend lint/test and Swift package checks if focused tests pass.
6. Refresh safe local reports with `make final-local-report-refresh-local`, accepting the expected exit 2 while external resources remain missing.
7. Merge back to `main`, commit, push, and clean up the P0.236 worktree.
