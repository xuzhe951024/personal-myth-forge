# P0.68 iOS Local Network Readiness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add source-verified iOS local network privacy readiness for physical iPhone backend demos.

**Architecture:** Extend existing source-only mobile project checks and iOS showcase acceptance to require `NSLocalNetworkUsageDescription`, then add the plist key. The change stays entirely in tracked source and does not run or mutate Xcode, signing, keychain, provider config, or global machine settings.

**Tech Stack:** SwiftPM project checker, iOS `Info.plist`, backend pytest source acceptance, static HTML evidence.

---

### Task 1: SwiftPM Project Check Gate

**Files:**
- Modify: `apps/mobile/ios/Sources/PersonalMythForgeMobileProjectChecks/main.swift`
- Modify later: `apps/mobile/ios/App/Info.plist`

- [ ] **Step 1: Write failing Swift project check.**

Add:

```swift
try requirePlistText(plist, key: "NSLocalNetworkUsageDescription")
```

near the existing camera/photo plist requirements.

- [ ] **Step 2: Run and confirm RED.**

Run:

```bash

```

Expected: fails with `Missing required Info.plist key: NSLocalNetworkUsageDescription`.

- [ ] **Step 3: Add the plist key.**

Add:

```xml

<string>Connect to the local Personal Myth Forge backend during an iPhone demo.</string>
```

- [ ] **Step 4: Re-run and confirm GREEN.**

Run:

```bash
swift run --package-path apps/mobile/ios PersonalMythForgeMobileProjectChecks
```

Expected: `PersonalMythForgeMobileProjectChecks passed`.
