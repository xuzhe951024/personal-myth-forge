# P0.343 External Ledger Preview Before Apply Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Keep `final-external-action-ledger` from surfacing direct resource apply writer actions while the apply-preview gate is still present.

**Architecture:** Add constants, promote the blocked apply-preview action into `operator_actions`, and then run a post-dedupe filter in `final_external_action_ledger.py`. The filter only removes exact `make final-apply-resources` roots when `make final-resource-apply-preview` is already listed.

**Tech Stack:** Python 3.10, FastAPI backend package, pytest, ruff.

- [x] **Step 3: Add helpers**

Add these helpers near `_dedupe()`:

```python
def _is_blocked_apply_preview_action(action: dict[str, Any]) -> bool:
    return (
        action.get("id") == "preview_final_resource_apply"
        and action.get("status") in {"missing", "blocked"}
    )


def _prefer_apply_preview_before_apply(actions: list[str]) -> list[str]:
    action_roots = {_operator_action_root(action) for action in actions}
    if FINAL_RESOURCE_APPLY_PREVIEW_ACTION not in action_roots:
        return actions
    return [
        action
        for action in actions
        if _operator_action_root(action) != FINAL_RESOURCE_APPLY_ACTION
    ]


def _operator_action_root(action: str) -> str:
    return action.split(" | ", 1)[0].strip()
```
