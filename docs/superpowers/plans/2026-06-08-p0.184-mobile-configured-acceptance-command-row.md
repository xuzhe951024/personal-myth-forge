# P0.184 Mobile Configured Acceptance Command Row Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Show `make final-acceptance-configured` in the iPhone Final Launch `Commands` rows whenever configured acceptance is available.

**Architecture:** Add a small private command-selection helper inside `FinalLaunchMobileSummaryBuilder`. The helper promotes the configured acceptance wrapper command for configured reports, fills the remaining slots from existing report command order, deduplicates, sanitizes, and keeps the existing four-row cap.

**Tech Stack:** Swift, SwiftPM contract tests, existing ProjectChecks source gate, local Makefile verification.

Add this private helper to `FinalLaunchMobileSummaryBuilder`:

```swift
private static func commandRows(from report: FinalDemoLaunchReport) -> [String] {
    let configuredAcceptanceCommand = "make final-acceptance-configured"
    var selected: [String] = []

    if isConfiguredMode(report.mode),
       report.commands.contains(configuredAcceptanceCommand) {
        selected.append(configuredAcceptanceCommand)
    }

    for command in report.commands where selected.count < 4 {
        if !selected.contains(command) {
            selected.append(command)
        }
    }

    return selected.map(sanitize)
}

private static func isConfiguredMode(_ mode: String) -> Bool {
    mode.trimmingCharacters(in: .whitespacesAndNewlines).lowercased() == "configured"
}
```
