# P0.163 Mobile Xcode Build Evidence Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an explicit, sanitized evidence command for the local iOS Xcode build gate.

**Architecture:** Mirror `mobile_deploy_preflight_evidence` with a focused backend module, CLI command, and Make target. Keep Xcode invocation explicit and separate from safe default refresh, while letting `ios_device_evidence_bundle` consume a saved report when present.

**Tech Stack:** Python 3.10, FastAPI backend package, pytest, Ruff, Make, shell-based existing iOS Xcode build gate.

---

## File Structure

- Create `services/backend/src/myth_forge_api/mobile_xcode_build_evidence.py`
  - Runs `apps/mobile/ios/scripts/xcode_build_gate.sh` through an injectable runner.
  - Builds a sanitized report with classification, bounded stdout/stderr, operator actions, and safety metadata.
- Create `services/backend/tests/test_mobile_xcode_build_evidence.py`
  - Covers ready, license blocker, missing Xcode blocker, runner injection, CLI output, and Make target exposure.
- Modify `services/backend/src/myth_forge_api/cli.py`
  - Add `mobile-xcode-build-evidence` subcommand and handler.
- Modify `Makefile`
  - Add `.PHONY: mobile-xcode-build-evidence` and command writing `.local/mobile-xcode-build-evidence.json`.
- Modify `services/backend/src/myth_forge_api/ios_device_evidence_bundle.py`
  - Read saved Xcode evidence if present and use it for the `xcode_build_gate` slot.
- Modify `services/backend/tests/test_ios_device_evidence_bundle.py`
  - Verify saved Xcode evidence overrides generic final acceptance details.
