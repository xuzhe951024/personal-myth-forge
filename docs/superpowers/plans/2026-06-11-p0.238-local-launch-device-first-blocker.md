# P0.238 Local Launch Device First Blocker Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `final-demo-launch-local` point its top-level first blocker at the local iPhone device path instead of provider-key resource apply when no-key local launch can continue.

**Architecture:** Keep launch phase generation unchanged. Add local-mode first-blocker selection logic that filters out the continuable `apply_final_resources` phase before choosing the first top-level phase blocker. In local mode, return the selected device phase directly so provider/resource nested hints stay in their embedded reports instead of polluting the device next action. Configured mode keeps the existing strict ordering.

**Tech Stack:** Python 3.10, FastAPI backend package, pytest, existing final demo launch report builders.

- [ ] **Step 2: Filter continuable apply phase in local mode**

Add helper:

```python
def _first_blocker_phases(*, mode: LaunchMode, phases: list[dict[str, Any]]) -> list[dict[str, Any]]:
    if mode != "local":
        return phases
    return [
        phase
        for phase in phases
        if not (
            phase.get("id") == "apply_final_resources"
            and phase.get("status") in {"missing", "blocked", "partial"}
        )
    ]
```

Use it before `_first_phase_blocker(...)`:

```python
candidate_phases = _first_blocker_phases(mode=mode, phases=phases)
```

Return local phase blockers without provider/resource nested hints:

```python
if phase_blocker is not None:
    if mode == "local":
        return phase_blocker
    return _phase_blocker_with_nested_hint(phase_blocker, nested_blocker)
```
