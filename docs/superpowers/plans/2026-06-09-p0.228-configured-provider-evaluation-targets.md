# P0.228 Configured Provider Evaluation Targets Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Make targets for configured Meshy/OpenAI evaluation evidence and
surface those targets in configured handoff reports.

**Architecture:** Keep the existing backend CLI evaluation commands unchanged.
Make targets become stable operator entry points; reports reference those Make
targets while still recording the same evidence files.

**Tech Stack:** Make, backend Python report builders, pytest, Swift
ProjectChecks.
