> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the local iOS deploy runbook list the concrete device deploy action before provider key actions.

**Architecture:** Keep the report shape unchanged and adjust only operator action ordering. Local mode promotes required deploy config slot actions before final resource/provider actions; configured mode keeps the existing provider-first behavior.

**Tech Stack:** Python, FastAPI backend support modules, pytest, project Make targets.

def test_ios_deploy_runbook_local_operator_actions_start_with_device_blocker(
    tmp_path: Path,
) -> None:
    repo_root = _repo(tmp_path)
    resources = repo_root / "services/backend/.local/final-resources.env"

    report = build_ios_deploy_runbook_report(mode="local", repo_root=repo_root)
    actions = report["operator_actions"]

        "provide DEVELOPMENT_TEAM in Deployment.local.xcconfig; "
        "rerun make mobile-deploy-preflight"

    provider_action = (
        "provide MESHY_API_KEY in final-resources.env; "
        "rerun make final-resources-preflight"
    )

    assert actions[0] == device_action
    assert provider_action in actions
    assert actions.index(device_action) < actions.index(provider_action)

- [ ] **Step 3: Implement local-mode promotion**

In `services/backend/src/myth_forge_api/ios_deploy_runbook.py`, add a small helper:

```python
def _local_device_slot_actions(input_slots: list[dict[str, Any]]) -> list[str]:
    device_slot_ids = {
        "development_team",
        "product_bundle_identifier",
        "backend_base_url",
        "final_launch_mode",
    }
    return [
        str(slot["operator_action"])
        for slot in input_slots
        if slot["id"] in device_slot_ids and slot["status"] in {"missing", "blocked"}
    ]
```

Then update `_operator_actions()` so `mode == "local"` extends actions with `_local_device_slot_actions(input_slots)` before final resource and provider actions, while the later generic slot loop skips those same local device slot actions to avoid duplicates.
