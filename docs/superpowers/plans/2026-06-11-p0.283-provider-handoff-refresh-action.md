# P0.283 Provider Handoff Refresh Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Promote child report `next_commands` into concrete final-local refresh operator actions so provider handoff shows a make target instead of a generic review instruction.

**Architecture:** Extend the refresh step summary with a sanitized `next_command` fallback derived from each child report. Keep command precedence unchanged: child `next_action` and `first_blocker` commands win, then `next_command`, then generic review text.

**Tech Stack:** Python 3.10, FastAPI backend package, pytest, existing Make targets.
