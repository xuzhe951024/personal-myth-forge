# P0.101 Print Fulfillment Approval Receipt Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an iPhone print fulfillment receipt and local approval gate before third-party print handoff.

**Architecture:** Add a pure Swift mobile-core builder that derives fulfillment status from the current session, quote, loading/error state, and local approval flag. Render it inside `PrintQuoteReviewView` with a local approval binding owned by `ForgeRootView`, then extend existing source gates and visual regression evidence.

**Tech Stack:** Swift 6, SwiftUI, SwiftPM executable tests, FastAPI source-gate tests, static HTML visual regression.

---

- Create `apps/mobile/ios/Sources/PersonalMythForgeMobileCore/PrintFulfillmentReceipt.swift` for status, rows, receipt, and builder.
- Create `apps/mobile/ios/App/PrintFulfillmentReceiptView.swift` for the compact receipt panel.
- Modify `apps/mobile/ios/App/PrintQuoteReviewView.swift` to accept `fulfillmentReceipt` and `isPrintQuoteApproved`.
- Modify `apps/mobile/ios/App/ForgeRootView.swift` to own/reset approval state and build the receipt.
- Modify source gates in `PersonalMythForgeMobileProjectChecks`, `ios_showcase_acceptance.py`, and `test_ios_showcase_acceptance.py`.
- Add roadmap and visual regression files.
