# P0.598 Mobile Live Consent Configured Plan

## Goal

Surface configured evidence plan live-provider commands directly in the iPhone Live
Provider Consent panel.

## Steps

1. Read current git state, product docs, roadmap, saved `.local` readiness reports,
   and the existing Live Provider Consent implementation.
2. Add a RED Swift contract test for a `configured_plan` row with a guarded Meshy
   command from the final configured evidence plan.
3. Update the Live Provider Consent builder and source/project gates.
4. Run focused Swift and backend acceptance tests.
5. Refresh safe local readiness reports, then commit, merge, and push.

## Safety

This phase only changes mobile presentation, tests, and docs. It must not run live
provider commands, write provider secrets, submit print orders, or touch Xcode
signing/global state.
