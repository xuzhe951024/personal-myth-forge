# P0.62 Treatstock Print Quote Adapter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a real Treatstock print quote adapter path and a downloadable local 3MF print candidate URL, while keeping order placement and live calls opt-in.

**Architecture:** Extend the existing print provider boundary with `TreatstockPrintProvider`, reuse existing `PrintQuote`/`PrintCandidate` models, add one local print candidate asset endpoint for demos, and wire readiness/factory/API handling. Tests use fake httpx clients, so automated checks never call Treatstock.

**Tech Stack:** Python 3.10, FastAPI, httpx, pytest, SwiftPM smoke checks, static HTML/PNG evidence.
