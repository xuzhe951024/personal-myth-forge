# P0.56 Backend Env Handoff Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a safe project-local writer for `services/backend/.env` so final Meshy/OpenAI provider resources can be supplied explicitly without committing secrets or making live provider calls.

**Architecture:** Add one POSIX shell script under `services/backend/scripts`, expose it through a Make target, and test it by executing the real script inside temporary git repos. Keep responsibility narrow: the writer creates ignored backend env config; existing provider readiness, handoff, and final acceptance commands remain responsible for validation.

**Tech Stack:** POSIX shell, Make, Python pytest subprocess tests, python-dotenv-backed settings parsing, SwiftPM source checks, static HTML evidence.
