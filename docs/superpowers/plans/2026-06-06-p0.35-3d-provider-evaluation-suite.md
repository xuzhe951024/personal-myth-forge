# P0.35 3D Provider Evaluation Suite Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a repeatable 20-case 3D provider evaluation suite so real Meshy
handoff runs can report quality-review inputs, route coverage, asset variants,
latency, failures, and safety without provider secret or raw media leakage.

**Architecture:** Move 3D evaluation report construction out of `cli.py` into a
focused backend evaluation module. Keep `evaluate-3d` backward compatible for
custom prompt files, and add `--suite default-v0` for the canonical 20-case
suite. Reports include generated asset provenance and manual review fields but
do not trigger live provider calls unless the caller selects a real provider.

**Tech Stack:** Python 3.10, dataclasses, Pydantic domain models, pytest,
FastAPI backend CLI, static HTML evidence, SwiftPM regression checks.

Add `run_three_d_evaluation` and helpers:

```python
def run_three_d_evaluation(
    *,
    provider: ThreeDProviderForEvaluation,
    selected_provider: str,
    suite_name: str,
    cases: tuple[ThreeDEvaluationCase, ...],
) -> dict[str, object]:
    case_reports = [_run_case(provider, case) for case in cases]
    return _sanitize_report(
        {
            "kind": "three_d_evaluation_report",
            "suite": suite_name,
            "provider": selected_provider,
            "total_cases": len(cases),
            "succeeded": sum(1 for case in case_reports if case["status"] == "succeeded"),
            "failed": sum(1 for case in case_reports if case["status"] == "failed"),
            "coverage": _coverage(case_reports),
            "review_rubric": REVIEW_RUBRIC,
            "safety": {
                "raw_media_in_report": False,
                "provider_secrets_in_report": False,
                "local_paths_in_report": False,
            },
            "cases": case_reports,
        }
    )
```

Use `_session_id_for_case(case.case_id)` based on a safe case id, not a raw
prompt hash. `_run_case` should:

- start a timer
- call `provider.generate_game_asset(ThreeDGenerationRequest(...))`
- collect `variant_roles` from `asset.variants`
- set `scene_loadable_variant` when any variant has `is_scene_loadable`
- include `asset.generation_provenance.model_dump(mode="json")` when present
- include `manual_review=dict(MANUAL_REVIEW_EMPTY)`
- catch exceptions and return a failed case with sanitized error

Use this sanitizer:

```python
def _safe_text(message: str) -> str:
    replacements = [
        r"Authorization\s*[=:]\s*Bearer\s+[A-Za-z0-9._~+/\-=:-]+",
        r"Bearer\s+[A-Za-z0-9._~+/\-=:-]+",
        r"raw=[^\s,;\"']+",
        r"api[_-]?key\s*[=:]\s*[^\s,;\"']+",
        r"data:[A-Za-z0-9.+-]+/[A-Za-z0-9.+-]+;base64,[A-Za-z0-9+/=_-]+",
        r"file://[^\s,;\"']+",
    ]
    sanitized = message
    for pattern in replacements:
        sanitized = re.sub(pattern, "[redacted]", sanitized, flags=re.IGNORECASE)
    return sanitized
```
