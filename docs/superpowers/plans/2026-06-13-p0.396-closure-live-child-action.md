# P0.396 Closure Packet Live Child Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `final-launch-closure-packet` use concrete live provider and print handoff child actions from the external action ledger.

**Architecture:** Keep the change inside `myth_forge_api.final_launch_closure_packet`. The external ledger already exposes optional `operator_action` values; the closure packet should preserve that field when compacting actions and prefer it in top-level operator actions.

**Tech Stack:** Python 3.10, FastAPI backend utilities, pytest, existing local report builders.
