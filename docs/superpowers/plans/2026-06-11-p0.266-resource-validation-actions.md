# P0.266 Resource Validation Actions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Preserve validation rerun commands in final resource requirement operator actions.

**Architecture:** Add one formatting helper in `final_resource_requirements.py` that combines `_blocker_command(requirement)` with `requirement["validation_command"]`. Also preserve `validation_command` when `final_local_report_refresh.py` promotes a blocker into `next_action`, so its existing `_action_command()` formatter can append the rerun instruction.

**Tech Stack:** Python 3.10, pytest, FastAPI backend report builders, ignored `.local` handoff reports.

- [x] **Step 2: Add `_blocker_action()`**

Add after `_blocker_command()`:

```python
def _blocker_action(requirement: dict[str, Any]) -> str:
    command = _blocker_command(requirement)
    validation_command = str(requirement.get("validation_command", "")).strip()
    if validation_command:
        return f"{command}; rerun {validation_command}"
    return command
```
