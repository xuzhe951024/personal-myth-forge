# P0.11 Capture-Aware 3D Generation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Pass uploaded capture media into backend 3D generation so from-capture sessions can use real object imagery as provider input.

**Architecture:** Introduce `ThreeDGenerationRequest` and capture media source types, make providers consume that request, add safe local capture media reads, and route Meshy image-source requests through `/openapi/v1/image-to-3d`.

**Tech Stack:** FastAPI, Pydantic v2, pytest, httpx MockTransport, SwiftPM unchanged for regression checks, Codex Browser static visual evidence.

---

## Environment Constraint

Do not call real Meshy APIs or require real API keys. Do not run global Xcode or
device configuration. Tests use local files and `httpx.MockTransport`.
