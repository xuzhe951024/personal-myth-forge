# P0.157 Final Launch Closure Packet Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a read-only final launch closure packet that turns scattered final resource, external action, device evidence, live-cost, and acceptance reports into one ordered final-acceptance checklist visible in backend JSON and mobile final launch status.

**Architecture:** Compose existing backend reports into `final_launch_closure_packet_report`, expose it via CLI/Make, embed it in `final_demo_launch`, decode it in Swift, render compact rows in `FinalLaunchStatusView`, and update acceptance/visual gates.

**Tech Stack:** FastAPI backend Python modules/tests, Swift Codable/mobile summary tests, SwiftUI app source gate, static visual regression artifacts.

- [x] **Step 2: Implement status, summary, operator actions, and safety**

Status rules:

- Any missing/blocked required resource, safe local write, device, or final
  acceptance section makes the packet `blocked`.
- Manual/global or live-cost sections make the packet `partial` when required
  blocking sections are ready.
- All ready sections make the packet `ready`.
