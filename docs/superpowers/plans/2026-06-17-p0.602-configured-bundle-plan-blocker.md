# P0.602 Configured Bundle Plan Blocker

## Goal

Keep configured evidence bundle next actions aligned with the actual configured plan
blocker before surfacing live provider cost actions.

## Steps

1. Inspect current git state, readiness reports, configured bundle code, and tests.
2. Add a RED test where the configured plan is blocked by
   `mobile_deploy_preflight` while live evidence is also blocked.
3. Prefer configured plan blockers when the configured plan status is `blocked`.
4. Preserve source blocker metadata and local validation actions in the bundle
   current blocker and operator actions.
5. Run focused tests, lint, safe local refresh, commit, merge, and push.

## Safety

This phase only changes read-only report prioritization and test coverage. It must not
run Meshy/OpenAI calls, write secrets, place print orders, or touch Xcode signing.
