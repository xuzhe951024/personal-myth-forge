**Architecture:** Add focused regression coverage in `test_final_configured_preflight.py`, then reuse the shared operator action normalizer `prefer_iphone_reachable_backend_url_handoff_actions()` inside `final_configured_preflight._operator_actions()`. **Tech Stack:** Python 3.10, pytest, ruff, FastAPI backend package.

### Task 2: Reuse Shared Backend URL Preference Normalization

**Files:**
- Modify: `services/backend/src/myth_forge_api/final_configured_preflight.py`

- [ ] **Step 1: Import the helper**

Add `prefer_iphone_reachable_backend_url_handoff_actions` to the existing `myth_forge_api.operator_actions` import:

```python
from myth_forge_api.operator_actions import (
    FINAL_RESOURCE_APPLY_ACTION,
    add_final_resource_validation_command,
    add_mobile_deploy_validation_command,
    normalize_operator_action,
    prefer_iphone_reachable_backend_url_handoff_actions,
)
```

- [ ] **Step 2: Apply it after dedupe**

Change the return expression in `_operator_actions()` from:

```python
    return _drop_unblock_fallback_actions(
        _prefer_apply_preview_before_apply(
            _dedupe(
                [
                    add_final_resource_validation_command(
                        add_mobile_deploy_validation_command(
                            normalize_operator_action(action)
                        )
                    )
                    for action in actions
                ]
            )
        )
    )
```

to:

```python
    normalized_actions = _dedupe(
        [
            add_final_resource_validation_command(
                add_mobile_deploy_validation_command(normalize_operator_action(action))
            )
            for action in actions
        ]
    )
    return _drop_unblock_fallback_actions(
        _prefer_apply_preview_before_apply(
            prefer_iphone_reachable_backend_url_handoff_actions(normalized_actions)
        )
    )
```
