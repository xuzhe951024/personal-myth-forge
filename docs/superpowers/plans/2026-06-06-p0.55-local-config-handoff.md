> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a safe project-local writer for `Deployment.local.xcconfig` so final iPhone deployment values can be supplied explicitly and validated by the existing deploy preflight.

**Architecture:** Add one POSIX shell script under the existing iOS scripts directory, expose it through a Make target, and verify it with real subprocess pytest coverage in temporary git repos. Keep validation split: the writer creates ignored local config; `deploy_preflight.sh` remains responsible for backend `/health`.

**Tech Stack:** POSIX shell, Make, pytest subprocess tests, SwiftPM project source checks, static HTML evidence.

- [ ] **Step 1: Implement the writer script**

Create a POSIX shell script with:

```sh
#!/usr/bin/env sh
set -eu

SCRIPT_DIR=$(CDPATH= cd -- "$(dirname -- "$0")" && pwd)
IOS_ROOT=$(CDPATH= cd -- "$SCRIPT_DIR/.." && pwd)
CONFIG_DIR="$IOS_ROOT/Config"
LOCAL_CONFIG="$CONFIG_DIR/Deployment.local.xcconfig"

team="${DEVELOPMENT_TEAM:-}"
bundle_id="${PRODUCT_BUNDLE_IDENTIFIER:-}"
backend_url="${PMF_BACKEND_BASE_URL:-}"

while [ "$#" -gt 0 ]; do
  case "$1" in
    --team) team="${2:-}"; shift 2 ;;
    --bundle-id) bundle_id="${2:-}"; shift 2 ;;
    --backend-url) backend_url="${2:-}"; shift 2 ;;
    --help) usage; exit 0 ;;
    *) printf '%s\n' "Unknown option: $1" >&2; exit 2 ;;
  esac
done
```

Validate missing values, loopback URL, and tracked local config. Write via
`tmp_file="$LOCAL_CONFIG.tmp.$$"` followed by `mv "$tmp_file" "$LOCAL_CONFIG"`,
then `chmod 600 "$LOCAL_CONFIG"` when chmod is available.
