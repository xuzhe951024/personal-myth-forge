# P0.7 iOS Client Scaffold Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a locally testable Swift iOS client scaffold that exercises the P0.6 capture-to-myth backend contract without requiring global Xcode configuration changes.

**Architecture:** Add `apps/mobile/ios` as a SwiftPM package with a pure Swift core module and an executable contract-test target. Keep SwiftUI app source as source-only scaffold for a later full-Xcode target, while validating models, multipart upload, API requests, capture id validation, and reducer behavior with `swift run`.

**Tech Stack:** Swift 6.3, Swift Package Manager executable targets, Foundation `URLRequest`, FastAPI backend contract fixtures, static HTML/Browser visual evidence.

---

## Environment Constraint

The local Swift compiler is available, but `swift test` cannot import `XCTest`
while the active developer directory points at Command Line Tools. Do not run
`xcode-select` or change the global developer directory in this plan. Use:

```bash
swift run --package-path apps/mobile/ios PersonalMythForgeMobileCoreContractTests
```
