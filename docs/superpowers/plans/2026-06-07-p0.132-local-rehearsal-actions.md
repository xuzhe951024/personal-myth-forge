> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the synthetic `final_rehearsal_local` step expose concrete local source report actions instead of falling back to a coarse review command.

**Architecture:** Keep report generation read-only. Aggregate missing/blocked local source report actions inside `_final_rehearsal_local`, then let the existing top-level action router include those nested actions.

**Tech Stack:** Python report builder, pytest, Makefile smoke verification, ignored `.local` JSON reports.

```python
def _local_rehearsal_operator_actions(
    local_sources: list[dict[str, Any]],
) -> list[str]:
    actions: list[str] = []
    for source in local_sources:
        status = str(source["status"])
        if status == "missing":
            actions.append(f"refresh {source['id']}: {source['command']}")
            continue
        if status != "blocked":
            continue
        nested_actions = source.get("operator_actions")
        if isinstance(nested_actions, list) and nested_actions:
            actions.extend(f"{source['id']}: {action}" for action in nested_actions)
        else:
            actions.append(f"review {source['id']}: {source['command']}")
        if len(actions) >= 6:
            break
    return _dedupe(actions)[:6]
```
