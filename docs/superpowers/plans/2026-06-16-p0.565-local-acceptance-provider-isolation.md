# P0.565 Local Acceptance Provider Isolation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prevent local final acceptance app checks from inheriting live Meshy/OpenAI provider configuration.

**Architecture:** Add a small context manager for local acceptance FastAPI app provider overrides, then use it in the local asset and capture-scene handoff checks. The context manager patches imported FastAPI route factory symbols and restores them in `finally`.

**Tech Stack:** Python, FastAPI TestClient, pytest, existing provider factories and local providers.

### Task 2: Add Local Acceptance App Provider Patch

- [ ] **Step 1: Create the context manager**

```python
@contextmanager
def local_acceptance_app(*, capture_store: CaptureStore | None = None) -> Iterator[FastAPI]:
    settings = Settings(
        three_d_provider="local",
        npc_provider="local",
        print_provider="local",
        myth_session_storage_dir=temp_dir,
    )
    patch api_main factory symbols to local factory closures
    yield api_main.app
    restore factory symbols
```

- [ ] **Step 2: Use it in local asset handoff acceptance**

```python
with local_acceptance_app() as app:
    client = _test_client(app)
```

- [ ] **Step 3: Use it in capture scene handoff acceptance**

```python
store = LocalCaptureStore(root_dir=capture_storage_dir)
with local_acceptance_app(capture_store=store) as app:
    client = _test_client(app)
```
