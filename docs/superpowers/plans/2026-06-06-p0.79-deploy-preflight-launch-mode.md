> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make iOS deploy preflight validate `PMF_FINAL_LAUNCH_MODE` before physical-device demos.

**Architecture:** Reuse `deploy_preflight.sh`'s existing `get_value` function so base `Deployment.xcconfig` provides the default and ignored local config can override it. Keep validation in the shell preflight because this is an operator deploy safety check, not app runtime behavior.

**Tech Stack:** POSIX shell, pytest subprocess tests, SwiftPM ProjectChecks, Markdown/HTML static visual evidence.

```python
def test_deploy_preflight_blocks_invalid_final_launch_mode(script_repo: Path) -> None:
    write_local_config(
        script_repo,
        backend_url="http://192.168.1.10:8080",
        final_launch_mode="live",
    )

    result = run_preflight(script_repo)

    assert result.returncode == 2
    assert "PMF_FINAL_LAUNCH_MODE must be local or configured." in result.stderr


def test_deploy_preflight_passes_with_configured_final_launch_mode(script_repo: Path) -> None:
    with health_server('{"status":"ok"}') as backend_url:
        write_local_config(
            script_repo,
            backend_url=backend_url,
            final_launch_mode="configured",
        )
        result = run_preflight(script_repo)

    assert result.returncode == 0
    assert "Final launch mode: configured" in result.stdout
```

Update `write_local_config` to accept `final_launch_mode: str | None = None` and write the line only when provided.
