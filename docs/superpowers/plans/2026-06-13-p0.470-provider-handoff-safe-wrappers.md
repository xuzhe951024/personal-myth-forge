# P0.470 Provider Handoff Safe Wrappers Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make provider/resource/configured evidence report Make targets accept usable blocked reports without treating missing external resources as Make failures.

**Architecture:** Keep existing Python report builders and CLI exit semantics. Add thin project-local shell wrappers that normalize exit code 2 to Make success, then point Make targets at those wrappers and add source gates.

**Tech Stack:** Make, POSIX shell, FastAPI backend CLI, pytest.

- [ ] **Step 1: Add shell wrappers**

Each wrapper uses this shape, with the CLI command and output path adjusted per target:

```sh
#!/usr/bin/env sh
set -eu

mkdir -p services/backend/.local
set +e
(
  cd services/backend
  uv run python -m myth_forge_api.cli final-resources-preflight \
    --repo-root ../.. \
    --output .local/final-resources-preflight.json
)
status=$?
set -e

if [ "$status" -eq 0 ] || [ "$status" -eq 2 ]; then
  printf '%s\n' "accepted final resources preflight exit code $status"
  printf '%s\n' "wrote services/backend/.local/final-resources-preflight.json"
  exit 0
fi

printf '%s\n' "final resources preflight failed before writing a usable report: exit $status" >&2
exit "$status"
```
