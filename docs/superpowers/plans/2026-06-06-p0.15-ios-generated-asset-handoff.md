# P0.15 iOS Generated Asset Handoff Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a SwiftPM-tested iOS generated asset preparation layer that downloads/caches backend-provided 3D asset URIs and hands SceneKit a local URL when the format is directly loadable.

**Architecture:** Keep provider keys on the backend. Add a Foundation-only mobile core asset preparation service with downloader/cache protocols, then wire `Artifact3DPreviewView` to use it asynchronously while falling back to proxy geometry for unsupported or conversion-required formats.

**Tech Stack:** Swift 6, SwiftPM, SwiftUI, SceneKit, Foundation URLSession and FileManager, existing FastAPI backend unchanged.

---

Do not run `xcode-select`, simulator builds, device builds, archive builds,
signing commands, or any global Xcode configuration. P0.15 verification is

```swift

try await testArtifactAssetPreparerDownloadsRemoteUSDZForSceneKit()
try await testArtifactAssetPreparerCachesRemoteGLBButRequiresConversion()

```

Add tests that verify:

- local `file:///tmp/relic.usdz` returns `.localSceneReady` and `sceneURL`.
- remote `https://cdn.example.com/relic.usdz` downloads bytes, stores
  `myth_test-meshy.usdz`, and returns `.cachedSceneReady`.
- remote `https://cdn.example.com/relic.glb` downloads bytes, stores

- [ ] **Step 3: Implement asset preparation service**

- `PreparedArtifactAssetStatus`
- `PreparedArtifactAsset`
- `ArtifactAssetDownloader`
- `ArtifactAssetCache`
- `URLSessionArtifactAssetDownloader`
- `FileSystemArtifactAssetCache`
- `ArtifactAssetPreparer`

The preparer must:

- Return local scene-ready state for existing `file://` SceneKit formats.
- Download and cache `http://` / `https://` assets.
- Mark cached `dae`, `scn`, `usd`, and `usdz` as SceneKit-ready.
- Mark cached `glb` and `gltf` as conversion-required.
- Use deterministic safe filenames: `<sessionId>-<provider>.<format>`.
- Return failure/unsupported statuses as values rather than throwing to the app.
