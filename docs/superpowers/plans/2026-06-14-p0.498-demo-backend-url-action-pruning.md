# P0.498 Demo Backend URL Action Pruning Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prefer the iPhone-reachable backend URL handoff over the generic backend URL deploy-config handoff in final demo launch top-level operator actions.

**Architecture:** Keep shared action dedupe/checklist behavior intact, then apply a narrow post-normalization filter in `myth_forge_api.final_demo_launch._operator_actions()` for top-level launch actions only. Resource requirements, deploy config writers, preflight commands, provider gates, and cost-consent actions remain unchanged.

**Tech Stack:** Python 3.10, pytest, ruff, FastAPI backend package.

```python
def _prefer_lan_backend_url_action(actions: list[str]) -> list[str]:
    action_roots = {_command_root(action) for action in actions}
    if FINAL_DEMO_LAN_BACKEND_URL_ACTION not in action_roots:
        return actions
    return [
        action
        for action in actions
        if _command_root(action) != FINAL_DEMO_GENERIC_BACKEND_URL_ACTION
    ]
```
