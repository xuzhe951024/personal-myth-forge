> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a manual configured final acceptance Make target that operators can run after providing real provider and iOS resources.

**Architecture:** Follow the existing local report wrapper pattern: a shell script normalizes exit code `0` and blocked-report exit code `2`, while Make exposes the script as a named target. Keep the target outside `final-rehearsal-local` so local rehearsal never triggers live provider calls.

**Tech Stack:** POSIX shell wrappers, GNU Make, pytest, fake `uv` test shim, SwiftPM mobile checks.

- [ ] **Step 1: Add configured wrapper**

Create `services/backend/scripts/write_final_acceptance_configured.sh`:

```sh
#!/usr/bin/env sh
set -eu

mkdir -p services/backend/.local
set +e
(
  cd services/backend
  uv run python -m myth_forge_api.cli final-acceptance \
    --profile quick \
    --provider-mode configured \
    --require-real-core \
    --allow-live-provider-calls \
    --repo-root ../.. \
    --output .local/final-acceptance-configured.json
)
status=$?
set -e

if [ "$status" -eq 0 ] || [ "$status" -eq 2 ]; then
  printf '%s\n' "accepted configured final acceptance exit code $status"
  printf '%s\n' "wrote services/backend/.local/final-acceptance-configured.json"
  exit 0
fi

printf '%s\n' "configured final acceptance failed before writing a usable report: exit $status" >&2
exit "$status"
```
