# P0.152 Configured Consent Preview Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make configured evidence consent needs visible before upstream API/key resource blockers are resolved.

**Architecture:** Add a backend summary field that distinguishes planned live-provider consent from the current `consent_required` status count. Decode and render that field in the existing iPhone Final Launch `Configured Evidence` group.

**Tech Stack:** Python report builders and pytest, Swift Codable models, SwiftPM contract/project checks.
