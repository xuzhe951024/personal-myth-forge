# P0.307 External Ledger Action Normalization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Normalize final external action ledger top-level operator actions so blocked rows surface concrete handoff commands instead of internal `unblock <id>:` wrappers.

**Architecture:** Preserve action row provenance in `action_groups` and `actions_by_id`. Change only the top-level `_operator_actions()` output for blocked rows, using the shared `normalize_operator_action()` function.

**Tech Stack:** Python 3.10, pytest, FastAPI backend support modules, project Make targets.
