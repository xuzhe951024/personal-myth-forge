# P0.36 Media-Backed 3D Evaluation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a guided-scan smoke evaluation suite that sends source images
through the existing 3D provider contract and reports single-image/multi-image
provenance without exposing raw media.

**Architecture:** Extend the P0.35 evaluation module with
`ThreeDEvaluationSourceImage`, optional source images on each case, and a
`guided-scan-smoke-v0` built-in suite. The CLI selects the new suite with
`--suite guided-scan-smoke-v0`; the report includes source image counts and
roles but keeps data URIs out of output.

**Tech Stack:** Python 3.10, dataclasses, pytest, FastAPI backend CLI, existing
3D provider adapters, static HTML evidence, SwiftPM regression checks.
