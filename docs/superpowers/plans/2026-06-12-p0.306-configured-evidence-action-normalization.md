# P0.306 Configured Evidence Action Normalization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Normalize configured provider evidence top-level operator actions from internal dependency labels into concrete Make/key handoff commands.

**Architecture:** Keep configured evidence step rows and blocker metadata unchanged, but normalize only top-level operator action strings. Add missing configured unblock mappings to the shared `operator_actions` module, then reuse that normalizer from both configured evidence reports.

**Tech Stack:** Python 3.10, pytest, FastAPI backend support modules, project Make targets.

```python
def _dedupe_operator_actions(values: list[str]) -> list[str]:
    seen: set[str] = set()
    result: list[str] = []
    for value in values:
        normalized = normalize_operator_action(value)
        if normalized in seen:
            continue
        seen.add(normalized)
        result.append(normalized)
    return result
```

Leave `_commands()` using the existing `_dedupe()` so command lists are not
rewritten as operator prose.
