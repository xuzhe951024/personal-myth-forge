# P0.562 Local Refresh Print Readiness Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `final-local-report-refresh` retain the final showcase backend-auto print fulfillment prerequisite in its top-level operator actions.

**Architecture:** Keep the behavior inside the existing action-selection path. `_run_step()` already stores each child report's `operator_actions` on `report_operator_actions`; `_operator_actions()` already collects selected report actions through `_selected_step_report_actions()`. The change is to allow `final_showcase_readiness` to contribute selected child actions using the same provider/print/device filters as `final_demo_launch_local`, and to preserve distinct validation-aware actions when one command root has different rerun targets.

**Tech Stack:** Python 3.10, FastAPI backend package, pytest, Make targets.

```python
def _validation_action_key(action: str) -> tuple[str, str] | None:
    command = _operator_action_root(action)
    root, separator, validation_root = command.partition("; rerun ")
    if not separator:
        return None
    validation_key = (
        validation_root.strip()
        if validation_root.strip() == FINAL_LOCAL_REPORT_PRINT_READINESS_VALIDATION
        else ""
    )
    return root.strip(), validation_key
```
