# P0.129 Mobile External Action Ledger Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the iPhone Final Launch surface decode and display the backend final external action ledger.

**Architecture:** Extend the existing Final Launch mobile payload path: `PMFModels.swift` decodes the backend report, `FinalLaunchMobileSummary.swift` converts it into sanitized string rows, and `FinalLaunchStatusView.swift` renders those rows in the existing grouped-line style. Backend source acceptance verifies the wiring without requiring runtime provider calls or Xcode/global actions.

**Tech Stack:** Swift Codable, SwiftPM contract tests, SwiftUI, Python source acceptance tests.
