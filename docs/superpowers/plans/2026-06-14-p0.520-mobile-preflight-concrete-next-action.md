# P0.520 Mobile Preflight Concrete Next Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make mobile deploy preflight evidence expose blocked remediation as a complete validation-aware next-action command.

**Architecture:** Reuse the existing `_validation_aware_action()` helper in `mobile_deploy_preflight_evidence.py`. The report continues storing raw checks and structured `validation_command`, but top-level `next_action.command` uses the same normalized command chain as `operator_actions[0]`.

**Tech Stack:** Python 3.10, pytest, ruff, Make evidence writer wrapper.

- [ ] **Step 1: Write the failing test**

```python
def test_preflight_evidence_next_action_promotes_validation_aware_writer_command(
    tmp_path: Path,
) -> None:
    result = build_mobile_deploy_preflight_evidence_report(
        repo_root=tmp_path,
        returncode=2,
        stdout="",
        stderr=(
            "Missing DEVELOPMENT_TEAM in Deployment.local.xcconfig.\n"
            "PMF_BACKEND_BASE_URL must be reachable from iPhone, not loopback.\n"
        ),
    )

    expected_command = (
        "DEVELOPMENT_TEAM=YOUR_TEAM_ID make mobile-write-deploy-config-auto; "
        "rerun make mobile-deploy-preflight"
    )

    assert result.report["next_action"]["command"] == expected_command
    assert result.report["next_action"]["validation_command"] == (
        "make mobile-deploy-preflight"
    )
    assert result.report["operator_actions"][0] == expected_command
```
