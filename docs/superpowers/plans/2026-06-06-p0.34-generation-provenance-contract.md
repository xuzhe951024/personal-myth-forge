# P0.34 Generation Provenance Contract Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add safe generation provenance to generated assets so final
acceptance can prove whether 3D generation came from text, one image, or a
multi-image guided scan without exposing private media.

**Architecture:** Extend the backend `GeneratedAsset` contract with optional
`generation_provenance`, populate it inside 3D providers, surface it in
acceptance summaries, and decode/render it in the iOS app. Keep the field
optional for backward compatibility with older stored sessions and fixtures.

**Tech Stack:** Python 3.10, Pydantic, pytest/httpx MockTransport, Swift
Codable/SwiftUI, SwiftPM compile checks, static HTML screenshot evidence.
