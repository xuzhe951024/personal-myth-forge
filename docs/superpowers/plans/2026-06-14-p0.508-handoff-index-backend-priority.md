# P0.508 Handoff Index Backend Priority Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `final-handoff-index.operator_actions` put `backend-device-demo` immediately after the first mobile deploy handoff and before backend URL/provider/print handoffs.

**Architecture:** Keep final handoff lanes, source reports, and device bundles unchanged. Adjust the existing `_prioritize_final_handoff_operator_actions` bucket order only when the first action is already a mobile deploy handoff, so backend-device-demo precedes iPhone-reachable backend URL actions in the physical-device flow without changing missing-report behavior.

**Tech Stack:** Python 3.10, FastAPI backend report builders, pytest.

def _is_mobile_deploy_handoff_action(action: str) -> bool:
    root, _detail_suffix = _split_detail_suffix(action)
    return (
        "rerun make mobile-deploy-preflight" in root
        and not _is_backend_device_demo_action(root)
        and "mobile-xcode-build" not in root
