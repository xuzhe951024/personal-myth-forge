# P0.337 iOS Writer Action Lists Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ensure final demo launch and iOS device launch rehearsal top-level operator actions prefer the safe iOS deploy writer command over the older manual Team ID action.

**Architecture:** Centralize the writer-preference rule in `operator_actions.py`, then call it from the report-specific action dedupe paths. Keep report semantics unchanged; only suppress the superseded action when the writer command is already present.

**Tech Stack:** Python 3.10, FastAPI backend package, pytest, ruff.

- [ ] **Step 3: Implement the helper**

Add constants for the validated writer root and validated manual Team ID root, plus:

```python
def prefer_mobile_deploy_writer_action(actions: list[str]) -> list[str]:
    command_roots = {_action_command_root(action) for action in actions}
    if MOBILE_WRITE_DEPLOY_CONFIG_AUTO_VALIDATED_ACTION not in command_roots:
        return actions
    return [
        action
        for action in actions
        if _action_command_root(action) != DEPLOYMENT_TEAM_VALIDATED_ACTION
    ]
```

The helper must split ` | ` detail suffixes before comparing roots.
