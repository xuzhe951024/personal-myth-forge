# P0.274 Configured Provider Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make final configured preflight expose the provider handoff through `make provider-handoff` in its top-level operator actions.

**Architecture:** Reuse the existing shared `normalize_operator_action()` helper inside the configured preflight top-level action aggregator, then apply the existing final-resource validation helper and dedupe. This preserves child report evidence and readiness status logic while improving the human handoff command surface.

**Tech Stack:** Python, FastAPI backend report builders, pytest, ruff.
