# P0.571 Rehearsal Ready Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop final readiness reports from repeating the backend-device-demo action after the iOS device launch rehearsal has already produced usable partial local evidence.

**Architecture:** Keep the saved rehearsal report intact, but normalize the readiness row for a `partial` rehearsal with no blocker into a device-lane-ready row. Preserve the original saved status for audit and leave real blocked, missing, stale, malformed, and live-provider blockers unchanged.

**Tech Stack:** Python 3.10, backend report builders, pytest, Make-based local evidence refresh.

```python
def test_ios_device_launch_rehearsal_readiness_treats_unblocked_partial_rehearsal_as_ready(
    tmp_path: Path,
) -> None:
    repo_root = tmp_path / "repo"
    report_path = _write_saved_rehearsal_readiness_report(repo_root, status="partial")
    payload = json.loads(report_path.read_text(encoding="utf-8"))
    payload["first_blocker"] = None
    payload["next_action"] = None
    payload["operator_actions"] = [
        "start backend-device-demo before device checks: make backend-device-demo; "
        "rerun make mobile-deploy-preflight"
    ]
    payload["sequence"] = [
        {
            "id": "final_rehearsal_local",
            "label": "Local final rehearsal",
            "status": "partial",
            "command": "make final-rehearsal-local",
            "classification": "saved_report_set",
        },
        {
            "id": "final_configured_preflight",
            "label": "Configured preflight",
            "status": "ready",
            "command": "make final-configured-preflight",
            "classification": "saved_report",
        },
        {
            "id": "final_handoff_index",
            "label": "Final handoff index",
            "status": "ready",
            "command": "make final-handoff-index",
            "classification": "saved_report",
        },
        {
            "id": "ios_device_launch_certificate",
            "label": "iOS device launch certificate",
            "status": "ready",
            "command": "make ios-device-launch-certificate",
            "classification": "saved_report",
        },
    ]
    report_path.write_text(json.dumps(payload), encoding="utf-8")

    result = build_ios_device_launch_rehearsal_readiness_report(repo_root=repo_root)
    sequence = {row["id"]: row for row in result.report["sequence"]}

    assert result.exit_code == 0
    assert result.report["status"] == "ready"
    assert result.report["first_blocker"] is None
    assert result.report["next_action"] is None
    assert result.report["operator_actions"] == []
    assert sequence["final_rehearsal_local"]["status"] == "ready"
    assert sequence["final_rehearsal_local"]["saved_status"] == "partial"
```

### Task 2: Normalize Usable Partial Rehearsal Rows

**Files:**
- Modify: `services/backend/src/myth_forge_api/ios_device_launch_rehearsal_readiness.py`

- [x] **Step 1: Add an unblocked-partial predicate**

Add a helper that returns `True` only when the saved rehearsal report has:

```python
status == "partial"
first_blocker is None
next_action is None
```

and the row being normalized is `final_rehearsal_local` with a non-blocked status.

- [x] **Step 2: Normalize the row while preserving audit state**

When the predicate is true, copy the row, set `saved_status` to the original
status, set `status` to `ready`, and remove row-level `operator_actions` that
only repeat backend-device-demo.

- [x] **Step 3: Suppress fallback report-level backend action**

When the saved rehearsal report is unblocked partial, `_operator_actions` should
return no backend-device-demo action for that report. Provider consent actions
from other reports should still be preserved by their own builders.
