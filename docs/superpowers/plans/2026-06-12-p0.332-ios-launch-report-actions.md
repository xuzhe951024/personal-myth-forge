# iOS Launch Report Actions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add explicit top-level `first_blocker` and `next_action` payloads to
iOS device launch certificate and rehearsal reports.

**Architecture:** Keep the existing launch gate sequencing and operator action
generation unchanged. Add small helper functions that derive action payloads
from the existing gate/blocker structures and include them in each report before
sanitization.

**Tech Stack:** Python 3.10, pytest, FastAPI backend CLI reports.
