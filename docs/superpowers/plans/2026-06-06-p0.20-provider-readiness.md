# P0.20 Provider Readiness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a backend and mobile provider readiness preflight that safely summarizes key/API integration state without exposing secrets.

**Architecture:** Add typed readiness models and a pure backend readiness builder, expose it through `GET /v1/provider-readiness`, then decode and display the same contract in the Swift mobile app. Keep provider secrets backend-only; local stubs remain demo-ready but not real-provider-ready.

**Tech Stack:** FastAPI, Pydantic, Swift 6, SwiftPM, SwiftUI, pytest, static HTML evidence.

---
