# P0.404 Configured Preflight Actions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove synthetic `unblock ...` fallback actions from final configured preflight operator actions.

**Architecture:** Keep nested configured preflight reports unchanged, but filter the final operator action list after existing normalization/dedupe and apply-preview preference. This preserves concrete provider/resource/iOS handoff commands while hiding old fallback labels.

**Tech Stack:** Python 3.10, backend report builders, pytest.

### Task 2: Filter Configured Preflight Fallback Actions

**Files:**
- Modify: `services/backend/src/myth_forge_api/final_configured_preflight.py`

- [ ] **Step 1: Add filter helper**

Near `_operator_actions()`, add:

```python
def _drop_unblock_fallback_actions(actions: list[str]) -> list[str]:
    return [
        action
        for action in actions
        if not _operator_action_root(action).startswith("unblock ")
    ]
```

- [ ] **Step 2: Apply helper after existing action normalization**

Change `_operator_actions()` from:

```python
return _prefer_apply_preview_before_apply(
    _dedupe(
        [
            add_final_resource_validation_command(
```

to:

```python
return _drop_unblock_fallback_actions(
```
