# P0.272 Configured Preflight Resource Actions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make configured preflight top-level operator actions validation-aware for known `final-resources.env` handoff actions.

**Architecture:** Reuse the shared `add_final_resource_validation_command()` helper in `final_configured_preflight` after collecting child report actions and before dedupe. This preserves child report contents and status logic while making the operator-facing action list directly actionable.

**Tech Stack:** Python, FastAPI backend report builders, pytest, ruff.
