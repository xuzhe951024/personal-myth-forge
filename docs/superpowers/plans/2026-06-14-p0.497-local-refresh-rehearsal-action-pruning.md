# P0.497 Local Refresh Rehearsal Action Pruning Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove low-information `make final-rehearsal-local` command-root actions from final local report refresh top-level operator actions when concrete handoff actions already exist.

**Architecture:** Add a narrow post-deduplication filter in `myth_forge_api.final_local_report_refresh`. The filter only changes the returned `operator_actions`; report steps, command execution, safe blocked-report writes, first blocker, and next action stay unchanged.

**Tech Stack:** Python 3.10, pytest, ruff, FastAPI backend package.

Add near `_dedupe_operator_actions()`:

```python
def _drop_bare_rehearsal_when_specific_actions_exist(actions: list[str]) -> list[str]:
    has_specific_action = any(
        _action_command_root(action) != FINAL_LOCAL_REPORT_REHEARSAL_COMMAND
        for action in actions
    )
    if not has_specific_action:
        return actions
    return [
        action
        for action in actions
        if _action_command_root(action) != FINAL_LOCAL_REPORT_REHEARSAL_COMMAND
    ]
```
