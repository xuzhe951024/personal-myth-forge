# P0.97 Mobile Final Showcase Launch Digest Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the iPhone `Final Showcase` card summarize both the local myth loop and the Final Launch handoff state.

**Architecture:** Keep `FinalLaunchMobileSummary` as the source of truth for launch readiness, and pass it into `FinalShowcaseSummaryBuilder` from `ForgeRootView`. The builder appends launch-facing stages and folds them into the top-card overall status while preserving old behavior when no launch summary is supplied.

**Tech Stack:** SwiftPM mobile core contract tests, SwiftUI app compile check, Swift source guards, backend source acceptance, static HTML visual regression.

var stages = [
    captureStage(captureSelection),
    threeDStage(session),
    npcStage(session: session, npcTickHistoryCount: npcTickHistoryCount),
    printStage(session: session, printQuote: printQuote),
    resourcesStage(readiness: providerReadiness, error: providerReadinessError),
]
if let finalLaunchSummary {

    stages.append(operatorHandoffStage(finalLaunchSummary))

}
