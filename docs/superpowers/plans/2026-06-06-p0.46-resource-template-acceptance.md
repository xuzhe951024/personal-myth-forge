# P0.46 Resource Template Acceptance Plan

## Goal

Add a source-only acceptance gate proving that final resource templates expose all
operator-provided slots and remain safe to commit.

## Baseline

Command:

```bash
cd services/backend && uv run pytest tests/test_resource_handoff.py tests/test_final_acceptance.py tests/test_cli.py -q
```

Result:

```text
30 passed
```

## Implementation Steps

1. Add failing tests for a new `resource_template_acceptance` module.
   - Passing fixture with complete `.env.example`, iOS deployment example, and
     ignored local destination files.
   - Missing backend key fails with a precise `missing_keys` entry.
   - Missing iOS key fails with a precise `missing_keys` entry.
   - Unsafe template content is redacted from the report and flips safety flags.

2. Implement `run_resource_template_acceptance(repo_root=None)`.
   - Parse dotenv-style backend templates.
   - Parse xcconfig-style iOS templates.
   - Check `.gitignore` using source text, not global git state.
   - Sanitize report text before returning.

3. Add CLI command:
   - `resource-template-acceptance`
   - `--repo-root`
   - `--output`

4. Wire into final acceptance.
   - Add an in-process check after iPhone backend handoff.
   - Extend test injection with a resource-template runner.
   - Update quick/full summary expectations.

5. Update resource handoff.
   - Add `MYTH_SESSION_STORAGE_DIR` as a backend local storage slot.
   - Add the new CLI command to the operator command list.

6. Update checked-in templates.
   - Add `MYTH_SESSION_STORAGE_DIR=` to `.env.example`.

7. Verification.
   - Targeted tests.
   - `make backend-lint`
   - `make backend-test`
   - Quick final acceptance expected blocked only by local iOS requirements.
   - Generate a small verification HTML/PNG artifact for the new report.
