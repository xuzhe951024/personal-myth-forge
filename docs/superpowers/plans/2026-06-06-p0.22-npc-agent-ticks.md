# P0.22 NPC Agent Ticks Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a stateless NPC agent tick loop so the mobile demo can ask the backend what the NPCs do next after the initial myth session.

**Architecture:** Reuse existing `NPCAgentTrace`, `NPCReaction`, and `WorldResolution` contracts. Add backend request/response models and a deterministic local tick runtime, expose it through `POST /v1/npc-ticks`, then add Swift decoding, API client wiring, and a compact SwiftUI tick panel. Keep live OpenAI ticks and persistence as future provider implementations.

**Tech Stack:** FastAPI, Pydantic, pytest, SwiftPM, SwiftUI, existing no-global-Xcode verification gate.
