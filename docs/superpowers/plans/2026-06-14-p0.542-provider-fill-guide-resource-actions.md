# P0.542 Provider Fill Guide Resource Actions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make final provider/key handoff actions prefer the canonical fill-guide provider chain over bare missing-key commands.

**Architecture:** Add a shared preference helper in `operator_actions.py`, then use it in final handoff aggregators after existing normalization and print/deploy preference passes. Update the provider key capability to point at the same canonical chain when provider resources are incomplete.

**Tech Stack:** Python 3.12, FastAPI backend support modules, pytest, project Make wrappers.

        "rerun make provider-handoff; rerun make live-provider-evidence"
