# P0.450 iOS Certificate Backend Handoff Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Keep the validated backend-device-demo handoff visible in the iOS device launch certificate top-level `operator_actions`.

**Architecture:** The existing certificate builder already collects child actions from final handoff reports and then prioritizes a capped top-level checklist. This phase adds a backend-device-demo classifier and includes that class in the child-action selection and final priority ordering.

**Tech Stack:** Python 3.10, FastAPI backend package, pytest, ruff via `make backend-lint`.

def _preferred_backend_device_demo_handoff_actions(actions: list[str]) -> list[str]:
    backend_actions = [
        action for action in actions if _is_backend_device_demo_handoff_action(action)
    ]
    validated_actions = [
        action for action in backend_actions if "mobile-deploy-preflight" in action.lower()
    ]
    return validated_actions or backend_actions
