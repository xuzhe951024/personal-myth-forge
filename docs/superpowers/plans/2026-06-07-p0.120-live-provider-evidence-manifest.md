# P0.120 Live Provider Evidence Manifest Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a safe, read-only manifest that proves which configured Meshy/OpenAI/Treatstock/iOS live evidence files are present, missing, blocked, or ready for final iPhone demo review.

**Architecture:** Follow the existing readiness-report pattern. Add one backend module that reads ignored local JSON evidence, one CLI/Make target, one `final-demo-launch` embedding, and one iPhone summary section. The command never runs live providers; it only reports the commands the operator may explicitly run later.

**Tech Stack:** Python 3.10, FastAPI service modules, Pydantic-adjacent plain JSON reports, pytest, SwiftPM, Swift Codable models, SwiftUI source acceptance.

```python
def _status_from_payload(slot: EvidenceSlot, payload: dict[str, Any]) -> str:
    if payload.get("kind") != slot.expected_kind:
        return "blocked"
    if slot.slot_id == "provider_handoff":
        return "ready" if payload.get("core_real_ready") is True else "blocked"
    if slot.slot_id == "three_d_evaluation_configured":
        return "ready" if int(payload.get("failed", 0)) == 0 and int(payload.get("succeeded", 0)) > 0 else "blocked"
    if slot.slot_id == "npc_evaluation_configured":
        return "ready" if int(payload.get("failed", 0)) == 0 and int(payload.get("succeeded", 0)) > 0 else "blocked"
    if slot.slot_id == "final_acceptance_configured":
        return "ready" if payload.get("overall_status") in {"passed", "ready"} else _partial_or_blocked(payload.get("overall_status"))
    if slot.slot_id == "final_demo_launch_configured":
        return "ready" if payload.get("overall_status") in {"ready", "partial"} else _partial_or_blocked(payload.get("overall_status"))
    return "blocked"
```
