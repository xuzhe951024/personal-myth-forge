**Goal:** Make the final launch closure packet's top-level device action bundle follow final showcase readiness ordering.

**Architecture:** `final_launch_closure_packet.py` already builds both `device_evidence` and `showcase_readiness`. This phase changes only the top-level closure packet bundle selection: prefer `showcase_readiness.device_action_bundle` when present, fall back to `device_evidence.device_action_bundle` or evidence slots otherwise.

**Tech Stack:** Python 3.10, pytest, FastAPI backend Make targets.

```python
def _device_action_bundle(
    *,
    device_evidence: dict[str, Any],
    showcase_readiness: dict[str, Any],
) -> dict[str, Any]:
    if _normalized_status(str(device_evidence.get("status", "missing"))) == "ready":
        source_bundle = device_evidence.get("device_action_bundle")
        source_report = "ios_device_evidence_bundle"
        fallback_status = str(device_evidence.get("status", "missing"))
        fallback_slots = device_evidence.get("evidence_slots", [])
    else:
        source_bundle = showcase_readiness.get("device_action_bundle")
        source_report = "final_showcase_readiness"
        fallback_status = str(showcase_readiness.get("status", "missing"))
        fallback_slots: Any = []
        if not isinstance(source_bundle, dict):
            source_bundle = device_evidence.get("device_action_bundle")
            source_report = "ios_device_evidence_bundle"
            fallback_status = str(device_evidence.get("status", "missing"))
            fallback_slots = device_evidence.get("evidence_slots", [])
```

Preserve the old fallback behavior for evidence slots, and keep ready device evidence authoritative when it has already passed.
