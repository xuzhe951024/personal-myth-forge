> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expose a sanitized top-level print fulfillment `next_action` in backend readiness reports and iPhone Final Launch summary rows.

**Architecture:** Reuse the existing `first_blocker` selected from print fulfillment readiness checks and derive an optional top-level `next_action` from it. Keep Swift decoding backward-compatible by adding optional model fields and only rendering the row when present.

**Tech Stack:** Python FastAPI backend helpers, pytest, Swift Codable models, SwiftPM contract tests.

Add:

```swift
private static func printFulfillmentReadinessNextActionRow(
    _ action: PrintFulfillmentReadinessNextAction
) -> String {
    var parts = ["Next action: \(action.command)"]
    parts.append("| \(action.id): \(action.status)")
    if let classification = action.classification, !classification.isEmpty {
        parts.append("| \(classification)")
    }
    if !action.detail.isEmpty {
        parts.append("| \(action.detail)")
    }
    return sanitize(parts.joined(separator: " "))
}
```
