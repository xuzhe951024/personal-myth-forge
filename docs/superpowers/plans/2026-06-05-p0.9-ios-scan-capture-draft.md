# P0.9 iOS Scan Capture Draft Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a SwiftPM-verifiable iOS capture draft layer for single-photo, photo-set, and future ARKit scan uploads.

**Architecture:** Keep capture validation in `PersonalMythForgeMobileCore`, transform valid drafts into existing backend `ObjectCaptureMetadata` plus `[CaptureUpload]`, and leave real camera/ARKit runtime integration for a later full-Xcode iteration. Use static mobile storyboard evidence for visual regression.

**Tech Stack:** Swift 6.3, Swift Package Manager executable contract tests, SwiftUI app shell source, FastAPI backend contract, Codex Browser visual evidence.

---

## Environment Constraint

Do not run `xcode-select` or require simulator/device builds. P0.9 local
verification uses SwiftPM, backend tests, `plutil`, project checks, and Browser
against a static storyboard.

- Create `apps/mobile/ios/Sources/PersonalMythForgeMobileCore/CaptureDraft.swift`: capture modes, media draft, validation, upload payload conversion.
- Modify `apps/mobile/ios/Sources/PersonalMythForgeMobileCore/PersonalMythForgeAPIClient.swift`: support scan asset content types and generated scan filenames.
- Modify `apps/mobile/ios/Sources/PersonalMythForgeMobileCoreContractTests/main.swift`: red/green contract tests for capture drafts and scan uploads.
- Modify `apps/mobile/ios/App/CaptureFormView.swift`: mode labels and scan readiness source UI.
- Modify `apps/mobile/ios/App/ForgeRootView.swift`: capture mode state.
- Modify `apps/mobile/ios/PersonalMythForge.xcodeproj/project.pbxproj`: core package product already links new core source through SwiftPM; no app target source change needed.
