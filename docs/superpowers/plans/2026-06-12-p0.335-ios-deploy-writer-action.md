# P0.335 iOS Deploy Writer Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Surface the project-local iOS deploy config writer command as the first safe action when the final iPhone launch path needs both Team ID and LAN backend URL.

**Architecture:** Keep the behavior inside existing report/action helpers. `mobile_deploy_preflight_evidence.py` chooses the combined writer action for the common blocked state; `operator_actions.py` recognizes that command as requiring `make mobile-deploy-preflight` validation.

**Tech Stack:** Python report builders, pytest, existing Make targets.
