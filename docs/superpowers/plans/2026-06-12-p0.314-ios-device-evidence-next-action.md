# P0.314 iOS Device Evidence Next Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add structured `first_blocker` and `next_action` fields to the iOS device evidence bundle and expose them through the iOS final launch summary.

**Architecture:** The backend derives both fields from the existing ordered `evidence_slots`, then sanitizes them with the rest of the report. Swift models decode the optional fields, and `FinalLaunchMobileSummary` renders short redacted rows when present.

**Tech Stack:** Python/FastAPI backend report builder, pytest, Swift Package contract executable.

- [ ] **Step 2: Add helpers**

Implement:

```python
def _first_blocker(slots: list[dict[str, Any]]) -> dict[str, Any] | None:
    for slot in slots:
        if slot["required"] and slot["status"] != "ready":
            return dict(slot)
    return None


def _next_action(first_blocker: dict[str, Any] | None) -> dict[str, Any] | None:
    if first_blocker is None:
        return None
    return {**first_blocker, "source": "first_blocker"}
```
