# P0.261 Final Handoff Concrete Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `final-handoff-index.json` surface the concrete saved local final demo launch `next_action` as the first local rehearsal operator action.

**Architecture:** Preserve lane status and blocker details, but carry a separate concrete operator action from saved source reports into the local rehearsal lane. `_operator_actions()` then uses that concrete action before falling back to detail text.

**Tech Stack:** Python 3.10, FastAPI backend package, pytest, project-local `.local` JSON readiness reports.

```python
def _saved_report_operator_action(payload: dict[str, Any]) -> str:
    next_action = payload.get("next_action")
    if isinstance(next_action, dict):
        command = str(next_action.get("command", "")).strip()
        validation_command = str(next_action.get("validation_command", "")).strip()
        if command and validation_command:
            return f"{command}; rerun {validation_command}"[:240]
        if command:
            return command[:240]
    return _first_operator_action(payload)
```

Use this helper in `_source_report()` to populate `operator_actions` when structured `next_action` exists.
