# P0.33 Meshy Multi-Image Guided Scan Regression

Date: 2026-06-06

## Scope

P0.33 updates `MeshyThreeDProvider` so guided scans with multiple prepared
reference images use Meshy's Multi-Image to 3D endpoint:

- 2-4 supported JPEG/PNG source images -> `/openapi/v1/multi-image-to-3d`
- 5+ supported JPEG/PNG source images -> first four images sent to multi-image
- 1 supported image -> existing `/openapi/v1/image-to-3d`
- 0 supported images -> existing text-to-3D preview/refine path

Official API reference checked during design:

- https://docs.meshy.ai/en/api/multi-image-to-3d

The Meshy docs state that `image_urls` accepts 1-4 public URLs or data URIs for
the same object from different angles. Automated tests use mocked HTTP transport
and do not make live Meshy calls.

## Provider Behavior Verified

The focused Meshy provider tests verify:

- two supported source images create a task at
  `/openapi/v1/multi-image-to-3d`
- the request payload sends `image_urls` with both data URIs
- the payload requests `target_formats=["glb", "usdz"]`
- `enable_pbr` and `should_texture` are enabled
- GLB/USDZ response URLs still map into existing generated asset variants
- five supported source images are capped to the first four
- unsupported image content types are filtered before routing
- existing single image-to-3D and text-to-3D paths continue passing
