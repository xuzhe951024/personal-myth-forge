# P0.77 Final Operator Handoff Regression

## Scope

P0.77 adds a read-only backend `final_operator_handoff` report and embeds it in
`/v1/final-demo-launch`. The iPhone `Final Launch Status` panel renders a new
`Next` section from `FinalLaunchMobileSummary.handoffRows`.

## Verified Behavior

- Backend handoff report produces stable ordered steps for final resources,
  unified apply, LAN backend startup, local final acceptance, iOS deploy
  preflight, Xcode build gate, and configured live-provider acceptance.
- Saved final acceptance blockers promote `mobile_deploy_preflight` and
  `mobile_xcode_build` into handoff step status and next actions.
- Configured final acceptance is marked as live/consent-gated when real
  providers are configured.
- `/v1/final-demo-launch` embeds `final_operator_handoff`.
- Swift models decode the handoff payload with `summary.live`, `steps`,
  `next_actions`, and safety flags.
- Mobile summary redacts unsafe handoff text before rendering.
- `FinalLaunchStatusView` renders a read-only `Next` section.

## Safety Boundaries

- No API request executes handoff commands.
- No iPhone UI control executes handoff commands.
- No Xcode, signing, simulator, device deployment, archive, Apple SDK license,
  or provider command is run by this feature.
- Provider secrets, raw media, checkout/payment URLs, file URLs, and absolute
  local paths are redacted.
