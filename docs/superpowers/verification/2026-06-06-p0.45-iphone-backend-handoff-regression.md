# P0.45 iPhone Backend Handoff Regression

## Scope

P0.45 adds an explicit iPhone backend launch handoff:

- `make backend-device-demo`
- `ios_backend_handoff_acceptance`
- final acceptance integration
- resource handoff command update

The work does not run Xcode, install to a device, start a persistent backend in
tests, mutate global developer settings, or call provider APIs.

## Remaining External Blockers

- Real iPhone validation still needs local signing config, Apple SDK license
  acceptance, macOS firewall/network permission, and a physical device on the
  same LAN.
- Live Meshy/OpenAI validation still needs backend-only keys, quota confirmation,
  and `--allow-live-provider-calls`.
- Real print fulfillment remains intentionally unimplemented.
