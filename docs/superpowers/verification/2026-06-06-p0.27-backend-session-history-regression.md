## Endpoint Evidence

New backend endpoints:
- `GET /v1/myth-sessions/{session_id}`
- `GET /v1/myth-sessions/{session_id}/history`

Persistence behavior:
- `POST /v1/myth-sessions` saves the returned session.
- `POST /v1/myth-sessions/from-capture` saves the returned session.
- `POST /v1/npc-ticks` appends the generated tick to backend history.
- `LocalMythSessionStore` keeps at most 24 ticks sorted by tick index.

Safety behavior:
- Store rejects invalid session ids before path construction.
- Store writes under `MYTH_SESSION_STORAGE_DIR`, defaulting to
  `services/backend/.local/myth-sessions`.
- Store sanitizes raw media data URIs, bearer tokens, API keys, and raw provider
  payload markers before writing JSON.
- Store does not persist raw capture media, provider keys, or raw personal source
  bodies.
