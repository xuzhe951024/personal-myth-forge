# P0.28 Mobile Backend History Sync Regression

Date: 2026-06-06

## Scope

P0.28 wires backend myth session history into the visible iOS demo restore flow.
The app still restores the local `DemoRunSnapshot` immediately, then
non-blockingly requests backend history for valid `myth_<16 hex>` session ids.

This is not production account sync, server-authoritative NPC memory, provider
key handling on mobile, raw capture persistence, real device installation, or
print fulfillment.

## App Sync Contract

- `MythSessionID` validates the backend `myth_<16 hex>` id contract before
  history sync.
- `DemoRunSnapshot(history:savedAt:)` converts backend `MythSessionHistory`
  into the compact local snapshot format and keeps the existing bounded tick
  behavior.
- `ForgeRootView` restores the local snapshot first, then calls
  `getMythSessionHistory(sessionId:)` without blocking the visible demo state.
- Successful sync replaces the visible ready session and NPC tick history, then
  refreshes the local snapshot.
- Failed sync leaves the local restored run visible and shows:
  `Backend history not reachable; using local demo state.`
- `DemoSnapshotStatusView` shows backend history progress and fallback status.

## Safety Review

- No provider keys are stored or displayed in the mobile app.
- No raw photos, scan binaries, data URIs, or local file URLs are added to the
  mobile snapshot path.
- No raw personal email, chat, calendar, or document bodies are persisted.
- Backend history sync failure is non-blocking and keeps the local demo state
  visible.
