execute Unity movement, install on an iPhone, run live provider calls, or expose

## Implemented Checks

`npc_agent_provider_acceptance` now requires:

- `OpenAINPCDirector`, `OpenAINPCOutput`, `text_format=OpenAINPCOutput`, and
  `agent_runtime="openai_structured_runtime"` in the NPC director provider.
- `OpenAINPCTickRuntime`, `OpenAINPCTickOutput`,
  `runtime_name = "openai_tick_structured_runtime"`, and
  `text_format=OpenAINPCTickOutput` in the tick runtime provider.
- `NPC_PROVIDER=openai` factory routing for both initial NPC reactions and
  stateless NPC ticks.
- provider readiness capabilities for `structured_agent_traces` and
  `structured_agent_ticks`.
- resource handoff coverage for `NPC_PROVIDER`, `OPENAI_API_KEY`, and
  backend-only key handling.
- backend/mobile docs that mobile receives generated NPC state, not provider
  keys.
