# P0.30 Bounded Autonomous NPC Run Regression

Date: 2026-06-06

## Scope

P0.30 adds a user-triggered bounded NPC autonomy run:

- Backend endpoint: `POST /v1/myth-sessions/{session_id}/autonomy-runs`.
- Request: `step_count` defaults to `3` and is bounded to `1...3`.
- Response: `NPCAutonomyRun` summary plus updated `MythSessionHistory`.
- iOS control: `Run Autonomy` beside `Advance Village`.
- Runtime boundary: foreground bounded action only, not a background daemon.

## Safety Review

- Provider keys remain backend-only.
- Mobile autonomy requests send a session id and bounded `step_count`, not raw
  media or provider secrets.
- Stored history sanitization still covers raw data URIs, bearer tokens, API
  keys, and raw provider payload markers before local JSON persistence.
- Personal Context Skill constraints remain unchanged: only user-approved
  capsule summaries should enter this flow, not raw email, chat, calendar, or
  document bodies.
- The autonomy run is explicitly bounded to 1-3 ticks and user-triggered; it is
  not an always-on NPC loop.
