# P0.15 iOS Generated Asset Handoff Regression

Date: 2026-06-06

## Scope

P0.15 verifies that the iOS app source now has a SwiftPM-tested generated asset
handoff path: remote generated asset URIs can be downloaded, cached locally, and
represented as a SceneKit-ready local URL when the format is `dae`, `scn`,
`usd`, or `usdz`.

This is not an Xcode simulator run, device install, signing pass, ARKit runtime
capture, GLB runtime import, provider-side USDZ export, or Unity village import.

## Code Evidence

- `ArtifactAssetPreparation.swift` defines `ArtifactAssetPreparer`,
  `ArtifactAssetDownloader`, `ArtifactAssetCache`,
  `URLSessionArtifactAssetDownloader`, and `FileSystemArtifactAssetCache`.
- Core contract tests cover local USDZ, remote USDZ cache and SceneKit handoff,
  remote GLB cache with conversion-required status, and unsupported URI handling.
- `Artifact3DPreviewView` starts a session-scoped preparation task and attempts
  `SCNScene(url:options:)` when a prepared local scene URL exists.
- `PersonalMythForgeMobileProjectChecks` requires the new core service and app
  source wiring.

## Remaining Gaps

- Provider-side USDZ export or GLB-to-USDZ conversion.
- Runtime GLB import.
- Xcode simulator/device deployment and signing.
- Live ARKit mesh capture runtime.
- Unity village import and NPC spatial behavior around the asset.
