# P0.59 Capture Generation Readiness Regression Evidence

## Scope

P0.59 replaces static scan readiness copy in the mobile capture form with a
tested SwiftPM mobile-core generation readiness summary.

Implemented behavior:

- Guided scan with too few images reports `waiting`.
- Ready guided scan reports `multiImage`, full source image count, and the
  provider-selected source count capped at 4.
- ARKit scan package reports `scanAsset` with scan plus reference counts.
- Missing live 3D provider configuration reports `needsAttention` and names
  `MESHY_API_KEY`.
- Provider readiness errors are redacted before display.
- `ForgeRootView` computes readiness from current capture selection and backend
  provider readiness, then passes title, snake_case route label, and detail into
  `CaptureFormView`.

This iteration does not call Meshy, create provider keys, run Xcode, install on
device, or change backend provider APIs.
