# P0.76 Final Acceptance Readiness Regression

## Scope

P0.76 makes the latest saved local final acceptance report visible in the
iPhone `Final Launch Status` panel. The backend embeds a read-only
`final_acceptance_readiness` report in `/v1/final-demo-launch`.

## Behavior Covered

- Missing saved final acceptance report maps to `missing`.
- Blocked saved final acceptance report lists compact blocker rows.
- Ready saved final acceptance report maps to `ready`.
- Malformed saved report maps to a safe blocked state.
- Backend API handling does not run final acceptance, Xcode, deployment,
  signing, or provider commands.
- The mobile model decodes `final_acceptance_readiness`.
- `FinalLaunchMobileSummary` renders `acceptanceRows` for missing, blocked, and
  ready states.
- `FinalLaunchStatusView` shows an `Acceptance` section.
- Unsafe strings are redacted before mobile display.
