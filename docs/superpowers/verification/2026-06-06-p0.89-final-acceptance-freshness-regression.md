# P0.89 Final Acceptance Freshness Regression

Date: 2026-06-06
Branch: `codex/p0.89-final-acceptance-freshness`

## Scope

P0.89 adds a read-only freshness guard for saved final acceptance reports. The
backend now compares `services/backend/.local/final-acceptance-local.json`
against the current git HEAD commit timestamp when git metadata is available.
If the saved report is older than the current revision, final acceptance
readiness becomes blocked with a `final_acceptance_freshness` blocker.

The iPhone final launch panel now decodes and displays the freshness state in
the `Launch Receipt` and `Acceptance` rows.

No final acceptance command, deploy command, provider call, Xcode command,
signing config, license command, or global system state changed.
