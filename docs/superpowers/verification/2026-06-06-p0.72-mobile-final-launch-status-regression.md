# P0.72 Mobile Final Launch Status Regression

## Scope

P0.72 adds a read-only iPhone `Final Launch Status` panel backed by the existing
`/v1/final-demo-launch` report.

## Behavior Covered

- Swift core builds `FinalLaunchMobileSummary` from `FinalDemoLaunchReport`.
- Missing report maps to waiting.
- Partial launch report maps to waiting and exposes phase rows, command rows,
  and final resource actions.
- Ready launch report maps to ready.
- Unsafe report strings are redacted before display.
- `ForgeRootView` renders `FinalLaunchStatusView` below Device Preflight.
- Xcode project source membership includes `FinalLaunchStatusView.swift`.
- Backend source acceptance requires the mobile summary builder, view, root
  wiring, and contract tests.

## Safety

The app still does not read provider keys, `.env`, or
`final-resources.env`. It consumes only the sanitized backend final launch
report and applies mobile redaction before rendering phase details, resource
actions, and commands.
