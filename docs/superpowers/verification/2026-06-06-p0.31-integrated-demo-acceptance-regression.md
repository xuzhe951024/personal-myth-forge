# P0.31 Integrated Demo Acceptance Regression

Date: 2026-06-06

## Scope

P0.31 adds a backend CLI acceptance harness for final API/key handoff:

- `demo-acceptance --provider-mode local --npc-steps 3 --output ...`
- `demo-acceptance --provider-mode configured --require-real-core --npc-steps 3 --output ...`
- local mode forces local 3D and local NPC providers in memory and makes no live
  Meshy/OpenAI calls
- strict configured mode exits `2` before provider calls until core providers
  are real-provider-ready
- reports summarize provider readiness, generated asset/scene variant, NPC
  runtime/ticks, timings, and safety policy

## Safety Review

- Provider keys remain backend-only.
- Local acceptance mode overrides provider selection in memory and does not
  mutate `.env` or process-global environment variables.
- Strict configured mode exits before provider calls when Meshy/OpenAI keys are
  missing.
- Reports are sanitized for bearer tokens, API keys, data URIs, `raw=` payload
  markers, and file URLs.
- The report does not include raw media, local filesystem paths, or raw personal
  source bodies.
- iOS signing/device gates remain separate and still require local user action
  outside Codex.
