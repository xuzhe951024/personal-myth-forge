# P0.57 ARKit Scan Package Regression

Date: 2026-06-06

## Scope

P0.57 adds a SwiftPM-verifiable mobile-core bridge that packages one
ARKit/Object Capture scan export plus optional reference images into a ready
`CaptureMediaSelection(mode: .arkitScan)`.

## Red-Green Evidence

- RED: `swift run --package-path apps/mobile/ios PersonalMythForgeMobileCoreContractTests`
  failed because `ARKitScanPackageBuilder`, `ARKitScanExportFile`,
  `ARKitScanReferenceImageFile`, and `ARKitScanPackageBuilderError` did not
  exist.
- GREEN: the same command passed after adding
  `ARKitScanPackageBuilder.swift`.

## Implemented Checks

- Scan assets accept GLB, USDZ, and octet-stream content types.
- Reference images accept HEIC, HEIF, JPEG, and PNG content types.
- Content types are trimmed and lowercased.
- Scan export is first in the output media list.
- Reference images are sorted by filename and capped at 11.
- Files above `CaptureDraft.maxFileBytes` are rejected.
- `ForgeRootView` references the bridge when combining ARKit scan assets and
  reference images.
- `PersonalMythForgeMobileProjectChecks` reads the builder and contract tests.
