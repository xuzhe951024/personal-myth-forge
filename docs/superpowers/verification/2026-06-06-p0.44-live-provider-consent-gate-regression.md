# P0.44 Live Provider Consent Gate Regression

## Scope

P0.44 adds an explicit `--allow-live-provider-calls` consent gate to
`demo-acceptance` and `final-acceptance`. The default path must not contact live
Meshy/OpenAI providers even when provider keys are configured.

## Remaining External Blockers

- Real Meshy/OpenAI provider validation still requires user-provided backend-only
  keys, quota confirmation, and `--allow-live-provider-calls`.
- iOS device deployment still requires local signing config, Apple SDK license
  acceptance, LAN backend URL, and real iPhone validation.
- Real print fulfillment remains intentionally unimplemented.
