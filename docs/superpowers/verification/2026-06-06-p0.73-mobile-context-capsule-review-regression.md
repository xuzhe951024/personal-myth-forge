# P0.73 Mobile Context Capsule Review Regression

## Scope

P0.73 adds an explicit iPhone context capsule review and approval gate before
`Forge Myth`.

## Behavior Covered

- Swift core builds `ContextCapsuleReview` from manually entered summary fields.
- Missing theme or tone maps to waiting.
- Complete but unapproved capsule maps to waiting.
- Approved complete capsule maps to ready.
- Unsafe context text is redacted before display.
- Capture form renders `ContextCapsuleReviewView`.
- `Forge Myth` is disabled unless media is ready and the capsule is approved.
- `ForgeRootView.forgeMyth` guards against unapproved capsules.
- Editing theme or tone clears approval.
- iOS showcase acceptance requires the context capsule review feature.

## Safety

The app still does not read raw personal data sources. It sends only manually
entered and explicitly approved `currentTheme` and `desiredTone` summary fields
to the backend. No provider keys, `.env` files, emails, chats, calendars,
documents, or raw private bodies are read by this flow.
