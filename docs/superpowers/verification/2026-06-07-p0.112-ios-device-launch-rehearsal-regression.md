# P0.112 iOS Device Launch Rehearsal Regression

P0.112 adds `make ios-device-launch-rehearsal`, a safe project-root wrapper for
regenerating the final ignored JSON evidence set before the physical iPhone demo
operator pass.

## Scope

- `services/backend/scripts/write_ios_device_launch_rehearsal.sh`
- `myth_forge_api.ios_device_launch_rehearsal`
- `myth_forge_api.cli ios-device-launch-rehearsal`
- `make ios-device-launch-rehearsal`
- `services/backend/.local/ios-device-launch-rehearsal.json`

## Verification

Focused checks:

```bash
cd services/backend
uv run pytest tests/test_ios_device_launch_rehearsal.py tests/test_final_rehearsal_scripts.py -q
uv run pytest tests/test_resource_template_acceptance.py tests/test_ios_showcase_acceptance.py -q
cd ../..
swift run --package-path apps/mobile/ios PersonalMythForgeMobileProjectChecks
```

Full branch checks:

```bash
make backend-test
make backend-lint
swift run --package-path apps/mobile/ios PersonalMythForgeMobileProjectChecks
swift run --package-path apps/mobile/ios PersonalMythForgeMobileCoreContractTests
make ios-device-launch-rehearsal
```

The rehearsal target exits `0` when each subcommand exits `0` or `2` and the
summary report is written. It treats exit `2` as a usable blocked-report state,
not as a wrapper failure.

## Safety

The wrapper runs only local/no-key or read-only report commands and writes
ignored JSON reports. It does not call live providers, apply resources, write
backend/iOS config, start servers, run deploy preflight, run Xcode, touch
signing/keychain, accept Apple licenses, or mutate global machine state.
