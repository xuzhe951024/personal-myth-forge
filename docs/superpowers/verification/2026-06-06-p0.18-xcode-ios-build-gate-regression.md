# P0.18 Xcode iOS Build Gate Regression

## Scope

P0.18 adds a repeatable Xcode build gate for
`apps/mobile/ios/PersonalMythForge.xcodeproj` without changing global Xcode
settings. It is an unsigned generic iOS build gate, not a simulator launch,
device install, archive, signing pass, or ARKit runtime test.

## Baseline Gates

```bash
swift run --package-path apps/mobile/ios PersonalMythForgeMobileProjectChecks
```

Result:

```text
PersonalMythForgeMobileProjectChecks passed
```

```bash
swift run --package-path apps/mobile/ios PersonalMythForgeMobileCoreContractTests
```

Result:

```text
PersonalMythForgeMobileCoreContractTests passed
```

```bash
swift build --package-path apps/mobile/ios --product PersonalMythForgeMobileAppCompileCheck
```

Result:

```text
Build of product 'PersonalMythForgeMobileAppCompileCheck' complete!
```

```bash
make backend-lint
```

Result:

```text
All checks passed!
```

```bash
make backend-test
```

Result:

```text
67 passed, 1 warning
```

The warning is the existing FastAPI/Starlette `TestClient` deprecation notice
for `httpx`; it is not new to P0.18.

## Xcode Gate

```bash
make mobile-xcode-build
```

Current machine result before Apple SDK license acceptance:

```text
apps/mobile/ios/scripts/xcode_build_gate.sh
You have not agreed to the Xcode license agreements. Please run 'sudo xcodebuild -license' from within a Terminal window to review and agree to the Xcode and Apple SDKs license.
make: *** [mobile-xcode-build] Error 69
```

This proves the repository command reaches Apple's `xcodebuild`. It remains an
environment license gate until the user accepts the license in Terminal; Codex
did not run `sudo xcodebuild -license`.
