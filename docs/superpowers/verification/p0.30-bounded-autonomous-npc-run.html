Backend Ownership

  • Each step uses the latest stored MythSessionHistory.
  • Each generated tick is appended before the next request.
  • Response includes NPCAutonomyRun plus updated history.
  • The app applies applyBackendHistory(run.history).

Fallback

Run unavailable foreground
The visible myth stays on screen and the UI reports that the autonomy run is not reachable yet.

Safety Boundary

  • Not a background daemon or infinite NPC loop.
  • No provider keys in mobile requests.
  • No raw media, local paths, or personal source bodies.
