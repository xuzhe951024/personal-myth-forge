# P0.16 Asset Variant Contract Regression

Date: 2026-06-06

## Scope

P0.16 verifies that backend generated asset metadata can expose a primary GLB
game asset plus a SceneKit-loadable USDZ iOS scene variant, and that the mobile
core selects the USDZ variant before falling back to the primary GLB.

No simulator, physical iPhone, Xcode signing, ARKit runtime, Unity import, real
Meshy key run, or print provider fulfillment was executed.

## Acceptance Notes

- Backend `GeneratedAsset` remains backward-compatible while adding
  `variants`.
- Local 3D generation now returns GLB primary metadata plus a USDZ
  `ios_scene_asset` stub.
- Meshy requests now include `target_formats: ["glb", "usdz"]` for text and
  image generation, and `model_urls.usdz` becomes the iOS scene variant when
  present.
- Swift `GeneratedAsset` decodes older responses without `variants` as an empty
  array.
- `ArtifactAssetPreparer` chooses `preferredSceneVariant` before the primary GLB
  and preserves the selected variant URI in `PreparedArtifactAsset.sourceURI`.
