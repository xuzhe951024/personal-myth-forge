# P0.64 Mobile Final Launch Readiness Regression

## Scope

P0.64 exposes the sanitized final demo launch profile through
`GET /v1/final-demo-launch?mode=local|configured` and wires the local-mode report
into the iOS device preflight summary. The mobile app now shows a `Final Launch`
row alongside backend, provider, local demo, and saved NPC checks.

The endpoint and mobile client are read-only. They do not write `.env`, write
iOS deploy config, start servers, call Meshy/OpenAI/Treatstock, run Xcode, sign
the app, or mutate global machine state.

## Safety Checks

- API response contains no `sk-` values, `Authorization`, raw media, checkout
  URLs, or `/Users/` paths.
- Swift HTTP error handling continues to sanitize secret-like provider failures.
- Device preflight redacts unsafe final launch errors before encoding/display.
- Live provider calls remain behind configured final acceptance consent.
