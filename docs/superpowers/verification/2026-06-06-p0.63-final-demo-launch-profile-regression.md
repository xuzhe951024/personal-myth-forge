# P0.63 Final Demo Launch Profile Regression

## Scope

P0.63 adds a final demo launch profile report and `make final-demo-launch`.
The report consolidates resource handoff, backend device serving, provider
readiness, local/configured final acceptance, iOS deploy preflight, and the
Xcode build gate. It is a static operator report: it does not start servers,
call providers, write secrets, or mutate global machine state.

## Launch Report Smoke

Local no-key launch:

```bash
cd services/backend
uv run python -m myth_forge_api.cli final-demo-launch \
  --mode local \
  --repo-root ../.. \
  --output /tmp/personal-myth-forge-final-demo-launch-p0.63-local.json
```

Result: exit `0`, `overall_status=partial`. Provider keys are not required in
local mode; the report still surfaces iPhone deploy config and Apple SDK license
actions.

Configured launch before final keys:

```bash
cd services/backend
uv run python -m myth_forge_api.cli final-demo-launch \
  --mode configured \
  --repo-root ../.. \
  --output /tmp/personal-myth-forge-final-demo-launch-p0.63-configured.json
```

Result: exit `2`, `overall_status=blocked`, because Meshy/OpenAI provider
selection/keys and iOS deploy values have not been supplied yet.

Quick final acceptance:

```bash
cd services/backend
uv run python -m myth_forge_api.cli final-acceptance \
  --profile quick \
  --provider-mode local \
  --repo-root ../.. \
  --output /tmp/personal-myth-forge-final-acceptance-p0.63.json
```

Result: exit `2`, `10 passed`, `2 blocked`, `0 failed`. The blocked checks are
the known local iOS environment gates:

- `mobile_deploy_preflight`: `blocked_by_local_ios_deploy_config`
- `mobile_xcode_build`: `blocked_by_apple_sdk_license`

## Safety Checks

- Launch report output contains no provider key values.
- Launch report output contains no local absolute paths.
- The command does not call Meshy, OpenAI, Treatstock, Sculpteo, Xcode, signing,
  keychain, or Apple license tools.
- Live provider acceptance remains explicit through
  `--allow-live-provider-calls`.
