# P0.114 Rehearsal Final Launch Sync Regression

P0.114 makes `make ios-device-launch-rehearsal` leave the iPhone-facing final
launch payload synchronized with the just-written rehearsal report.

Behavior covered:

- Wrapper command: `make ios-device-launch-rehearsal`
- First refreshed artifact: `services/backend/.local/ios-device-launch-rehearsal.json`
- Final synced artifact: `services/backend/.local/final-demo-launch-local.json`
- Mobile surface: `Final Launch Status` -> `Launch Rehearsal`

Verification commands:

```bash
cd services/backend
uv run pytest tests/test_final_rehearsal_scripts.py tests/test_resource_template_acceptance.py tests/test_ios_showcase_acceptance.py -q
cd ../..
swift run --package-path apps/mobile/ios PersonalMythForgeMobileProjectChecks
make ios-device-launch-rehearsal
jq '.ios_device_launch_rehearsal_readiness | {kind,status,summary,safety}' services/backend/.local/final-demo-launch-local.json
```

Visual evidence:

- `docs/superpowers/verification/p0.114-rehearsal-final-launch-sync.html`
- `docs/superpowers/verification/assets/p0.114-rehearsal-final-launch-sync-390x844.png`

Safety scope:

- No provider calls
- No backend/iOS config writes
- No server start
- No deploy preflight execution
- No Xcode, signing, keychain, or Apple license mutation
- No command execution from the iPhone app
