# P0.48 Capture Scene Handoff Acceptance Regression

## Scope

P0.48 adds an integrated local acceptance gate for the mobile showcase path:

1. Upload guided scan media with `POST /v1/object-captures`.
2. Create a myth session with `POST /v1/myth-sessions/from-capture`.
3. Verify `generation_provenance.input_mode == "multi_image"` with two source
   images.
4. Download the returned backend-served `game.glb` and `scene.dae` assets.
5. Confirm the report contains no raw media, `local-capture://` URIs, local
   filesystem paths, provider bearer tokens, or provider secrets.

The gate runs entirely with the deterministic local provider path. It does not
call Meshy, OpenAI, Treatstock, Sculpteo, Apple services, or external network
endpoints.

## Safety

The new `capture_scene_handoff_acceptance` report confirms:

- Guided scan media count is two.
- Generation provenance remains multi-image.
- Primary game asset URL is backend HTTP and ends with `game.glb`.
- iOS scene asset URL is backend HTTP and ends with `scene.dae`.
- Downloaded GLB starts with `glTF`.
- Downloaded DAE contains `COLLADA`.
- No live provider calls.
- Temporary FastAPI capture-store patch is restored.
- Report contains no `data:image`, `local-capture://`, `/tmp`, `/Users/`,
  `Authorization`, or `Bearer` markers.
