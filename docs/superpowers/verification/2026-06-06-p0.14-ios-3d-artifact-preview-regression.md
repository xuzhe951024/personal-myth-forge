# P0.14 iOS 3D Artifact Preview Regression

Date: 2026-06-06

## Scope

P0.14 verifies that the iOS app source now contains a SwiftPM-compiled SceneKit
artifact preview surface for generated 3D assets. This evidence is source-level
and static Browser evidence only; it is not an Xcode simulator run, device
install, ARKit runtime capture, or real Meshy GLB import.

## Code Evidence

- `ArtifactPreviewState` classifies generated asset preview readiness in the
  SwiftPM mobile core.
- `Artifact3DPreviewView` imports SceneKit, builds an `SCNScene`, and renders it
  through SwiftUI `SceneView(scene:)`.
- `ArtifactSummaryView` embeds `Artifact3DPreviewView(session: session)`.
- `PersonalMythForgeMobileProjectChecks` requires the new source file, Xcode
  target membership, SceneKit usage, and artifact summary wiring.
- `PersonalMythForgeMobileAppCompileCheck` compiles the app source against the
  mobile core without changing global Xcode settings.

## Remaining Gaps

- Actual provider asset download, cache, and local file handoff.
- GLB runtime conversion or provider-side USDZ path for direct SceneKit loading.
- Full Xcode simulator/device deployment and signing.
- Live ARKit mesh capture runtime.
- Richer 3D village scene with NPC movement around the artifact.
