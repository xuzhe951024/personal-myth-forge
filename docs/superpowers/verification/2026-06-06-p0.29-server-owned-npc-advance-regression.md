# P0.29 Server-Owned NPC Advance Regression

Date: 2026-06-06

## Scope

P0.29 adds a server-owned NPC advance path for stored myth sessions:

```http
POST /v1/myth-sessions/{session_id}/npc-ticks
```

The backend loads `MythSessionHistory`, derives the next tick index and recent
events from stored history, runs the configured NPC tick runtime, appends the
tick, and returns updated `MythSessionHistory`. The mobile app prefers this path
for valid backend session ids and applies the returned history to the visible
demo state.

This is not a background autonomous loop, production account memory, Unity
movement execution, provider-key provisioning, or print fulfillment.

## Server-Owned Advance Contract

- `POST /v1/myth-sessions/{session_id}/npc-ticks` requires only the session id.
- The backend loads stored history and returns updated `MythSessionHistory`.
- The next tick index comes from stored tick history.
- Recent events come from the latest stored tick, or the initial session world
  changes when there are no ticks.
- Existing stateless `POST /v1/npc-ticks` remains available.

## Mobile Contract

- `advanceMythSessionHistory(sessionId:)` validates `myth_<16 hex>` before
  network transport.
- `ForgeRootView` uses `advanceBackendNPCTick` for backend-valid sessions.
- Returned history flows through `applyBackendHistory(history)`.
- `advanceStatelessNPCTick` remains as the compatibility fallback for legacy
  sessions.

## Safety Review

- No provider keys are sent to mobile.
- No raw photos, scan binaries, data URIs, or local file URLs are added to the
  mobile request or snapshot path.
- No raw personal email, chat, calendar, or document bodies are persisted.
- Provider failures are sanitized before HTTP response details are returned.
- Backend advance failure leaves the current visible myth session on screen.
